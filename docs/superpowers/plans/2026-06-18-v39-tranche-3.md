# v39 — Tranche 3: Table of Contents (`toc`) Component

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the **one genuine component gap** the RetroUI study found — a **Table of Contents** with
scroll-spy. cascivo has ~140 components but no TOC. Build it the cascivo way: signal-driven (no
`useState`/`useEffect`), controlled `items`-first API, a `<nav>` landmark of anchor links with the active
heading tracked via `useSignal` + `useSignalEffect`-driven `IntersectionObserver`, keyboard-navigable, WCAG
AA, i18n-defaulted label. Ship the full kit — `toc.tsx` + `toc.module.css` + `toc.meta.ts` + `toc.test.tsx`
— export it from `@cascivo/react`, and let `pnpm regen` add it to `registry.json` so MCP/docs/Storybook pick
it up automatically.

**Architecture:** Follow an existing navigation component (e.g. `breadcrumb` or `side-nav`) for file layout,
manifest shape, and CSS Module conventions. Per CLAUDE.md: no banned hooks; visual states via CSS
(`:hover`/`:focus-visible`/`[aria-current]`); DOM observation via `useSignalEffect` only; in React apps a
component reading `signal.value` during render calls `useSignals()` first.

**Tech Stack:** signal-driven TSX + CSS Modules; `@cascivo/core` (`useSignal`, `useSignalEffect`,
`useSignals`, `useRef`); `@cascivo/i18n` for the label; Vitest + Testing Library.

### Task 2: Implement `toc.tsx` (controlled, signal-driven)

**Files:** `packages/components/src/toc/toc.tsx`

- [ ] **Step 1:** Props: `items: { id: string; label: string; level?: number }[]` (controlled, required);
      `activeId?: string` (controlled override); `onActiveChange?: (id: string) => void`;
      `labels?: { nav?: string }` (overrides the i18n default); standard `className`/`...rest`. Render a
      `<nav aria-label={…}>` containing an ordered/unordered list of `<a href={'#'+id}>` links, indented by
      `level` via a CSS custom property (`--toc-level`), `aria-current="location"` on the active link.
- [ ] **Step 2:** Active-item state: `const active = useSignal(activeId ?? items[0]?.id)`. If `activeId` is
      provided (controlled), sync it during render (`active.value = activeId` — no-op if unchanged, per the
      CLAUDE.md controlled-prop pattern). Call `useSignals()` as the first statement so React apps re-render.
- [ ] **Step 3:** Scroll-spy via `useSignalEffect` + `IntersectionObserver`: observe each `#id` target,
      update `active.value` to the topmost intersecting heading, call `onActiveChange` through a synced ref
      (`onActiveChangeRef.current = onActiveChange` during render; read inside the effect). **No `useEffect`.**
      Disconnect the observer in the effect's cleanup. Guard for SSR/no-DOM (`typeof IntersectionObserver`).
- [ ] **Step 4:** Click on a link is the browser's native anchor jump (no JS needed); ensure focus management
      and `prefers-reduced-motion` are respected if you add smooth scroll (CSS `scroll-behavior`, not JS).

### Task 3: Optional `useTocFromRegion` helper

**Files:** `packages/components/src/toc/use-toc-from-region.ts`

- [ ] **Step 1:** Export `useTocFromRegion(ref: RefObject<HTMLElement>, opts?): Signal<TocItem[]>` that scans
      headings (`h2`–`h4`, configurable) inside the region and returns a signal of `items` (id from the
      heading's `id`, label from text, level from tag). Use `useSignal` + `useSignalEffect` (observe DOM
      mutations only if needed; a one-shot scan is fine for static content). Keep it thin — it only derives
      `items` to feed `<Toc items=… />`; it does not own active state.
- [ ] **Step 2:** Document it as opt-in convenience; the controlled `items` API remains primary/testable.
