### Task 4: Add pagination to A11yMatrix.tsx

- [ ] **Step 1: Verify DataTable `pagination` prop type**

  The DataTable in `packages/components/src/data-table/data-table.tsx` accepts a `pagination`
  prop. Confirm the expected shape by reading the type:

  ```sh
  grep -n "pagination" packages/components/src/data-table/data-table.tsx | head -10
  ```

  Expected shape (from implementation): `{ pageSize: number, pageSizeOptions?: number[] }`.

- [ ] **Step 2: Add `pagination` prop to the `DataTable` in A11yMatrix.tsx**

  Find the `<DataTable>` element in `A11yMatrix.tsx`:

  ```tsx
  <DataTable
    columns={COLUMNS}
    rows={rows.value}
    getRowId={(r) => r.name}
    density="compact"
    stickyHeader
    title="Keyboard and ARIA matrix"
    description="Role, WCAG level, and keyboard interaction per registry entry, generated from each component.meta.ts."
  />
  ```

  Replace with:

  ```tsx
  <DataTable
    columns={COLUMNS}
    rows={rows.value}
    getRowId={(r) => r.name}
    density="compact"
    stickyHeader
    pagination={{ pageSize: 10, pageSizeOptions: [10, 25, 50] }}
    title="Keyboard and ARIA matrix"
    description="Role, WCAG level, and keyboard interaction per registry entry, generated from each component.meta.ts."
  />
  ```

  No other changes needed in this file — the DataTable component handles page reset internally
  when `rows` changes (the `pageSignal` is bounded by `pageCount` which recalculates on each
  render). When the user changes the category filter and `rows.value` shrinks, the current page
  is automatically clamped.
