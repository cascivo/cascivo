**Goal:** A machine-readable, CI-validated directory of community registries exists and is
published; directory namespaces work in the CLI with zero config; a docs Directory page and
a starter template make authoring + submitting a registry a 30-minute exercise; MCP tools
reach community registries.

**Architecture:** `directory/registries.json` in this repo is the source of truth →
validated by a dedicated CI job (reusing `@cascivo/registry` validators) → copied into
`apps/docs/public/r/registries.json` at generate time → consumed by the docs Directory
page, the CLI namespace fallback, and the MCP server.

**Tech stack:** unchanged.

### Task 2: CI validation job (the verified tier)

**Files:**

- Create: `.github/workflows/directory.yml` (or extend the existing CI workflow — check
  repo convention first)
- Create: `scripts/directory/validate.ts`

- [ ] **Step 1:** `scripts/directory/validate.ts` (runs in CI on PRs touching
      `directory/`): for each entry — `validateDirectory` (hard fail) → fetch the live
      `registry.json`/index from `registryUrl` (4 retries) → `validateIndex` (hard fail) →
      **verified-tier checks** per item: `meta` present with complete `accessibility`
      block (role + wcag + keyboard), license set, files reachable (HEAD), `cascade
registry build`-conformant flat layout. Network-only failures report as a distinct
      soft status (risk #4); schema failures block merge.
- [ ] **Step 2:** `verified: true` may only be set when the verified-tier checks pass —
      the script errors if an entry claims `verified` but checks fail. (Human review of
      the PR remains the final gate; document in Task 4's guide.)
- [ ] **Step 3:** Run the script locally against the seed directory — green.
- [ ] **Step 4: Commit** — `ci(directory): validate registry submissions, enforce verified tier`

## Tranche exit criteria

1. `directory/registries.json` validated in CI; an intentionally-broken fixture PR check
   fails with named errors; verified flag enforced by checks.
2. `cascade add @cascade/button` works with zero config (directory fallback), offline
   failure mode is a clear error.
3. Directory page renders entries with verified badges and filters; built with cascade
   components.
4. A fresh clone can follow CONTRIBUTING-REGISTRY.md start-to-finish: starter builds,
   installs locally, submission process documented.
5. MCP `list_registries` + namespaced `get_component` green in tests; default behavior
   unchanged.
6. Root gate + drift check exit 0.
