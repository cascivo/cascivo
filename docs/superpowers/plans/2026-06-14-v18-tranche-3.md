> Current-state assumptions — RE-VERIFY before starting:
>
> - `packages/core/src/signals.ts` exports `useSignal`, `useComputed`, `useSignalEffect`, `useSignals`,
>   `signal`, `computed`, `effect`, `batch`. These are the only state primitives allowed.
> - `CLAUDE.md` already documents the controllable pattern by hand:
>   `const isOpen = useSignal(open); isOpen.value = open` and the `onCloseRef` ref idiom. T3 codifies

>   listeners all go inside `useSignalEffect` with cleanup returns.
> - `useRef` allowed only for DOM/callback refs.

**Goal:** Five small, foundational hooks in `@cascivo/core`: `useControllableSignal` (the documented
controlled/uncontrolled pattern, codified), `useMediaQuery`, `useScrollLock`, `useId`, `useClipboard`.
Each tested, each obeying the signal rules.

**Architecture:** New files in `packages/core/src/`:
`controllable.ts`, `media-query.ts`, `scroll-lock.ts`, `use-id.ts`, `clipboard.ts` (+ `*.test.ts(x)`),
exported from `index.ts`.

**Tech stack:** signals only. `useId` may wrap React's `useId` for SSR-safe ids. No new deps.

### Task 1: `useControllableSignal`

**Files:** Create: `packages/core/src/controllable.ts` + test.

- [ ] **Step 1:** Implement `useControllableSignal<T>({ value, defaultValue, onChange })` → returns a
      `Signal<T>` plus a setter that (a) when `value` is provided (controlled), mirrors it into the signal
      each render (`sig.value = value`) and routes writes through `onChange`; (b) when undefined
      (uncontrolled), owns the signal seeded from `defaultValue` and still calls `onChange` on writes.
      This is the codification of the `CLAUDE.md` pattern.
- [ ] **Step 2:** Use a ref to keep `onChange` current (the `onCloseRef.current = onChange` idiom) so the
      setter always calls the latest callback without `useEffect`.
- [ ] **Step 3:** Test: controlled mode reflects prop changes; uncontrolled mode updates locally; both
      fire `onChange`; switching is not supported (assert documented behavior — controlled vs uncontrolled
      is fixed for the component's life, like React).

### Task 2: `useMediaQuery`

**Files:** Create: `packages/core/src/media-query.ts` + test.

- [ ] **Step 1:** Implement `useMediaQuery(query: string): ReadonlySignal<boolean>` — seed from
      `matchMedia(query).matches` (SSR-safe: default `false` when `window`/`matchMedia` absent), then
      subscribe to changes inside `useSignalEffect`, updating the signal; clean up the listener on
      teardown.
- [ ] **Step 2:** Test (jsdom): mock `matchMedia`, assert initial value and that a simulated change event
      updates the signal. Assert SSR-safe default when `matchMedia` is undefined.
