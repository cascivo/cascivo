**Goal:** A functional **Cascade Flow** dashboard: a list of process instances (`DataTable` with state),
a process detail view with a **hand-built SVG flow diagram** whose active token advances via the sim
engine, an incidents list, an instance history `Timeline`, a process-variables `TreeView`, and a Tasklist
where a user task can be claimed and completed via a `Form` (mutation persists across reload).

**Architecture:** New `apps/examples/flow/` mirroring T1 (vite config + aliases incl.
`@cascivo/example-kit`, `data/`, `api/`, `App.tsx`, `sections/`, a `components/FlowDiagram.tsx`,
`test/`). No kit changes expected.

**Tech stack:** unchanged. The token-movement uses `useSimulation`/`useSignalEffect`; `useSignals()`
first; no `useState`/`useEffect`/`useContext`. Strings via `@cascivo/i18n`.

## Tranche exit criteria

1. `apps/examples/flow` runs and builds with no prior full build.
2. The flow diagram is hand-built SVG (no library); the token advances via the sim engine and pauses;
   movement is deterministic and keyboard/axe-clean.
3. Instances/incidents load via mock-api with loading/empty states; Timeline + TreeView render an
   instance's history and variables.
4. Claiming/completing a task persists across reload and rolls back on injected failure.
5. Playwright smoke + axe pass; no `useState`/`useEffect`/`useContext`; `useSignals()` first; strings via
   `@cascivo/i18n`; mobile-clean 320–414.
