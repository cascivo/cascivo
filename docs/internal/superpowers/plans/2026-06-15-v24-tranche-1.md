# v24 — Tranche 1: SideNav Groups + Flow Nav Restructure

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extend `SideNav` with an optional `groups` prop that renders labelled section headers above groups of items. Wire the Flow app's navigation to use it, producing a CONSOLE section (Dashboard, Clusters, Organization) and a CLUSTERS section (one named workspace item with an env `Badge`).

**Architecture:** `SideNavProps` gains `groups?: SideNavGroup[]`. When `groups` is provided, the component renders each group as a `<section>` with an optional `<h3>` label above its items. Flat `items` prop continues to work for all existing callers. The Flow `App.tsx` migrates from `items` to `groups`.

**Tech Stack:** `packages/components/src/side-nav/side-nav.tsx`, `@cascivo/react`, Vitest.

- [ ] **Step 2: Write failing test for group rendering**

  In `packages/components/src/side-nav/side-nav.test.tsx`, add:

  ```tsx
  it('renders section headers when groups are provided', () => {
    render(
      <SideNav
        groups={[
          { label: 'CONSOLE', items: [{ label: 'Dashboard' }] },
          { label: 'CLUSTERS', items: [{ label: 'Dev Cluster' }] },
        ]}
      />,
    )
    expect(screen.getByText('CONSOLE')).toBeInTheDocument()
    expect(screen.getByText('CLUSTERS')).toBeInTheDocument()
    expect(screen.getByText('Dashboard')).toBeInTheDocument()
    expect(screen.getByText('Dev Cluster')).toBeInTheDocument()
  })

  it('renders no section headers when groups have no label', () => {
    render(<SideNav groups={[{ items: [{ label: 'Dashboard' }] }]} />)
    expect(screen.getByText('Dashboard')).toBeInTheDocument()
    // No spurious header element
    expect(screen.queryByRole('heading')).toBeNull()
  })

  it('existing flat items prop still works when groups is absent', () => {
    render(<SideNav items={[{ label: 'Instances' }, { label: 'Incidents' }]} />)
    expect(screen.getByText('Instances')).toBeInTheDocument()
  })
  ```

- [ ] **Step 4: Add `SideNavGroup` type and `groups` prop**

  In `packages/components/src/side-nav/side-nav.tsx`, add after the `SideNavItem` interface:

  ```ts
  }
  ```

  Change `items` to optional: `items?: SideNavItem[]`.

- [ ] **Step 5: Update the SideNav render function to handle groups**

  Inside the component body, derive the effective items before the existing render logic:

  ```tsx
  // Resolve items: groups take precedence over flat items
  const effectiveGroups: SideNavGroup[] =
    props.groups ?? (props.items ? [{ items: props.items }] : [{ items: [] }])
  ```

  Replace the existing `{items.map(...)}` JSX with:

  ```tsx
  {
    effectiveGroups.map((group, gi) => (
      <section key={gi} className={styles['group']}>
        {group.label && <h3 className={styles['groupLabel']}>{group.label}</h3>}
        {group.items.map((item, ii) =>
          /* existing item render JSX unchanged, keyed by ii */
          renderItem(item, ii),
        )}
      </section>
    ))
  }
  ```

  Extract the existing per-item render into a `renderItem(item, key)` helper function inside the
  component to avoid duplication.

- [ ] **Step 4: Update Flow App.tsx to use navGroups**

  Replace the `navItems` array in `apps/examples/flow/src/App.tsx` with:

  ```tsx
  const navGroups: SideNavGroup[] = [

      label: t(msg.navGroupConsole),
      items: [
        {
          label: t(msg.navDashboard),
          active: currentSection.value === 'instances',
          onClick: (e) => {
            e.preventDefault()
            currentSection.value = 'instances'
          },
        },
        {
          label: t(msg.navClustersNav),
          active: false,
        },
        {
          label: t(msg.navOrganization),
          active: false,
        },
      ],
    },
    {
      label: t(msg.navGroupClusters),
      items: [
        {
          label: t(msg.navClusterItem),
          active: false,
          // env badge rendered via icon slot
          icon: (
            <Badge size="sm" variant="secondary">
              Dev
            </Badge>
          ),
          items: [
            { label: t(msg.navInstances), href: '#', active: currentSection.value === 'instances' },
            { label: t(msg.navIncidents), href: '#', active: currentSection.value === 'incidents' },
            { label: t(msg.navTasklist), href: '#', active: currentSection.value === 'tasklist' },
          ],
          onClick: (e) => {
            e.preventDefault()
          },
        },
      ],
    },
  ]
  ```

  Pass `navGroups={navGroups}` instead of `navItems={navItems}` to `<AppShell>`.

  Import `Badge` from `@cascivo/react`.
