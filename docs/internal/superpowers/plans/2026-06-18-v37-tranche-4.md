# v37 — Tranche 4: Flattened Published Types

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ensure the published `@cascivo/react` `.d.ts` no longer expose the internal monorepo layout
(#4) — "Go to definition" should land inside the published surface, not in
`dist/types/packages/components/src/…`.

**Architecture:** A `flatten-types.mjs` step already exists
(`packages/react/scripts/flatten-types.mjs`, run by the build script). The report was filed against
published `0.1.0`, which still leaked `packages/.../src` paths — so this tranche is **verify the fix
actually ships**, not build from scratch (Decision 6). We add a regression check that greps the emitted
`dist/**/*.d.ts` for leaked source paths and fix the flatten step only if the check fails.

**Tech Stack:** Node check script; the existing `tsc` + flatten build pipeline.

---

## Files

- Modify: `packages/react/scripts/flatten-types.mjs` — only if the check below fails
- Create: `packages/react/scripts/check-types-flat.mjs` — grep `dist/**/*.d.ts` for leaked paths
- Modify: `packages/react/package.json` — run the check after build

### Task 1: Reproduce the leak against the current build

**Files:** build output `packages/react/dist`

- [ ] **Step 1:** Clean-build `@cascivo/react` (`pnpm exec vp run @cascivo/react#build`). Inspect
      `dist/index.d.ts` and `dist/**/*.d.ts` for references to `packages/`, `/src/`, or
      `./types/packages/...` re-exports (the exact shapes the report cites:
      `export * from './types/packages/react/src/index'` → `../../components/src/button/button`).
- [ ] **Step 2:** Record whether the existing `flatten-types.mjs` already removes these. Two outcomes:
      (a) flat already → T4 is just the regression check; (b) still leaking → fix the flatten step.

### Task 2: Add the regression check

**Files:** `packages/react/scripts/check-types-flat.mjs`

- [ ] **Step 1:** Write a script that scans `dist/**/*.d.ts` and exits non-zero if any declaration
      references `packages/` source paths or re-exports through a `types/packages/...` tree. Keep the
      allowed surface explicit (a flat `index.d.ts` plus colocated declarations).
- [ ] **Step 2:** Wire it into `packages/react/package.json` `build` (after `flatten-types`) so a leak
      fails the build, and into the CI type/build gate.

### Task 3: Fix the flatten step (only if Task 1 found a leak)

**Files:** `packages/react/scripts/flatten-types.mjs`

- [ ] **Step 1:** Roll the declarations up to a single published entry that resolves within the package
      (bundle/rewrite the re-export chain so `dist/index.d.ts` does not point at
      `./types/packages/.../src`). Prefer fixing the existing script over adding a new tool; if a bundler
      (api-extractor / dts-bundle) is warranted, justify it and follow the dependency policy (latest stable).
- [ ] **Step 2:** Re-run Task 1's inspection; confirm "Go to definition" targets resolve inside the
      published surface.
