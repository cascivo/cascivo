# v33 — Tranche 3: Demo Back-Link + Blocks Site Chrome & Live Previews

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Two navigation/chrome fixes. (1) Add a "← Back to cascivo" link to the demo apps' mock banner so users can return to the landing page. (2) Give the Blocks pages the standard site header/footer, and replace the green placeholder card images with live scaled previews of the real blocks.

**Architecture:** The demo back-link lives in the shared `AppShell` mock banner; demos build with `base: './'` and mount at `/demos/<slug>/`, so the link target is the site root `/`. Block cards embed the existing bare `/blocks/preview/<name>` route in a scaled, non-interactive iframe. Both Blocks pages get wrapped in the same chrome other landing pages use.

**Tech Stack:** React, `@cascivo/i18n` (`defineMessages`/`t`) for the banner copy, `@cascivo/core` signals, CSS modules / `blocks.css`.

### Task 3: Replace green placeholder images with live preview iframes

**Files:** Modify `apps/landing/src/pages/blocks/BlocksPage.tsx`, `apps/landing/src/pages/blocks/blocks.css`

- [ ] **Step 1: Replace the card `<img>` with a scaled preview iframe.** In the `displayed.value.map(...)` card body, swap:

  ```tsx
  <div className="block-card__screenshot">
    <img src={entry.meta.screenshot.light} … />
  </div>
  ```

  for:

  ```tsx
  <div className="block-card__screenshot">
    <iframe
      className="block-card__preview"
      src={`/blocks/preview/${entry.meta.name}`}
      title={`${entry.meta.displayName} preview`}
      loading="lazy"
      tabIndex={-1}
      aria-hidden="true"
    />
  </div>
  ```

  The card remains a single `<a>` link; the iframe is non-interactive.

- [ ] **Step 2: Style the scaled preview** in `blocks.css`. Keep `.block-card__screenshot` as the fixed 16:9 frame with `overflow: hidden` and `pointer-events: none`. Render the iframe at a wider logical width and scale it down so a desktop layout fits the card:

  ```css
  .block-card__screenshot {
    aspect-ratio: 16 / 9;
    overflow: hidden;
    pointer-events: none;
    background: var(--cascivo-color-surface-raised);
  }
  .block-card__preview {
    inline-size: 1280px;
    block-size: 800px;
    border: 0;
    transform: scale(0.25);
    transform-origin: top left;
  }
  ```

  Tune `inline-size`/`scale` so the preview fills the card width cleanly. Remove the old `.block-card__screenshot img` rule if nothing else uses it.

- [ ] **Step 3: Confirm clickability.** `pointer-events: none` on the frame + `tabIndex={-1}`/`aria-hidden` on the iframe means clicks fall through to the wrapping `<a href="/blocks/<name>">`. Keyboard focus skips the iframe.

- [ ] **Step 4: Note (do not delete) the dead green-pixel assets.** The 1×1 PNGs in `apps/landing/public/blocks/screenshots/` are now unused by the card grid. Per CLAUDE.md, flag them in the commit message but leave them — `meta.screenshot.*` may still be referenced elsewhere.
