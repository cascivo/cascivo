# v26 — Tranche 1: Console Mobile — Burger Menu + Sidebar Overlay

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a hamburger button to the left side of the `RelayConsole` titlebar and make the
`SideNav` overlay the content area on mobile instead of disappearing. Desktop layout is unchanged.

**Architecture:** Two files only. `RelayConsole.tsx` gains a `sidebarOpen` signal and a hamburger
button; `landing.css` gains a scoped mobile block under `.console-frame`. The `SideNav` component
is unchanged — the overlay is pure CSS positioning applied from the outside.

**Tech Stack:** `useSignal`, `useSignals`, `useSignalEffect` from `@cascivo/core`. CSS custom
properties. No new components.

### Task 2: Add `sidebarOpen` signal and hamburger button

- [ ] **Step 1: Add the signal**

  In `apps/landing/src/demo/RelayConsole.tsx`, inside `RelayConsole()`:

  ```tsx
  const sidebarOpen = useSignal(false)
  ```

  Place it directly after the existing `const activeNav = useSignal(0)`.

- [ ] **Step 2: Add the hamburger button to the titlebar**

  The current titlebar JSX is:
  ```tsx
  <header className="console-titlebar">
    <span className="console-brand">Relay</span>
    <div className="console-titlebar-actions">…</div>
  </header>
  ```

  Add a hamburger button as the FIRST element inside the header, before the brand:

  ```tsx
  <header className="console-titlebar">
    <button
      type="button"
      className="console-burger"
      aria-label={sidebarOpen.value ? 'Close sidebar' : 'Open sidebar'}
      aria-expanded={sidebarOpen.value}
      aria-controls="console-sidebar"
      onClick={() => { sidebarOpen.value = !sidebarOpen.value }}
    >
      <span className="console-burger-bar" />
      <span className="console-burger-bar" />
      <span className="console-burger-bar" />
    </button>
    <span className="console-brand">Relay</span>
    <div className="console-titlebar-actions">…</div>
  </header>
  ```

- [ ] **Step 3: Add `id` and `aria-hidden` to the SideNav wrapper**

  The `<SideNav>` is inside `.console-body` directly. Wrap it in a `<div>` so we can target
  it precisely with CSS and ARIA:

  ```tsx
  <div
    id="console-sidebar"
    className={`console-sidebar${sidebarOpen.value ? ' console-sidebar--open' : ''}`}
    aria-hidden={!sidebarOpen.value}
  >
    <SideNav items={navItems} ariaLabel="Relay" defaultCollapsed />
  </div>
  ```

- [ ] **Step 4: Add a scrim for mobile overlay**

  After the `</div>` that closes `console-body`, before the closing `</section>`, add:

  ```tsx
  {sidebarOpen.value && (
    <div
      className="console-scrim"
      aria-hidden="true"
      onClick={() => { sidebarOpen.value = false }}
    />
  )}
  ```

- [ ] **Step 5: Close sidebar on Escape**

  Add a `useSignalEffect` after the existing effects (or alongside `activeNav`):

  ```tsx
  useSignalEffect(() => {
    const handler = (e: KeyboardEvent) => {
      if (e.key === 'Escape') sidebarOpen.value = false
    }
    document.addEventListener('keydown', handler)
    return () => document.removeEventListener('keydown', handler)
  })
  ```

- [ ] **Step 6: Add `data-sidebar-open` to `console-frame`**

  The `console-frame` `<div>` needs to reflect sidebar state so CSS can target it:

  ```tsx
  <div
    className="console-frame"
    data-sidebar-open={sidebarOpen.value ? '' : undefined}
  >
  ```

### Task 3: Add CSS for the hamburger and mobile overlay

In `apps/landing/src/landing.css`, find the `/* ── Console (Relay) ──────────────────────────────────────── */` block.

- [ ] **Step 1: Add burger button styles (always visible)**

  Add after `.console-microcopy`:

  ```css
  /* ── Console burger button ──────────────────────────────── */
  .console-burger {
    display: none; /* hidden on desktop; shown by mobile query below */
    flex-direction: column;
    gap: 4px;
    inline-size: 2.75rem;
    block-size: 2.75rem;
    align-items: center;
    justify-content: center;
    background: none;
    border: none;
    cursor: pointer;
    padding: var(--cascivo-space-2);
    color: var(--cascivo-color-text);
    flex-shrink: 0;
    border-radius: var(--cascivo-radius-sm);
  }

  .console-burger:focus-visible {
    outline: 2px solid var(--cascivo-color-accent);
    outline-offset: 2px;
  }

  .console-burger-bar {
    display: block;
    inline-size: 1.25rem;
    block-size: 2px;
    background: currentColor;
    border-radius: 1px;
    transition: transform 200ms ease, opacity 200ms ease;
  }

  @media (prefers-reduced-motion: reduce) {
    .console-burger-bar {
      transition: none;
    }
  }

  /* ── Console sidebar wrapper ─────────────────────────────── */
  .console-sidebar {
    display: contents; /* transparent on desktop — SideNav inside acts normally */
  }

  /* ── Console scrim ───────────────────────────────────────── */
  .console-scrim {
    display: none; /* hidden on desktop */
  }
  ```

- [ ] **Step 2: Add mobile-specific overlay styles**

  Find the `/* ── Console (Relay) mobile ───────────────────────────── */` block and **replace** the
  existing content of that block with:

  ```css
  /* ── Console (Relay) mobile ───────────────────────────── */
  @media (max-width: 47.99rem) {
    .console-body {
      min-block-size: auto;
      /* single column: no sidebar column; sidebar overlays */
      grid-template-columns: 1fr;
    }

    .console-main {
      padding: var(--cascivo-space-4);
    }

    .console-burger {
      display: flex;
    }

    /* Sidebar: positioned absolutely inside console-frame */
    .console-frame {
      position: relative;
      overflow: hidden;
    }

    .console-sidebar {
      display: block;
      position: absolute;
      inset-block: 0;
      inset-inline-start: 0;
      z-index: 20;
      transform: translateX(-100%);
      transition: transform 250ms ease;
      /* fill full height; let SideNav scroll if needed */
      overflow-y: auto;
    }

    @media (prefers-reduced-motion: reduce) {
      .console-sidebar {
        transition: none;
      }
    }

    .console-sidebar.console-sidebar--open {
      transform: translateX(0);
    }

    /* Scrim: covers content area behind the open sidebar */
    .console-scrim {
      display: block;
      position: absolute;
      inset: 0;
      background: oklch(0 0 0 / 0.45);
      z-index: 19;
    }
  }
  ```
