# v45 — Tranche 2: Full Catalog Generation, Exports & Tests

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn the T1 generator output into the **real, expanded public surface**. Re-export the generated
chromicons icons from `packages/icons/src/index.tsx` alongside the existing 61 (no duplicate names — existing
wins), taking the catalog to **~450**, bump `VERSION`, and add tests that hold the invariants: every icon
renders a stroked 24×24 `currentColor` SVG with the `aria-hidden` default, there are no duplicate exports, and
**tree-shaking still holds** at the larger size.

**Architecture:** `generated.tsx` (from T1) already contains `createIcon` exports with collisions skipped.
`index.tsx` re-exports them with a single `export * from './generated'` placed **after** the existing inline
exports, so the hand-authored 61 remain authoritative and the surface only grows. `package.json` is already
`"sideEffects": false`, so named re-exports stay individually tree-shakeable; a guard test makes that durable.

**Tech Stack:** TSX (`@cascivo/icons`); Vitest + Testing Library (`packages/icons/src/index.test.tsx`); `vp
pack` build.
