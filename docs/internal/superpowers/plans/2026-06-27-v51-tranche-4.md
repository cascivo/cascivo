# v51 — Tranche 4: Interaction & Motion

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the charts feel alive — the four interaction/motion capabilities Recharts has and cascivo lacks:
**click/select events** (C-F7 — today only the legend has an `onClick`), a **brush range-selector** (C-F8), **`syncId`
cross-chart tooltip linking** (C-F9), and **reduced-motion-gated entrance/update animation** (C-F10). This is the
heaviest tranche; it wants T1–T3 settled first. Lens: **feature parity**.

**Architecture:** most interaction belongs in `core/chart-frame.tsx`, not per-chart. The frame already owns the
focusable keyboard layer (Arrow/Home/End/Escape over `resolvedTooltip.points`), the `focusedIndex` signal, the
`aria-live` region, the focus ring, and `nearest()` hit-testing. So:
- **Events:** the focusable `<div role="application">` already has `onPointerMove`/`onKeyDown` setting `focusedIndex`.
  Add `onClick` + Enter/Space → invoke an `onSelect(point)` callback the frame receives; charts pass an
  `onDatumClick`/`onSelect(datum, series)` that the frame translates from the focused `ChartPoint` back to the source
  datum (the `point.seriesId` + index encoding the charts already build for tooltips).
- **`syncId`:** a tiny `core/sync.ts` module holds a `Map<syncId, Signal<number | null>>` (the shared hovered
  x-index). A frame with a `syncId` writes its `focusedIndex` into the shared signal and reads it in a
  `useSignalEffect` to mirror the hover — charts in the same group highlight the same x. No prop-drilling, just a
  module-level signal registry (cleaned up by ref-count).
- **Brush:** a `chrome/brush.tsx` strip below the plot with two draggable + keyboard-operable handles defining a
  `[startIndex, endIndex]` window (a signal); the chart slices its data to the window before scaling. Handles are
  ≥44px touch targets under `pointer: coarse`.
- **Motion:** CSS keyframes in `core/motion.module.css`, **entirely inside** `@media (prefers-reduced-motion:
  no-preference)`: line-draw via `stroke-dasharray`/`stroke-dashoffset`, bar-grow via `transform: scaleY`, area-reveal
  via a clip. Marks get a `data-animate` attribute / class; when reduced-motion is set, no animation runs and the
  static frame is correct. Any JS timing uses `useSignalEffect`, never `useEffect`.

**Tech Stack:** `core/chart-frame.tsx`, new `core/sync.ts` + `chrome/brush.tsx` + `core/motion.module.css`; signals
from `@cascivo/core`. No motion dependency. No `useState`/`useEffect`.

### Task 1: Click / select events

**Files:** `core/chart-frame.tsx`, the charts

- [ ] **Step 1:** Add `onSelect?: (point: ChartPoint) => void` to `ChartFrameProps`; in the focusable layer add
      `onClick` → `if (focusedIndex.value != null) onSelect(points[focusedIndex.value])` and handle `Enter`/`Space`
      in the existing `onKeyDown` the same way. Set `cursor: pointer` when `onSelect` is present.
- [ ] **Step 2:** Each chart adds `onDatumClick?: (datum: Datum, series: Series) => void`; it maps the frame's
      `ChartPoint` (via `seriesId` + the index encoded in `point.id`, as the tooltip formatter already does in
      `line-chart.tsx`) back to the source `datum`/`series` and calls the user callback.
- [ ] **Step 3:** Keep it keyboard-reachable (Enter/Space) and announced — no mouse-only affordance.

### Task 2: `syncId` cross-chart linking

**Files:** `core/sync.ts`, `core/chart-frame.tsx`

- [ ] **Step 1:** `sync.ts`: `getSyncSignal(id)` returns a module-level `Signal<number | null>` per id (lazily
      created, ref-counted); `releaseSyncSignal(id)` decrements. No globals beyond the `Map`.
- [ ] **Step 2:** `ChartFrameProps` gains `syncId?: string`. In a `useSignalEffect`, when local `focusedIndex`
      changes, write it to the shared signal; in another effect, when the shared signal changes and differs from
      local, mirror it into `focusedIndex` (guard against feedback loops). Subscribe/release around the component
      lifetime using `useSignalEffect` cleanup (not `useEffect`).
- [ ] **Step 3:** `sync.test.ts`: two frames with the same `syncId` mirror a focused index; a different id does not;
      releasing cleans the map.

### Task 3: Brush range-selector

**Files:** `chrome/brush.tsx`, the charts (Line/Area/Bar)

- [ ] **Step 1:** `Brush({ length, count, window })` renders a mini track with two handles bound to a
      `Signal<[number, number]>` (start/end indices). Pointer-drag + ArrowLeft/Right (and Shift for the other handle)
      move the handles; handles meet a ≥44px touch target under `pointer: coarse`.
- [ ] **Step 2:** A chart with `brush` renders the `Brush` below the plot and slices its series to `[start, end]`
      before computing scales, so the main plot zooms to the window.
- [ ] **Step 3:** `aria`: the brush is a labelled slider group ("Showing items N–M of T"); fully keyboard-operable.
- [ ] **Step 4:** `brush.test.tsx`: dragging/arrowing narrows the window and the chart re-slices; defaults to the full
      range.

### Task 4: Reduced-motion-gated animation

**Files:** `core/motion.module.css`, `core/chart-frame.tsx`, the charts

- [ ] **Step 1:** `motion.module.css`: define `line-draw` (`stroke-dashoffset` → 0), `bar-grow` (`transform: scaleY(0)`
      → 1, `transform-origin: bottom`), and `area-reveal` keyframes — **all** rules nested inside
      `@media (prefers-reduced-motion: no-preference)`; outside that block, no animation (static).
- [ ] **Step 2:** Charts tag marks with the motion class (e.g. `data-animate`); the static rendered geometry is the
      end state, so reduced-motion users see the final frame with zero motion.
- [ ] **Step 3:** No infinite/looping animation; no layout shift; durations short (≤400ms). Update animation (data
      change) is a transition on the same properties, also gated.
- [ ] **Step 4:** Test: with `prefers-reduced-motion: reduce` simulated, marks carry no animating styles and render at
      final geometry (assert the static `d`/transform is correct regardless of motion).
