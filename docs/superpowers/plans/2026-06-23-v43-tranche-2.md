# v43 — Tranche 2: Viewport (Pan/Zoom) + Background

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Render the pane the graph lives in. Ship `<Flow>`/`<FlowCanvas>` — a single CSS-transformed pane
driven by the `viewport` signal — with **drag-to-pan** (reuse `@cascivo/core` `useDraggable`), **wheel/pinch
zoom** (clamped pointer math in `useSignalEffect`), and **fit-view** (the engine's `graphBounds` +
`useResizeObserver` for the container size). Ship `<FlowBackground>` rendering dots / grid / cross **purely in
CSS** (`background-image` gradients scaled with the viewport via `background-size`) — no SVG, no per-cell DOM.

**Architecture:** The pane applies `transform: translate(var(--flow-tx), var(--flow-ty)) scale(var(--flow-scale))`
where the three custom properties derive from the `viewport` signal (T1). Pan uses `useDraggable` on the pane
background, converting the pointer offset (zoom-corrected) into `viewport.x`/`y`. Zoom attaches a `wheel`
listener in `useSignalEffect`, computing a clamped `zoom` around the pointer focal point and writing
`viewport`. `fit-view` reads `graphBounds(nodes)` and the container size (`useResizeObserver`) to compute the
`viewport` that frames the graph. Per CLAUDE.md: `useSignals()` first; no `useState`/`useEffect`; all listeners
in `useSignalEffect` with cleanup.

**Tech Stack:** signal-driven TSX + CSS Modules; `@cascivo/core` (`useDraggable`, `useResizeObserver`,
`useControllableSignal`, `useSignal`/`useComputed`/`useSignalEffect`/`useSignals`, `cn`); the T1 engine
(`useFlow`, `transform.ts`, `geometry.ts`); Vitest + Testing Library.
