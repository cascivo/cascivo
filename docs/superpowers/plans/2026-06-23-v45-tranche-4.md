# v45 — Tranche 4: Docs Icon Gallery Page (`/icons`)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the catalog **browsable in the docs**. Add an `IconsPage` at `/icons` — a searchable, responsive
grid rendered from `icons.catalog.json` (T3), with click-to-copy of the exact
`import { Name } from '@cascivo/icons'` snippet and size/color/theme controls — modeled on the existing
`TokensPage`. Register the route, the side-nav entry, the sitemap, and SEO.

**Architecture:** `TokensPage` is the template: it imports a generated catalog JSON and renders sections with
live previews, all signal-driven. `IconsPage` reads `icons.catalog.json`, renders each entry as a tile
(the icon via its `@cascivo/icons` export or via the catalog's inner-svg, name, copy button), with a
`useSignal`-backed search query filtering by name/keywords/category and `useSignal` size/color controls. Per
CLAUDE.md (docs app is Preact, signals are natively reactive): **no `useState`/`useEffect`**; copy uses
`useClipboard` from `@cascivo/core`.

**Tech Stack:** Preact + `@cascivo/core` signals (`useSignal`, `useComputed`, `useSignals`), `@cascivo/icons`,
`@cascivo/components` (input/card/etc. for the controls), `useClipboard`; the docs router
(`preact-iso`), `nav.ts`/`Layout.tsx`, sitemap + `seo.ts`.
