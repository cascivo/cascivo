# v20 Tranche 1 — Responsive token foundation

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development or
> superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.
>
> Start here. This tranche lays the DNA: a canonical breakpoint scale (as documented values + JS
> constants), clamp()-based fluid type, and coarse-pointer touch-target tokens — all in
> `@cascivo/tokens`. Nothing in this tranche changes a component's look; it only adds the vocabulary the
> later tranches consume. Desktop computed values must not move.
>
> Current-state assumptions — RE-VERIFY before starting:
>
> - `packages/tokens/src/index.css` has spacing/type/etc. but **no** `--cascivo-screen-*`/breakpoint
>   tokens. Control-height tokens exist (~168–170): `--cascivo-control-height-sm: 1.75rem`,
>   `-md: 2.25rem`, `-lg: 2.75rem`.
> - Fixed type tokens `--cascivo-text-xs … -3xl` exist; `clamp()` is **not** used in the token layer
>   (only in `apps/docs/src/app.css`).
> - `packages/tokens/src/functions.css` has `--cascivo-scale(...)` (Chrome 133+, with static fallback).
> - Tokens are exported as **source CSS** (no `dist`) — confirm `packages/tokens/package.json` `exports`.

**Goal:** `@cascivo/tokens` ships (1) a canonical `--cascivo-screen-{sm,md,lg,xl}` scale documented in
one place, (2) matching JS constants for `matchMedia`/Playwright, (3) clamp()-based fluid type tokens with
the fixed values preserved as fallback, and (4) coarse-pointer touch-target tokens. A snapshot proves no
desktop computed value changed.

**Architecture:** `packages/tokens/src/index.css`, new `packages/tokens/src/screens.ts`,
`packages/tokens/package.json` (export the new entry), optionally `functions.css`. No component changes.

**Tech stack:** unchanged. clamp() is universal (no `@function` needed for the fluid path, but if a
`@function`/`if()` form is added it MUST have a static fallback line — `fallback:check`).

### Task 1: Canonical breakpoint scale (CSS doc-block)

**Files:** Modify `packages/tokens/src/index.css`

- [ ] **Step 1:** Add a clearly-commented block documenting the canonical scale as the single source of
      truth. Because `@media`/`@container` cannot read `var()`, these are **documentation + custom
      properties for JS/`calc` use only**, not for media conditions:
      `--cascivo-screen-sm: 30rem; --cascivo-screen-md: 40rem; --cascivo-screen-lg: 64rem;
      --cascivo-screen-xl: 80rem;`. Re-verify `40rem` and `64rem` are the values already load-bearing in
      `columns/grid/header` and `app-shell` respectively before locking the scale.
- [ ] **Step 2:** Add a comment header above the block stating the rule: "Breakpoints are a lint-guarded
      constant. Use these exact values in `@media`/`@container`; do not invent off-scale widths.
      `@media`/`@container` cannot read these custom properties — copy the value." Point to
      `scripts/checks/breakpoint.test.ts` (added in T2).

### Task 2: JS breakpoint constants

**Files:** Create `packages/tokens/src/screens.ts`; modify `packages/tokens/package.json`

- [ ] **Step 1:** Create `screens.ts` exporting a typed, frozen object — both rem strings and px numbers,
      since `matchMedia` needs px and Playwright needs numbers:
      `export const SCREEN = { sm: { rem: '30rem', px: 480 }, md: { rem: '40rem', px: 640 },
      lg: { rem: '64rem', px: 1024 }, xl: { rem: '80rem', px: 1280 } } as const`. Add a helper
      `minWidth(bp)` / `maxWidth(bp)` returning the media-query string for `matchMedia` reuse.
- [ ] **Step 2:** Export `screens.ts` from `@cascivo/tokens` (add to `package.json` `exports` — e.g.
      `"./screens"` — mirroring how the package already exports its CSS; keep it source, no `dist`).
      Confirm no app needs a new vite alias (tokens resolve to source).
- [ ] **Step 3:** Add a unit test asserting `SCREEN` rem/px pairs agree (e.g. `lg.px === 1024` and
      `lg.rem === '64rem'` at 16px root) so the two representations can't drift.

### Task 3: clamp()-based fluid type tokens

**Files:** Modify `packages/tokens/src/index.css`

- [ ] **Step 1:** For the display/heading sizes (re-verify which exist: likely `--cascivo-text-2xl`,
      `-3xl`, and any `-4xl`), add fluid variants using `clamp(min, vw-based-preferred, max)` where `max`
      equals the **current fixed value** so the desktop ceiling is unchanged. Name them
      `--cascivo-text-2xl-fluid` etc. (additive — do NOT remove the fixed tokens; they are the fallback
      and remain in use by components that want fixed sizing).
- [ ] **Step 2:** Choose `min` ≈ one step down and a `vw` coefficient so the value reaches `max` around
      the `md`–`lg` range. Document the chosen formula in a comment. Keep it conservative — this is type
      that scales, not type that shrinks to unreadable on phones (respect a sensible `min`).
- [ ] **Step 3 (optional, only if trivially clean):** Add `--cascivo-space-fluid-*` for one or two
      section-gap tokens using the same clamp() approach. Skip if it complicates more than it helps —
      fluid space is explicitly optional in the roadmap.

### Task 4: Coarse-pointer touch-target tokens

**Files:** Modify `packages/tokens/src/index.css`

- [ ] **Step 1:** Add a comfortable touch-target token: `--cascivo-target-min-coarse: 2.75rem` (44px),

- [ ] **Step 2:** Confirm `scripts/checks/target-size.test.ts` still passes unchanged (it asserts the

## Tranche exit criteria

1. `index.css` documents the `--cascivo-screen-{sm,md,lg,xl}` scale with the "lint-guarded constant" rule
   and the "cannot read in media queries" caveat.
2. `@cascivo/tokens` exports `SCREEN` (rem + px) and `minWidth/maxWidth` helpers; a test guards rem/px
   agreement.
3. Fluid type tokens exist as additive `clamp()` variants whose `max` equals the prior fixed value; fixed
   tokens are untouched.
4. `--cascivo-target-min-coarse` (44px) exists; `target-size.test.ts` still passes.
5. A snapshot/test confirms no desktop computed value regressed. No component look changed in this tranche.
6. No `useEffect`/`useState` introduced; any `@function`/`if()` form added has a static fallback.
