**Goal:** Replace the confusing horizontal `BarChart` in the "Same app, same axe run, three libraries" section (`#axe`) with a pass/fail scorecard: three status cards (cascivo, shadcn/ui, Carbon) that read PASS at 0 violations and FAIL with the violation count + failing rule names. The data is `[0, 0, 1]` — a bar chart collapses two of three bars to invisibility; a scorecard makes the comparison legible and surfaces Carbon's single `label` failure.

**Architecture:** Pure presentational change in `AxeComparison.tsx`. The `AXE` data source, the rule list, and the disclosure text are unchanged. No chart primitive; cards are plain markup + tokens. Removing the `BarChart` import drops `@cascivo/charts` from this file.

**Tech Stack:** React, existing `AXE` data (`./data`), CSS in `accessibility.css` (or `landing.css` if that's where the page's styles live — match the existing import).

- [ ] **Step 3: Render a scorecard grid** in place of the chart. One card per library:

  ```tsx
  <div className="axe-scorecard">
    {LIBS.map((lib) => {
      const v = axe[lib].violations
      const pass = v === 0
      return (
        <div key={lib} className="axe-card" data-status={pass ? 'pass' : 'fail'}>
          <div className="axe-card__head">
            <span className="axe-card__lib">{LIB_LABELS[lib]}</span>
            <span className="axe-card__badge">{pass ? 'PASS' : 'FAIL'}</span>
          </div>
          <span className="axe-card__count">
            {v} {v === 1 ? 'violation' : 'violations'}
          </span>
          <span className="axe-card__rules">
            {axe[lib].rules.length > 0 ? axe[lib].rules.join(', ') : 'WCAG 2.2 AA — clean'}
          </span>
        </div>
      )
    })}
  </div>
  ```

  Use a real badge/role pattern that conveys status to assistive tech (don't rely on colour alone — the "PASS"/"FAIL" text carries the meaning).
