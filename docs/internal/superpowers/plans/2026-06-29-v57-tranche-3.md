# v57 — Tranche 3: Documentation Depth — Descriptions, Prop Search, API Reference

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix the *true* half of the "sparse documentation" complaint. Prop tables and Cmd+K search already render
(`PropsTable.tsx`, `marketing/search/`), so the report's "no prop table / no search" is inaccurate (C-3) — but
**~34% of components have zero prop descriptions** (~43% partial) and search indexes only name+description. Backfill
descriptions to 100% with a guard, index props/variants/sizes in search, and add a central API reference, so a
developer never again reads source to learn what a prop does (M-3, Master Decision 4).

**Architecture:** metadata backfill + a guard + a search-index extension + one new docs page.
- **Backfill**: fill `PropMeta.description` (`packages/core/src/types.ts:15`, optional) for every prop across every
  `*.meta.ts`. Source the text from each component's TS types, JSDoc, and existing examples — concise, behavioral
  ("what it does," not "the X prop").
- **Guard**: `docs:coverage` (`scripts/registry/docs-coverage.ts`) fails if any component prop lacks a `description`,
  so coverage cannot regress.
- **Search**: extend `apps/site/src/marketing/search/buildIndex.ts` to index `props[].name`, `variants`, `sizes`
  (deep-linking to the component page), plus a small synonym map so `Flex`/`Box`/`HStack`/`VStack` resolve to `Stack`
  and friends (feeds T5's discoverability fix too).
- **API reference**: `apps/site/src/pages/ApiReferencePage.tsx` at `/docs/api` — a single manifest-generated table of
  every component × its props/variants/sizes, filterable, so "which components take `loading`?" is answerable in one
  page.

**Tech Stack:** `*.meta.ts` manifests, `scripts/registry/`, `apps/site` (Preact — signals natively reactive, no
`useSignals()` ceremony), `@cascivo/i18n` for the new page's chrome. No new runtime dependency.
