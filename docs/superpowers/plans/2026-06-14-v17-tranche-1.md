**Goal:** A `/guides` route renders a `GuidesPage` with a working hero, a "Guides" nav link that
highlights when active, a shared `guides/data.ts` content module, a `landing.css` section shell, and a
mobile-harness entry covering the new page. The page renders with zero horizontal overflow at
320–414px even with only the hero present.

**Architecture:** New `apps/landing/src/pages/GuidesPage.tsx` + `apps/landing/src/pages/guides/`
folder (`GuidesHero.tsx`, `data.ts`). Edits to `App.tsx`, `Header.tsx`, `landing.css`,
`test/mobile.spec.ts`. No package changes.

**Tech stack:** unchanged. Mobile-first CSS. `useSignals()` only if a component reads a signal.

## Tranche exit criteria

1. Navigating to `/guides` renders `GuidesPage` with the hero; `document.title` is "Guides — cascivo".
2. A "Guides" link appears in the desktop header nav AND the mobile drawer; it highlights on `/guides`.
3. `guides/data.ts` exports the hero content as a typed module; the hero reads from it.
4. The page renders with zero horizontal overflow at 320/375/390/414 (mobile spec passes for `/guides`).
5. No `useState`/`useEffect`/`useContext` anywhere in the new files; `useSignals()` only if a signal is
   actually read.
6. No hardcoded color values in the new CSS (all token-based).
