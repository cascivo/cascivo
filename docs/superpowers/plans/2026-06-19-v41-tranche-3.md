# v41 — Tranche 3: `@cascivo/core` Interaction Hooks

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Adopt the user-facing `@heroui/use-*` hooks cascivo's core lacks — **`useDisclosure`**,
**`useInfiniteScroll`**, **`useDraggable`** — as pure, signal-driven additions to `@cascivo/core`. These are
the DX gap the HeroUI study found: HeroUI ships them as standalone packages; cascivo currently re-implements
open/close per-component and has no infinite-scroll or drag primitive. Build all three from
`useSignal`/`useSignalEffect`/`useRef` (the `clipboard.ts` template) — **no `useState`/`useEffect`** — export
them from `index.ts`, and unit-test each deterministically.

**Architecture:** Follow `packages/core/src/clipboard.ts` for the hook shape (`'use client'`, signal state,
`useRef` for timers/observers, `useSignalEffect` cleanup, typed `Options`/`Return` interfaces) and
`packages/core/src/controllable.ts` for the controllable-signal pattern (so `useDisclosure` accepts an
optional controlled `isOpen`). SSR/no-DOM guarded throughout.

**Tech Stack:** signal hooks only — `@cascivo/core` `useSignal`/`useSignalEffect`/`ReadonlySignal` + `useRef`;
Vitest + Testing Library (`renderHook`, mocked `IntersectionObserver`, synthetic pointer events).

### Task 2: `useDisclosure`

**Files:** `packages/core/src/disclosure.ts`, `disclosure.test.ts`

- [ ] **Step 1:** Signature: `useDisclosure(options?: { defaultOpen?: boolean; isOpen?: boolean; onOpenChange?: (open: boolean) => void })`
      returning `{ isOpen: ReadonlySignal<boolean>, open(): void, close(): void, toggle(): void }`. Back the state
      with `useControllableSignal` (controlled when `isOpen` is passed, else internal, seeded from `defaultOpen`).
- [ ] **Step 2:** `open`/`close`/`toggle` set the signal and call `onOpenChange` through a synced ref
      (`onOpenChangeRef.current = onOpenChange` during render; read inside the setter). No `useState`/`useEffect`.
- [ ] **Step 3:** Tests: default closed; `open`/`close`/`toggle` flip `isOpen.value`; controlled `isOpen` prop is
      respected and `onOpenChange` fires; passing a new controlled value updates the read.

### Task 3: `useInfiniteScroll`

**Files:** `packages/core/src/infinite-scroll.ts`, `infinite-scroll.test.ts`

- [ ] **Step 1:** Signature: `useInfiniteScroll(options: { hasMore: boolean; onLoadMore: () => void; isEnabled?: boolean; rootMargin?: string })`
      returning `{ sentinelRef: RefObject<HTMLElement>, rootRef?: RefObject<HTMLElement> }` (a sentinel element the
      caller places at the end of the list; optional scroll-root).
- [ ] **Step 2:** In a `useSignalEffect`: if disabled / no DOM (`typeof IntersectionObserver === 'undefined'`) or
      no sentinel, return. Else create an `IntersectionObserver` (root = `rootRef.current ?? null`, `rootMargin`)
      that calls `onLoadMore` (via a synced ref) when the sentinel intersects **and** `hasMore`. Observe the
      sentinel; **disconnect in the effect's cleanup**. Re-run when `hasMore`/`isEnabled` change (read them in the
      effect via synced refs/signals). No `useEffect`.
- [ ] **Step 3:** Tests: mock `IntersectionObserver`; firing an intersect entry while `hasMore` calls
      `onLoadMore`; while `!hasMore` it does not; disabling disconnects the observer (deterministic — drive the
      mock, no real scroll/timers).

### Task 4: `useDraggable`

**Files:** `packages/core/src/draggable.ts`, `draggable.test.ts`

- [ ] **Step 1:** Signature: `useDraggable(options?: { isDisabled?: boolean; onDragEnd?: (offset: { x: number; y: number }) => void; axis?: 'x'|'y'|'both' })`
      returning `{ handleRef, targetRef, offset: ReadonlySignal<{ x: number; y: number }>, isDragging: ReadonlySignal<boolean>, reset(): void }`.
      The caller applies `offset` as a CSS `translate` on the target (no animation library).
- [ ] **Step 2:** In a `useSignalEffect`: attach `pointerdown` on `handleRef.current` (guard: DOM present, not
      disabled). On pointerdown record the start point + capture the pointer; on `pointermove` update
      `offset.value` (respecting `axis`) and set `isDragging`; on `pointerup`/`pointercancel` release capture, clear
      `isDragging`, and call `onDragEnd` (synced ref). **Remove all listeners in the cleanup.** No `useEffect`.
- [ ] **Step 3:** Respect motion preferences: the hook itself does no animation (it sets a transform offset); any
      *settle/snap-back* transition is the consumer's CSS, disabled under `prefers-reduced-motion`. `reset()` zeroes
      the offset.
- [ ] **Step 4:** Tests: dispatch synthetic `pointerdown`→`pointermove`→`pointerup` on the handle; assert
      `offset.value` tracks the delta (respecting `axis`), `isDragging` toggles, `onDragEnd` fires with the final
      offset, `reset()` zeroes it, and `isDisabled` attaches no listeners (deterministic — synthetic events, no
      real drag).
