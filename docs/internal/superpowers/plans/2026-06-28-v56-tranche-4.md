**Goal:** Address the feedback's Bottleneck 3 ("context & scope isolation for large-scale teams") with the *smallest*
honest fix: a **disposable signal scope** that owns a set of signals/computeds/effects and tears them all down in one
call, so switching org/workspace/repo-group cleans up without leaks or stale subscriptions â€” instead of accumulating
state in a single global signal root (M-4, Master Decision 4).

This is **not** a state-management library, a store, or a `Context` (banned by CLAUDE.md). It is a thin ownership
wrapper over `@cascivo/core` signals/effects.

**Architecture:** one module in `@cascivo/core`.
- **`createScope()`** â†’ `{ signal, computed, effect, dispose }`. `scope.signal(initial)` and `scope.computed(fn)` are
  thin pass-throughs to the existing `signal`/`computed` that **register** the created instance with the scope.
  `scope.effect(fn)` wraps `effect` and stores its disposer. `dispose()` runs every stored effect-disposer and drops
  references to the owned signals/computeds, so a workspace switch (`scope.dispose()` then `createScope()` for the new
  workspace) leaves nothing running. `dispose()` is idempotent.
- **`useScope()`** â†’ creates a scope once (`useRef`-stable) and disposes it on unmount via `useSignalEffect` cleanup,
  binding a scope's lifetime to a component (e.g. a per-workspace panel).

The value proposition over a bare global signal: **all** of a workspace's derived state and effects are disposed
together, deterministically, when the user leaves it â€” no orphaned `effect` subscriptions firing against unmounted UI,
no manual teardown bookkeeping.

**Tech Stack:** `packages/core/src/` â€” `signal`/`computed`/`effect` from `./signals.ts`, `useRef`,
`useSignalEffect`. No new dependency.
