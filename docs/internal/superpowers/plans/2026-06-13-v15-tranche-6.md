> Landing mobile (T5) is done; bring the docs to parity. The docs app is **Preact** (signals are
> natively reactive there — no `useSignals()` needed), but `useEffect` is still banned; DOM side
> effects use `useSignalEffect`.
>
> Current-state assumptions — RE-VERIFY before starting:
>
> - `apps/docs/src/app.css` (~302 lines, global), **no `@media`** in it. Layout via `@cascivo/layouts`
>   AppShell (`packages/layouts/src/app-shell/app-shell.module.css`, single `@media (max-width: 64rem)`).
> - `.doc-page` fixed `max-width: 760px` (`app.css:72`), no <375px handling. `.page` padding
>   `--cascivo-space-6`. Tables/code rely on `overflow-x:auto`.
> - Pages in `apps/docs/src/pages/*` (Home, ComponentPage, ChartsPage, PlaygroundPage, Benchmarks,
>   LayoutsPage, DirectoryPage, ContextExplorerPage, WhyCascivoPage, perf tables, AiPage). Layout in
>   `apps/docs/src/Layout.tsx`. The AppShell nav is shared — changing it affects every page.

**Goal:** The docs render world-class and with **zero horizontal overflow** at 320/375/390/414px:
off-canvas nav with focus trap, fluid type, responsive tables/code/charts, container-query sections,
and a small-viewport AppShell pass. Verified by Playwright + overflow assertion in the PR.

**Architecture:** Mobile-first additions to `app.css` + the shared AppShell module; reuse the T5
nav pattern (drawer/scrim/focus-trap) where the AppShell SideNav doesn't already cover it. Apply the
same fluid type scale + breakpoints established in T5 for consistency across both apps.

**Tech stack:** modern CSS, Preact signals (no `useSignals()` needed in Preact; no `useEffect`),
Playwright.
