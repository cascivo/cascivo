# v52 — Tranche 5: Composable Primitives Toolkit (the visx story)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make cascivo's engine what visx *is* — a **documented, composable toolkit** of primitives you can assemble
into a custom chart (V-5/V-6) — without inverting the library away from batteries-included components (Decision 1).
cascivo already *exports* scales/shapes/stats and (after T1–T3) gradients/patterns/curves/glyphs/brush/voronoi, but
they're undocumented and not positioned as a "build your own chart" surface. Also fixes V-6: a wrapping/measured
`Text` primitive (`@visx/text`) so axis/labels stop hand-rolling overflow. Lens: **visx**.

**Architecture:** this tranche is mostly **API hardening + documentation** over primitives that now exist. Curate a
public toolkit surface in `index.ts` (scales: `linearScale`/`bandScale`/`logScale`/`timeScale`; shapes: `linePath`/
`areaPath`/`arcPath`/`stackSeries`/`splitDefined` + curves; chrome: `Axis`/`GridLines`/`Glyph`/`ChartDefs`/`Brush`;
engine: `nearest`/`voronoi`/`stats`; the `ChartFrame` shell), each with JSDoc usage. Add `chrome/text.tsx` — a `Text`
that measures and wraps to a max width (canvas-measured or `<tspan>` greedy wrap), reused by `Axis` for long category
labels (replacing the manual `labelEvery` thinning where wrapping reads better). The deliverable's centerpiece is a
docs page: **"Build a custom chart from primitives"** — compose `ChartFrame` + a scale + a shape + an axis + an
annotation into a working chart, proving the escape hatch.

**Tech Stack:** `packages/charts/src/index.ts` (curated exports + JSDoc), `chrome/text.tsx`; `apps/site` docs. No new
dependency; no behavior change to the prebuilt components beyond optional `Text` wrapping in `Axis`.

### Task 1: Wrapping Text primitive

**Files:** `packages/charts/src/chrome/text.tsx`, `chrome/axis.tsx`

- [ ] **Step 1:** `Text({ x, y, width, children, anchor, ... })` greedily wraps the string to `width` (measure via a
      shared offscreen canvas context, cached) emitting `<tspan>` lines; falls back to a single line when `width` is
      omitted. Honor `dominant-baseline`/`text-anchor`.
- [ ] **Step 2:** `Axis` gains an optional `wrap?: number` (max label width); when set, long category labels wrap
      instead of being thinned. Default keeps today's `labelEvery` behavior.
- [ ] **Step 3:** `text.test.tsx`: a long string wraps into multiple `<tspan>`; a short one stays single-line; the
      measured wrap is deterministic in jsdom (mock `measureText`).
