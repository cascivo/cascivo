# v40 — Tranche 3 (OPTIONAL): Cascade View Language (`cvl`) — Compact DSL ↔ `ViewConfig`

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

> **⚠️ This tranche is OPTIONAL and may be cut cleanly.** It does not affect T1/T2. Build it **only** as a
> deliberate positioning bet — see the cost/benefit note below — not by default.

**Goal:** Give cascivo a compact, line-oriented serialization of the render tree — the lesson from OpenUI
Lang, which is up to ~67% smaller than the equivalent JSON. Add `encode(ViewConfig): string` and
`parse(string): { config, errors }` with a **lossless round-trip**, keeping `ViewConfig` JSON canonical.

**Cost/benefit (read before starting):** The token saving is **real but only bites at volume**, and
`@cascivo/render` is today a *showcase* (landing/docs/examples), not a metered production path — so do **not**
justify this on cost. The legitimate justification is **positioning**: a benchmarkable "our generative-UI
format is ~50% cheaper than JSON" headline that upgrades the landing pitch. Weigh that against the permanent
cost: a hand-written DSL + parser + grammar to maintain in lockstep with `ViewConfig` forever. If that bet
isn't clearly worth it, **skip this tranche** — T1+T2 already deliver the core value.

**Architecture:** `ViewConfig`/`ComponentNode` (`component`, `props`, `bind: $data.*`, `events: $actions.*`,
`children: ComponentNode[] | string | TranslationRef`). `cvl` is a 1:1 lossless projection into a
line-oriented DSL inspired by OpenUI Lang's `identifier = Expression` form (e.g. `root = Stack([title, tbl])`
/ `title = Text("Hello")`). Lives as a **submodule** in `packages/render/src/lang/` (exported from
`@cascivo/render`), not a new package. The parser is **partial/invalid-tolerant** (structured errors, never
throws) so untrusted/streamed input degrades gracefully.

**Tech Stack:** Hand-written TypeScript encoder/parser (no Zod, no parser-generator dep); Vitest via vite+.

### Task 1: Pin the `cvl` grammar against `ViewConfig`

**Files:** `packages/render/src/types.ts` (read), `packages/render/src/lang/grammar.ts`

- [ ] **Step 1:** Enumerate every shape that must round-trip: `$schema`/`version`/`view.layout`,
      `view.regions` (named arrays), and `ComponentNode.{component, props, bind, events, children}`; all
      `PropValue` forms (string/number/boolean/null/`TranslationRef`/array/object); `DataRef`/`ActionRef`
      strings; `children` as array | string | `TranslationRef`.
- [ ] **Step 2:** Define the line-oriented grammar (comment-documented) with explicit sigils so encode/parse
      are unambiguous: e.g. `@data.users` ↔ `$data.users`, `@act.openUser` ↔ `$actions.openUser`, `t"key"{…}`
      ↔ `TranslationRef`. Record the mapping table in a comment — it is the contract the T1 prompt's `cvl`
      section documents.

### Task 2: Implement `encode(config): string`

**Files:** `packages/render/src/lang/encode.ts`

- [ ] **Step 1:** Serialize `ViewConfig` to `cvl`: emit `$schema`/`version`/`layout` headers only when
      present, one block per region, each node as a component call with `props`/`bind`/`events`/`children`.
- [ ] **Step 2:** Encode every `PropValue` variant (nested arrays/objects, `TranslationRef`) and both ref
      kinds. Deterministic output (stable key ordering) for reproducible round-trip + benchmark.

### Task 3: Implement `parse(src): { config, errors }`

**Files:** `packages/render/src/lang/parse.ts`

- [ ] **Step 1:** Parse `cvl` back into a `ViewConfig`, reconstructing `bind`/`events`/`children`/`$t` exactly
      as `encode` produced them.
- [ ] **Step 2:** Be partial/invalid-tolerant: return `{ config, errors: { path, message }[] }` for malformed
      or **truncated** input (unknown line shapes, unterminated calls, dangling refs) — **never throw**, so
      untrusted LLM output and any future streaming degrade gracefully. Truncated input yields the valid prefix
      + a "truncated" error.
- [ ] **Step 3:** Export `encode`, `parse`, and lang types from `lang/index.ts` and re-export from
      `packages/render/src/index.ts`.

### Task 4: Lossless round-trip + token benchmark

**Files:** `packages/render/src/lang/roundtrip.test.ts`, `packages/render/src/lang/tokens.test.ts`,
`packages/render/README.md`

- [ ] **Step 1:** Fixtures covering every shape (minimal; with `layout`; multiple regions; nested `children`;
      `bind`/`events`; `TranslationRef` props + children; array/object props) — reuse existing `ViewConfig`
      fixtures from `schema.test.ts`/`validate.test.ts` where possible. Assert `parse(encode(config)).config`
      deep-equals `config`, and `encode` is idempotent.
- [ ] **Step 2:** Compare a token/size proxy for `JSON.stringify(config)` vs `encode(config)`; if a tokenizer
      dep is undesirable, use a documented proxy and state the method. Assert `cvl` is materially smaller
      (target ≥40% on representative views). Record the measured number + the `cvl` syntax (one worked example)
      in `packages/render/README.md`, crediting the OpenUI Lang study.
