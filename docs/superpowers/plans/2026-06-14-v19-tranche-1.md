# v19 Tranche 1 — SEO metadata foundation

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development or
> superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.
>
> Start here. Fix the stale headline numbers, give every route its own head (title + description +
> canonical + og), and add the crawler primitives (`robots.txt`, `sitemap.xml`, JSON-LD). This tranche
> sets up the per-route head model that T3 (prerender) renders into static HTML.
>
> Current-state assumptions — RE-VERIFY before starting:
>
> - `index.html` hardcodes "97+ components" and "five themes" in og:description; real values are
>   **118 components / 10 themes** (`jq '.components | length' registry.json`; `ls packages/themes/src/*.css`).
> - `App.tsx` sets `document.title` per route (~line 67) from a `ROUTES` record but changes no other
>   meta. Routes: `/`, `/accessibility`, `/performance`, `/guides`, `/og`.
> - `index.html` has no canonical, no robots meta, no JSON-LD, no `twitter:title`/`twitter:description`.
> - `apps/landing/public/` has no `robots.txt` or `sitemap.xml`.

**Goal:** `index.html` carries correct, non-stale numbers and a complete base head (canonical, robots,
twitter:title/description, JSON-LD). A small `seo.ts` defines per-route head metadata; `App.tsx` applies
it on navigation (title + description + canonical + og:* + twitter:*). `robots.txt` and `sitemap.xml`
ship in `public/`. A drift check guards the hardcoded counts.

**Architecture:** `index.html`, new `apps/landing/src/seo.ts`, `App.tsx`, `public/robots.txt`,
`public/sitemap.xml`. No package changes.

**Tech stack:** unchanged. If `App.tsx`'s head application reads a signal it needs `useSignals()`;
otherwise plain DOM writes are fine in the existing route-change path (mirror how `document.title` is
already set — do NOT introduce `useEffect`).

## Tranche exit criteria

1. `index.html` shows 118 components / 10 themes; has canonical, robots, twitter:title/description,
   og:image:alt + width/height, and valid `SoftwareApplication`/`WebSite` JSON-LD.
2. Navigating between routes updates title, description, canonical, og:*, twitter:* (observable via
   DevTools Elements while clicking nav links).
3. `robots.txt` and `sitemap.xml` are emitted to `dist/` and list the four public routes.
4. A drift/unit check fails if a hardcoded component/theme count diverges from source.
5. No `useEffect`/`useState`/`useContext` introduced; per-route head runs in the existing route path.
