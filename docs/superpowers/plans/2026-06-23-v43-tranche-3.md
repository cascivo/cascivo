# v43 — Tranche 3: Nodes (Drag, Select, Handles)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Render the nodes of the graph. Ship `<FlowNode>` — an **HTML** box positioned inside the viewport
pane by CSS custom properties (`--flow-x`/`--flow-y`), draggable via `@cascivo/core` `useDraggable`
(zoom-corrected so it tracks the pointer at any zoom), selectable (click → `data-selected`, the visual handled
by CSS), containing **arbitrary children** (any themed cascivo component) — and `<FlowHandle>`, the connection
ports (`source`/`target`, at `top|right|bottom|left`) that edges route to. ≥44px coarse targets,
keyboard-focusable.

**Architecture:** `<FlowNode>` reads its `position` from the node signal (T1 store) and applies
`transform: translate(var(--flow-x), var(--flow-y))`. Dragging wires `useDraggable` on the node; the pointer
offset is divided by `viewport.zoom` (zoom-correction) before writing the node's position via the store's
`updateNode`. Selection is a click handler toggling `data-selected` (CSS draws the ring). `<FlowHandle>` renders
a small absolutely-positioned port; the engine's `handleAnchor` (T1) reports its `{x,y}` so T4 edges can route
to it. Per CLAUDE.md: `useSignals()` first; no `useState`/`useEffect`; drag listeners live inside
`useDraggable`'s `useSignalEffect`; visual hover/focus/active/selected handled by CSS.

**Tech Stack:** signal-driven TSX + CSS Modules; `@cascivo/core` (`useDraggable`, `useSignal`/`useComputed`/
`useSignalEffect`/`useSignals`, `cn`); the T1 engine (`useFlow`, `geometry.handleAnchor`, `transform`);
`@cascivo/i18n` for any default aria-label; Vitest + Testing Library.
