**Goal:** Two new plain-CSS layout primitives in `packages/layouts/src/`: `Masonry`
(native CSS masonry where supported, multi-columns fallback everywhere, zero JS measurement)
and `AutoGrid` (media-query-free responsive grid via `auto-fill`/`minmax(min(…,100%),1fr)`).

**Architecture:** Pure presentational components in the grid house style — a className, a
couple of private custom properties, CSS does all the work. No state of any kind.

**Tech stack:** unchanged.

- [ ] **Step 1:** Component:

```tsx
'use client'
import { cn } from '@cascade-ui/core'
import type { HTMLAttributes } from 'react'
import styles from './auto-grid.module.css'

type SpaceStep = 1 | 2 | 3 | 4 | 5 | 6 | 8 | 10 | 12

export interface AutoGridProps extends HTMLAttributes<HTMLDivElement> {
  /** Minimum track size before items wrap to fewer columns. */
  min?: string
  gap?: SpaceStep
}

export function AutoGrid({ min = '16rem', gap = 4, className, style, ...props }: AutoGridProps) {
  return (
    <div
      className={cn(styles['auto-grid'], className)}
      style={{
        ['--_min' as string]: min,
        ['--_gap' as string]: `var(--cascade-space-${gap})`,
        ...style,
      }}
      {...props}
    />
  )
}
```

- [ ] **Step 2:** CSS:

```css
@layer cascade.component {
  .auto-grid {
    display: grid;
    grid-template-columns: repeat(auto-fill, minmax(min(var(--_min, 16rem), 100%), 1fr));
    gap: var(--_gap, var(--cascade-space-4));
  }
}
```

No media queries, no container queries needed — intrinsically responsive.

- [ ] **Step 1:** Component — same shape as AutoGrid with `cols?: number` (default 3) and
      `gap?: SpaceStep` (default 4) driving `--_cols`/`--_gap`.

- [ ] **Step 2:** CSS — universal columns fallback first, native masonry as progressive
      enhancement (adjust to Step-0 browser findings; if a browser supports both syntaxes,
      the LAST matching `@supports` block must be the preferred one):

```css
@layer cascade.component {
  /* Fallback: CSS multi-columns. Items flow column-major (top-to-bottom per column). */
  .masonry {
    column-count: var(--_cols, 3);
    column-gap: var(--_gap, var(--cascade-space-4));
  }

  .masonry > * {
    break-inside: avoid;
    margin-block-end: var(--_gap, var(--cascade-space-4));
  }

  /* Native masonry (grid syntax — Firefox lineage). Row-major fill. */
  @supports (grid-template-rows: masonry) {
    .masonry {
      display: grid;
      column-count: initial;
      grid-template-columns: repeat(var(--_cols, 3), minmax(0, 1fr));
      grid-template-rows: masonry;
      gap: var(--_gap, var(--cascade-space-4));
    }

    .masonry > * {
      margin-block-end: 0;
    }
  }

  /* Native masonry (display syntax — WebKit/Chrome lineage). */
  @supports (display: masonry) {
    .masonry {
      display: masonry;
      column-count: initial;
      masonry-template-tracks: repeat(var(--_cols, 3), minmax(0, 1fr));
      gap: var(--_gap, var(--cascade-space-4));
    }

    .masonry > * {
      margin-block-end: 0;
    }
  }
}
```

(Property names inside the `display: masonry` block follow the in-flux spec —
**re-verify against MDN at implementation time** and fix to whatever shipped. If neither
native syntax is in any target browser yet, keep both blocks as inert progressive
enhancement; the fallback is the supported implementation.)

## Tranche exit criteria

   the module CSS (grep-verified).

   fallback elsewhere; both paths eyeballed; ordering caveat in the manifest description.
3. Zero state in either primitive (no hooks, no signals — static components).
4. Logical properties only in new CSS; tokens/`--_*` knobs per house style.
5. Registry gains exactly `layout/auto-grid` + `layout/masonry`; root gate exits 0.
