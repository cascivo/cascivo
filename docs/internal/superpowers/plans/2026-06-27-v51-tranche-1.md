# v51 — Tranche 1: Annotation Layer (ReferenceLine / ReferenceArea / ReferenceDot)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give every cartesian chart the single most-requested dashboard primitive Recharts has and cascivo lacks —
the ability to draw a **target/threshold line**, a **shaded reference band**, and a **labelled marker**. Today
`grep -ri 'reference\|threshold\|annotation' packages/charts/src` hits only Meter internals; no chart can show a goal
line. This tranche adds three chrome components and an opt-in `annotations` prop on Line/Area/Bar/Scatter/Combo. Lens:
**feature parity** (C-F1). Highest-leverage feature in the roadmap.

**Architecture:** annotations are geometry over the chart's existing scales. `chrome/axis.tsx` already takes a `scale`
with `.map(value) → pixel`; the new primitives do the same. A `ReferenceLine` at `y=goal` is a horizontal line at
`yScale.map(goal)`; at `x=date` a vertical line at `xScale.map(date)`. `ReferenceArea` is a `<rect>` between two
mapped bounds; `ReferenceDot` is a `<circle>` + optional label at `(xScale.map(x), yScale.map(y))`. The charts already
build these scales inside the `ChartFrame` render callback (see `line-chart.tsx` — `xScale`/`yScale` are in scope when
children render), so the annotations render in the same `<g transform="translate(margins.left, margins.top)">` group,
**behind the marks** (grid → annotations → marks → axes). Each annotation contributes an SVG `<title>`/`aria` text so
it is announced, and its value joins the chart `<desc>`.

**Tech Stack:** SVG + the existing scales (`linearScale`/`timeScale` from `engine/`); `@cascivo/i18n` for any default
label; tokens `--cascivo-chart-grid` (line), a muted band fill. No engine change. No new dependency.

### Task 1: The reference chrome primitives

**Files:** `packages/charts/src/chrome/reference.tsx`

- [ ] **Step 1:** Define the shared discriminated-union type:
      `type Annotation = { kind: 'line'; axis: 'x' | 'y'; value: number | Date; label?: string; color?: string;
      dash?: boolean } | { kind: 'area'; axis: 'x' | 'y'; from: number | Date; to: number | Date; label?: string;
      color?: string } | { kind: 'dot'; x: number | Date; y: number; label?: string; color?: string }`.
- [ ] **Step 2:** Implement `ReferenceLine` — for `axis: 'y'`, a full-width `<line x1=0 x2=innerW y1=y2=yScale.map(value)>`;
      for `axis: 'x'`, full-height vertical. Default stroke `var(--cascivo-chart-grid)`, `strokeWidth={1.5}`, dashed
      when `dash`. Optional end-label `<text>` (i18n-formatted) clamped inside the plot. Include
      `<title>{label ?? formatted value}</title>` for the a11y tree.
- [ ] **Step 3:** Implement `ReferenceArea` — a `<rect>` spanning `[map(from), map(to)]` on the given axis and the
      full extent on the other, filled with a low-opacity color (default a muted accent token), `aria-hidden` on the
      rect with the label in a `<title>`.
- [ ] **Step 4:** Implement `ReferenceDot` — a `<circle r=4>` at `(xScale.map(x), yScale.map(y))` + optional label;
      `<title>` for a11y.
- [ ] **Step 5:** Export `renderAnnotations(annotations, ctx)` that maps an `Annotation[]` to the right component,
      so each chart adds one line: `{renderAnnotations(annotations, { xScale, yScale, innerW, innerH })}`.

### Task 2: Wire `annotations` into the cartesian charts

**Files:** the five chart `*.tsx`

- [ ] **Step 1:** Add `annotations?: readonly Annotation[]` to each chart's `Props` interface.
- [ ] **Step 2:** Inside each chart's `ChartFrame` children callback, after grid lines and **before** the marks,
      render `{annotations && renderAnnotations(annotations, { xScale, yScale, innerW, innerH })}` in the
      `translate(margins.left, margins.top)` group. (Dots may render after marks so they sit on top — decide per
      primitive; line/area go behind.)
- [ ] **Step 3:** For `plain` charts, skip annotations (micro/inline charts have no room) — gate on `!plain`.
- [ ] **Step 4:** Append annotation summaries to the chart `<desc>`/fallback so screen-reader users get the
      thresholds too (e.g. "Target: 80").
