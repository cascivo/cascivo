**Goal:** A `Slot` component + an `asChild` mechanism in `@cascivo/core` that lets a component render its
single child element while merging its own props/refs onto it — the Radix-style composition primitive
shadcn relies on, implemented on cascivo's existing `composeRefs`/`mergeProps`.

**Architecture:** New `packages/core/src/slot.tsx` + `packages/core/src/slot.test.tsx`; export from
`packages/core/src/index.ts`.

**Tech stack:** React `cloneElement`/`isValidElement` + existing `composeRefs`/`mergeProps`/`cn`. No new
deps. No signals needed (pure composition).

---

### Task 1: Implement `Slot`

**Files:** Create: `packages/core/src/slot.tsx`.

- [ ] **Step 1:** Implement `Slot` — a component that takes `children` (expected to be a single React
      element) plus arbitrary props, and returns `cloneElement(child, mergedProps)` where merged props
      come from `mergeProps(slotProps, childProps)` and the ref is composed via `composeRefs(slotRef,
      childRef)`. `className` merges via `cn`. If `children` is not a single valid element, throw a clear
      error (fail fast — per CLAUDE.md error handling).
- [ ] **Step 2:** Handle ref forwarding: `Slot` is a `forwardRef`; its ref must compose with the child's
      own ref so consumers and the child both receive the DOM node. Reuse `composeRefs`.
- [ ] **Step 3:** Export a tiny helper for the `asChild` consumption pattern, e.g. a `Slottable`/`Slot`
      pairing or document the idiom: a component supporting `asChild` renders `const Comp = asChild ? Slot
      : 'button'` then `<Comp {...props} ref={ref}>{children}</Comp>`. Keep it minimal; do not build a
      full Radix `Slottable` multi-child system unless a consuming component (T5) needs it — start with the
      single-child case.

### Task 2: Test `Slot`

**Files:** Create: `packages/core/src/slot.test.tsx`.

- [ ] **Step 1:** Test prop merge: `Slot` props + child props both land on the rendered element;
      `className` is the union (`cn`); `style` merges.
- [ ] **Step 2:** Test event chaining: an `onClick` on both `Slot` and the child both fire (order:
      slot-then-child or as `mergeProps` defines — assert the actual contract).
- [ ] **Step 3:** Test ref composition: a ref passed to `Slot` and a ref on the child both receive the DOM
      node.
- [ ] **Step 4:** Test the failure path: non-element / multiple children throws the clear error.
