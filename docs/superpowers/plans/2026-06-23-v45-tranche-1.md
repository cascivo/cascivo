# v45 — Tranche 1: Icon Source Pipeline & Generator

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the regenerable pipeline that turns vendored **chromicons** SVG source into cascivo's owned
`createIcon` exports — the foundation every later tranche depends on. Vendor the chromicons SVG source (MIT)
under `packages/icons/svg/`, add `scripts/icons/generate.mjs` that normalizes each SVG and emits
`packages/icons/src/generated.tsx`, attribute Feather + chromicons in a `NOTICE`, and wire the generator into
`pnpm regen`. **No icon is exported to consumers yet (T2 does that); `createIcon`/`IconProps` are unchanged.**

**Architecture:** chromicons are stroked `viewBox="0 0 24 24"` SVGs — the same family as the existing 61
Feather-derived icons. `createIcon(name, children)` already accepts arbitrary SVG children and supplies
`stroke=currentColor`/`stroke-width:2`/`fill:none`/`aria-hidden`, so the generator only needs to extract each
SVG's **inner geometry** (paths/lines/circles), strip presentation attributes that `createIcon` already
provides, map the kebab-case filename to PascalCase, and emit a `createIcon('PascalName', <…inner…/>)` line.
The generator follows the existing generated-artifact precedent (tokens: `catalog:generate` →
`tokens.catalog.json`).

**Tech Stack:** Node ESM script (no new runtime dep for the package); plain string/DOM-free SVG parsing
(regex/lightweight parser) sufficient for chromicons' flat markup; output is TSX consumed by `vp pack`.

- [ ] **Step 1:** Read `create-icon.tsx` — confirm the wrapper supplies `viewBox="0 0 24 24"`, `fill="none"`,

      `aria-label`. These are the attributes the generator must **strip** from source SVGs (keep only inner

- [ ] **Step 2:** Read 10–15 representative inline icons in `index.tsx` to learn the emitted shape

- [ ] **Step 3:** Inspect how an existing generator is wired into `pnpm regen` (find the `regen` script in root

- [ ] **Step 4:** Locate the chromicons SVG source (repo `lifeomic/chromicons.com`, its `src`/icons directory)

### Task 3: Write `scripts/icons/generate.mjs`

**Files:** `scripts/icons/generate.mjs`

- [ ] **Step 1:** Read every `packages/icons/svg/*.svg`. For each: parse out the **inner** markup (everything
      inside `<svg …>…</svg>`), and strip presentation attributes that `createIcon` already supplies
      (`stroke`, `fill` when it's the default, `stroke-width`, `width`, `height`, `class`, `xmlns`). Keep
      geometry attributes (`d`, `points`, `cx/cy/r`, `x1/y1/x2/y2`, `rx`, `transform`). Assert each source is
      `viewBox="0 0 24 24"`; fail loudly on any that isn't (so off-grid icons are caught, not silently
      mis-rendered).
- [ ] **Step 2:** Convert each kebab filename → `PascalCase` (`alert-circle` → `AlertCircle`). Apply the
      **skip/alias list** seeded from Task 1 Step 2: if the PascalName already exists in the hand-authored set,
      **skip** it (existing wins). Make the skip list an explicit, readable constant at the top of the script.
- [ ] **Step 3:** Convert SVG element/attribute names to JSX (kebab attrs → camelCase: `stroke-linecap` →
      `strokeLinecap`, etc.; self-close void elements). Emit, sorted by name, a generated file
      `packages/icons/src/generated.tsx`:
      ```tsx
      // GENERATED by scripts/icons/generate.mjs — do not edit by hand.
      // Geometry: chromicons (MIT). See NOTICE.
      import { createIcon } from './create-icon'
      export const AlertOctagon = createIcon('AlertOctagon', <>…</>)
      // …
      ```
- [ ] **Step 4:** Make it **idempotent** and **deterministic** (stable sort, stable formatting) so re-running
      produces no diff. Print a summary (`N icons generated, M skipped as collisions`).
