# v2 Tranche 7 — Chart Parity Tail Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Milestone 4.3 — the remaining chart catalog (Bubble, Histogram, Boxplot, Heatmap, Treemap, Radar, Combo bar+line, Bullet) plus `ChartMeta` so the registry, MCP, docs, and the JSON renderer know chart props the same way they know component props.

**Demand-driven caveat (from the roadmap):** build the ones real dashboards use, skip novelty until asked. Before starting each chart task, check whether anything actually needs it (issues, user asks, block requirements). It is correct to leave chart tasks unstarted and close the tranche with a "deferred: no demand" note per chart — record that in the final report. ChartMeta (Tasks 1–2) is NOT optional — it unblocks the renderer/MCP/docs integration for the wave-1 charts regardless.

**Architecture:** every chart reuses the Tranche-4 engine (scales, shapes, ChartFrame, chrome, hover-via-effect zero-re-render pattern, fallback table, builtin ramp). New engine pieces are added only where a chart genuinely needs new math (histogram binning, boxplot quartiles, treemap squarify, radar polar grid) — each as pure, tested functions under `src/engine/`.

**Conventions:** as previous tranches. GPG broken — `git -c commit.gpgsign=false commit …`.

### Task 3: Histogram + Boxplot (statistical pair — shared engine math)

**Files:** `packages/charts/src/engine/stats.ts` + tests; chart dirs `histogram/`, `boxplot/`

- [ ] **Engine first (TDD, fast-check):**

```ts
/** Freedman–Diaconis bin width with sane fallbacks; returns inclusive-exclusive bins. */
export function binValues(
  values: readonly number[],
  binCount?: number,
): { x0: number; x1: number; count: number }[]
/** Five-number summary + outliers via 1.5×IQR fences. */
export function boxStats(values: readonly number[]): {
  min: number
  q1: number
  median: number
  q3: number
  max: number
  outliers: number[]
}
```

Properties: bins cover [min, max], counts sum to n, bins contiguous; boxStats quartiles ordered, outliers outside fences, median correct for odd/even n (example tests for known datasets).

- [ ] **Histogram** — props `data: number[]`, `bins?: number`, axes linear, bars via the BarChart rect pattern, tooltip shows bin range + count (`formatNumber`). Fallback table bin/count.
- [ ] **Boxplot** — props `series: { id, label, values: number[] }[]`, band scale across series, box/whisker/outlier-dot rendering from `boxStats`, tooltip = five-number summary. Fallback table per series.
- [ ] Meta files, gallery sections, visual snapshots, exports, regen. Gate + commit per chart.

### Task 5: Heatmap + Treemap + Radar + Bullet

- [ ] **Heatmap** (`heatmap/`) — props `data: { x: string; y: string; value: number }[]`, band scales both axes, cell fill via a sequential ramp interpolated between two chart tokens (`color-mix(in oklab, var(--cascivo-chart-1), var(--cascivo-color-surface) X%)` computed per cell — CSS does the mixing, JS supplies the percentage). Engine addition: `extent(values)` helper (trivial). Tooltip cell value; fallback = the matrix as a table.
- [ ] **Treemap** (`treemap/`) — engine `squarify(values, rect)` (squarified treemap, Bruls et al. — implement with worst-aspect-ratio rule, ~60 lines, property tests: areas proportional to values, tiles tile the rect without overlap within ε). Chart: flat one-level treemap (`data: { id, label, value }[]`); nested hierarchies deferred until asked (note in meta description). Labels clipped via `<clipPath>`.
- [ ] **Radar** (`radar/`) — engine `polarPoints(values, max, cx, cy, r)` mapping n axes to a closed polygon path; polar grid rings via `arcPath` full circles + spoke lines. Props `axes: string[]`, `series: { id, label, values: number[] }[]`, `max?: number`. Keyboard point nav skipped (no meaningful x-order) — fallback table carries the a11y load; document in meta.
- [ ] **Bullet** (`bullet/`) — Meter sibling: measure bar + target tick + qualitative ranges (`ranges: number[]` as background bands). HTML/SVG hybrid like Meter.
- [ ] Meta, gallery, snapshots, exports, regen each. Gate + commit per chart.
