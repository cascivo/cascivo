# v34 — Tranche 5: Dry-run Verification + Gate

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prove the pipeline is correct **without publishing**: audit each Tier‑1 package's tarball contents, dry-run `changeset version`, and run the full CI gate. No feature work — verification only, plus committing any drift the metadata edits produced.

**Architecture:** Two dry-runs that touch nothing on npm: (1) `pnpm pack` / `npm pack --dry-run` builds the exact tarball that would ship and lets us inspect its file list; (2) `changeset version` applied to a throwaway working tree confirms the version + changelog output, then is reverted. Finally the six-step CLAUDE.md gate.

**Tech Stack:** pnpm, npm pack, changesets, vite+ (`vp`).

---

## Files

- Verify only: Tier‑1 package tarballs, `registry.json`, README, changelog output
- Commit only if drift/gate produces changes

### Task 1: Build, then audit each tarball

- [ ] **Step 1:** `pnpm build` → exit 0 (tarball audit needs `dist/` present for the JS packages).
- [ ] **Step 2:** For each Tier‑1 package, produce the tarball file list, e.g.:

  ```bash
  pnpm --filter @cascivo/core exec npm pack --dry-run
  # ...repeat per package, or: pnpm -r --filter './packages/*' exec npm pack --dry-run
  ```

- [ ] **Step 3:** Confirm per package:
  - **JS/TS packages** (`core`, `icons`, `i18n`, `storage`, `react`, `mcp`, `registry`): tarball contains `dist/**` + `package.json` + (auto-included) `README`/`LICENSE`. **No** `src/`, `*.test.*`, `tsconfig*`, `vite.config.*`.
  - **`cascivo` (CLI)**: contains `dist/**` **and** `bin/cascivo.mjs`; the `bin` target resolves.
  - **CSS-only** (`tokens`, `themes`): contains `src/**` CSS exactly as referenced by `exports`; nothing built.
  - Every path referenced by each package's `exports`/`types`/`bin` is present in its tarball.
- [ ] **Step 4:** Confirm `npm` does not warn about a missing `LICENSE`/`license` (T2 added both). Record any leak or missing file and fix it back in T2's metadata (`files` allowlist) before continuing.

### Task 2: Dry-run `changeset version`

- [ ] **Step 1:** On a clean working tree, run `pnpm changeset version`. Inspect the diff:
  - Each Tier‑1 `package.json` `version` bumps `0.0.0` → `0.1.0`.
  - A `CHANGELOG.md` is created per Tier‑1 package from the initial-release summary.
  - Internal `workspace:^` ranges remain `workspace:^` in source (they are rewritten only at publish/pack time) — confirm no unexpected dependency rewrites.
  - **No** Tier‑2/Tier‑3 (private) package is touched.
- [ ] **Step 2:** Confirm the staged changeset `.changeset/initial-release.md` is consumed (deleted) by the version step — expected.
- [ ] **Step 3: Revert the dry-run** so source stays at `0.0.0` and the changeset is restored:

  ```bash
  git checkout -- . && git clean -fd .changeset packages '**/CHANGELOG.md'
  ```

  Verify `git status` is clean and `.changeset/initial-release.md` is back. (The real version bump happens in CI via the Version Packages PR — not in this branch.)

### Task 3: Full CI gate (mirrors CLAUDE.md "Gate Before Committing")

- [ ] **Step 1:** `pnpm exec vp check` → exit 0.
- [ ] **Step 2:** `pnpm build` → exit 0.

- [ ] **Step 4:** `pnpm test` → exit 0.
- [ ] **Step 5:** Drift — `pnpm regen && pnpm exec vp check --fix && git diff --exit-code` → exit 0 (commit regenerated `registry.json`/README if the CLI rename or metadata rippled into them and wasn't already committed in T2/T3).
- [ ] **Step 6:** `pnpm breakpoint:check` → exit 0.
