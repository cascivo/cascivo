# v20 Tranche 5 — Layout shells & blocks

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development or
> superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.
>
> Put the layout layer on the canonical scale and fix the "content vanishes" anti-pattern. AppShell's
> single `64rem` literal becomes the canonical `lg`; its aside content relocates instead of
> `display:none`; the demo blocks adopt container queries so they adapt to their slot, not just the
> viewport. ConsoleApp itself is T6 — this tranche is the shell + the other blocks.
>
> Current-state assumptions — RE-VERIFY before starting:
>
> - `app-shell.module.css:63` `@media (max-width: 64rem)` turns the sidenav into a drawer + scrim and sets
>   the aside `display: none` (line ~72 — content dropped).
> - `shell-state.ts:35` detects desktop via `matchMedia('(min-width: 64rem)')` — replace the literal with
>   `SCREEN.lg`/`minWidth('lg')` from `@cascivo/tokens` (T1).
> - Blocks with little/no own adaptation: `sidebar-app`, `users-table-page`, `settings-form-page`,
>   `stats-cards`, `dashboard-charts`, `empty-dashboard`, `login-page`, `notification-center`,
>   `page-with-breadcrumb` (re-verify which have CSS vs. inline styles).
> - Layout primitives already use `@container (max-width: 40rem)` (columns, grid) — these are on-scale
>   (`md`); leave them, just confirm `breakpoint:check` accepts them.

**Goal:** AppShell sources its breakpoint from the canonical `lg` (no raw `64rem`), and its aside content
**relocates** to a reachable disclosure/drawer below `lg` instead of disappearing. The demo blocks adapt
to small screens via container queries / the canonical scale, with no horizontal overflow and nothing
lost. All enrolled in the sweep.

**Architecture:** `app-shell.module.css`, `app-shell.tsx` (only if a relocation hook/landmark is needed),
`shell-state.ts` (literal → `SCREEN.lg`), and the block CSS/TSX listed above. Use canonical values only.

**Tech stack:** unchanged. `matchMedia` stays in `useSignalEffect`/the existing store; CSS-first for
layout.
