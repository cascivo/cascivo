# v45 — Tranche 3: Icon Catalog Manifest (`icons.catalog.json`) + AI Surface

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Produce the **metadata** the docs gallery (T4), the landing showcase (T5), and AI surfaces consume:
extend the generator to also emit **`icons.catalog.json`** — one entry per icon with `name`, `pascalName`,
`tags`, `keywords`, `category`, and the inner-SVG markup. This is where hugeicons' one genuinely-useful idea —
**rich per-icon search keywords/categories** — is adopted without taking the library. The component
`registry.json` stays untouched (icons are a package, not copy-paste components).

**Architecture:** The generator already parses every SVG (T1). Extend it to derive metadata: `pascalName` from
the filename, `keywords` from the kebab name tokens plus a small synonym map (e.g. `trash` → delete/remove/
bin), and `category` from the chromicons source grouping (All / App-UI / Science / Health) or a
filename-prefix heuristic. Write `icons.catalog.json` deterministically. This mirrors the tokens precedent
(`tokens.catalog.json` consumed by `TokensPage`).

**Tech Stack:** the T1 Node generator (`scripts/icons/generate.mjs`); JSON output; the docs search-index
builder and `llms.txt` generator (existing `scripts/*`).

### Task 2: Emit `icons.catalog.json`

**Files:** `scripts/icons/generate.mjs`, `icons.catalog.json`

- [ ] **Step 1:** For each generated **and** existing icon, build an entry:
      `{ name: 'alert-circle', pascalName: 'AlertCircle', category: 'app-ui', tags: [...], keywords: [...], svg: '<inner markup>' }`.
      Derive `keywords` from the kebab tokens + a small, explicit synonym map; derive `category` from the
      chromicons grouping (fallback `'app-ui'`). Include the existing 61 (category `'general'`/`'app-ui'`) so
      the catalog is complete, not chromicons-only.
- [ ] **Step 2:** Write `icons.catalog.json` sorted by `name`, with a stable shape
      (`{ generatedAt?: omit-or-fixed, count, icons: [...] }`). **Do not** embed a volatile timestamp that
      breaks the drift gate — either omit it or make it deterministic (match how `tokens.catalog.json` handles
      this).
- [ ] **Step 3:** Run `pnpm regen`; confirm `icons.catalog.json` is produced, `count` matches the exported
      icon count from T2, and a second run yields no diff.

### Task 3: Wire icons into search + `llms.txt`

**Files:** docs search-index builder, `llms.txt` generator

- [ ] **Step 1:** Append icon entries to the docs search index (name + keywords → `/icons?q=<name>` or an
      anchor) so an icon is findable from the global search dialog.
- [ ] **Step 2:** Add a short **icons** section/note to `llms.txt` (count + "browse at /icons" + that they are
      `@cascivo/icons` named exports) so agents know the catalog exists. Keep it concise; the source of truth
      is `icons.catalog.json`.
- [ ] **Step 3:** Confirm `registry.json` is **not** modified (icons remain a package, not copy-paste
