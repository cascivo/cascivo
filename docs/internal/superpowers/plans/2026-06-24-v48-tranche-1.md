# v48 — Tranche 1: Editor Publish Fix (unblocks Phase 4)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `@cascivo/editor` publish a usable tarball so `@lifosy/ui`'s CodeMirror → Cascivo `CodeEditor`
migration (Phase 4) can proceed. The published `0.1.1` contains only `package.json`/`README`/`LICENSE` — **no
`dist/`** — so every entry point (`@cascivo/editor`, `@cascivo/editor/styles.css`) resolves to a missing file. The
verified root cause is that **`@cascivo/editor` is missing from the root `build:release` filter** in `package.json`
(the `vp run -F …` list the release workflow builds before `changeset publish`), so its `dist/` is never built in
CI. Fix: (1) add `-F @cascivo/editor` to `build:release`; (2) add a defensive `prepack`/`prepublishOnly` that runs
the package `build` so a stray manual publish can't ship an empty tarball; (3) add a `npm pack --dry-run` assertion
that the three `dist/` entry points are present; (4) bump to `0.1.2` via a changeset so consumers aren't served the
cached-empty `0.1.1`. **Decline** the feedback's suggestion to populate the registry `files[]` — the editor is
npm-installed by design.

**Architecture:** Root `release` = `pnpm build:release && changeset publish`. `build:release` =
`vp run -F @cascivo/core -F @cascivo/i18n -F @cascivo/icons -F @cascivo/charts -F @cascivo/mcp -F @cascivo/react
-F @cascivo/registry -F @cascivo/storage -F @cascivo/themes -F @cascivo/tokens -F cascivo build` — **ten packages,
no editor.** `.github/workflows/release.yml` runs `regen` + `vp check --fix` (no `pnpm build`) then
`changesets/action` with `publish: pnpm run release`. The editor's own `build`
(`vp build && node scripts/flatten-types.mjs && node scripts/check-types-flat.mjs`) is correct but never invoked.
`packages/editor/package.json` declares `"files": ["dist"]`, `types`/`exports` → `./dist/*`, `private: false`, and
has **no** `prepack`. The registry generator (`scripts/registry/generate.ts:127`) marks `type: 'editor'`
`isNpmInstalled` → `install: "@cascivo/editor"`, `files: []` — intentional; not a bug.

**Tech Stack:** root `package.json` scripts (pnpm/`vp run -F`); `npm pack --dry-run` (Node `scripts/check-pack.mjs`,
parse the file list); changesets (`.changeset/*.md`, patch bump); the editor package's existing `build` chain. No
new runtime deps, no chart/editor source changes.
