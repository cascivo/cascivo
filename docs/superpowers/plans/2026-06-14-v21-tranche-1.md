**Goal:** A working `@cascivo/example-kit` (`createMockApi`, `createSimulation`, `seededRandom`,
`<AppShell>`) and a runnable, functional **Cascade Deploy** app that loads projects/deployments via the
mock-api (with loading/empty/error states), persists at least one mutation across reload via
`@cascivo/storage`, and shows a "live" build progressing via `createSimulation`.

**Architecture:** New `apps/examples/kit/` (package `@cascivo/example-kit`, exports source) and
`apps/examples/deploy/` (Vite app mirroring `json-playground`). Per-app `vite.config.ts` aliases all
`dist`-exporting `@cascivo/*` packages **and** `@cascivo/example-kit` to their `src` entries.

**Tech stack:** unchanged. `useSignals()` first for signal reads; `createSimulation`/mock-api
subscriptions use `useSignalEffect`; NO `useState`/`useEffect`/`useContext`. Strings via `@cascivo/i18n`.

### Task 2: `seededRandom` + `createMockApi`

**Files:** Create `apps/examples/kit/src/seeded-random.ts`, `src/mock-api.ts` (+ tests)

- [ ] **Step 1:** `seededRandom(seed: number)` — a tiny deterministic PRNG (e.g. mulberry32) returning
      `{ next(): number, int(min,max), pick<T>(arr), bool(p) }`. Pure, no globals. Unit test: same seed →
      same sequence.
- [ ] **Step 2:** `createMockApi<T>(opts)` — wraps data-producing functions so each returns a `Promise`
      resolving after a configurable latency (default ~300–600ms, jittered via a seeded RNG), with an
      optional `errorRate`/manual `failNext()` to inject a rejection. Export a small helper to flip a
      `latency`/`errorRate` signal at runtime (so apps can add a "simulate slow/failing API" toggle).
- [ ] **Step 3:** Tests: resolves with the given data after latency; `failNext()` rejects once; latency
      is deterministic under a fixed seed (use fake timers, no wall-clock flakiness).

### Task 3: `createSimulation`

**Files:** Create `apps/examples/kit/src/simulation.ts` (+ test)

- [ ] **Step 1:** `createSimulation({ tickMs, seed, onTick })` returning `{ running: Signal<boolean>,
      start(), stop(), toggle() }`. It drives `onTick(rng)` on an interval; consumers mutate their own
      signals inside `onTick`. The interval lives behind a `useSignalEffect`-friendly API — provide a
      `useSimulation(sim)` hook that starts/stops via `useSignalEffect` and cleans up the interval on
      unmount. **No `useEffect`.**
- [ ] **Step 2:** Deterministic: the same seed + tick count produces the same sequence. Pausable: `stop()`
      halts mutations.
- [ ] **Step 3:** Test with fake timers: N ticks produce the expected deterministic mutations; `stop()`
      stops them; the cleanup clears the interval.

5. No `useState`/`useEffect`/`useContext` anywhere; signal reads call `useSignals()` first; strings via
   `@cascivo/i18n`. Mobile-clean at 320–414.
