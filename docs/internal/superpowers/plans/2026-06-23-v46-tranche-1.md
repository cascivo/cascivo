**Goal:** Build the two foundations every later tranche needs. (1) Replace the inline `handleKeyDown` in
`code-editor.tsx` with a small **keymap dispatch** (`Map<chord, command>`), re-expressing the existing
`Tab`/`Shift-Tab` indent as commands with **no behavior change**. (2) Add an **owned signal history** â€”
snapshots of `{ text, selectionStart, selectionEnd }`, coalesced on idle/word boundaries â€” so `Mod-Z`/
`Mod-Shift-Z` restore text **and** selection and history **survives a programmatic `value` write** (the blocker:
native textarea undo is destroyed the moment the app sets `value` for a remote/reset sync). Addresses gap-analysis
finding **#3 (robust undo/redo)** and lays the dispatch spine for save (#5), find (#2), and extensibility (#10).

**Architecture:** The editor is a `<textarea>` over a highlighted `<pre>`, state held in a `text` signal via
`useControllableSignal`. Native undo works while typing but is wiped by any programmatic `value=` set. The owned
history is a signal-held ring buffer: every committed edit pushes a snapshot; `Mod-Z` pops to the previous and
writes it back through the textarea (so native state stays coherent) restoring the caret; a new edit after an
undo truncates the redo tail. Coalescing groups rapid character typing into one undo step (CodeMirror-like) using
a time/Ň‚boundary heuristic â€” no timers that leak; use the edit's nature (single char vs. paste/structural). The
keymap is `Map<string, Command>` where the chord is a normalized string (`'Mod-z'`, `'Shift-Tab'`) and a
`Command` is `(ctx) => boolean` (returns whether it handled the event). `Mod` resolves to `metaKey` on macOS,
`ctrlKey` elsewhere.

**Tech Stack:** owned TypeScript (no new deps); `@cascivo/core` signals (`useSignal`); `useRef` for the textarea;
Vitest + `@testing-library/user-event` for key/typing simulation. No `useState`/`useEffect`.
