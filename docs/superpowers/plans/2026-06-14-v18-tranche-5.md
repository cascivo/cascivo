**Goal:** Generate and verify the 10 Tier-1 parity components, each built on the v18 primitives, each
passing the component checklist and exported from `@cascivo/react`. Regenerate `registry.json` so the
catalog reflects them.

**Architecture:** Factory output → `packages/components/src/<name>/` (`.tsx` + `.module.css` + `.meta.ts`
+ `.test.tsx`) → exported from `packages/react/src/index.ts` → `pnpm regen` updates `registry.json` +
context/catalog/parity.

**Tech stack:** unchanged. Components consume `@cascivo/core` primitives; signal rules apply.

### Task 1: A11y substrate — Label + Field

**Files:** Factory → `packages/components/src/{label,field}/`; modify `packages/react/src/index.ts`.

- [ ] **Step 1:** Generate `Label` — associates with a control via `htmlFor`/`useId`; supports `asChild`
      (`Slot`); a11y per spec. Verify checklist; export.
- [ ] **Step 2:** Generate `Field` — composes Label + control + description + error message with correct
      `aria-describedby`/`aria-invalid` wiring (uses `useId`). This is the substrate Tier-2 form
      components reuse. Verify checklist; export.
- [ ] **Step 3:** Confirm both pass `vp run @cascivo/components#test` and are in `@cascivo/react`.

---

### Task 2: Grouping + roving focus — Button Group + Toggle Group + Icon Button

**Files:** Factory → `packages/components/src/{button-group,toggle-group,icon-button}/`; export.

- [ ] **Step 1:** Generate `Icon Button` — icon-only button, required `aria-label`, sizes/variants
      mirroring `button`; `asChild` via `Slot`. Verify; export.
- [ ] **Step 2:** Generate `Button Group` — visually joined buttons; optional `RovingFocus` for keyboard
      traversal. Verify; export.
- [ ] **Step 3:** Generate `Toggle Group` — single/multiple selection built on `RovingFocus` +
      `useControllableSignal`; `data-state` for pressed (CSS-driven visuals). Verify; export.

### Task 3: Feedback + overlay-lite — Inline Loading + Notification

**Files:** Factory → `packages/components/src/{inline-loading,notification}/`; export.

- [ ] **Step 1:** Generate `Inline Loading` — Carbon-style status (`active`/`finished`/`error`) with

      export.
- [ ] **Step 2:** Generate `Notification` — inline + actionable variants (the toast variant already
      exists as `toast`; do NOT duplicate it — Notification is the static/inline/actionable surface).
