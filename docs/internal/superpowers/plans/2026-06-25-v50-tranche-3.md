**Goal:** Make every interactive header control meet the 44px touch-target minimum on touch devices. Four header
controls are `2.25rem` (36px) with no `pointer: coarse` override, undercutting the WCAG 2.2 AA target-size rule (SC
2.5.8) the marketing surface itself advertises. Add a coarse floor; leave desktop density untouched. Lens:
**architect + business** (it's a conformance claim the site prints).

**Architecture:** `.header-theme-cycle` + `.header-peek-toggle` (`landing.css:3409`, `2.25rem` square),
`.header-icon-link` (`:3445`, `2.25rem` square), `.header-search-btn` (`:3469`, `block-size: 2.25rem`, padded inline)
ŌĆö none has a `@media (pointer: coarse)` rule. The project's established pattern is `.btn` (`:2836`):
`@media (pointer: coarse) { min-block-size: var(--cascivo-target-min-coarse, 2.75rem) }`. The hamburger
`.mobile-nav-toggle` is already `44px`.

**Tech Stack:** `landing.css` only; token `--cascivo-target-min-coarse` (fallback `2.75rem` = 44px). No markup/JS.
