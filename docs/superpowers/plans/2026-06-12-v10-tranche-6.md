**Goal:** Ship the six typography components (`heading`, `text`, `code`, `blockquote`,
`list`, `prose`) — cascade's first typography set, all presentational (no signals, no FSMs,
no i18n chrome; children only) — queue the eight v11-candidate specs to the dark factory
backlog, regenerate every derived artifact, and walk the full ROADMAP-V10 Definition of
Done with evidence.

**Architecture:** Six new directories under `packages/components/src/`, each the full house
anatomy (tsx + module.css + meta + test) plus a Storybook story, an exports-map entry, and
a react re-export. Heading decouples semantics (`level` → `h1`–`h6`) from visuals (`size`,
derived from `level` when absent: 1→2xl, 2→xl, 3→lg, 4→md, 5→sm, 6→sm — master plan risk
11). Prose styles descendant HTML it doesn't control via descendant selectors in its module
CSS (the Bulma "content" pattern). Backlog work is pure JSON editing. The final task is
verification only.

**Tech stack:** unchanged — React 18+, tokens-only CSS with logical properties, vitest +
@testing-library/react, vp (vite+) toolchain.

```tsx
'use client'
import { cn } from '@cascade-ui/core'
import type { HTMLAttributes } from 'react'
import styles from './heading.module.css'

export type HeadingLevel = 1 | 2 | 3 | 4 | 5 | 6
export type HeadingSize = 'sm' | 'md' | 'lg' | 'xl' | '2xl'

/** Visual size derived from semantic level when `size` is not given. */
const DERIVED_SIZE: Record<HeadingLevel, HeadingSize> = {
  1: '2xl',
  2: 'xl',
  3: 'lg',
  4: 'md',
  5: 'sm',
  6: 'sm',
}

export interface HeadingProps extends HTMLAttributes<HTMLHeadingElement> {
  /** Semantic heading level — which hN element renders. */
  level?: HeadingLevel
  /** Visual size, decoupled from level. Defaults from level. */
  size?: HeadingSize
}

export function Heading({ level = 2, size, className, children, ...props }: HeadingProps) {
  const Tag = `h${level}` as const
  return (
    <Tag
      data-size={size ?? DERIVED_SIZE[level]}
      className={cn(styles['heading'], className)}
      {...props}
    >
      {children}
    </Tag>
  )
}
```
