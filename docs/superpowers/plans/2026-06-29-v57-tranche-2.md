# v57 — Tranche 2: Standalone-Install Guard (regression proof)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make "every registry component installs and builds standalone" a **guarded invariant**, not a one-time T1
fix. The five `use-popover` importers were a sample; statically prove the *whole* registry is closed under internal
imports, fix any stragglers, and wire a guard + a clean-room smoke test into regen/CI so the blocker can never reship
(M-2, Master Decision 3).

**Architecture:** two scripts + a CI hook; no product-code change beyond straggler manifests.
- **`deps:check`** (`scripts/registry/deps-check.ts`): for each registry component, statically parse every source file
  in its `files[]` (re-resolved from source), collect **relative** imports (`./`, `../`), map each to the registry
  component that owns the target file, and **fail** if the owning component is neither the component itself nor in its
  transitive `registryDependencies`. Catches "imports a sibling it never declared" — the exact T1 bug class.
- **`standalone-smoke`** (`scripts/registry/standalone-smoke.ts`): for a representative set, copy the component's
  fully-resolved file set (self + transitive registry deps) into a temp dir mirroring the install layout, drop a
  minimal `tsconfig` with the `@cascivo/*` externals stubbed, and run `tsc --noEmit`. Proves the *installed* tree
  actually compiles — the thing the report could not do.

**Tech Stack:** `scripts/registry/` (Node + the existing TS source-walk the generator uses), `tsc`, the registry
data. Reuse `generate.ts`'s file discovery so "what ships" and "what's checked" share one definition. No new runtime
dependency.

### Task 1: The import-graph checker

**Files:** `scripts/registry/deps-check.ts`

- [ ] **Step 1:** Build a map `sourceFile → owningComponentName` by reusing the generator's directory walk (the same
      `isSourceFile` set T1 widened), so every shippable file is attributed to exactly one registry component.
- [ ] **Step 2:** For each component, read each of its source files and extract import specifiers (a lightweight
      regex/`ts` scan for `import ... from '...'` + `export ... from '...'` + dynamic `import('...')`). Keep only
      **relative** specifiers (`./`/`../`); resolve each to an absolute source path (try `.ts`/`.tsx`/`/index.*`).
- [ ] **Step 3:** For each resolved target, find its owning component. **Fail** (non-zero exit, list every violation)
      when the owner is not the importing component itself and not in the importer's transitive
      `registryDependencies`. Build the transitive set by walking `registryDependencies` from `registry.json`.
- [ ] **Step 4:** Ignore imports that resolve to npm packages (`@cascivo/*`, `react`, etc. — non-relative) — those are
      `dependencies`, validated elsewhere. Also flag the inverse smell: a declared `registryDependencies` entry that
      is **never** imported (warn, don't fail — keeps declarations honest).

### Task 2: Run the audit, fix stragglers

**Files:** the audit output → any `*.meta.ts` it implicates

- [ ] **Step 1:** Run `deps:check` against `main` post-T1. The five popover importers must pass (T1 declared them).
      Triage every remaining violation: each is a component importing a sibling/shared module it does not declare.
- [ ] **Step 2:** For each real violation, add the missing `registryDependencies` to the offending manifest (same fix
      shape as T1 Task 3). For any private `.ts` helper that should **not** ship (internal-only, not imported by any
      shipped file), confirm it is genuinely unreferenced by shipped files and exclude it from `files[]` via a naming
      convention (e.g. `*.internal.ts`) handled in `isSourceFile` — document the choice.
- [ ] **Step 3:** Re-run until `deps:check` is clean. Record the full list of components touched in the commit body.

### Task 3: Clean-room install + tsc smoke test

**Files:** `scripts/registry/standalone-smoke.ts`

- [ ] **Step 1:** For a representative set (`shell-header`, `side-nav`, `menu` — all transitive-`popover` consumers —
      plus one leaf like `button`), compute the full resolved file set (self + transitive registry deps), copy the
      source files into a temp dir preserving relative layout (mirroring the CLI's install structure from T1 Task 4).
- [ ] **Step 2:** Write a minimal `tsconfig.json` (strict, matching repo settings) with `paths` aliasing `@cascivo/*`
      to lightweight type stubs (or `skipLibCheck` + ambient module decls) so the smoke test isolates *internal*
      resolution, not npm-dep typing. Run `tsc --noEmit`; fail on any unresolved-module error.
- [ ] **Step 3:** Assert specifically that the `shell-header` tree resolves `../popover/use-popover` (the original
      failure). This is the acceptance test that the report's blocker is gone.
