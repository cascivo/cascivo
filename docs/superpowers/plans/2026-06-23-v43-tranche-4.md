# v43 — Tranche 4: Edges (Paths, Markers, Animation)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Connect the nodes. Ship the **SVG edge layer** and `<FlowEdge>` — an edge that computes its path
from the source→target handle anchors (T3 geometry) using **owned bezier / straight / smoothstep** path math
(no D3), draws an **arrowhead marker**, and supports an optional **HTML label** at the path midpoint. Ship the
**animated** edge variant — the "animate flows" half of the brief — as a CSS `stroke-dashoffset` keyframe
("marching ants"), with an optional `offset-path` particle as progressive enhancement, each with a **static
fallback** and **disabled under `prefers-reduced-motion: reduce`**.

**Architecture:** A single `<svg>` layer sits in the viewport pane (absolutely positioned, `overflow: visible`,
`pointer-events: none` except on edge paths). `engine/path.ts` exposes pure `d`-string builders
(`bezierPath`, `straightPath`, `smoothStepPath`) taking `{ source, target, sourcePosition, targetPosition }`.
`<FlowEdge>` reads the two handle anchors (via the T3 `geometry.handleAnchor` + node positions from the store),
builds the `d`, renders a `<path>` + a reusable `<marker>` arrowhead, and (if `label`) an HTML label positioned
at the computed midpoint. Animation is CSS-only on the path. Per CLAUDE.md: `useSignals()` first; no
`useState`/`useEffect`; animation is progressive enhancement + reduced-motion-safe with static fallbacks.

**Tech Stack:** signal-driven TSX + CSS Modules + SVG; owned path math (no D3-shape); `@cascivo/core`
(`useSignal`/`useComputed`/`useSignals`, `cn`); the T1/T3 engine (`handleAnchor`, store); Vitest + Testing
Library.
