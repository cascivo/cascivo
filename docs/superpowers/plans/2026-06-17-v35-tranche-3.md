**Goal:** Eliminate the `color-contrast` Lighthouse failure on the badge elements in the Deploys table. This tranche is **gated on T1 being deployed** — the contrast failure may be purely a cascading artifact of the MIME bug (theme tokens undefined → transparent/unresolved colors → bad contrast). Run Lighthouse after T1 deploys before making any changes.

**Affected element:** `tbody > tr._row_9q3jn_168 > td > span._badge_4ffzi_2` — multiple rows in the Deploys DataTable section of the `RelayConsole` demo. The badge variants in use are `success`, `secondary`, `destructive`, and `outline` (from `DeploysRegion.tsx:14-18`).

**Token pairs to audit (light theme):**

| Badge variant | Background token                     | Text token                            |
| ------------- | ------------------------------------ | ------------------------------------- |
| `success`     | `--cascivo-color-success-subtle`     | `--cascivo-color-success-content`     |
| `secondary`   | `--cascivo-color-bg-subtle`          | `--cascivo-color-text-subtle`         |
| `destructive` | `--cascivo-color-destructive-subtle` | `--cascivo-color-destructive-content` |
| `outline`     | `transparent`                        | `--cascivo-color-text`                |

WCAG 2.1 AA requires: 4.5:1 for normal text (< 18pt / < 14pt bold), 3:1 for large text.

### Task 3: Fix failing token pair(s)

Two approaches — use whichever achieves 4.5:1 with the smallest change:

**Approach A (preferred): Darken/strengthen the content token in light.css**

If `--cascivo-color-success-content` is too light, darken it. For example, if `success-subtle` is `oklch(0.90 0.08 145)` and `success-content` is `oklch(0.45 0.15 145)`, the contrast may be borderline. Shift `success-content` to `oklch(0.35 0.15 145)` (darker).

- [ ] **Step 1:** Adjust the content token value to achieve ≥ 4.5:1 against the subtle background.
- [ ] **Step 2:** Verify the adjusted color still looks correct in context (green on light green for success, etc.). Run `pnpm exec vp run @cascivo/landing#build` and preview to visual-check.

**Approach B (fallback): Add explicit contrast overrides in badge.module.css**

If adjusting the shared semantic token would break other usages:

- [ ] **Step 1:** Add `@media (prefers-contrast: normal)` overrides (or use `@layer` specificity) in `packages/components/src/badge/badge.module.css` to set explicit color values for the failing variant that meet 4.5:1 against their background.
- [ ] **Step 2:** Keep the override minimal — only the failing variant(s), only the `color` property.
