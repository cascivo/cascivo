# v52 — Tranche 3: Interaction Toolkit — Brush, Zoom/Pan & Voronoi Hover

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add visx's signature: **first-class interaction** (V-1/V-2). A keyboard-operable **Brush** range-selector
(`@visx/brush`; deferred from v51), **zoom/pan** (`@visx/zoom`), and **voronoi-mesh** precise nearest-point hover
(`@visx/voronoi`) for dense scatter/line. Today there's hover + keyboard nav + `onSelect`, but no range-select, no
pan/zoom, and only rectilinear `nearest` hit-testing. Lens: **visx**. This is the heaviest interaction tranche; it
wants T1/T2 settled.

**Architecture:** most of this lands in `core/chart-frame.tsx`, which already owns the focusable layer, `focusedIndex`,
the `aria-live` region, and `nearest()` hit-testing.
- **Brush** (`chrome/brush.tsx`): a strip below the plot with two drag- and keyboard-operable handles bound to a
  `Signal<[startIndex, endIndex]>`; a chart with `brush` slices its data to the window before scaling. Handles meet a
  ≥44px touch target under `pointer: coarse`; the brush is a labelled slider group ("Showing N–M of T").
- **Zoom/pan**: a `transform` (scale + translate) signal in `chart-frame`, driven by wheel (zoom), drag (pan), pinch,
  and keyboard (`+`/`-`/arrows/`0`-to-reset), applied to the marks group; axes re-tick to the visible domain. A reset
  control is always present and keyboard-reachable.
- **Voronoi** (`engine/voronoi.ts`): a dependency-free Delaunay/Fortune-lite tessellation producing, for a set of
  points, the nearest-point regions; `chart-frame` uses it (when `hover="voronoi"`) so a pointer anywhere snaps to the
  truly-nearest mark — far better than rectilinear `nearest` on dense or irregular scatter. The keyboard path is
  unchanged (Arrow traverses points in order).

**Tech Stack:** `core/chart-frame.tsx`, new `chrome/brush.tsx` + `engine/voronoi.ts`; signals from `@cascivo/core`;
`useSignalEffect` for wheel/drag/pointer listeners. No new dependency (voronoi hand-rolled, Decision 4).
