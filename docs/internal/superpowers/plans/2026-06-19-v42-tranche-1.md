# v42 — Tranche 1: `comparison` Component

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the clearest, most distinctive component gap the Web Awesome study found — a **`comparison`**
before/after comparer (Web Awesome's `wa-comparison`). Two stacked, slotted layers (`before`/`after`, e.g.
images) revealed by a draggable divider whose position is driven by cascivo's existing **`useDraggable`**
(v41) applied as a CSS `clip-path`/`inset` reveal — **no animation library**. Keyboard-accessible
(`role="slider"`, Arrow nudge, Home/End), `position` (0–100, controllable), `orientation`
(`horizontal`/`vertical`), reduced-motion-safe. Ship the full kit — `comparison.tsx` + `comparison.module.css`
+ `comparison.meta.ts` + `comparison.test.tsx` — export it from `@cascivo/react`, add it to `_all-metas.ts`,
and let `pnpm regen` add it to `registry.json`.

**Architecture:** Reuse `packages/core/src/draggable.ts` (`useDraggable` → `{ x, y }` signal + handle/target
refs) for the pointer drag; convert the offset to a clamped 0–100 `position` signal via `useComputed`. Apply
the reveal with CSS `clip-path: inset(...)` (or `inset-inline-end`) on the top layer, the divider line/handle
absolutely positioned at `position%`. Add a keydown handler on the handle for Arrow/Home/End. Per CLAUDE.md:
no banned hooks; visual states via CSS; pointer/keyboard side effects via `useSignalEffect` (inside
`useDraggable`) — never `useEffect`; `useSignals()` first.

**Tech Stack:** signal-driven TSX + CSS Modules; `@cascivo/core` (`useDraggable`, `useControllableSignal`,
`useSignal`, `useComputed`, `useSignals`, `cn`); `@cascivo/i18n` for the divider aria-label default; Vitest +
Testing Library.

### Task 2: Implement `comparison.tsx` (signal-driven)

**Files:** `packages/components/src/comparison/comparison.tsx`

- [ ] **Step 1:** Props: `before` + `after` (ReactNode slots, or `slot="before"/"after"` children),
      `position?` (0–100, controllable via `useControllableSignal`, default 50), `onPositionChange?`,
      `orientation?: 'horizontal' | 'vertical'` (default horizontal), `keyboardStep?` (default 5), `label?`
      (a11y), `className`/`...rest`.
- [ ] **Step 2:** `useSignals()` first. `const position = useControllableSignal({ value: props.position, defaultValue: 50, onChange: onPositionChange })`.
      Call `useDraggable` on the handle; in a `useComputed`/handler, convert the pointer offset (relative to the
      container's measured size) into a clamped 0–100 and write it to `position`. Container carries
      `data-orientation` and a `--position` custom property = `${position}%`.
- [ ] **Step 3:** Render: container `<div>` with the `after` layer as the base and the `before` layer clipped
      via `clip-path: inset(...)` keyed off `--position`; an absolutely-positioned divider line + handle
      (`role="slider"`, `aria-valuemin=0`, `aria-valuemax=100`, `aria-valuenow={position}`,
      `aria-label`/`aria-orientation`).
- [ ] **Step 4:** Handle keydown: Arrow keys ±`keyboardStep` (orientation-aware), Home → 0, End → 100,
      PageUp/PageDown ±big step; clamp 0–100. No `useEffect` — the only side effects are `useDraggable`'s pointer
      listeners (in `useSignalEffect`) and the React `onKeyDown` prop.
