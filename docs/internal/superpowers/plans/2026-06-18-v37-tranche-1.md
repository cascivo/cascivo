# v37 — Tranche 1: Unblock — Export Fix + cascade→cascivo Rename

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove the single blocker (#1) that forces every strict-bundler consumer to patch
`@cascivo/react`, and finish the **cascade → cascivo** rename (#2/#5) that caused it — across CSS layer
names, JS/JSDoc brand strings, and package descriptions — then add a guard so it can never regress.

**Architecture:** #1 is a one-line `exports` correction. #2 is a mechanical sweep: `@layer cascade.*`
appears in **119** `packages/components/src/**/*.module.css` files and in every `packages/themes/src/*.css`
(and possibly `packages/tokens/src/*.css`). The rename is breaking for the public CSS layer API, so it is
documented and guarded. Token custom properties are already `--cascivo-*` — only the **layer** names and
prose brand strings change.

**Tech Stack:** vite+ (`vp`); a small Node guard script.

---
