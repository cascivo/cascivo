# v53 — Tranche 3: Data Pipeline — Transforms & Encode

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give cascivo ECharts' `dataset` + `transform` + `encode` data pipeline (E-4): pure, composable helpers to
**filter / sort / aggregate / bin / regress** raw rows and **encode** a 2-D table into the `series`/`data` a chart
consumes. Today the caller must prepare everything by hand. Lens: **ECharts**. These are **opt-in pre-processors**
(Decision 4) — the component contract is unchanged.

**Architecture:** `engine/transform.ts` exports pure `(rows) => rows` (or `rows → aggregated`) functions composable
left-to-right; `engine/dataset.ts` maps a `{ rows, encode }` to chart-ready series. Nothing here touches a component —
a user writes `<BarChart {...encode(aggregate(filter(rows, …), …), {x:'category', y:'total'})} />`. Regression returns
the fitted points (for an overlay line/annotation) and the coefficients/R². All dependency-free (least-squares solved
in-engine, matching the hand-rolled natural-spline math from v52).

**Tech Stack:** pure TypeScript in `engine/`; no React, no signals, no dependency.

### Task 1: Transforms

**Files:** `engine/transform.ts`

- [ ] **Step 1:** `filter(rows, pred)`, `sort(rows, key, dir)`, `aggregate(rows, { groupBy, ops })` where ops =
      `{ field: 'sum'|'mean'|'min'|'max'|'count'|'median' }` → grouped rows. Pure; stable.
- [ ] **Step 2:** `bin(values, { bins?, range? })` → histogram buckets `{ x0, x1, count, values }` (the histogram
      primitive, reusable by the Histogram chart).
- [ ] **Step 3:** `regression(points, { type })` for `linear` | `exponential` | `polynomial(order)` via least-squares;
      return `{ predict(x), points, r2, coefficients }`. Guard degenerate inputs.
- [ ] **Step 4:** `transform.test.ts`: aggregate sums/means a known set; bin partitions correctly (edge values land in
      the right bucket); linear regression recovers a known slope/intercept and R²≈1 on collinear points; degenerate
      inputs don't throw.

### Task 2: Encode (table → series)

**Files:** `engine/dataset.ts`

- [ ] **Step 1:** `encode(rows, mapping)` maps a table to chart inputs: `{ x, y }` → `{ series:[{id,label,data}], x, y }`
      for cartesian charts; a `series` key splits rows into multiple series; a `category`/`value` form → pie/bar data.
- [ ] **Step 2:** Return a spreadable object so `<LineChart {...encode(rows, {x:'date', y:'value', series:'team'})} />`
      Just Works.
- [ ] **Step 3:** `dataset.test.ts`: a flat table encodes into the expected single series; a `series` key splits into
      multiple; missing fields are handled gracefully.
