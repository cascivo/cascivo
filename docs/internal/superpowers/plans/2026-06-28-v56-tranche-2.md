# v56 — Tranche 2: Virtualized `LogViewer` (components)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the missing piece the feedback's Bottleneck 2 actually needs — a **virtualized monospace log/stream
console** that renders a continuous build/deploy log without mounting a DOM node per line. Backed by the T1
`StreamBuffer` (or any `Signal<readonly LogLine[]>`), it windows the DOM, auto-follows the tail, and stays keyboard-
and screen-reader-accessible (M-2, Master Decision 2).

**Architecture:** one component in `packages/components`, mirroring the windowing model `DataTable` already uses.
- **`LogViewer`** props: `lines: ReadonlySignal<readonly LogLine[]> | readonly LogLine[]`, `rowHeight?` (default from
  a token), `overscan?` (default 8), `follow?`/`onFollowChange?` (controlled pin-to-bottom), `ansi?: boolean`,
  `search?: string`, `maxHeight?`, `labels?` (i18n override), plus container HTML attrs.
- **`LogLine`** = `{ id: string | number; text: string; level?: 'info' | 'warn' | 'error' | 'debug' }`. ANSI parsing
  (when `ansi`) splits a line into colored spans via a tiny SGR-16 parser; otherwise `level` drives the row color.
- **Virtualization:** a `useSignal` scroll-top + a `useComputed` visible `[start, end]` range from `rowHeight` +
  viewport height; only `lines.slice(start - overscan, end + overscan)` mount, absolutely positioned inside a spacer
  sized to `total * rowHeight`. Same approach as `data-table.tsx` (windowSize/overscan).
- **Auto-follow:** a `follow` signal. When `true`, a `useSignalEffect` scrolls to the bottom on every `lines` change
  (via the container ref). A scroll handler sets `follow=false` when the user scrolls up, and `true` when they return
  to within one row of the bottom. The whole thing is a 2-state `useMachine` (`following` ↔ `paused`) only if it
  genuinely simplifies — otherwise a single `follow` signal (prefer the signal; no machine for controlled state per
  CLAUDE.md).

**Tech Stack:** `@cascivo/core` (`useSignal`/`useComputed`/`useSignalEffect`/`useRef`-for-DOM, the T1
`StreamBuffer`), `@cascivo/i18n` (strings), CSS Modules in `@layer cascade.component`, monospace + `--cascivo-*`
tokens, logical properties.

### Task 1: Windowed rendering core

**Files:** `log-viewer.tsx`, `log-viewer.module.css`

- [ ] **Step 1:** Read `packages/components/src/data-table/data-table.tsx` virtualization (windowSize/rowHeight/
      overscan/visible-range slice) and mirror its windowing math. Accept `lines` as a signal or a plain array;
      normalize to a `ReadonlySignal` internally.
- [ ] **Step 2:** Render an outer scroll container (`overflow:auto`, `max-block-size`), an inner spacer sized to
      `total * rowHeight`, and only the visible rows (`start-overscan`..`end+overscan`) absolutely positioned at
      `top = index * rowHeight`. Monospace font, `white-space: pre`, horizontal scroll for long lines. Only visible
      rows mount — assert this in tests.
- [ ] **Step 3:** Row coloring: `data-level` attribute drives CSS for `info|warn|error|debug` via semantic
      `--cascivo-color-*` tokens. When `ansi`, parse SGR-16 escape codes into spans (a small pure function +
      `data-ansi-*` classes); fall back to plain text on parse failure (fail soft, never throw on a log line).

### Task 2: Auto-follow, search, copy, a11y, i18n

**Files:** `log-viewer.tsx`, `log-viewer.module.css`

- [ ] **Step 1:** `follow` signal (controllable via `follow`/`onFollowChange` using `useControllableSignal`). A
      `useSignalEffect` scrolls the container to the bottom whenever `lines` changes **and** `follow` is true. A
      scroll handler flips `follow` off on scroll-up and back on return-to-bottom (within one `rowHeight`). A "Jump to
      latest" affordance appears when paused; clicking it re-engages follow and scrolls down.
- [ ] **Step 2:** `search` filters/highlights matching lines (highlight via `<mark>`); show a result count. Keep it
      O(visible) for highlight; filtering may scan `lines` but stays off the render hot path (compute in a
      `useComputed`).
- [ ] **Step 3:** Copy-all button (`useClipboard` from core) copies the full buffer text. All controls ≥44px under
      `@media (pointer: coarse)` via `var(--cascivo-target-min-coarse, 2.75rem)`.
- [ ] **Step 4:** A11y: container `role="log"` `aria-live="polite"` `aria-label` (from i18n); keyboard scrolling works
      natively (focusable container, `tabindex=0`); the follow toggle + copy + search are real buttons/inputs with
      labels. All user-visible strings default from `@cascivo/i18n` (`t(builtin.logViewer.*)`), overridable via
      `labels`. **Never** hardcode English.
- [ ] **Step 5:** Empty state (no lines) renders the i18n `empty` string; a `lineCount` status is exposed for SR via
