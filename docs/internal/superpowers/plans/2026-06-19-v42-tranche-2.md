# v42 — Tranche 2: `qr-code` Component

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the second net-new component the Web Awesome study found — a **`qr-code`** encoder
(`wa-qr-code`). Encode a URL/short string into a scannable **SVG** module matrix, signal-driven and themeable:
`value`, `size`, `errorCorrection` (`L|M|Q|H`), `radius`, `fill`/`background` (default
`currentColor`/`transparent`), and an a11y `label`. Encoding is done by a **minimal MIT QR routine vendored as
owned source** (`qr-code/encode.ts`) — **no runtime npm dependency**, preserving cascivo's no-runtime-deps and
copy-paste-owned model. Ship the full kit + the encoder, export from `@cascivo/react`, add to `_all-metas.ts`,
`pnpm regen`.

**Architecture:** `encode.ts` is a small, pure, dependency-free function `encode(value, ecLevel) → boolean[][]`
(the module matrix). `qr-code.tsx` calls it during render (pure, no side effects) and maps the matrix to SVG
`<rect>`s (or a single `<path>` for fewer nodes), sized by `size`, coloured from `fill`/`background` tokens,
wrapped in an `<svg role="img" aria-label>`. Per CLAUDE.md: no banned hooks (the component is effectively pure
— `useSignals()` only if a prop is read from a signal); CSS for any visual state.

**Tech Stack:** signal-driven TSX + CSS Modules; vendored owned-source encoder (TS, MIT-attributed in a header
comment); `@cascivo/i18n` for the default aria-label; Vitest + Testing Library.

- [ ] **Step 2:** Call `encode(value, errorCorrection)` during render → matrix. Map to SVG: a `viewBox` of
