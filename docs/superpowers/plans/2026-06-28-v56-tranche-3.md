# v56 — Tranche 3: Live Chart Binding + Multi-Axis (charts)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the *only* true chart gap the feedback pointed at — there is no documented way to feed a **live
metrics stream** (poll/SSE) into a chart, and **multi-axis lives only in `ComboChart`**. The engine already decimates
(`engine/decimate.ts`) and time-scales (`engine/scale-time.ts`) and the charts are already signal-driven and zero-dep
(C-2) — so this is a **thin binding helper + a multi-axis generalization + a cookbook**, never an engine rewrite
(M-3, Master Decision 3).

**Architecture:** two small additions inside `@cascivo/charts`.
- **`useStreamSeries`/`bindStream`** (`src/stream/use-stream-series.ts`): wraps a T1-style capped time-series signal
  for chart consumption. `useStreamSeries<P>({ capacity, decimateTo?, source })` returns a `ReadonlySignal<readonly
  P[]>` of the current window. `source` is a subscribe function `(push: (point: P) => void) => () => void` (the seam a
  poll loop or an `EventSource` plugs into). On each `push`, append O(1) (reuse `createStreamBuffer` from
  `@cascivo/core`), and when `decimateTo` is set, run `engine/decimate.ts` so the **rendered** point count is capped
  regardless of ingestion rate. `bindStream` is the non-hook factory form for non-component callers.
- **Multi-axis on Line/Area** (`charts/line-chart`, `charts/area-chart`): promote the existing `ComboChart`
  `secondAxis?: boolean` pattern to a per-series `axis?: 'left' | 'right'` plus a `secondAxis` config (label, format,
  scale). Render a right-hand y-axis + gridlines when any series targets `'right'`. Default unchanged (single left
  axis) — pure progressive addition, no breaking change.

**Tech Stack:** `@cascivo/charts` (`engine/decimate.ts`, `engine/scale-time.ts`, `core/chart-frame.tsx`,
`chrome/grid-lines.tsx`), `@cascivo/core` (`createStreamBuffer` from T1, `useSignalEffect` for the subscription
lifecycle), `@cascivo/i18n` for any axis labels. No new dependency.
