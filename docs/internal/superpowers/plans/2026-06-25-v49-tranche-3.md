# v49 ‚Äî Tranche 3: One Conversion Path + Headline Polish

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the single action you want a visitor to take unmistakable. Today the primary CTA splits ‚Äî Hero says
"Get started" ‚Üí `/guides`; CtaBand says "Start building" ‚Üí `/docs` ‚Äî and three different `npx` commands compete across
Hero/QuickStart/CtaBand. The zero-setup `@cascivo/react` route is hidden behind a `Collapsible`, so a buyer asking
"can I just install it?" may never see it. And the headline's "Fluent in agent." reads idiomatically off. This
tranche unifies the verb + destination, picks one hero command, surfaces the npm route, and tightens the headline.
Lens: **business + designer.**

**Architecture:** `Hero.tsx` renders `<LinkButton href="/guides">Get started</LinkButton>` + a secondary GitHub
button, then `<CopyCommand command="npx cascivo add button" />`. `CtaBand.tsx` renders
`<LinkButton href="/docs">Start building</LinkButton>` + `<CopyCommand command="npx cascivo init" />`. `QuickStart.tsx`
shows `init` ‚Üí `add` ‚Üí import as three numbered steps, then hides the `@cascivo/react` npm install inside
`<Collapsible trigger="Or use components directly from npm ‚Äî no copy needed">`. The headline is in `Hero.tsx:12`.

**Tech Stack:** Preact + signals; existing `LinkButton`/`CopyCommand`/`Collapsible` components; `landing.css`. No new
deps.
