# v43 — Tranche 7: Final Gate (Regen, Drift, CI, Sweep)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prove the `@cascivo/flow` package is fully wired and ships clean. Run `pnpm regen` (the registry
gains the `flow/*` entries with `install: '@cascivo/flow'`), confirm the drift gate is clean, run every check
(`breakpoint:check`, `fallback:check`, `brand:check`), run the full CI gate **including the cold-cache
sequential `pnpm ready:ci`** (which proves the source aliases let docs/landing/storybook build without a prior
full build), and grep-sweep every registration surface to confirm no flow primitive was missed. Flip
`docs/ROADMAP-V43.md` status to shipped.

**Architecture:** No new product code — this is the integration + verification tranche. The one risk this
tranche specifically guards is the **workspace-alias build-ordering bug** called out in CLAUDE.md: because
`@cascivo/flow`'s root export resolves to `./dist/`, the perf/storybook/landing builds (which run **without** a
prior `pnpm build`) will fail to resolve it unless the four source aliases added in T1 are correct.
`pnpm ready:ci` is the canonical reproduction.

**Tech Stack:** `pnpm regen`, `vp check`, `vp run -r build/check/test`, `pnpm ready` / `pnpm ready:ci`,
`breakpoint:check`, `fallback:check`, `brand:check`, `git diff --exit-code`, grep.
