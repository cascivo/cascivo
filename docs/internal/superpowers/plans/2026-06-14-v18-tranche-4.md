> Current-state assumptions — RE-VERIFY before starting:
>
> - T2 (`Slot`) and T3 (`useControllableSignal`, `useMediaQuery`, `useScrollLock`, `useId`) are shipped.
> - `packages/core/src/focus-scope.tsx` and `portal.tsx` already exist — read them for the established

**Goal:** Four interaction primitives in `@cascivo/core` — `DismissableLayer` (outside-click + Escape),
`RovingFocus`/`useRovingFocus` (arrow-key roving tabindex), `Presence` (mount/exit with animation), and an
anchor-positioning helper — each tested, each signal-driven. These unblock the overlay/menu/navigation
components in T5/T6.

**Architecture:** New `packages/core/src/`:
`dismissable-layer.tsx`, `roving-focus.tsx`, `presence.tsx`, `anchor.tsx` (+ `*.test.tsx`), exported from
`index.ts`.

**Tech stack:** signals + `useSignalEffect` for all DOM work. Reuse `composeRefs`, `FocusScope`,
`useControllableSignal`. No new deps.

### Task 1: `DismissableLayer`

**Files:** Create: `packages/core/src/dismissable-layer.tsx` + test.

- [ ] **Step 1:** Implement `DismissableLayer` — wraps children, takes `onDismiss` + flags
      (`disableOutsidePointer?`, `escapeDismisses?`). Inside `useSignalEffect`: add document
      `pointerdown`/`click` and `keydown` (Escape) listeners; call `onDismiss` when a pointer lands
      outside the layer's root node or Escape is pressed; remove listeners on teardown. Use a ref for the
      root node; keep `onDismiss` current via a ref (T3 idiom).
- [ ] **Step 2:** Support nested layers (a dropdown inside a modal): only the top-most layer responds to
      Escape. A simple module-level layer stack is acceptable; keep it minimal and tested.
- [ ] **Step 3:** Test: outside pointer triggers `onDismiss`; inside pointer does not; Escape triggers it;
      nested layers dismiss top-first. Use `@testing-library` + user events.

---

### Task 2: `RovingFocus` / `useRovingFocus`

**Files:** Create: `packages/core/src/roving-focus.tsx` + test.

- [ ] **Step 1:** Implement roving tabindex: a `useRovingFocus({ orientation, loop })` hook (and/or a
      `RovingFocus` provider-free wrapper using signals, NOT context) that tracks the active index in a
      signal and exposes per-item props (`tabIndex`, `onKeyDown`, `ref`). Arrow keys move focus
      (Home/End jump); `loop` wraps. Orientation `horizontal`/`vertical`/`both`.
- [ ] **Step 2:** Because cascivo forbids `useContext`, share state via a signal returned from the hook
      that each item reads (the parent passes item registration down, or items self-register by ref into a
      signal array). Pick the simplest approach that a Tier-1 consumer (Toggle Group) and a Tier-2
      consumer (Menu/Tree) can both use; document it.
- [ ] **Step 3:** Test: arrow navigation moves `tabIndex`/focus across items; Home/End; looping on/off;
      vertical vs horizontal key sets.

---

### Task 3: `Presence`

**Files:** Create: `packages/core/src/presence.tsx` + test.

- [ ] **Step 1:** Implement `Presence` — given a `present: boolean | Signal<boolean>`, render children
      while present AND while an exit animation/transition is still running, then unmount. Use
      `useSignalEffect` to listen for `animationend`/`transitionend` on the child node; no `useEffect`.
      Mirror the controlled-prop sync pattern (`useControllableSignal` or the manual `sig.value = present`).
- [ ] **Step 2:** Expose the data attribute hook (`data-state="open|closed"`) so CSS drives the animation
      (per CLAUDE.md: visual states in CSS). The component only manages mount/unmount timing.
- [ ] **Step 3:** Test: `present:true` mounts; flipping to `false` keeps the node until
      `transitionend`/`animationend`, then unmounts; `present:false` initially renders nothing.

### Task 4: Anchor-positioning helper

**Files:** Create: `packages/core/src/anchor.tsx` + test.

- [ ] **Step 1:** Implement a small `useAnchorPosition({ anchorRef, placement })` (or `Anchor`/`Anchored`
      wrapper) that positions a floating element relative to an anchor. Primary path: CSS anchor
      positioning (`anchor-name`/`position-anchor`) emitted via inline style/attrs — progressive. Fallback
      path: a JS `getBoundingClientRect` placement inside `useSignalEffect` (recompute on
      scroll/resize), behind a feature check (`CSS.supports('anchor-name: --x')`).
- [ ] **Step 2:** Keep it small — NOT a full collision/flip/shift engine (deferred in the roadmap).
      Support the common placements (top/bottom/left/right + start/end). Document the fallback contract.
- [ ] **Step 3:** Test the JS fallback math (mock `getBoundingClientRect`); assert the supports-gated path
      selection. Keep deterministic (no real layout).
