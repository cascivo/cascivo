`breakpoint:check` (which lints CSS only). This tranche deletes the orphans (after T1 lifted any salvageable copy),
aligns the JS breakpoint, and runs the full gate. Lens: **architect.**

**Architecture:** Orphan confirmation (re-verify before deleting — sections may have been wired by an earlier
tranche): none of `Features`/`ComponentGrid`/`ExamplesGallery`/`JsonPlayground` are imported by any
`App.tsx`/`pages/*`/`sections/*` file. `Header.tsx:199` reads `useMediaQuery('(max-width: 47.99rem)')` to decide
whether to wire the hamburger/drawer; the canonical scale (CLAUDE.md) is sm 30 / md 40 / lg 64 / xl 80 rem. The CSS
that hides the inline nav and shows the drawer lives in `landing.css` and must use the **same** breakpoint as the JS
or the button goes dead in the gap.

**Tech Stack:** `apps/site` Preact + signals; `useMediaQuery` from `@cascivo/core`; the project gate (`pnpm ready` /
`ready:ci`, `breakpoint:check`, `fallback:check`, and the new `links:check` from T5). No new deps.
