# v47 — Tranche 5: Wrap-Mode Handling + Very-Large-Document Strategy + Measurement

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the last two holes and prove the win. (1) Under `wrap` (soft-wrap on), DOM windowing is
**disabled** because row heights are variable, so every row must render — the O(n) **render** cost is irreducible
without wrap-aware pixel virtualization (engine territory, out of scope). But the `LineStateIndex` from T2–T4 still
makes **edits** cheap: only the changed suffix re-tokenizes until reconvergence. Confirm the wrap path routes
edits through the same index invalidation, **accept and document** the O(n) render limit, and optionally surface
soft guidance ("disable wrap above ~N lines") — **never** hide content (no `display:none`). (2) Evaluate the
worker-offload option (`PERFORMANCE.md` option #3) and **explicitly defer** it, recording the trigger that would
reopen it (a real, sustained 100k+-line editing target) in both `PERFORMANCE.md` and the meta `whenNotToUse`.
(3) Re-run `bench-large-doc.mjs`, record **before/after** numbers (e.g. the 50k-line keystroke from ~587 ms →
bounded by the changed suffix), and rewrite the "Where the limits are" cliff/floor section of `PERFORMANCE.md` to
reflect the windowed path. Addresses gap-analysis findings **`wrap` O(n) render (🟡)** and **worker offload
(scope)** from the master plan.

**Architecture:** Windowing constants are `VIRTUALIZE_THRESHOLD = 1000` and `OVERSCAN = 12`; windowing is
**disabled when `wrap` is true** (variable row heights make the spacer-padding math unsound). After T3/T4 the
per-render tokenization cost is **O(viewport)** and per-edit cost is **O(changed suffix until reconvergence)** via
`LineStateIndex.invalidateFrom(changedLine)` driven by v46's `diff()`. Under `wrap` the same index invalidation
applies — edits stay bounded — but `renderRows` emits **all** rows, so rendering is O(n) regardless of the
tokenization win. The bench script measures cold/warm/keystroke across 500–100k lines on the `markdown` grammar
with mostly-unique lines. Today's measured ceilings: comfortable ≤5k lines; 10k keystroke ~106 ms; 50k ~587 ms;
the cliff is `MAX_CACHE = 5000`. T5 changes **no** render-virtualization behavior — it confirms the edit path,
measures, and documents.

**Tech Stack:** owned TypeScript in `@cascivo/editor` (no new runtime deps); the T2–T4 `LineStateIndex` +
`tokenizeRange`; `node --experimental-strip-types` for `bench-large-doc.mjs`; Markdown for `PERFORMANCE.md`; the
`ComponentMeta` `whenNotToUse` array in `code-editor.meta.ts`. No `useState`/`useEffect`; no new components or
props.
