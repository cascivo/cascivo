# v30 — Tranche 3: Image Optimization

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Serve all raster images in the landing app as WebP/AVIF with responsive `srcset`, explicit `width`/`height` attributes (to prevent CLS), and `loading="lazy"` on every below-fold image. Three files currently have `<img>` tags: `ExamplesGallery.tsx`, `ExamplesPage.tsx`, and `ExampleDetailPage.tsx`.

**Architecture:** Install `vite-imagetools` as a dev dependency. For each screenshot `<img>`, use the imagetools query string API to request WebP output at an appropriate width. The screenshots in `apps/landing/public/screenshots/` are served as static public assets — imagetools cannot transform files in `public/`. For those, provide explicit WebP pre-generated copies alongside the originals and use `<picture>` with `<source type="image/webp">`. For images referenced directly by Vite's module graph (imported in TSX), imagetools handles conversion at build time.

**Note on `public/` images:** Vite imagetools only processes images that flow through the Vite module graph (i.e., `import img from './foo.png?format=webp'`). Images in `public/` are copied verbatim. For screenshots in `public/screenshots/`, the optimization strategy is:

1. Determine the current screenshot format (PNG vs. JPEG).
2. Accept the current format and focus on adding `width`/`height` and `loading="lazy"` to prevent CLS — these are zero-cost and eliminate the most impactful perf issue.
3. For WebP conversion of `public/` screenshots, add a `pnpm screenshots:optimize` npm script that runs `sharp-cli` or `squoosh-cli` to produce `.webp` variants alongside the PNGs. This is a stretch goal within T3 if time permits.

**Tech Stack:** `vite-imagetools` (dev dep), `<picture>` element, `loading="lazy"`, `width`/`height` attributes.

  ```tsx
    <img src={screenshot} alt="..." width={1280} height={800} loading="lazy" decoding="async" />
  </picture>
  ```

  Commit generated `.webp` files alongside the existing PNGs in `public/screenshots/`.
