> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `@cascivo/editor` a first-class **AI-first** citizen and close the roadmap. Teach the registry
generator and the MCP server about a new `'editor'` entry type so `CodeEditor`/`Highlight` flow automatically
into `registry.json`, `context.json`, `llms.txt`, the specs/parity/readme/sitemap artifacts, and the MCP
`list_components`/`get_component` tools. Then run the full CI gate (including `pnpm ready:ci` for the cold-cache,
build-ordering check the new workspace package warrants), the drift gate, and a grep sweep proving the editor
reached every registration surface.

**Architecture:** `scripts/registry/generate.ts` scans `ROOTS[]` for `<name>/<name>.meta.ts` and tags each entry
with a `type` + `install`. Add `'editor'` to `EntryType` and a root
`{ dir: packages/editor/src/editor, type: 'editor', prefix: 'editor/' }` with `install: '@cascivo/editor'` (the
charts root is the template). `packages/mcp/src/server.ts` `list_components` `type` enum gains `'editor'`
(`CodeEditor`'s `category:'inputs'` and `Highlight`'s `category:'display'` are already valid category values).
Everything downstream of `registry.json` (`context:generate`, `llms:generate`, `specs:generate`,
`parity:generate`, `readme:*`, `sitemap:generate`) regenerates from it via `pnpm regen`. Per CLAUDE.md: drift
gate must be clean and committed.

**Tech Stack:** the registry/MCP generators + `pnpm regen`; the full `pnpm ready` / `pnpm ready:ci` gate.
