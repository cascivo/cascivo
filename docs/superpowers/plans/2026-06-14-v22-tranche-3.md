**Goal:** Five detail pages at `/examples/<slug>` — one reusable template fed by `DEMOS` data — each
with: a hero (name + "feels like" + tagline), a screenshot gallery (light + dark, desktop + mobile from
T4), a "What it proves" block, component-coverage chips, a prominent "Open live demo →" CTA linking to
`/demos/<slug>/`, and the mock-demo disclaimer. Each is prerendered with a unique head and listed in the
sitemap.

**Architecture:** One `src/pages/ExampleDetailPage.tsx` template that reads the current slug from the
pathname and renders the matching `DEMOS` entry. Register five `ROUTES['/examples/<slug>']` entries (or
a single pattern-matched detail route) in `App.tsx`; add five `ROUTE_HEAD['/examples/<slug>']` entries
and five `PRERENDER_ROUTES` entries; add five sitemap `<url>`s.

**Tech stack:** unchanged. `useSignals()` first if reading signals; tokens; mobile-first; i18n.
