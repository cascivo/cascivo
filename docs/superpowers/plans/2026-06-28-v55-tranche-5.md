# v55 — Tranche 5: Gallery UI + AI Discovery

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make templates **discoverable** — by humans (a gallery) and by agents (MCP + a skill) (M-6/M-7). The gallery
is a **static** page in `apps/site` rendered from the build-time `public/marketplace.json` (T4) with **no runtime
backend calls** (ROADMAP-V55 Q3 / Master Decision 3). MCP gains `list_templates`/`get_template`/`add_template`, and a
`cascivo:add-template` skill (plus a "start from a template" path in `cascivo:design-page`) lets agents pull a whole
page composition.

**Architecture:** two surfaces.
- **Gallery** (`apps/site/src/pages/MarketplacePage.tsx`): read `public/marketplace.json` at build time; render
  screenshot cards (light/dark per theme), a category + framework + tag filter, a verified badge, a copyable
  `cascivo add <installSpec>` command, and a "Live demo" link (the author's Pages deploy). All client-side filtering
  over the baked JSON — no fetch to any server.
- **AI discovery** (`packages/mcp/src/{server,templates}.ts`): `list_templates` (filter by category/tag/framework over
  the same `marketplace.json`), `get_template` (resolve one template's components + files + demo), `add_template`
  (delegate to `cascivo add <template>` via the existing `add_to_project` CLI-wrapper pattern). A
  `cascivo:add-template` skill; `cascivo:design-page` offers "start from a template, then adapt."

**Tech Stack:** `apps/site` (Preact + cascivo, signals natively reactive — but follow CLAUDE.md: signals only, no
`useState`/`useEffect`); `@cascivo/mcp`; `skills/`. The catalog input is `public/marketplace.json` from T4. No backend,
no new runtime dependency.
