### Task 3: Audit story files for missing `Primary` export

The docs → storybook URL pattern is `?path=/story/{category}-{name.toLowerCase()}--primary`.
Every component story file must export `Primary` as a named story.

- [ ] **Step 1: List all component story files**

  ```sh
  ls apps/storybook/stories/*.stories.tsx | grep -v chart | grep -v PlainChart
  ```

- [ ] **Step 2: Check which files are missing a `Primary` export**

  ```sh
  for f in apps/storybook/stories/*.stories.tsx; do
    grep -l "^export const Primary" "$f" > /dev/null 2>&1 || echo "MISSING Primary: $f"
  done
  ```

  Note every file reported as missing `Primary`.

- [ ] **Step 3: Add `Primary` export to each missing file**

  For each file without `Primary`, open it and look at what the first story export is. Add
  a `Primary` alias that re-uses the existing canonical variant. Examples:

  If the first export is `Default`:
  ```ts
  export const Primary: Story = {} // same as Default
  export const Default: Story = {}
  ```

  If the first export is `Showcase` or some other name:
  ```ts
  export const Primary: Story = {} // canonical default for storybook link
  export const Showcase: Story = { args: { … } }
  ```

  `Primary: Story = {}` with no `args` renders the component with the `meta.args` defaults,
  which is the intended canonical first view. Do not duplicate complex `args` — keep it empty
  and let `meta.args` carry the defaults.

  If `meta.args` is empty and the component requires args to render, copy the simplest
  existing story's args:
  ```ts
  export const Primary: Story = { args: { children: 'Button' } }
  ```

- [ ] **Step 4: Verify no stories broke**

  Run Storybook:
  ```sh
  pnpm exec vp run @cascivo/storybook#dev
  ```

  Click through the newly added `Primary` stories to confirm they render without errors.

### Task 4: Add docs dark theme default

- [ ] **Step 1: Open `apps/docs/src/theme.ts`**

  Find:
  ```ts
  export const theme = persistedSignal<Theme>(STORAGE_KEY, 'light')
  ```

- [ ] **Step 2: Change the default to `'dark'`**

  ```ts
  export const theme = persistedSignal<Theme>(STORAGE_KEY, 'dark')
  ```

  That is the only change. `persistedSignal` reads from `localStorage` first; if a value
  is stored, it is used. The default only applies when no value has been stored (first visit).

- [ ] **Step 3: Verify**

  Open the docs in an incognito window (no localStorage) in the dev server:
  ```sh
  pnpm exec vp run @cascivo/docs#dev
  ```

  Expected: the page renders with `data-theme="dark"` on `<html>`. The docs should look dark.
  Open DevTools → Application → Local Storage. Confirm no `cascade-theme` key exists in a
  fresh session (or if it was cleared). Reload. Still dark.

  Open a normal window where localStorage may have `cascade-theme = "light"`. Confirm the
  stored preference is still used — the page renders light.

### Task 5: Add "View in Storybook" link to ComponentPage

- [ ] **Step 1: Open `apps/docs/src/pages/ComponentPage.tsx`**

  Read the file. The component renders `<article class="doc-page">` with several `<section>`
  blocks: Preview, Examples, Props, States/Variants/Sizes, Tokens, Accessibility.

- [ ] **Step 2: Derive the storybook URL**

  Add a helper at the top of the file (after imports):

  ```tsx
  function storybookUrl(category: string, name: string): string {
    const slug = `${category}-${name.toLowerCase().replace(/[^a-z0-9]/g, '')}`
    return `https://storybook.cascivo.com/?path=/story/${slug}--primary`
  }
  ```

- [ ] **Step 3: Add the link in the Preview section**

  In the `<section class="doc-section">` that contains the Preview, add the link after the
  `<div class="preview">` block:

  ```tsx
  <section class="doc-section">
    <h2>Preview</h2>
    <div class="preview">{Demo ? <Demo /> : <span class="muted">No live preview.</span>}</div>
    <a
      class="storybook-link"
      href={storybookUrl(entry.category, meta.name)}
      target="_blank"
      rel="noopener noreferrer"
    >
      View in Storybook →
    </a>
  </section>
  ```

- [ ] **Step 4: Add CSS for `.storybook-link`**

  In `apps/docs/src/app.css` (or wherever the docs component styles live), add:

  ```css
  .storybook-link {
    display: inline-block;

    font-size: 0.875rem;
    color: var(--cascivo-color-accent, #3b82f6);
    text-decoration: none;
  }

  .storybook-link:hover,
  .storybook-link:focus-visible {
    text-decoration: underline;
  }
  ```
