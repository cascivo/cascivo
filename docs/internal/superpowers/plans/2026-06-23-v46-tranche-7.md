# v46 — Tranche 7: Try the New Editor — Docs & Storybook Variants

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make every v46 capability **tryable** — a hands-on showcase of the new editor across **both** the docs
app (`apps/docs/src/pages/EditorPage.tsx`) and Storybook
(`apps/storybook/stories/editor/code-editor.stories.tsx`), in a full spread of variants. T6 wires the demos
minimally for the gate; **this tranche is the comprehensive, interactive surface**: undo/redo, find & replace,
`Mod-S` save, per-instance theme with **live switching** (Zen mode), bracket matching, active-line gutter, custom
keymap, a decorations provider, the imperative `CodeEditorHandle`, large-document/virtualized editing, and a
Markdown notes scenario. Every new prop gets a Storybook control so reviewers can poke it without writing code.

**Architecture:** Storybook already has `Editor/CodeEditor` stories (Basic, WithLineNumbers, ReadOnly, Wrap,
Languages, Themed) and an `Editor/Highlight` story; the docs `EditorPage` already has CodeEditor + Highlight +
Languages + Options + Themeable sections. This tranche **extends both** — new `argTypes` for the v46 props, new
named stories per feature, and new `EditorPage` sections — reusing the same sample-string + grid patterns already
in those files. Interactive stories that need state (the controlled/handle/theme-switch demos) use a small
render-function wrapper that follows each app's reactivity rules: **Storybook is a React app** (call
`useSignals()` first when reading a signal during render; no `useState`/`useEffect`), **docs is Preact** (signals
are natively reactive; still no banned hooks).

**Tech Stack:** Storybook (`@storybook/react-vite`) — `Meta`/`StoryObj`, `argTypes`, render functions; the docs
Preact page; `@cascivo/core` signals (`useSignal`, `useSignals`) for the interactive demos; `@cascivo/editor`
public API from T1–T6 (`CodeEditor`, `CodeEditorHandle`, `EditorTheme`, `Decoration`, `KeyMap`, `Highlight`). No
new runtime deps. Depends on **T1–T6** (all new props/exports must exist).
