# v44 — Tranche 1: `@cascivo/editor` Package Scaffold + Highlight Engine

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up the new standalone package **`@cascivo/editor`** (mirroring `@cascivo/charts`) and the
**owned, zero-dependency highlight engine** it is built on: a per-line, restartable tokenizer + a grammar
registry + a small grammar set (`plaintext`, `json`, `javascript`, `typescript`, `css`, `html`, `markdown`,
`bash`). No React surface yet (that is T2) — this tranche delivers a buildable, tested package and a
deterministic tokenizer.

**Architecture:** Copy the `@cascivo/charts` package shape (`package.json`, `vite.config.ts`, `tsconfig.json`,
`scripts/flatten-types.mjs`, `scripts/check-types-flat.mjs`, `setup.ts`, `css-modules.d.ts`, `readme.body.md`)
and swap `charts`→`editor`, `cssFileName:'charts'`→`'editor'`. The engine is pure functions: a token type union,
a **restartable** `tokenize(line, startState) → { tokens, endState }` (state carried across lines so block
comments/strings/template literals continue correctly), and a `registerGrammar`/`getGrammar` registry. Each
grammar is a small, individually-importable module (regex/state-machine rules) so consumers tree-shake to the
languages they use. Per CLAUDE.md: zero runtime deps; the engine is plain TS (no React, no signals).

**Tech Stack:** TypeScript (strict), vite+ (`vp`) lib build, Vitest. No runtime dependencies.

### Task 3: Implement the tokenizer engine

**Files:** `packages/editor/src/engine/types.ts`, `tokenize.ts`, `registry.ts`

- [ ] **Step 1:** `types.ts`: a `TokenKind` union
      (`'keyword'|'string'|'number'|'comment'|'function'|'type'|'operator'|'punctuation'|'variable'|'tag'|'attr'|'regexp'|'boolean'|'property'|'plain'`),
      a `Token = { kind: TokenKind; value: string }`, a `GrammarState` (opaque per-grammar continuation, e.g. a
      small string/enum for "inside block comment"/"inside template literal"), and a `Grammar` interface
      (`name: string`, `tokenizeLine(line: string, state: GrammarState): { tokens: Token[]; state: GrammarState }`,
      `initialState: GrammarState`).
- [ ] **Step 2:** `tokenize.ts`: a `tokenize(grammar, line, startState)` wrapper that delegates to
      `grammar.tokenizeLine`, plus a `tokenizeDocument(grammar, text)` helper that splits on `\n` and **carries
      `endState` from each line into the next** (so multi-line constructs continue), returning per-line token
      arrays. Pure + deterministic; no DOM. (The memoization cache is added in T4 — keep the function shape
      memo-friendly: input `(grammar, line, startState)` → output `{ tokens, endState }`.)
- [ ] **Step 3:** `registry.ts`: a module-level `Map<string, Grammar>` with `registerGrammar(grammar)`,
      `getGrammar(name): Grammar` (falling back to `plaintext`), and `listGrammars()`. Register the built-ins
      from Task 4 at import time of each grammar module (or via an explicit `registerBuiltins()` called from
      `index.ts` — choose the tree-shakeable option: grammars self-register only when imported).

### Task 4: Implement the grammar set

**Files:** `packages/editor/src/grammars/{plaintext,json,javascript,typescript,css,html,markdown,bash}.ts`

- [ ] **Step 1:** `plaintext.ts`: a trivial grammar that emits one `plain` token per line (the fallback).
- [ ] **Step 2:** `json.ts`: strings, numbers, booleans/null, property keys, punctuation. `css.ts`: selectors,
      properties, values, at-rules, comments (`/* */` with state carry). `bash.ts`: comments (`#`), strings,
      keywords (`if/then/fi/for/do/done/…`), variables (`$VAR`/`${...}`), operators.
- [ ] **Step 3:** `javascript.ts` + `typescript.ts`: keywords, strings (single/double/template with state carry
      across lines), numbers, line `//` + block `/* */` comments (state carry), regexp (best-effort),
      function/identifier, punctuation; `typescript.ts` extends `javascript` with type keywords
      (`interface/type/enum/implements/…`). Keep these grammars **small** — token-class accuracy, not full
      parsing.
- [ ] **Step 4:** `html.ts`: tags, attributes, attribute strings, comments (`<!-- -->` state carry), text.
      `markdown.ts`: headings, emphasis/strong, inline code + fenced code blocks (state carry), links, lists.
- [ ] **Step 5:** Each grammar `registerGrammar`s itself on import. Export them from `src/index.ts` under a
      `grammars` namespace and re-export the engine API (`tokenize`, `tokenizeDocument`, `registerGrammar`,
      `getGrammar`, `listGrammars`, types).
