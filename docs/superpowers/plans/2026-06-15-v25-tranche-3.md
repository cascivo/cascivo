# v25 — Tranche 3: ThemeDemo Animation

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the static 10-card theme grid with a single-card animated cycler that auto-advances through all ten themes every 1.2 s with a CSS cross-fade transition and dot indicators for manual navigation.

**Architecture:** `ThemeDemo.tsx` gains `useSignals()` + `useSignal<number>` for the active theme index and `useSignalEffect` with `setInterval` for auto-advance. The `SignupCard` renders once inside a single `data-theme` container whose value changes each interval tick. Clicking a dot writes to the signal and the `setInterval` continues from the new index. CSS: `transition: opacity 0.3s` on the pane; the dot indicators use `:checked` or `data-state="active"` for styling. The ten-pane grid (`theme-demo-grid`) is removed; the `theme-demo-pane` class is repurposed for the single container.

**Tech Stack:** `useSignal`, `useSignals`, `useSignalEffect` from `@cascivo/core`, `@cascivo/components/button` (optional, for the card), `landing.css`.

- [ ] **Step 2: Write the new ThemeDemo.tsx**

  Replace the contents of `apps/landing/src/sections/ThemeDemo.tsx` with:

  ```tsx
  'use client'
  import { useSignal, useSignals, useSignalEffect } from '@cascivo/core'
  import { Button } from '@cascivo/components/button'
  import { Input } from '@cascivo/components/input'
  import { Checkbox } from '@cascivo/components/checkbox'
  import { Card, CardContent, CardHeader, CardTitle } from '@cascivo/components/card'

  const THEMES = [
    'light',
    'dark',
    'warm',
    'flat',
    'minimal',
    'midnight',
    'pastel',
    'brutalist',
    'corporate',
    'terminal',
  ] as const

  type Theme = (typeof THEMES)[number]

  function SignupCard() {
    return (
      <Card variant="elevated">
        <CardHeader>
          <CardTitle>Create account</CardTitle>
        </CardHeader>
        <CardContent>
          <div className="theme-demo-form">
            <Input label="Email" placeholder="you@example.com" />
            <Input label="Password" type="password" />
            <Checkbox label="Send me product updates" defaultChecked />
            <Button>Sign up</Button>
          </div>
        </CardContent>
      </Card>
    )
  }

  export function ThemeDemo() {
    useSignals()
    const activeIdx = useSignal(0)

    useSignalEffect(() => {
      const id = setInterval(() => {
        activeIdx.value = (activeIdx.value + 1) % THEMES.length
      }, 1200)
      return () => clearInterval(id)
    })

    const activeTheme: Theme = THEMES[activeIdx.value] ?? 'light'

    return (
      <section className="section" data-reveal="">
        <h2>One form, ten personalities</h2>
        <p className="section-sub">
          The exact same markup rendered inside ten <code>data-theme</code> containers. Themes
          override only the semantic token layer — no component changes.
        </p>
        <div className="theme-demo-cycler">
          <div
            key={activeTheme}
            className="theme-demo-pane theme-demo-pane--single"
            data-theme={activeTheme}
          >
            <div className="theme-demo-label">{activeTheme}</div>
            <SignupCard />
          </div>
        </div>
        <div className="theme-demo-dots" role="group" aria-label="Select theme">
          {THEMES.map((theme, i) => (
            <button
              key={theme}
              type="button"
              className="theme-demo-dot"
              aria-label={theme}
              aria-pressed={activeIdx.value === i}
              data-state={activeIdx.value === i ? 'active' : undefined}
              onClick={() => {
                activeIdx.value = i
              }}
            />
          ))}
        </div>
      </section>
    )
  }
  ```

### Task 2: Update landing.css for the cycler layout

- [ ] **Step 1: Decide on cross-fade approach**

  Using `key={activeTheme}` on the pane div causes React to unmount+remount the element on
  theme change, which does NOT produce a cross-fade — it produces an instant swap. For a true
  cross-fade, remove `key` from the pane div so React keeps the DOM node and only the
  `data-theme` attribute changes. CSS `transition` on `color`, `background-color`, and
  `border-color` will then animate between values.

  Update `ThemeDemo.tsx`: remove `key={activeTheme}` from the inner div:

  ```tsx
  <div
    className="theme-demo-pane theme-demo-pane--single"
    data-theme={activeTheme}
  >
  ```

- [ ] **Step 2: Add/update theme-demo CSS in landing.css**

  Open `apps/landing/src/landing.css`. Locate the `.theme-demo-grid` block. Remove it (the grid
  is no longer used). Add the following CSS instead:

  ```css
  .theme-demo-cycler {
    display: flex;
    justify-content: center;
    margin-block: var(--cascivo-space-6);
  }

  .theme-demo-pane--single {
    inline-size: min(28rem, 100%);
    /* Animate token-driven colour changes as data-theme switches */
    transition:
      background-color 0.3s ease,
      color 0.3s ease,
      border-color 0.3s ease;
  }

  .theme-demo-dots {
    display: flex;
    justify-content: center;
    gap: var(--cascivo-space-2);
    flex-wrap: wrap;
    margin-block-start: var(--cascivo-space-4);
  }

  .theme-demo-dot {
    inline-size: 0.625rem;
    block-size: 0.625rem;
    min-block-size: 2.75rem; /* ≥44 px touch target via padding trick — use padding instead */
    padding: 1.0625rem; /* 17px each side → 34px + 10px dot = 44px touch target */
    background-clip: content-box;
    border-radius: 50%;
    border: none;
    cursor: pointer;
    background-color: var(--cascivo-color-border);
    transition: background-color 0.2s ease, transform 0.2s ease;
  }

  .theme-demo-dot[data-state='active'] {
    background-color: var(--cascivo-color-accent);
    transform: scale(1.3);
  }
  ```

  Note: the touch target for the dots is ≥44 px via padding. The dot visual is 10 px; padding
  of 17 px on each side makes the clickable area 44 px, satisfying the
  `@media (pointer: coarse)` touch-target requirement without a separate rule.
