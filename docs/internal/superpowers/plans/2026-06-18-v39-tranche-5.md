# v39 — Tranche 5: Blocks Expansion + Docs & Final Gate

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Adopt RetroUI's "blocks" idea — pre-composed page sections — for the handful cascivo lacks, then
document the whole v39 roadmap and run the full gate. cascivo already ships 8 blocks (`app-shell`,
`auth-login`, `auth-signup`, `dashboard-overview`, `dashboard-table`, `marketing-features`,
`marketing-hero`, `settings-profile`). T5 adds high-value RetroUI-style sections — **reuse-only**: each
composes **existing** components and introduces **no** new primitive (master-plan Decision 7).

**Architecture:** Follow the existing `packages/components/src/blocks/*` pattern (composition TSX + manifest;
`types.ts` for any shared block type). Blocks are signal-driven where interactive and obey the same authoring
rules (no banned hooks; CSS for visual states; i18n-defaulted strings; `useSignals()` in React apps).

**Tech Stack:** reuse-only compositions of shipped components; vite+ (`vp`) for the gate.

### Task 2: Implement the blocks (reuse-only)

**Files:** `packages/components/src/blocks/{pricing,site-footer,faq,testimonials}/*`

- [ ] **Step 1:** For each block: create the composition TSX importing only shipped components; accept a small
      typed `data`/content prop (e.g. `tiers`, `links`, `questions`, `items`) with sensible defaults; render
      responsively (mobile-first; ≥44px coarse targets; no off-scale breakpoints; never `display:none` data).
- [ ] **Step 2:** Strings default from `@cascivo/i18n`; `labels`/content props override. No hardcoded English
      fallbacks. If interactive (e.g. `faq` via `Accordion`), the interactivity lives in the reused component —
      the block adds none of its own banned hooks.
- [ ] **Step 3:** Add a `*.meta.ts` manifest per block matching the existing blocks' manifest shape; update
      `types.ts` only if it enumerates block kinds.
