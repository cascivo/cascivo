# v33 — Tranche 2: Home Carousel Cross-Fade

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the hard cut between demos in the "Drive it, don't read about it." carousel (`#examples`) with a smooth cross-fade. The screenshot and info block should fade in (opacity + slight motion) on every switch — auto-advance, prev/next, and dot clicks alike — and swap instantly under `prefers-reduced-motion: reduce`.

**Architecture:** Re-key the fading content on `activeIdx` so React remounts it on each switch, re-triggering a CSS entrance `@keyframes`. No second image layer, no JS animation loop.

**Tech Stack:** `@cascivo/core` signals (already in use), CSS keyframes in `landing.css`.

### Task 2: Add the cross-fade keyframes

**Files:** Modify `apps/landing/src/landing.css`

- [ ] **Step 1: Locate the `.examples-carousel*` block** (the `.examples-carousel`, `.examples-screenshot-frame`, `.examples-carousel-info`, `.examples-nav*` rules and the `@media (min-width: 64rem)` grid override).

- [ ] **Step 2: Add a `.examples-fade` rule** with `display: contents` so children keep their existing grid placement, plus the entrance animation applied to the fading children (or to a wrapper that does not break the grid). Because `display: contents` elements can't be animated directly, apply the keyframes to `.examples-screenshot-frame` and `.examples-carousel-info` themselves, triggered by their remount:

  ```css
  @media (prefers-reduced-motion: no-preference) {
    .examples-fade .examples-screenshot-frame,
    .examples-fade .examples-carousel-info {
      animation: examples-fade-in 360ms ease-out both;
    }
    .examples-fade .examples-carousel-info {
      animation-delay: 60ms; /* slight stagger after the image */
    }
  }

  @keyframes examples-fade-in {
    from {
      opacity: 0;
      transform: translateY(0.5rem);
    }
    to {
      opacity: 1;
      transform: translateY(0);
    }
  }
  ```

  Under `prefers-reduced-motion: reduce` no animation is declared, so the swap is instant.

- [ ] **Step 3: Verify** the controls don't re-animate, the image doesn't reflow the layout, and there is no horizontal overflow at 320/360/390/414.
