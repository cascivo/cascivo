> Current-state assumptions — RE-VERIFY before starting:

> - T1 is merged and on `main` — `usePopover` exists, MultiSelect will consume it
> - `factory-backlog.json` has no T2 items queued yet
> - The factory skill's integrate step exports to `@cascade-ui/react` (verify in SKILL.md before queuing)
> - Edge case 13: confirm factory SKILL.md has the `@cascade-ui/react` export step

**Goal:** Add eight new input-category components via the dark factory pipeline. All follow the
copy-paste registry model. MultiSelect floats on `usePopover` (T1 prerequisite). Factory runs
sequentially through the backlog queue; humans review design + a11y before each item flips to
`done`. End state: 8 new inputs exported from `@cascade-ui/react`, all in registry, all with
stories and docs demos.

**Architecture:** Every component queued in `factory-backlog.json` with a detailed spec. The
`/factory` skill picks items in priority order: scaffold → verify → integrate → review. The agent
(dark factory) generates four files per component (tsx, module.css, meta.ts, test.tsx), runs
`vp check` + `vp run @cascade-ui/components#test`, and opens a PR. Human reviews design and a11y;
human flips `review` → `done`.

**Tech stack:** Same as components package — signals, no hooks, CSS Modules, `@cascade-ui/i18n`
for strings, `@cascade-ui/core` for signals.

**Conventions:** Each factory spec below is the ground truth for the dark factory agent. Paste the
spec directly into the factory backlog item's `spec` field. The factory self-heals up to 5
attempts on lint/type/test failures before escalating.

**Context paths:**

- Factory skill: `.claude/skills/factory/SKILL.md`
- Backlog: `factory-backlog.json`
- Popover hook: `packages/components/src/popover/use-popover.ts`
- Existing input pattern: `packages/components/src/input/`
