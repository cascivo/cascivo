# v31 — Tranche 2: Form Utilities (Progress, Kbd, Filter)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship three new components: `Progress` (determinate/indeterminate bar), `Kbd` (keyboard shortcut display), and `Filter` (signal-driven toggle-button group for faceting). All three follow the standard four-file component pattern and are registered in `_all-metas.ts` and `packages/react/src/index.ts`.

**Architecture:** `Progress` wraps the native `<progress>` element — indeterminate state is triggered automatically when no `value` attribute is set, which activates the browser's native indeterminate animation. `Kbd` is a pure display component: styled `<kbd>` element, no signals. `Filter` uses a `useSignal<string[]>` for the selected state and emits `onChange` on selection change.

**Tech Stack:** `useSignals` from `@cascivo/core` for `Filter` (reads signal during render). CSS Modules. No new npm dependencies.

  ```tsx
  import styles from './progress.module.css'

  export type ProgressVariant = 'primary' | 'info' | 'success' | 'warning' | 'error'
  export type ProgressSize = 'sm' | 'md' | 'lg'

  export interface ProgressProps {
    /** 0–100. Omit for indeterminate state. */
    value?: number
    variant?: ProgressVariant
    size?: ProgressSize
    /** Accessible label */
    'aria-label'?: string
    /** id of element describing the progress */
    'aria-describedby'?: string
    className?: string
  }

  export function Progress({
    value,
    variant = 'primary',
    size = 'md',
    className,
    ...aria
  }: ProgressProps) {
    return (
      <progress
        className={[styles.progress, className].filter(Boolean).join(' ')}
        data-variant={variant}
        data-size={size}
        value={value}
        max={value !== undefined ? 100 : undefined}
        {...aria}
      />
    )
  }
  ```

- [ ] **Step 1: Create `filter.tsx`**

  ```tsx
  import { useSignal, useSignals } from '@cascivo/core'
  import styles from './filter.module.css'

  export type FilterVariant = 'pill' | 'outline'

  export interface FilterOption {
    label: string
    value: string
  }

  export interface FilterProps {
    options: FilterOption[]
    value?: string[]
    defaultValue?: string[]
    onChange?: (selected: string[]) => void
    /** Allow selecting multiple options at once */
    multi?: boolean
    variant?: FilterVariant
    'aria-label'?: string
    className?: string
  }

  export function Filter({
    options,
    value,
    defaultValue = [],
    onChange,
    multi = false,
    variant = 'pill',
    className,
    ...aria
  }: FilterProps) {
    useSignals()
    const selected = useSignal<string[]>(value ?? defaultValue)
    selected.value = value ?? selected.value

    function toggle(val: string) {
      const current = selected.value
      let next: string[]
      if (multi) {
        next = current.includes(val) ? current.filter(v => v !== val) : [...current, val]
      } else {
        next = current.includes(val) ? [] : [val]
      }
      selected.value = next
      onChange?.(next)
    }

    return (
      <div
        role="group"
        className={[styles.filter, className].filter(Boolean).join(' ')}
        data-variant={variant}
        {...aria}
      >
        {options.map(opt => (
          <button
            key={opt.value}
            type="button"
            className={styles.item}
            data-selected={selected.value.includes(opt.value) || undefined}
            onClick={() => toggle(opt.value)}
            aria-pressed={selected.value.includes(opt.value)}
          >
            {opt.label}
          </button>
        ))}
      </div>
    )
  }
  ```
