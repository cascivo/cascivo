# v49 — Tranche 1: Truth-in-Advertising (manifest sample, derived headings, WCAG, voice)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove every place the landing page tells a small lie about itself. The AI-first proof sample mis-states the
library's own manifest key (`a11y:` where the real `ComponentMeta` uses `accessibility:`); two headings hardcode
counts that drift ("Twelve themes", "Five reasons it feels different"); and the internal accessibility contract
(CLAUDE.md, 2.1 AA) disagrees with the shipped claim (2.2 AA). This tranche fixes correctness in-place — no layout or
visual change — and lifts any orphan copy worth keeping before T6 deletes the orphan files. Lens: **architect +
designer.**

**Architecture:** `AdvantageCarousel.tsx` AiPanel renders a `CodeSnippet` literal with `a11y: { role: 'button',
wcag: 'AA' }` (`:152`). The real schema (CLAUDE.md `ComponentMeta`; `packages/components/src/button/button.meta.ts:33`)
uses `accessibility: { role, wcag, keyboard }`. The carousel heading "Five reasons it feels different" (`:250`) is a
string literal though `ADVANTAGES` has a `.length`. `ThemesPanel` headline "Twelve themes, one token swap" (`:97`) is
literal though `__CASCIVO_THEME_COUNT__` exists (`vite.config.ts:246`) and `THEMES` is enumerable (`theme.ts`). The
marketing surface standardizes on **WCAG 2.2 AA** (route-head, ProofTeasers, accessibility pages); CLAUDE.md says
2.1 AA.

**Tech Stack:** Preact + `@cascivo/core` signals in `apps/site`; the build-time `define` macros in `vite.config.ts`;
existing `CodeSnippet` component. No new deps. No component-library changes.
