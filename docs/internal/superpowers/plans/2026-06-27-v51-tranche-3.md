# v51 — Tranche 3: Axis, Scale & Stacking Completeness

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the data/axis correctness gaps that stop cascivo charts from matching real Recharts usage: **axis
titles + explicit domain control** (C-F5), **percent/100% stacking** (C-F3), **`connectNulls` gap handling** (C-F4),
and **generalized per-series axis assignment** (C-F6). Some of these are latent bugs — a `null` y currently produces a
`NaN` path segment. Lens: **feature parity**.

**Architecture:**
- **Axis titles/domain:** `chrome/axis.tsx` `AxisProps` today is `{scale, orientation, length, format?, tickCount?,
  labelEvery?, transform?}` — no title, no domain. Charts derive the domain internally (`Math.min/Math.max` in
  `line-chart.tsx` etc.). Add an optional `label?: string` (rendered as a rotated/positioned axis title) and let charts
  accept `domain?: [number, number] | 'auto'` / `yMin`/`yMax` that override the derived extent before building the
  scale.
- **Percent stacking:** `engine/stacked.ts` produces stacked offsets; `bar-chart.tsx` `mode` is `'grouped' |
  'stacked'`. Add a `'percent'` mode that normalizes each category's segments to sum to 1 (the y-axis becomes 0–100%).
  Give `AreaChart` a stacked + percent mode too (it has none today).
- **`connectNulls`:** `line-chart.tsx`/`area-chart.tsx` map every datum straight into `linePath`; a `null`/missing y
  yields `NaN`. Split the point list into contiguous defined runs and emit a separate path per run (a real gap) unless
  `connectNulls` bridges them. `engine/shape.ts` `linePath` may need a `defined`-aware variant or the charts segment
  before calling it.
- **Per-series axis:** today only `combo-chart.tsx` has `secondAxis`. Add a per-series `axis?: 'left' | 'right'` so a
  Line/Area/Bar series can bind to a right-hand scale; render a second `Axis` when any series opts right (mirroring
  ComboChart's `lineYScale` + extra right margin).

**Tech Stack:** `chrome/axis.tsx`, `engine/{stacked,shape}.ts`, the affected charts; tokens for the axis-title fill.
No new dependency.

### Task 1: Axis titles + explicit domain

**Files:** `chrome/axis.tsx`, the charts that own axes

- [ ] **Step 1:** Add `label?: string` to `AxisProps`; render it as an axis title — x-axis: centered below the ticks;
      y-axis: rotated `-90°`, centered along the axis. Use `var(--cascivo-color-foreground-muted)`, `0.75rem`.
- [ ] **Step 2:** Add `xAxisLabel?: string` / `yAxisLabel?: string` to Line/Area/Bar/Scatter/Combo props, forwarded

      use it instead of the derived `min/max` when building the scale. Default stays auto. Clamp/guard so an explicit
      domain that excludes data still renders (no `NaN`).

### Task 2: Percent / 100% stacking

**Files:** `engine/stacked.ts`, `bar-chart.tsx`, `area-chart.tsx`

- [ ] **Step 1:** Extend the stacking helper to support a `'percent'` offset: per category, divide each segment by the
      category total so segments sum to 1; the y-scale domain becomes `[0, 1]` rendered as `0–100%`.
- [ ] **Step 2:** `BarChart` `mode` → `'grouped' | 'stacked' | 'percent'`; in `'percent'` the y-axis formats as `%`
      and the tooltip shows share (+ raw value).
- [ ] **Step 3:** `AreaChart` gains `mode?: 'overlap' | 'stacked' | 'percent'` (today areas just overlap); stacked &
      percent reuse the same offsets.
- [ ] **Step 4:** `stacked.test.ts`: percent offsets sum to 1 per category; a zero-total category doesn't divide by
      zero (renders empty, no `NaN`).

### Task 3: `connectNulls` gap handling

**Files:** `engine/shape.ts` (or the charts), `line-chart.tsx`, `area-chart.tsx`

- [ ] **Step 1:** Treat `y == null`/`NaN` as a gap. Split each series into contiguous defined runs; render one
      `linePath`/area path per run so the line **breaks** at the gap (the honest default).
- [ ] **Step 2:** Add `connectNulls?: boolean` (default `false`); when `true`, bridge runs into a single path.
- [ ] **Step 3:** Tooltip/label/point hit-testing skips undefined points (don't emit a `ChartPoint` for a gap).
- [ ] **Step 4:** `shape.test.ts` (or chart test): a series `[1, null, 3]` renders two segments by default, one when
      `connectNulls`; no `NaN` in any `d` attribute.
