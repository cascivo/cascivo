# Support

Thanks for using cascivo. Here's where to go, depending on what you need.

## Questions & help

- **Documentation** — [cascivo.com](https://cascivo.com): component
  reference, adopter guides, theming, and the AI/MCP layer. Start here.
- **Getting started** — [docs/GETTING-STARTED.md](docs/GETTING-STARTED.md) and
  the [docs index](docs/README.md).
- **Migrating from shadcn/ui** — [docs/MIGRATING-FROM-SHADCN.md](docs/MIGRATING-FROM-SHADCN.md).
- **Discussions & questions** — [GitHub Discussions](https://github.com/cascivo/cascivo/discussions)
  for how-to questions, ideas, and sharing what you built.

## Bugs & feature requests

- **Bug reports** and **feature requests** — open an issue with the matching
  template: [New issue](https://github.com/cascivo/cascivo/issues/new/choose).
  Search [existing issues](https://github.com/cascivo/cascivo/issues) first to
  avoid duplicates.

## Security

Please do **not** open a public issue for security problems. Follow
[SECURITY.md](SECURITY.md) — use GitHub's private vulnerability reporting or the
maintainer email listed on the npm package page.

## Contributing

Want to fix a bug, add a component, or propose a theme? See
[CONTRIBUTING.md](CONTRIBUTING.md) and the authoring rules in
[CLAUDE.md](CLAUDE.md). Run `pnpm ready` before opening a pull request.
