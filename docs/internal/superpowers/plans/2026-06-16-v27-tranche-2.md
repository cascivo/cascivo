# v27 — Tranche 2: Sticky Navbar + Scroll Indicator + GitHub Icon

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the landing header sticky so it stays above the fold at all scroll positions. Add
a 2 px scroll progress bar directly below the header. Replace the "GitHub" text link with an SVG
icon button.

**Architecture:** CSS changes only for sticky + progress bar. A `scrollRatio` signal + passive
scroll listener writes a CSS custom property to the progress bar DOM element (no React re-render).
GitHub icon is an inline SVG in `Header.tsx`.

**Tech Stack:** CSS `position: sticky`, CSS custom properties, `useSignalEffect`, passive scroll
listener, inline SVG.

### Task 3: Add the scroll progress bar

- [ ] **Step 1: Add `.scroll-progress` element to `Header.tsx`**

  `Header.tsx` currently returns a `<>` fragment containing `<ShellHeader .../>`, a scrim
  `<div>`, and the mobile drawer `<nav>`. Add the progress bar element immediately after
  `<ShellHeader>`:

  ```tsx
  import { useRef } from 'react'
  // ... existing imports ...

  export function Header() {
    // ... existing signals ...
    const progressRef = useRef<HTMLDivElement>(null)

    // Scroll progress — write to CSS custom property directly to avoid re-renders
    useSignalEffect(() => {
      const update = () => {
        const el = progressRef.current
        if (!el) return
        const scrollable = document.body.scrollHeight - window.innerHeight
        const ratio = scrollable > 0 ? Math.min(window.scrollY / scrollable, 1) : 0
        el.style.setProperty('--scroll-ratio', String(ratio))
      }
      window.addEventListener('scroll', update, { passive: true })
      update() // initial state
      return () => window.removeEventListener('scroll', update)
    })

    return (
      <>
        <ShellHeader ... />
        <div ref={progressRef} className="scroll-progress" aria-hidden="true" />
        {/* existing scrim and drawer */}
      </>
    )
  }
  ```

- [ ] **Step 2: Add CSS for the progress bar**

  In `apps/landing/src/landing.css`, add after the sticky header block:

  ```css
  /* ── Scroll progress bar ────────────────────────────────── */
  .scroll-progress {
    position: sticky;
    top: var(--cascivo-shell-header-block-size, 3.5rem);
    z-index: 199;
    block-size: 2px;
    background: transparent;
    /* Stays in normal flow; sits below the sticky header */
  }

  .scroll-progress::after {
    content: '';
    display: block;
    block-size: 100%;
    background: linear-gradient(
      to right,
      var(--cascivo-color-accent),
      var(--cascivo-brand-primary, var(--cascivo-color-accent))
    );
    transform: scaleX(var(--scroll-ratio, 0));
    transform-origin: inline-start;
    transition: transform 80ms linear;
  }

  @media (prefers-reduced-motion: reduce) {
    .scroll-progress::after {
      transition: none;
    }
  }
  ```

  The `::after` approach uses `scaleX` on a compositor-promoted property — GPU-animated, no
  layout cost.

### Task 4: Replace the GitHub text link with an SVG icon

- [ ] **Step 1: Locate the GitHub entry in `NAV_LINKS`**

  In `Header.tsx`, the `NAV_LINKS` array has:

  ```ts
  { label: 'GitHub', href: 'https://github.com/cascivo/cascivo' },
  ```

  The `ShellHeader` `nav` prop renders these items. Depending on how `ShellHeader` renders nav
  items, the GitHub link may render as plain text. There are two approaches:
  - If `ShellHeader` accepts a `end` slot (it does — currently used for the theme dots), move
    the GitHub link into `end` as an icon `<a>` element.
  - If `ShellHeader` supports per-item icon overrides in `nav`, use that.

  The cleanest approach: remove `GitHub` from `NAV_LINKS` and add it to the `end` slot alongside
  the theme switcher (which will be replaced in T3). For T2, the `end` slot becomes:

  ```tsx

        href="https://github.com/cascivo/cascivo"

        aria-label="GitHub"
  ```
