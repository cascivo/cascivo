# v48 — Tranche 6: Docs, Meta, Registry, Storybook, Changesets, Regen & Gate

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land the documentation, demos, release plumbing, and final gate for the v48 charts work, and verify both
consumer-feedback acceptance checklists are satisfied (or remaining items explicitly deferred with a note). Add the
new pie capabilities (donut center/thickness/size, empty-state, `value (pct%)` tooltip) and stacked-bar capabilities
(`toStackedSeries`, per-segment tooltip) to the docs `ChartsPage` and Storybook; cut the `@cascivo/charts` minor
changeset (the editor `0.1.2` patch changeset already landed in T1); run `pnpm regen` + the full gate + `pnpm
ready:ci`; flip `docs/ROADMAP-V48.md` to Shipped.

**Architecture:** `apps/docs/src/pages/ChartsPage.tsx` is the dogfooded docs surface (Preact); Storybook stories live
under `apps/storybook/stories/charts/` and are partly generated — check whether stories are authored or emitted by
`stories:generate` (`pnpm regen` includes it) and follow the existing convention. Both apps build **without** a prior
full build, so the charts source alias must stay intact in their vite configs (per CLAUDE.md's workspace-alias rule
— charts exports source, so likely no alias needed, but verify the build). Changesets drive the version bumps;
`pnpm regen` keeps `registry.json`/llms/README/schema/context/specs in sync; the drift gate fails on any uncommitted
regen output.

**Tech Stack:** `apps/docs` (Vite + Preact), `apps/storybook` (Storybook + vite), changesets, `pnpm regen` chain,
vite+ (`vp`) for check/build/test, `pnpm ready` / `pnpm ready:ci` for the full gate. React stories call
`useSignals()` (no Babel signals transform in the React apps).
