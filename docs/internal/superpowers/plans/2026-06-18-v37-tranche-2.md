# v37 — Tranche 2: Theming Path + Base Layer

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make a themed setup a **one-import** story, stop double-loading `@cascivo/tokens` (#3), and
ship the one rule that activates the font token (#6) so plain consumer markup renders in the sans stack
instead of falling back to serif.

**Architecture:** Today `light.css` and `dark.css` each `@import '@cascivo/tokens'`, so light+dark pulls
tokens twice (`packages/themes/src/light.css:4`, `dark.css:4`); there is no aggregate bundle. We add a
`@layer cascivo.base` (font-family/line-height/color on `html`/`body`) shipped by `@cascivo/themes`, an
`all` bundle that imports tokens once + base + light/dark, and we de-dupe the per-theme token import so a
multi-theme setup loads tokens a single time. Layer ordering: `cascivo.base` < `cascivo.theme` <
`cascivo.component` so components and consumer styles still win (consistent with #5).

**Tech Stack:** CSS only (`@cascivo/themes` is CSS-only, no build step); docs prose.
