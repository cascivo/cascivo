**Goal:** Make templates *installable* â€” the user-visible payoff before any marketplace machinery (M-3). After this

**Architecture:** three touch points.

  closure **plus** the template's own `files` (the existing closure walk already pulls components; templates just also
  carry page/fixture files to copy). No new address kinds â€” `bare`/`github`/`namespace`/`url` all already resolve a
  template JSON.
- **add** (`commands/add.ts`): a template branch â€” install the component closure (existing behavior), then write the
  template's page/fixture `files` to `meta.fileRoles` targets (default under `src/`), and record `{ name, version }` in
  the lockfile under a `templates` section.
- **create / view** (`commands/{create,view}.ts`): `create --template <spec>` scaffolds a new app (existing `create`
  scaffold) then runs the template install into it; `view <template>` prints the bundled components + files + a demo
  link before installing.

**Tech Stack:** `packages/cli/src/utils/resolve.ts`, `commands/{add,create,view}.ts`; the registry types/validator from

### Task 1: Template-aware resolution

**Files:** `utils/resolve.ts`

- [ ] **Step 1:** After fetching + parsing an item, if `isTemplateItem(item)`, validate it (`validateTemplate`) and
      fail fast with the collected errors if invalid. The four address kinds (`bare`/`github`/`namespace`/`url`) need
      no change â€” a template JSON resolves like any item.
- [ ] **Step 2:** Return a result distinguishing the **component closure** (from `registryDependencies`, resolved by
      the existing transitive walk) from the **template's own files** (`item.files` with their `meta.fileRoles`). Do
      **not** inline component source into the template (Decision 2).
- [ ] **Step 3:** Test (`resolve.test.ts`): resolving a template returns its component closure + its page/fixture
      files; a missing component dependency surfaces a clear error; a malformed template fails validation pre-install.

### Task 2: `add` a template

- [ ] **Step 4:** Test: `add` a template writes the page/fixture files to targets, installs the closure, records the
