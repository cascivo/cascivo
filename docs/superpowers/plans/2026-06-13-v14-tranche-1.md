**Goal:** Every chart looks right on every theme. A researched, CVD-safe base palette
(Okabe-Ito-grounded) maps onto cascade's 8 series slots; each of the ten themes overrides all
eight with a set tuned for its surface; a test proves presence, contrast, and color-vision
distinguishability across all themes.

**Architecture:** A documented base palette + per-theme overrides in `@cascivo/themes`. A
pure CVD-simulation + contrast utility in `scripts/checks` powers a verification test. No
runtime dependency is added to shipped packages — the CVD math lives in the test layer.

**Tech stack:** unchanged. Colors stay oklch (the existing token format).

## Tranche exit criteria

1. `docs/specs/chart-palette.md` documents the research lineage, the Okabe-Ito mapping, the
   subset guidance, and the redundant-encoding rule with citations.
2. The base palette + all ten themes define all 8 `--cascivo-chart-*`, each theme tuned to its
   surface.
3. The verification test passes: presence (all 8 per theme), contrast (≥ documented ratio vs
   theme chart background), and CVD distinguishability (protan/deutan/tritan) — and fails if a
   theme drops a color or ships an indistinguishable pair (prove by temporarily breaking one).
4. The v9 parity test stays green; charts read colors only via `var(--cascivo-chart-N)`.
5. Root gate + drift check exit 0.
