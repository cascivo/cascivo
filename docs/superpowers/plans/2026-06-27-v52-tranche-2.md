# v52 — Tranche 2: Glyphs & Richer Annotations

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the two visx primitives that make charts more *encodable*: **point-glyph shapes** (V-3 — visx `Glyph`)
so a scatter/line point can encode a second categorical dimension by shape, and **richer annotations** (V-4 — visx
`Annotation`/`Threshold`/`Marker`, nivo annotations) — a note with a connector/leader-line + label, and a
threshold/difference band — beyond v51's `ReferenceLine`/`Area`/`Dot`. Today scatter renders a plain `<circle>`. Lens:
**visx + nivo**.

**Architecture:** glyphs are small centered SVG paths. A `chrome/glyph.tsx` exports `Glyph({ shape, x, y, size,
color })` for `circle | square | diamond | triangle | cross | star` and a `glyphPath(shape, size)` helper. Scatter/
Line accept `glyph?: GlyphShape | (datum) => GlyphShape` and render glyphs instead of bare circles. Annotations extend
`chrome/reference.tsx`: a new `note` kind draws a subject marker (dot/circle) + a connector line to an offset label
(clamped in-bounds), like `@visx/annotation`'s `Connector`/`Label`; a `threshold` kind shades the area *between two
series* (or a series and a constant) where one exceeds the other — the difference encoding `@visx/threshold` provides.
Both reuse the existing `AnnotationContext` (scales + inner dims).

**Tech Stack:** SVG `<path>`/`<line>`/`<text>` over the existing scales; i18n for any label default; tokens for glyph/
note color. No new dependency.

### Task 1: Glyph shape set

**Files:** `packages/charts/src/chrome/glyph.tsx`

- [ ] **Step 1:** `type GlyphShape = 'circle' | 'square' | 'diamond' | 'triangle' | 'cross' | 'star'`.
      `glyphPath(shape, size)` returns a centered path `d` (radius = size/2) for each; `circle` may return a `<circle>`
      instead for crispness.
- [ ] **Step 2:** `Glyph({ shape, x, y, size, color, ... })` renders the shape translated to `(x, y)` with
      `fill={color}`; pass-through `data-*`/`aria-hidden` (the value is in the a11y tree already).
- [ ] **Step 3:** `glyph.test.tsx`: each shape renders a non-empty mark centered at `(x, y)`; `size` scales it.

### Task 3: Note + threshold annotations

**Files:** `packages/charts/src/chrome/reference.tsx`
