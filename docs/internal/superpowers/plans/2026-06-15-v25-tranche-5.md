### Task 1: Add prebuilt option to QuickStart.tsx

The current section shows three CLI steps for copying components. Add a "or use prebuilt" block
below to show that `@cascivo/react` can be imported directly from npm without copying.

- [ ] **Step 1: Read the current QuickStart.tsx**

  Open `apps/landing/src/sections/QuickStart.tsx`. Note:
  - `STEPS` is an array of three objects: title, code, note
  - Each step renders a `<CopyCommand>` and a note paragraph
  - The section div uses `className="quickstart"`

- [ ] **Step 2: Add the prebuilt block**

  After the `<div className="quickstart">` block (which renders the three steps), add a new
  prebuilt section. Replace the full `QuickStart` function body with:

  ```tsx
  export function QuickStart() {
    return (
      <section className="section" id="quickstart" data-reveal="">
        <h2>Up and running in three steps</h2>
        <div className="quickstart">
          {STEPS.map((step, i) => (
            <div key={step.title} className="quickstart-step">
              <div className="quickstart-head">
                <span className="quickstart-num">{i + 1}</span>
                <h3>{step.title}</h3>
              </div>
              <CopyCommand command={step.code} />
              <p className="quickstart-note">{step.note}</p>
            </div>
          ))}
        </div>

        <div className="quickstart-prebuilt" aria-label="Prebuilt option">
          <h3 className="quickstart-prebuilt-heading">
            Or use components directly from npm — no copy needed
          </h3>
          <p className="quickstart-prebuilt-note">
            <code>@cascivo/react</code> ships a prebuilt distribution of all components. Install
            once and import anywhere. You don&apos;t own the source, but there&apos;s no setup and
            no copy step. Upgrade to owned code via the CLI at any time.
          </p>
          <div className="quickstart-prebuilt-steps">
            <div>
              <CopyCommand command="npm add @cascivo/react @cascivo/tokens @cascivo/themes" />
              <p className="quickstart-note">Install the prebuilt package and its peer assets.</p>
            </div>
            <div>
              <CopyCommand
                command={`import { Button } from '@cascivo/react'\nimport '@cascivo/themes/light.css'`}
              />
              <p className="quickstart-note">
                Import the component and a theme CSS file. No provider, no wrapper.
              </p>
            </div>
          </div>
        </div>
      </section>
    )
  }
  ```

### Task 2: Fix Header.tsx nav links

The "Components" and "Storybook" nav items in `Header.tsx` use relative SPA paths that produce
404s. Change them to the canonical external URLs.

- [ ] **Step 1: Open Header.tsx and locate NAV_LINKS**

  In `apps/landing/src/sections/Header.tsx`, find:

  ```ts
  const NAV_LINKS = [
    { label: 'Components', href: '/docs' },
    // ...
    { label: 'Storybook', href: '/storybook' },
    // ...
  ]
  ```

- [ ] **Step 2: Update the two hrefs**

  ```ts
  const NAV_LINKS = [
    { label: 'Components', href: 'https://docs.cascivo.com' },
    { label: 'Examples', href: '/examples', active: path.startsWith('/examples') },
    { label: 'Storybook', href: 'https://storybook.cascivo.com' },
    {
      label: 'Accessibility',
      href: '/accessibility',
      active: path.startsWith('/accessibility'),
    },
    {
      label: 'Performance',
      href: '/performance',
      active: path.startsWith('/performance'),
    },
    {
      label: 'Guides',
      href: '/guides',
      active: path.startsWith('/guides'),
    },
    { label: 'GitHub', href: 'https://github.com/cascivo/cascivo' },
  ]
  ```

- [ ] **Step 3: Confirm external links open in a new tab**

  Check whether `ShellHeader` adds `target="_blank" rel="noopener noreferrer"` for external hrefs
  automatically, or if this must be done manually. Open
  `packages/components/src/shell-header/shell-header.tsx` and search for `isExternal` or
  `target`. If `ShellHeader` does not auto-detect external links, the `Components` and `Storybook`
  links must be passed with `target` and `rel` attributes. Check the `ShellHeaderNavItem` type.

  If the nav items support `target`/`rel` props, add them:

  ```ts
  { label: 'Components', href: 'https://docs.cascivo.com', target: '_blank', rel: 'noopener noreferrer' },
  { label: 'Storybook', href: 'https://storybook.cascivo.com', target: '_blank', rel: 'noopener noreferrer' },
  ```

  If `ShellHeader` does not support `target`/`rel` on nav items, leave them without — external
  navigation without `target="_blank"` is still acceptable; it is a UX preference, not a
  correctness bug.

  Also check: the mobile drawer in `Header.tsx` renders its own `<a>` tags. These DO support
  `target` and `rel` directly. Update those to match the external pattern used in `Footer.tsx`:

  ```tsx
  {
    NAV_LINKS.map((link) => {
      const isExternal = /^https?:\/\//.test(link.href)
      return (
        <a
          key={link.href}
          href={link.href}
          className="mobile-nav-link"
          aria-current={link.active ? 'page' : undefined}
          {...(isExternal ? { target: '_blank', rel: 'noopener noreferrer' } : {})}
          onClick={() => {
            isNavOpen.value = false
          }}
        >
          {link.label}
        </a>
      )
    })
  }
  ```
