# v35 — Tranche 1: Theme CSS MIME Fix

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stop all ten theme CSS files from being served as `text/html`. Fix the extensionless `@import` statements in `landing.css` and add dual export entries to `packages/themes/package.json`. This is the highest-priority fix: it resolves the ten console errors, restores theme token resolution across the page, and likely eliminates the cascade effects on the color-contrast and unused-CSS audits.

**Root cause recap:** `apps/landing/src/landing.css` imports themes without `.css` extension (`@import '@cascivo/themes/light'`). Vite resolves these via the themes `package.json` export map at build time, but the built asset URL lacks the `.css` suffix. Cloudflare Pages matches the extensionless path against its SPA fallback rule, serving `index.html` as `text/html`. The browser refuses this with a MIME-type security error.

**Architecture:** Two files change. No source code, no component logic, no behaviour.
