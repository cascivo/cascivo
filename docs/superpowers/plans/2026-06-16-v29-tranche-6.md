### Task 5: Drift check

- [ ] **Step 1: Run regen and check for diff**

  Run: `pnpm regen`
  Run: `pnpm exec vp check --fix`
  Run: `git diff --exit-code`

  If `git diff` shows changes:
  - The regenerated files differ from what is committed.
  - This usually means `registry.json` or generated story files need updating.
  - If `registry.json` changed: review the diff. If the change is correct (e.g. a new field
    was added by the generator), commit it: `git add registry.json && git commit -m "chore: regen registry"`
  - If unexpected files changed: investigate the generator script for unintended side effects.
  - Re-run `pnpm regen && git diff --exit-code` until it exits 0.

- [ ] **Step 2: Confirm exit 0**

  Run: `pnpm regen && pnpm exec vp check --fix && git diff --exit-code`

### Task 6: Breakpoint check

- [ ] **Step 1: Run `pnpm breakpoint:check`**

  Run: `pnpm breakpoint:check`

  If it fails:
  - Read the output: it will name the file and the off-scale literal.
  - Common causes: a block CSS module used a hardcoded pixel value (`768px`) instead of the
    canonical rem scale, or used a non-canonical rem value.
  - Canonical values: `30rem` (480px / sm), `40rem` (640px / md), `64rem` (1024px / lg),
    `80rem` (1280px / xl).
  - Replace the off-scale value with the correct canonical rem value.
  - Re-run.

- [ ] **Step 2: Confirm exit 0**

  Run: `pnpm breakpoint:check`
  Expected: exits 0.
