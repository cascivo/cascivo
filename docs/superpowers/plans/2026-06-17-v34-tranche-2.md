# v34 — Tranche 2: Publish Metadata + LICENSE

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give every Tier‑1 package the publish metadata npm needs — `license`, `repository` (+`directory`), `homepage`, `bugs`, `author`, `keywords`, a correct `files` allowlist, and `publishConfig: { access: "public", provenance: true }` — and add a root `LICENSE` file.

**Architecture:** Additive `package.json` edits only. Existing `name`, `version`, `exports`, `types`, `bin`, `sideEffects`, and build scripts are preserved. The repo is `urbanisierung/cascivo`; per-package `repository.directory` points at `packages/<name>`.

**Tech Stack:** npm publish metadata, SPDX license (MIT per Decision 2), vite+ (`vp`).

---

## Files

- Create: `LICENSE`
- Modify: `packages/{core,tokens,themes,icons,i18n,storage,react,mcp,registry,cli}/package.json`
- Modify: root `package.json` (add `license`, `repository`, `author` for documentation/inheritance)

### Task 1: Add the root LICENSE

**Files:** Create `LICENSE`

- [ ] **Step 1:** Create a root `LICENSE` with the chosen license (Decision 2: MIT). Use the current year (`2026`) and the maintainer/org as copyright holder. If the holder name is unknown, use the GitHub org `urbanisierung` and flag for confirmation — do not invent a personal legal name.
- [ ] **Step 2:** Add `"license": "MIT"` to the root `package.json` (the monorepo root is `private: true`; this documents intent and lets tooling inherit).

### Task 2: Define the shared metadata block

The same fields repeat across all ten Tier‑1 packages, differing only by `description` (already present), `directory`, `homepage` anchor, and `keywords`.

- [ ] **Step 1:** For each Tier‑1 package add:

  ```jsonc
  "license": "MIT",
  "author": "<maintainer/org>",
  "repository": {
    "type": "git",
    "url": "git+https://github.com/urbanisierung/cascivo.git",
    "directory": "packages/<dir>"
  },
  "homepage": "https://github.com/urbanisierung/cascivo/tree/main/packages/<dir>#readme",
  "bugs": "https://github.com/urbanisierung/cascivo/issues",
  "keywords": ["cascivo", "design-system", "react", "css", "signals", /* + per-pkg */],
  "publishConfig": { "access": "public", "provenance": true }
  ```

  Place these alongside the existing fields; keep the file valid JSON (no trailing commas in the real file). Do not reorder or remove existing keys.

- [ ] **Step 2:** Tune `keywords` per package (e.g. `icons` → add `"icons","svg"`; `mcp` → `"mcp","ai"`; `themes` → `"theme","dark-mode"`; `cli` → `"cli"`). Keep them short and accurate.

### Task 3: Set the `files` allowlist (Decision 5)

**Files:** the ten Tier‑1 `package.json` files

- [ ] **Step 1:** Add `files` where missing:
  - `core`, `icons`, `i18n`, `storage`, `mcp`, `registry`, `react`: `"files": ["dist"]` (`react`/`mcp` already have it — leave as-is).
  - `cli`: `"files": ["dist", "bin"]`.
  - `tokens`, `themes`: `"files": ["src"]` already present (CSS-only) — leave as-is.
- [ ] **Step 2:** Cross-check each package's `exports`/`types`/`bin` targets against its `files` allowlist — every path referenced by `exports` must be inside a `files` entry, or it won't be in the tarball. In particular:
  - `react` `exports["./styles.css"]` → `./dist/cascade.css` (inside `dist` ✓).
  - `cli` `bin` → `./bin/cascivo.mjs` (inside `bin` ✓) and `exports` → `./dist/...` (inside `dist` ✓).
  - `tokens`/`themes` `exports` → `./src/*.css` (inside `src` ✓).
  - Note any mismatch for T5's tarball audit; do not change `exports` here.
