# v42 — Tranche 4: `@cascivo/core` Observer Hooks

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the Web Awesome observer-element DX gap the cascivo way. Web Awesome ships
`wa-resize-observer`, `wa-mutation-observer`, and `wa-intersection-observer` as **declarative elements**;
cascivo's idiom is **signal hooks** (it already has `useInfiniteScroll`, an IntersectionObserver sentinel).
Add three pure, signal-driven hooks to `@cascivo/core`:

- **`useResizeObserver`** — observe an element's content box; expose a signal of the latest size/entry.
- **`useMutationObserver`** — observe DOM mutations on an element subtree; expose a signal of the latest
  records (with an options arg: `childList`/`attributes`/`subtree`/…).
- **`useIntersectionObserver`** — generic intersection (not the infinite-scroll sentinel); expose a signal of
  the latest entry / `isIntersecting` (with `root`/`rootMargin`/`threshold` options).

Each returns a target `ref` + a signal; each is built from `useSignal`/`useSignalEffect`/`useRef` (the
`infinite-scroll.ts` template), SSR-guarded, observer disconnected in teardown, exported from `index.ts`, and
unit-tested. **Pure additions** — no component changes.

**Architecture:** Mirror `packages/core/src/infinite-scroll.ts`: a `useRef<Element>(null)` target, a
`useSignal` holding the latest observation, and a `useSignalEffect` that (a) guards `typeof <Observer> !== 'undefined'`
and a non-null ref, (b) constructs the observer with a callback that writes to the signal, (c) observes the
ref, and (d) returns a teardown that `disconnect()`s. No `useEffect`/`useState`. Keep each hook's option object
stable/destructured so the effect re-subscribes only when meaningful inputs change.

**Tech Stack:** pure signal hooks (`useSignal`, `useSignalEffect`, `useRef` from `@cascivo/core`); Vitest +
Testing Library + jsdom (mock/polyfill the observers where jsdom lacks them — deterministic, no real layout).

      `useSignalEffect`: guard `typeof ResizeObserver`, construct with a callback writing the latest entry +
      derived size to the signals, `observe(ref.current, { box })`, teardown `disconnect()`.
- [ ] **Step 2:** Test: mock `ResizeObserver` in jsdom; mount a component using the hook; fire the observer

### Task 4: `useIntersectionObserver`
