# v53 — Tranche 6: New Types — Candlestick, Polar & Gauge

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the three highest-demand ECharts types cascivo lacks (E-8): **Candlestick** (OHLC/financial), **Polar**
(a polar coordinate system → rose / polar bar-line), and **Gauge** (a true speedometer with a needle — distinct from
Meter's bar/gauge and RadialBar's rings). Lens: **ECharts**. Out of scope (per the roadmap): 3D/GL, geo/maps, force/
network, candlestick-GL, the exotic long tail.

**Architecture:** each follows the established chart contract (`ChartFrame`, `role="img"`, fallback table, optional
tooltip, theme colors, i18n, a `*.meta.ts` with `intent`, tests, exports, generated docs).
- **Candlestick** (`charts/candlestick`): OHLC data `{ t, open, high, low, close }[]`; a band scale over time, a linear
  value scale; each candle is a wick `<line>` (low→high) + a body `<rect>` (open↔close) colored up/down (close≥open vs
  close<open); optional volume bars on a secondary axis. Tooltip lists O/H/L/C.
- **Polar** (`engine/polar.ts` + `charts/polar`): a reusable polar coordinate system — `angleScale` (band/linear over
  the circle) + `radiusScale` (linear value → radius). The `Polar` chart plots bars (rose) or a line/area in polar
  space, reusing `arcPath`. Unlocks rose charts and polar bar/line.
- **Gauge** (`charts/gauge`): a speedometer — a value arc over a `[min, max]` sweep, threshold-colored zones, tick
  marks + labels, and a **needle** pointing at the value, with a center value readout. Distinct from Meter (bar/simple
  gauge) and RadialBar (concentric rings).

**Tech Stack:** SVG + existing `arcPath`/`linePath` + new `engine/polar.ts`; `ChartFrame`; `@cascivo/i18n`;
`--cascivo-chart-*` tokens. No new dependency.
