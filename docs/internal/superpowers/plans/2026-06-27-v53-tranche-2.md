# v53 — Tranche 2: visualMap — Data-Driven Color & Size Encoding

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add ECharts' `visualMap` (E-3) — a component that maps a numeric **data dimension** to **color** and/or
**size** on a **continuous** or **piecewise** scale, with an interactive **legend**. Today nothing maps a data value
to a visual channel beyond per-series palette assignment; Heatmap's color is ad-hoc. Lens: **ECharts**.

**Architecture:** a `chrome/visual-map.tsx` exposing `VisualMap` (the legend) + a `mapVisual(value)` resolver. Two
modes: `continuous` (a smooth ramp legend with draggable thumbs that **filter** the visible range) and `piecewise`
(discrete buckets, each a clickable legend swatch that toggles its segment). The channel is `color`, `size`, or both.
Color ramps come from `engine/ramp.ts` — sequential and diverging ramps interpolated between CVD-safe palette anchors
in oklch space, so a data→color encoding is color-vision-safe by construction (Decision 3). Charts accept a
`visualMap` prop describing `{ field?, min, max, mode, channel, pieces? }`; each mark resolves its color/size from its
value. The legend filtering writes a visible-range signal the chart reads to dim/hide out-of-range marks.

**Tech Stack:** SVG + `engine/ramp.ts` (oklch interpolation over palette anchors); signals for the legend's
filter range; tokens for the ramp anchors. No new dependency.

### Task 1: CVD-safe ramps

**Files:** `engine/ramp.ts`

- [ ] **Step 1:** `sequentialRamp(t)` / `divergingRamp(t)` (t ∈ [0,1]) interpolate in oklch between CVD-safe palette
      anchors; return a CSS color. `rampStops(n)` returns n evenly-spaced colors (for piecewise + the legend gradient).
- [ ] **Step 2:** `ramp.test.ts`: endpoints map to the anchors; midpoints are monotonic in lightness; n stops returns n
      colors; no `NaN`/empty colors.

### Task 2: VisualMap legend + resolver

**Files:** `chrome/visual-map.tsx`

- [ ] **Step 1:** `mapVisual(value, opts)` → `{ color?, size? }` for `continuous` (ramp at `(value-min)/(max-min)`) or
      `piecewise` (the bucket's color/size). Clamp out-of-range.
- [ ] **Step 2:** `VisualMap({ mode, min, max, channel, pieces?, range })` renders the legend: continuous → a gradient
      bar with two draggable thumbs writing a `range` filter signal; piecewise → clickable swatches toggling pieces.
      `role="group"`, labelled, keyboard-operable (Arrow on thumbs; Enter on swatches); ≥44px coarse thumbs.
- [ ] **Step 3:** `visual-map.test.tsx`: `mapVisual` returns the expected ramp color at min/mid/max; piecewise picks the
      right bucket; dragging a thumb narrows the range signal; toggling a piece updates it.
