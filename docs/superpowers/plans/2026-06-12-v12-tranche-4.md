**Goal:** Pain #4 (bundle weight, hydration cost, RSC incompatibility) is answered with
numbers cascade publishes and CI enforces: a per-component size table with budgets, a
machine-enforced RSC boundary matrix demonstrated in a real Next.js Server Component, and
hydration measurements — plus the standing note that the copy-paste path has no library
bundle at all.

**Architecture:** A measurement script (`scripts/size/measure.ts`) builds isolated
per-component bundles from `@cascade-ui/react`, compares against committed
`size-budgets.json`, and emits the data the docs Size page renders. The RSC matrix is a
committed `rsc-matrix.json` + an enforcement test + an example page. Hydration numbers
extend whatever `apps/bench` already measures.

**Tech stack:** unchanged; measurement uses the in-tree bundler (Rolldown via vp/vite
API) — no size-limit dependency unless the in-tree route proves unworkable (decide in
Task 1, record the decision).

  `"use client"` presence ⇔ matrix says `client`; fail on components missing from the
  matrix entirely (new components must declare — master-plan risk #9)
- Modify: factory templates/spec guidance so factory components declare classification
