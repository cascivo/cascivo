# v38 — Tranche 3: System Registration (CLI, Storybook, Docs)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Register the now-complete `cyberpunk` theme in the three non-landing surfaces that enumerate
themes: the **CLI** (`cascivo theme`), **Storybook** (toolbar switcher), and the **docs app** (theme type,
switcher list, legacy-migration guard, CSS import, and any enumerated references). The landing page is T4.

**Architecture:** Each surface keeps its own hard-coded theme list — there is no shared registry. This
tranche updates each list and its paired CSS `@import`. The docs app additionally has a `legacy`
migration guard in `theme.ts` that lists themes; the new name must be considered there so persisted/legacy
theme values resolve correctly.

**Tech Stack:** TS/TSX edits + CSS imports; `vp` build/check.
