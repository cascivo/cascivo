# v50 — Tranche 4: Mobile Menu Completeness + Header Density

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Relieve the crowded small-screen header and make the mobile menu complete. On home the header `end` slot
renders the peek-eye **+** theme **+** search **+** GitHub, plus brand and the hamburger — five controls + brand at
320–360px — while the drawer lists nav links only, so the (cramped) theme/search/GitHub controls have no roomier
home. Surface those controls inside the drawer and trim the small-screen header. Lens: **designer.**

**Architecture:** `Header.tsx` `end` = peek `Tooltip`/button (`currentPath === '/'` only) + theme `Dropdown` +
`SearchButton` + GitHub `<a className="header-icon-link">`. The drawer `<nav className="mobile-nav-drawer">` maps
`drawerLinks = [...PRIMARY_LINKS, ...RESOURCE_LINKS]` into `.mobile-nav-link`s. The theme store is `theme`/`setTheme`/
`THEMES` (`../../theme`); GitHub href is `GITHUB_HREF`; search opens via `searchOpen.value = true`. The drawer already
has a focus trap (`drawerRef`) that picks up `a[href], button:not([disabled])`, so adding buttons/links inside it
participates correctly.

**Tech Stack:** Preact + `@cascivo/core` signals; existing `Dropdown`/`SearchButton`/theme store; `landing.css`. No
component-library change.

### Task 2: Trim the small-screen header density (M-D2)

**Files:** `apps/site/src/marketing/sections/Header.tsx`, `apps/site/src/marketing/landing.css`

- [ ] **Step 1:** Now that theme/GitHub (and maybe search) live in the menu, reduce the `end` density **below the
      nav breakpoint** (`max-width: 39.99rem`). Pick the lightest-touch option that keeps everything reachable:
      - Preferred: keep **search** in the header (it's the highest-frequency action) + the hamburger, and let
        **theme + GitHub** live in the drawer — relocating, not hiding. If a control is moved out of the header on
        small screens, it must exist in the drawer (Task 1) so nothing leaves the a11y tree (CLAUDE "never hide
        content").

- [ ] **Step 2:** If the home-only **peek-eye** is the tightest offender at 320px, consider gating it to slightly
      wider small screens (it's a decorative easter-egg, not essential nav) — but keep it reachable; do not remove
      the feature. Decide against the real render.
- [ ] **Step 3:** Whatever moves, confirm it's controlled by the **same** `max-width: 39.99rem` boundary as the rest
      of the mobile nav (no new breakpoint).
