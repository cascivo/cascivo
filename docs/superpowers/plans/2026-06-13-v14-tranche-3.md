**Goal:** Make the performance story honest. First diagnose _why_ shadcn tabs is `0` and
whether cascade's per-component numbers are inflated by one-time runtime; then fix the
baseline so incrementals are true marginal costs; then present standalone + incremental +
amortized lenses with every `0` explained.

**Architecture:** Investigation note → bench-runner changes (baseline composition + new
metrics) → `perf-data.ts` selectors → `PerformancePage.tsx` table + copy. Symmetric treatment
across all three libs to keep comparisons fair.

**Tech stack:** unchanged — Node bench runner, gzip via `zlib`, Vite matrix builds, Playwright.
