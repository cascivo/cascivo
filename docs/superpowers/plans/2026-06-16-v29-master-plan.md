# v29 â€” Blocks (`/blocks`) â€” Master Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `/blocks` gallery and `/blocks/:name` detail page to the landing, backed by 8
production-ready UI blocks in `packages/components/src/blocks/`, CLI `block/` prefix support, and
screenshot-based gallery card thumbnails.

**Architecture:** All new block components live in `packages/components/src/blocks/<name>/`.
They import only from `@cascivo/react`. The landing adds two new routes (`/blocks` and
`/blocks/:name`), a single `blocks-data.ts` import seam, and a vite alias map for the 8 known
block source paths. `registry.json` gains a `blocks` array. The CLI `add` command resolves names
prefixed with `block/` to that array. Screenshots are committed PNGs generated by extending the
existing `gen-demo-screenshots.mjs` script.

**Tech Stack:** `@cascivo/core` (useSignal, useSignals, useSignalEffect), `@cascivo/react`
(Button, Input, Textarea, Select, Checkbox, Toggle, Card, Badge, Alert, Tabs), React 18,
`@container`, CSS modules, Vitest + @testing-library/react, Playwright for screenshot capture,
Node.js for registry script extension.

## Key Interfaces

### `BlockMeta` (`packages/components/src/blocks/types.ts`)

```ts
export type BlockCategory = 'auth' | 'dashboard' | 'marketing' | 'shell'

export type BlockMeta = {
  name: string // 'auth-login' â€” matches directory name
  displayName: string // 'Login Form'
  description: string // one-line prose
  category: BlockCategory
  tags: string[]
  screenshot: {
    light: string // '/blocks/screenshots/auth-login-light.png'
    dark: string // '/blocks/screenshots/auth-login-dark.png'
  }
}
```

### `BlocksData` entry (`apps/landing/src/pages/blocks/blocks-data.ts`)

```ts
export type BlockEntry = {
  meta: BlockMeta
  load: () => Promise<{ default: React.ComponentType }>
}

export const BLOCKS: BlockEntry[] = [
  /* 8 entries */
]
export const findBlock = (name: string): BlockEntry | undefined =>
  BLOCKS.find((b) => b.meta.name === name)
export const blocksByCategory = (cat: BlockCategory | 'all'): BlockEntry[] =>
  cat === 'all' ? BLOCKS : BLOCKS.filter((b) => b.meta.category === cat)
```

## Cross-Tranche Rules

1. Every block file must pass `grep -n 'useState\|useEffect\|useContext\|useReducer\|useLayoutEffect'`
   returning zero matches before the tranche is complete.
2. Every block CSS file must pass `pnpm breakpoint:check`.
3. `pnpm exec vp run @cascivo/components#test` must exit 0 at the end of T2, T3, and T4.
4. The landing must build (`pnpm exec vp run @cascivo/landing#build`) at the end of T1 (skeleton)
   and T5 (full).
5. `registry.json` entries must be added in alphabetical order within the `blocks` array.
