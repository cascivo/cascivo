# v48 — Tranche 2: PieChart Donut Center + Thickness/InnerRadius + Square Size

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close three genuine `PieChart` gaps the `lifosy` `DonutChart` needs (charts feedback C2/C3/C4): (1) **donut
center content** — `centerValue?: string` + `centerLabel?: string` (mirroring the consumer's API exactly) plus an
optional `centerSlot?: ReactNode`, rendered in the ring center only when `donut`; (2) **controllable thickness** —
`thickness?: number` (ring width) and/or `innerRadius?: number`, defaulting to today's hardcoded `outerR*0.6` so
existing donuts are pixel-identical, clamped to a valid range; (3) **square `size?: number` shorthand** that sets
`width===height`. All additive; the default render is unchanged.

**Architecture:** `pie-chart.tsx` computes `cx`/`cy`, `outerR = Math.min(cx,cy) - 8`, and `innerR = donut ?
outerR*0.6 : 0` inside the `ChartFrame` render-prop (`{({ width, height }) => …}`). Slices are `arcPath(cx, cy,
outerR, innerR, start, end)` from `engine/shape.ts`. There is no center element today. `ChartFrame` takes
`width`/`height` (default height 300 via `resolvedHeight`). The component is `useSignals()`-first; `hidden` is a
`useSignal(Set)`. Center text/slot must render **inside the SVG** (as `<text>`/`<foreignObject>`) or as an absolutely
positioned overlay in the frame — choose the SVG-native `<text>` path for the value/label (crisp, in the a11y tree,
theme-token fill) and reserve `centerSlot` (foreignObject/overlay) for arbitrary content.

**Tech Stack:** owned TSX in `@cascivo/charts`; `engine/shape.ts` `arcPath`; `@cascivo/core` signals; SVG `<text>`
for center value/label (fill via `--cascivo-color-*` tokens); Vitest + @testing-library/react; vite+ (`vp`).

### Task 2: Add `size` shorthand + thickness/innerRadius

**Files:** `packages/charts/src/charts/pie-chart/pie-chart.tsx`

- [ ] **Step 1:** Add `size?: number` to `PieChartProps`. When provided, it sets both width and height
      (`fixedWidth ?? size`, `height ?? size`) so a single number renders a square donut — resolve precedence so
      explicit `width`/`height` still win if both are passed (document `size` as the shorthand). Default behavior
      (neither given) unchanged.
- [ ] **Step 2:** Add `thickness?: number` and `innerRadius?: number`. Inside the render-prop, compute the effective
      inner radius: if `innerRadius` given → use it (clamped to `[0, outerR)`); else if `thickness` given →
      `outerR - thickness` (clamped); else → `outerR * 0.6` (today's default). Only applies when `donut`; `pie`
      stays `innerR = 0`.
- [ ] **Step 3:** Ensure the clamp prevents `innerR < 0` or `>= outerR` (which would produce a NaN/inverted arc).
      Default render (donut, no thickness/innerRadius) must be **byte-identical** to today.
