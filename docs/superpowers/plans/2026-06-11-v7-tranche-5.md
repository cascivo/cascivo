- [ ] **Step 1:** `a11y.spec.ts` — zero violations, light AND dark, including open-modal
      state:

```ts
import AxeBuilder from '@axe-core/playwright'
import { expect, test } from '@playwright/test'

for (const theme of ['light', 'dark'] as const) {
  test(`axe sweep — ${theme}`, async ({ page }) => {
    await page.goto('/')
    await page.evaluate((t) => {
      document.documentElement.dataset.theme = t
    }, theme)
    const scan = await new AxeBuilder({ page })
      .withTags(['wcag2a', 'wcag2aa', 'wcag21a', 'wcag21aa'])
      .analyze()
    expect(scan.violations).toEqual([])
  })
}

test('axe sweep — modal open', async ({ page }) => {
  await page.goto('/')
  await page.getByRole('button', { name: 'New deploy' }).click()
  const scan = await new AxeBuilder({ page }).withTags(['wcag2a', 'wcag2aa']).analyze()
  expect(scan.violations).toEqual([])
})
```

Violations are component or page bugs — fix them at the source (component fixes go to
`packages/components` with tests, per CLAUDE.md; page fixes in landing). `expect([]).toEqual`
prints the full violation objects on failure — read them.

- [ ] **Step 2:** `motion.spec.ts`:

```ts
import { expect, test } from '@playwright/test'

test.use({ reducedMotion: 'reduce' })

test('reduced motion: no view transition, no autonomous animation', async ({ page }) => {
  await page.goto('/')
  // theme switch still works, instantly
  await page.click('.header-theme-dot[data-theme-name="dark"]')
  await expect(page.locator('html')).toHaveAttribute('data-theme', 'dark')
  // nothing animates by itself: two screenshots 1.5s apart are identical

  const b = await page.screenshot({ fullPage: false })
  expect(a.equals(b)).toBe(true)
})
```

(If a chart renders sub-pixel anti-aliasing noise, scope the screenshot to `.console-frame`'s

### Task 4: Design review loop + copy edit

- [ ] **Step 1: Screenshot pass.** With the preview running, capture fullpage screenshots:
      light + dark + warm × 1440 and 390 (six images, Playwright one-liners or the browser
      tooling available). LOOK at them against this checklist (from the v5/v7 research):

- Density: console reads as real software (no oversized empty cards, no oceans of padding).
- Hierarchy: one H1; section heads consistent scale; mono eyebrows consistent.
- Alignment: console grid lines up across regions; KPI baselines align.
- Theme integrity: hairlines visible in dark; warm card distinct in all themes; charts
  legible in flat/minimal.
- Slop sweep: zero gradients/glassmorphism/emoji/icon-grids/testimonials anywhere.

Fix what fails; re-screenshot. Iterate until two consecutive passes produce no findings —
if it's not converging after 2 attempts, use the design-iterator pattern (multiple
screenshot-analyze-improve cycles) rather than one-shot tweaks.
