**Goal:** Restate all color tokens in the perceptually-uniform oklch color space; add semantic
radius/shadow/focus tokens; restyle the three existing themes to modern aesthetics; update chart
tokens. End state: no hardcoded hex in any component CSS, contrast gate green for all three
themes, visual regression baselines re-recorded (diff = restyle review).

**Architecture:** Three-level token system is preserved. Changes hit level 1 (primitives → oklch)
and level 2 (semantic: new radius/shadow/focus tokens). Level 3 (component tokens) auto-restyles
by inheritance — no per-component edits needed except fixing any hardcoded hex. The component-level
migration is script-assisted (sed + grep audit). Visual regression baseline re-record is a
deliberate PR that human reviews by approving the visual diff.

**Tech stack:** CSS (oklch), TypeScript (scripts), Playwright (visual regression snapshots),
`scripts/quality/contrast-check.ts` (extended).

**Conventions:** Only change token files and theme files in this tranche. Do not touch component
logic. Run `vp check` before each commit. Visual regression re-record is a separate commit clearly
labeled "chore: re-record visual regression baselines (T4 restyle)".

- Token file: `packages/tokens/src/index.css`
- Theme files: `packages/themes/src/light.css`, `dark.css`, `warm.css`
