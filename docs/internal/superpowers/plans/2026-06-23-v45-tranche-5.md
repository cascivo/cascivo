# v45 — Tranche 5: Landing Icon Showcase

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the catalog **visible on the landing page**. Add a lightweight `IconShowcase` section — a
headline stating the catalog size ("~450 icons, one import"), a responsive (optionally auto-scrolling,
reduced-motion-safe) grid of a representative subset, and a link to the `/icons` docs gallery. Lazy-loaded like
the other landing sections and registered in `SectionNav`.

**Architecture:** The landing app lazy-imports each section in `apps/landing/src/App.tsx` and lists them in
`SectionNav`. `IconShowcase` follows that pattern. **Critical CLAUDE.md rule:** the React landing gets **no**
signals Babel transform — if the section reads any `signal.value` during render it must call `useSignals()`
first, or it will not re-render. A static subset grid needs no signal; only add one if there is interactivity
(e.g. a "shuffle"/hover). Icons come straight from `@cascivo/icons` named imports (a curated representative
subset, ~24–40 glyphs) — tree-shaking keeps the bundle cost to just those.

**Tech Stack:** React + `@cascivo/icons` (named imports of a representative subset); `@cascivo/core`
(`useSignals` only if interactive); CSS module for the grid/marquee; the landing router for the `/icons`
external link (it points at the docs app).
