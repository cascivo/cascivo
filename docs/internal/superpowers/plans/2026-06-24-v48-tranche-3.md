# v48 — Tranche 3: PieChart Empty-State + Percentage Tooltip

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the two remaining `PieChart` polish gaps for the `lifosy` `DonutChart` (charts feedback C5/C6):
(1) a **visible empty state** — at `total === 0`, render a theme-aware `NO DATA` placeholder (configurable
`emptyLabel?`, defaulting from the `@cascivo/i18n` built-in catalog) instead of an empty/NaN arc; keep the existing
a11y fallback table. (2) a **percentage tooltip** — add a `tooltipFormat?: (p) => string` escape hatch and a richer
default that shows `value (pct%)` in the slice color (percent reaches the formatter). Both additive; the default
non-tooltip render and the existing a11y behavior are unchanged.

**Architecture:** `ChartFrame` already sets `data-state="empty"` (driven by the pie's `hasData`/`data-state` prop)
and renders the `fallback` table, but the SVG body is whatever `children` returns — currently an empty `<g>` at
`total===0`. The empty placeholder belongs in `ChartFrame` (shared, so other charts benefit) gated on
`data-state==='empty'`, OR in the pie's render-prop; prefer `ChartFrame` so it's consistent and keeps the pie lean.
For tooltips: `TooltipModel { points, format? }` (`data-point.ts`) already supports a `format` callback used by the
aria-live announcement and the visible tooltip; `defaultFormat = "label: value"`. The pie's `buildTooltip` builds
`ChartPoint`s with `label`/`value` but no percent. Add an optional `percent?: number` to `ChartPoint`, populate it
in `buildTooltip` (computed from `visible`/`total`), thread `tooltipFormat` into the `TooltipModel.format`, and make
the pie default formatter emit `value (pct%)`. Slice color in the tooltip text: pass the slice color through (e.g.
via `ChartPoint` `seriesId` → color lookup, or an added optional color field consumed by the tooltip renderer).

**Tech Stack:** owned TSX in `@cascivo/charts`; `core/chart-frame.tsx` + `core/data-point.ts`; `@cascivo/i18n`
built-in catalog for the empty label; `@cascivo/core` signals; Vitest + @testing-library/react; vite+ (`vp`).

- [ ] **Step 1:** Add `percent?: number` (and, if the tooltip renderer needs it, an optional `color?: string`) to
