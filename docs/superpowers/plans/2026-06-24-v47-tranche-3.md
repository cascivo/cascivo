# v47 — Tranche 3: Window-Scoped Tokenization in the View Path

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stop tokenizing the whole document on every render. Today `code-editor.tsx` calls
`tokenizeDocument(getGrammar(language), highlightText.value)` over the **entire** document and `renderRows` then
slices `[start, end)` — the work is O(document), which the T1 perf test proves is the limit. Switch the view path
to compute the **visible window first**, then call `tokenizeRange(grammar, allLines, start, end, index)` (the T2
engine entry) and feed **only the window's tokens** to `renderRows`. Do the same in read-only `Highlight`.
Highlighting output must be **byte-identical**; decorations, find-match highlight, bracket matching, and the
active-line gutter must still align. After this tranche the **T1 perf regression test goes GREEN**.

**Architecture:** The window math is unchanged (`VIRTUALIZE_THRESHOLD = 1000`, `OVERSCAN = 12`; windowing
disabled when `wrap` is on). The new ordering is: compute `total` from `highlightText.value` (a split count — no
token arrays built), derive `start`/`end` from `scrollTop`/`lineHeight`/`viewport`, then `tokenizeRange` over
`[start, end)` only. `tokenizeRange` returns rows aligned so index 0 corresponds to `start`; `renderRows` is
adapted to index relative to `start` while keeping **absolute-index keys** (windowing-safe). `Gutter count={total}`
and the `topPad`/`bottomPad` spacers keep using the **TOTAL** line count so scroll height is preserved. The
`LineStateIndex` is mutable infrastructure → held in a `useRef` (like v46's `createHistory()` handle), seeded on
mount and refreshed alongside the rAF-debounced `highlightText` signal. `Highlight` renders all rows today and has
no scroll metrics — its render set stays the whole doc, but it routes through `tokenizeRange(0, total)` so output
is identical; the measurable win lands in `CodeEditor`.

**Tech Stack:** owned TypeScript in `@cascivo/editor` (no new runtime deps); `@cascivo/core` signals/`cn`; the T2
`tokenizeRange` + `LineStateIndex`; the existing `renderRows`/`Gutter` view; `useRef` for the line-state index
(mutable infra); Vitest + Testing Library for output-unchanged + window-bounded tests. Signals only; no banned
hooks; `useSignals()` first.
