> Current-state assumptions — RE-VERIFY before starting:
>
> - Storybook 10.4.2 at `apps/storybook/`, addons: `['@storybook/addon-a11y']` (main.ts)
> - `.storybook/preview.tsx`: custom `withTheme` decorator wrapping stories in
>   `<div data-theme=…>`; `globalTypes.theme` toolbar with 6 items incl. 'side-by-side';
>   `parameters.layout: 'fullscreen'`
> - Defect being fixed: top-layer elements (Popover API — Menu, Tooltip, Modal, HeaderPanel,
>   Toast…) escape the wrapper div → unthemed; html/body never themed

> - `@cascade-ui/charts` is NOT a storybook dependency; 16 charts exist in
>   `packages/charts/src/charts/`

**Goal:** Working toolbar theme switcher (all five themes, including portal/popover content),
realistic story widths, complete category grouping (same taxonomy as docs), and stories for all
16 charts.

**Architecture:** Replace the custom theme decorator with `@storybook/addon-themes`'
`withThemeByDataAttribute` pointed at `html` (`parentSelector`), so the entire document including
the top layer inherits the theme. Keep a slim frame decorator for padding/background. Grouping
via explicit `title` in every story meta, prefixes derived from registry `meta.category`.

**Tech stack:** Storybook 10, @storybook/addon-themes, react-vite framework.

- [ ] **Step 2: Rewrite `preview.tsx`:**

```tsx
import type { Preview, Renderer } from '@storybook/react-vite'
import { withThemeByDataAttribute } from '@storybook/addon-themes'
import '@cascade-ui/themes/light'
import '@cascade-ui/themes/dark'
import '@cascade-ui/themes/warm'
import '@cascade-ui/themes/flat'
import '@cascade-ui/themes/minimal'

const preview: Preview = {
  decorators: [
    withThemeByDataAttribute<Renderer>({
      themes: {
        light: 'light',
        dark: 'dark',
        warm: 'warm',
        flat: 'flat',
        minimal: 'minimal',
      },
      defaultTheme: 'light',
      attributeName: 'data-theme',
      parentSelector: 'html',
    }),
  ],
  parameters: {
    layout: 'centered',
  },
}

export default preview
```

This deletes the custom `withTheme` decorator, the `globalTypes.theme` toolbar, the
`initialGlobals`, and the 'side-by-side' option (deferred — see roadmap Deferred list).

---### Task 2: Realistic story widths

**Files:**

- Modify: `apps/storybook/.storybook/preview.tsx` (decorator)
- Modify: individual story files that need `fullscreen`

- [ ] **Step 1: Global width frame.** `layout: 'centered'` (set in Task 1) centers stories but
      doesn't constrain width. Add a second decorator AFTER the theme decorator in
      `preview.tsx`:

```tsx
import React from 'react'
import type { Decorator } from '@storybook/react-vite'

const withFrame: Decorator = (Story, context) => {
  if (context.parameters['layout'] === 'fullscreen') return <Story />
  return (
    <div style={{ inlineSize: 'min(26rem, 90vw)' }}>
      <Story />
    </div>
  )
}

// decorators: [withThemeByDataAttribute(...), withFrame]
```

26rem ≈ a realistic form column; stories that already wrap themselves in narrower containers
are unaffected (the frame is a max, content can be narrower).

`Breadcrumb`, `Terminal`, `AiChat`.

import type { Meta, StoryObj } from '@storybook/react-vite'

const meta: Meta<typeof LineChart> = {
  title: 'Charts/LineChart',
  component: LineChart,
  parameters: { layout: 'fullscreen' },
  decorators: [
    (Story) => (
      <div style={{ inlineSize: 'min(40rem, 90vw)', padding: '2rem' }}>
        <Story />
      </div>
    ),
  ],
}
export default meta
type Story = StoryObj<typeof LineChart>
