# v53 — Tranche 7: Configurable Animation, Extension Hook & Comparison Docs

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land the closing polish: a **configurable animation** API (C-2 — Chart.js's default-on, tunable
animations), a minimal **extension hook** (C-3 — cascivo's owned-code answer to Chart.js plugins), and the **"vs
chart.js / vs echarts" comparison + recipe docs**. v52 added reduced-motion CSS transitions; this makes
duration/easing/per-property tunable and opens a draw seam, then documents the v53 surface. Lens: **Chart.js + docs**.

**Architecture:**
- **Animation config** (`core/chart-frame.tsx` + CSS): a `transition?: boolean | { duration?; easing?; properties? }`
  prop. When set, the frame writes the chosen duration/easing into CSS custom properties the
  `chart-frame.module.css` transition rules consume (e.g. `--cascivo-chart-anim-dur`), so the existing
  reduced-motion-gated transitions become tunable without JS animation loops. `false` disables; unset keeps the v52
  defaults. Still fully off under `prefers-reduced-motion`.
- **Extension hook** (`core/chart-frame.tsx`): `onBeforeDraw?`/`onAfterDraw?` render-prop slots receiving the resolved
  size + scales, so a user can inject custom SVG **behind** or **over** the marks (a watermark, a custom region, an
  extra series) — a tiny seam, not a plugin framework (Decision 8).
- **Docs:** a "cascivo vs chart.js vs echarts" comparison page (honest matrix: a11y, bundle/zero-deps, renderer
  SVG/Canvas, chart-type count, dataZoom/visualMap/transforms/toolbox, interaction, animation, owned code; cascivo
  leads a11y/deps/owned-code; ECharts leads breadth/3D/geo; Chart.js leads tiny-bundle simplicity) + a recipe gallery
  (folding in the v52 deferral): dataZoom, connected charts, visualMap heatmap, transform+encode, axis crosshair,
  decimated 100k-point line, toolbox export, candlestick, polar rose, gauge.

**Tech Stack:** `core/chart-frame.tsx` + CSS custom properties; `apps/site` (Preact) comparison + recipe pages reusing
the existing code-block/playground. No new dependency.
