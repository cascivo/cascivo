# v53 — Tranche 1: Zoom, Pan & Connect

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Finally ship the dashboard interaction cascivo has deferred twice (v51, v52): **dataZoom + inside zoom-pan +
chart connect** (E-1/E-2 — ECharts' signature). Today there is a v52 `Brush` (a fixed window) but no slider-dataZoom,
no wheel/drag zoom-pan inside the plot, and no `syncId`/`connect` so one chart drives others. Lens: **ECharts**.
Foundational — the crosshair (T4) builds on the zoomable plot.

**Architecture:** three pieces.
- **DataZoom slider** (`chrome/data-zoom.tsx`): generalizes `chrome/brush.tsx` — a track with two handles *and* a
  draggable body (move the whole window). Bound to a `Signal<[startIndex, endIndex]>`; resolution-independent (0–100
  viewBox) like the Brush; `role="slider"` handles, Arrow/Home/End, ≥44px coarse targets.
- **Inside zoom-pan** (`core/chart-frame.tsx`): a `transform` signal `{ k, tx }` (or a visible `[i0, i1]` index
  window) driven by wheel (zoom toward cursor), drag (pan), pinch, and keyboard (`+`/`-`, arrows, `0`-reset). Axes
  re-tick to the visible domain. An always-present, keyboard-reachable "Reset zoom" control. Listeners attach via
  `useSignalEffect` (no `useEffect`).
- **Connect/sync** (`core/sync.ts`): a module-level registry keyed by `syncId` holding a shared window + hovered-x
  signal; charts with the same `syncId` mirror each other's zoom window and hover. Ref-counted; cleaned via
  `useSignalEffect`.

**Tech Stack:** `core/chart-frame.tsx`, new `chrome/data-zoom.tsx` + `core/sync.ts`; signals from `@cascivo/core`;
`useSignalEffect` for wheel/drag listeners. No new dependency.
