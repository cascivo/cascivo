# v39 — Tranche 2: `arcade` Retro Theme (8-bit / Pixel)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a twelfth first-party theme — `arcade` — that captures RetroUI's literal identity: an **8-bit
/ pixel-arcade** aesthetic distinct from the existing `brutalist` (quiet/structural) and `cyberpunk` (v38,
dark/neon). Limited bright retro palette, a **pixel display font** (via T1's `--cascivo-font-display`),
blocky pixel borders, hard offset shadows, and a small set of **opt-in, reduced-motion-safe** pixel/CRT
effects (pixel-corner edges, scanline). Register it **everywhere a theme is registered** — the same ~12-file
sweep v38 used for `cyberpunk`. No component TSX changes; a drop-in CSS file gated by the parity + chart-CVD
tests.

> **Deferral note (master-plan Decision 4):** only T1's font token is a hard dependency. If the maintainer
> is theme-fatigued after v38's `cyberpunk`, T2 may be cut without blocking T3/T4/T5. If proceeding, follow
> below.

**Architecture:** A theme is `@import '@cascivo/tokens';` then
`@layer cascivo.theme { [data-theme='arcade'] { …--cascivo-* overrides… } }`. Author by copying a light,
high-chroma skeleton (or `brutalist.css` for the radius-0/hard-shadow structure) so the exact `--cascivo-*`
set is present, then re-color. `chart-palette.test.ts` runs CVD simulation — borrow a passing ramp, don't
hand-pick 8 retro hues.

**Tech Stack:** CSS-only (incl. progressive `@function`/`if()` with static fallbacks); vite+ (`vp`).

      Optionally set `--cascivo-font-sans` to a chunky grotesk; keep `-mono` for code.
- [ ] **Step 3:** Chart ramp — keep `brutalist.css`'s `--cascivo-chart-1..8` (already CVD-safe) as the start;
      adjust lightness/chroma minimally for the arcade base — **do not** replace all eight with retro hues.
      Set `--cascivo-chart-grid`/`-axis` to low-contrast values readable on the base.

### Task 5: Opt-in pixel/CRT effects (progressive, reduced-motion-safe)

**Files:** `packages/themes/src/arcade.css`

- [ ] **Step 1:** Add theme-scoped, **opt-in** effects behind marker classes/`data-` attributes (so they never
      fire on arbitrary components): e.g. a pixel-corner edge treatment and a subtle scanline overlay.
      `cyberpunk.css` (v38) is the reference for the opt-in marker + keyframe pattern — reuse it; do not

- [ ] **Step 2:** Every animated/`@function`/`if()` declaration has a **static fallback** immediately
      preceding it (`fallback:check`); wrap all keyframe effects so `@media (prefers-reduced-motion: reduce)`

- [ ] **Step 3:** If an effect needs a new custom property (e.g. a scanline color), add it to **all 12** theme
      files with a sane neutral value so `parity.test.ts` stays green (master-plan Cross-Tranche rule 2).
