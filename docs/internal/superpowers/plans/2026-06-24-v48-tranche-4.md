# v48 — Tranche 4: StackedBarChart Ergonomics + Per-Segment Tooltip + Label Thinning + Tiny Sizing

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the `BarChart` (`mode: 'stacked'`) gaps for the `lifosy` `StackedBarChart` (charts feedback
C8/C10/C11/C12): (1) **authoring ergonomics** — ship a pure, dependency-free `toStackedSeries(rows)` helper that
pivots row-oriented `{ label, segments: { key, value, color }[] }[]` into `BarChartSeries[]` + `x`/`y` accessors
(preserving per-segment `color`), and document both it and the manual pivot in a cookbook; (2) **stacked tooltip
breakdown** — a `tooltipFormat?` escape hatch plus a stacked default that shows `label · total` then each non-zero
segment in its layer color; (3) **X-label thinning** — confirm `xTicks` always includes the final category, or add
`xLabelEvery?`; (4) **tiny embedded sizing** — a test proving `plain` + explicit `width`/`height` render crisply
< 200px tall with no clipped axis. All additive; default grouped/stacked render unchanged.

**Architecture:** `bar-chart.tsx` takes column-oriented `series: BarChartSeries<Datum>[]` + `x`/`y` accessors;
`mode: 'stacked'` computes offsets via `stackSeries()` (`engine/shape.ts`) and honors per-series `color`
(`:195`/`:281`). The tooltip (`buildTooltip`, gated on `tooltip`) builds per-point `ChartPoint`s with `label`/`value`
(`x(d)`/`y(d)`) — no stacked breakdown. `xTicks`/`yTicks` are tick **counts** passed to `Axis`/`GridLines`. The new
helper is pure data transformation living in `engine/` (no React); it returns `{ series, x, y }` so a consumer writes
`<BarChart {...toStackedSeries(rows)} mode="stacked" … />`. The stacked tooltip default needs per-layer values at a
category — derivable from `series` + the focused category; thread `tooltipFormat` into `TooltipModel.format` (same
seam as the pie in T3) and build a multi-line default for stacked mode.

**Tech Stack:** owned TS/TSX in `@cascivo/charts`; `engine/stacked.ts` (pure); `engine/{shape,scale}.ts`;
`core/data-point.ts` `TooltipModel`; `@cascivo/core` signals; Vitest + @testing-library/react; vite+ (`vp`).

### Task 2: The `toStackedSeries` pivot helper

**Files:** `packages/charts/src/engine/stacked.ts`, `packages/charts/src/engine/stacked.test.ts`

- [ ] **Step 1:** Define the row types and helper:

      ```ts
      export interface StackedSegment { key: string; value: number; color?: string }
      export interface StackedRow { label: string; segments: readonly StackedSegment[] }
      // Returns props spreadable into <BarChart mode="stacked" />
      export function toStackedSeries(rows: readonly StackedRow[]): {
        series: BarChartSeries<{ x: string; y: number }>[]
        x: (d: { x: string; y: number }) => string
        y: (d: { x: string; y: number }) => number
      }
      ```

- [ ] **Step 2:** Implement: collect the **union of segment keys in first-seen order** across rows → one
      `BarChartSeries` per key (`id`/`label` = key, `color` = the first non-undefined color for that key); for each
      series, map every row to `{ x: row.label, y: segment.value ?? 0 }` (missing segments → 0 so stacks align).
      `x`/`y` are the identity accessors. Pure, dependency-free, no React import.
- [ ] **Step 3:** Tests (`stacked.test.ts`): ragged rows (a key missing in some rows → 0), color preserved per key,
      key order stable, output spreads into `BarChart` shape. Property-style check: total per category equals the sum
      of its segment values.
