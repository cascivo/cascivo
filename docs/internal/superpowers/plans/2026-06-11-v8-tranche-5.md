**Goal:** A `/layouts` gallery page in docs that renders every layout primitive, section and
block live — grouped, with descriptions and copy-paste `npx cascade add <name>` commands —
linked from the docs nav. Storybook stories for everything v8 added.

**Architecture:** One new docs page (`src/pages/LayoutsPage.tsx`) driven by `registry.json`
types (`layout`/`section`/`block`) for names/descriptions/commands, with a hand-written
preview map (entry name → JSX) for the live renders — mirroring how `demos.tsx` maps
component names to demos. Blocks that are full pages (console-app, login-page, …) render in
a scaled, non-interactive frame or as a constrained live mount — decide per block, favor
live.

**Tech stack:** unchanged — Preact docs app, preact-iso router.

- [ ] **Step 1:** `layout-previews.tsx` — the preview map:

```tsx

export const previews: Record<string, () => JSX.Element> = {
  'layout/auto-grid': () => <AutoGrid min="8rem">{…six bordered tiles…}</AutoGrid>,
  'layout/masonry': () => <Masonry cols={3}>{…varied-height tiles…}</Masonry>,
  'section/hero': () => <Hero variant="split" title="…" … />,

```

      Fixture content rules: realistic copy (anti-slop), varied tile heights for masonry,
      cascade Buttons in hero actions, StatsBand with real-looking numbers + sparkline
      trends. Section previews render at reduced scale if needed — prefer natural size in a

      frame, not the viewport.
