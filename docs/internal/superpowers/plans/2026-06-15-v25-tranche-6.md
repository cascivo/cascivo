**Goal:** Enhance `scripts/readme/generate.ts` to include (1) a shared link bar after each package title (cascivo.com · docs.cascivo.com · storybook.cascivo.com · GitHub) and (2) a richer footer with all three website links. The function inserts `SITE_LINKS` after the title/description block and replaces the current single-link footer. The rest of the script is unchanged. Per-package body content remains in `readme.body.md` files.

**Tech Stack:** Node.js + `fs` (existing). No new dependencies. `pnpm readme:generate` (part of `pnpm regen`) runs the updated script.

### Task 2: Audit and create missing readme.body.md files

Every package and app should have a `readme.body.md` with at least one sentence of custom
content. Without it, the generated README has only the title, description, link bar, install
block (for packages), and footer — no human-written context.

- [ ] **Step 1: Find all packages and apps with no readme.body.md**

  ```sh
  node -e "
    const { readdirSync, existsSync } = require('fs')
    const { join } = require('path')
    const root = process.cwd()
    const groups = ['packages', 'apps', 'apps/examples']
    for (const group of groups) {
      const base = join(root, group)
      if (!existsSync(base)) continue
      for (const entry of readdirSync(base, { withFileTypes: true })) {
        if (!entry.isDirectory()) continue
        const dir = join(base, entry.name)
        if (!existsSync(join(dir, 'package.json'))) continue
        const hasBody = existsSync(join(dir, 'readme.body.md'))
        const pkg = JSON.parse(require('fs').readFileSync(join(dir, 'package.json'), 'utf8'))
        if (!hasBody) console.log('MISSING:', pkg.name ?? dir)
      }
    }
  "
  ```

  This prints every workspace package/app that has no `readme.body.md`.
