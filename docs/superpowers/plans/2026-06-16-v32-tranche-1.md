### Task 1: Fix the landing router — skip `/demos/` paths

The landing SPA's `initRouter()` function intercepts all same-origin anchor clicks and calls `navigate()` to route them through the signal-based SPA. But `/demos/<slug>/` are separate assembled SPAs, not routes in the landing. The fix: add a guard so any click whose `pathname` starts with `/demos/` falls through to the browser's native navigation.

**Files:**
- Modify: `apps/landing/src/router.ts`

- [ ] **Step 1: Read `apps/landing/src/router.ts` in full**

  Locate the `initRouter()` function. Find the block of early-return guards that already handle cross-origin links, `target="_blank"`, and `download` attributes:

  ```ts
  if (url.origin !== location.origin) return
  if (a.target === '_blank') return
  if (a.hasAttribute('download')) return
  ```

- [ ] **Step 2: Add the `/demos/` guard after the existing guards**

  In `initRouter()`, immediately after the `download` guard, add:

  ```ts
  if (url.pathname.startsWith('/demos/')) return
  ```

  The resulting guard block in `initRouter()` must read:

  ```ts
  if (url.origin !== location.origin) return
  if (a.target === '_blank') return
  if (a.hasAttribute('download')) return
  if (url.pathname.startsWith('/demos/')) return
  e.preventDefault()
  navigate(url.pathname)
  ```

- [ ] **Step 3: Verify no other test or source file references this behavior**

  Run:

  ```bash
  grep -r "/demos/" apps/landing/src/ --include="*.ts" --include="*.tsx"
  ```

  Expected: only `data.ts` (the `liveHref` definitions like `'/demos/deploy/'`) and no test files that mock the router behavior. The router itself has no test file in this app.

### Task 2: Fix the SideNav CSS — split overflow axes

The `.sideNav` CSS rule uses `overflow: hidden`, blocking both horizontal overflow (needed for the collapse animation) and vertical overflow (scrolling to items below the fold). The fix: split into `overflow-x: hidden; overflow-y: auto`.

**Files:**
- Modify: `packages/components/src/side-nav/side-nav.module.css`

- [ ] **Step 1: Read `packages/components/src/side-nav/side-nav.module.css` in full**

  Locate the `.sideNav` rule at the top of the `@layer cascade.component { ... }` block. Confirm it currently contains:

  ```css
  overflow: hidden;
  ```

- [ ] **Step 2: Replace the single overflow declaration with axis-split declarations**

  In `.sideNav`, replace:

  ```css
  overflow: hidden;
  ```

  With:

  ```css
  overflow-x: hidden;
  overflow-y: auto;
  ```

  The full updated `.sideNav` rule must read (key properties shown — leave all other properties unchanged):

  ```css
  .sideNav {
    box-sizing: border-box;
    display: flex;
    flex-direction: column;
    inline-size: var(--cascivo-sidenav-inline-size, 16rem);
    min-block-size: 100%;
    padding: var(--cascivo-space-2);
    font-family: var(--cascivo-font-sans);
    background-color: var(--cascivo-sidenav-bg, var(--cascivo-color-surface));
    border-inline-end: 1px solid var(--cascivo-color-border);
    overflow-x: hidden;
    overflow-y: auto;
    transition: inline-size var(--cascivo-motion-emphasis);
    /* ... rest unchanged ... */
  }
  ```
