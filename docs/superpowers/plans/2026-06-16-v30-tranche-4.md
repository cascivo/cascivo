# v30 — Tranche 4: Critical CSS Inline

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stop the full `landing.css` (3,191 lines) from blocking first paint. Extract the styles required for above-the-fold content into an inline `<style>` block in `index.html`, then load the rest of `landing.css` asynchronously so the browser does not wait for the full CSS before painting the hero.

**Architecture:** Two-part change:

1. Identify and move the critical selectors (CSS custom properties, reset, header, hero) from `landing.css` into an inline `<style>` in `index.html`. The moved selectors are **deleted from `landing.css`** — they must not exist in both places.
2. Switch CSS loading from `import './landing.css'` in `main.tsx` to `<link media="print" onload="this.media='all'">` in `index.html`. Vite processes a CSS file referenced in an HTML `<link>` tag identically to one imported in JS — the resulting dist output is the same hashed `.css` file.

**Warning — critical CSS must be complete:** If any CSS custom property block (`--cascivo-*`) is missing from the inline style, the above-the-fold content renders with broken tokens before the async CSS loads. Include every `:root`, `[data-theme]`, and `*` rule.

**Tech Stack:** Inline `<style>` in HTML, `media="print"` async CSS loading trick, Vite HTML processing.

### Task 2: Add inline `<style>` to `index.html`

**Files:**

- Modify: `apps/landing/index.html`

- [ ] **Step 1: Read `apps/landing/index.html` in full**

  Locate the closing `</head>` tag.

- [ ] **Step 2: Add the inline style block**

  Immediately before `</head>`, add:

  ```html
  <style>
    /* Critical CSS — above the fold: CSS tokens, reset, header, hero */
    /* [paste the extracted critical CSS here] */
  </style>
  ```

  The inline style block must contain ALL of the critical selectors identified in Task 1.
  Order within the block must match the order they appear in `landing.css` to preserve
  cascade intent.

- [ ] **Step 3: Switch CSS loading to async**

  Below the inline `<style>` block, add the async CSS link:

  ```html
  <link rel="stylesheet" href="/src/landing.css" media="print" onload="this.media='all'" />
  <noscript><link rel="stylesheet" href="/src/landing.css" /></noscript>
  ```

  **Note:** Vite will rewrite this `href` at build time to the hashed output path, just as
  it does for a CSS import. The `media="print"` trick makes the link non-render-blocking:
  browsers download it but do not block paint on it. When it loads, `onload` fires and
  switches `media` back to `'all'`, applying the styles.
