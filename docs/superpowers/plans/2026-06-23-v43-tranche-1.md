# v43 — Tranche 1: Package Scaffold + Flow Engine

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up the new **`@cascivo/flow`** package (cloned from the `@cascivo/charts` package shape), wire
it into every monorepo surface that the registry/docs/MCP/storybook pipeline touches, and implement the
**engine** — a plain-signal state model (`useFlow` over `nodes`/`edges`/`viewport` signals), screen↔flow
coordinate transforms, and geometry helpers (node bounds, handle anchors, graph bounding box). **Nothing
renders yet**; the engine is pure and unit-tested in isolation. This tranche unblocks every later one.

**Architecture:** Copy `packages/charts/`'s `package.json`/`tsconfig.json`/`vite.config.ts`/`scripts/`/
`src/setup.ts`/`src/css-modules.d.ts` shape verbatim, renaming to `@cascivo/flow`. The `src/` layout mirrors
charts: `engine/` (state + math), `core/` (the pane + hooks, T2+), `flows/<name>/` (per-primitive, T2+). The
state model is **plain signals** (no Zustand, no MVX): `nodes: Signal<FlowNode[]>`, `edges: Signal<FlowEdge[]>`,
`viewport: Signal<{ x; y; zoom }>`, exposed via a `useFlow` hook with action helpers
(`setNodes`/`updateNode`/`setEdges`/`setViewport`), controllable via `useControllableSignal`. Per CLAUDE.md:
no banned hooks; any future DOM side effects use `useSignalEffect`.

**Tech Stack:** TypeScript + signal state (`@cascivo/core` `useSignal`/`useComputed`/`useControllableSignal`);
owned geometry math (no D3); Vitest. vite+ (`vp`) for build/test.

### Task 3: Implement the engine — types, store, transforms, geometry

**Files:** `packages/flow/src/engine/types.ts`, `store.ts`, `transform.ts`, `geometry.ts`

- [ ] **Step 1:** `types.ts`: `FlowNode` (`id`, `position {x,y}`, `data`, `type?`, `selected?`), `FlowEdge`
      (`id`, `source`, `target`, `sourceHandle?`/`targetHandle?`, `type?`, `animated?`, `label?`, `selected?`),
      `Viewport` (`x`, `y`, `zoom`), `HandlePosition` (`'top'|'right'|'bottom'|'left'`), `XYPosition`.
- [ ] **Step 2:** `store.ts` — `useFlow(options)`: `nodes`/`edges`/`viewport` signals (via
      `useControllableSignal` where controllable) + action helpers (`setNodes`, `updateNode(id, patch)`,
      `setEdges`, `addEdge`, `setViewport`). No `useState`/`useReducer`. Pure signal reads/writes.
- [ ] **Step 3:** `transform.ts` — `screenToFlow(point, viewport)` and `flowToScreen(point, viewport)`
      (account for `viewport.zoom` + `x`/`y` translation), plus `clampZoom(z, min, max)`. Pure functions.
- [ ] **Step 4:** `geometry.ts` — `nodeBounds(node, size)`, `handleAnchor(node, handle, size)` (the
      `{x,y}` where an edge attaches given a handle position), `graphBounds(nodes, sizes)` (the bounding box
      for fit-view). Pure functions.
