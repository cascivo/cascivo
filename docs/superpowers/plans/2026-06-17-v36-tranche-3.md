# v36 — Tranche 3: Hero Redesign (Above the Fold)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the above-the-fold experience a single clear message in the Linear/Apple mold: one
headline, one-line subhead, two CTAs (primary "Get started", secondary GitHub), the `npx cascivo add`
copy command, and **one** product visual. Generous whitespace, short copy, no competing elements.

**Architecture:** Edits `apps/landing/src/sections/Hero.tsx` and its styles in `landing.css`. The hero
stays eager (it holds the LCP element). The single visual is a lightweight element (e.g. a static
component preview or the theme-swatch motif) — not a heavy interactive demo (that lives in the
showcase, T4).

**Tech Stack:** React 18 island, `@cascivo/core` signals, vite+ (`vp`).

---

## Files

- Modify: `apps/landing/src/sections/Hero.tsx`
- Modify: `apps/landing/src/landing.css` (hero block: spacing, type scale, responsive)

### Task 1: Distill the hero copy and CTAs

**Files:** `apps/landing/src/sections/Hero.tsx`

- [ ] **Step 1:** Keep one `<h1>` (the existing `hero-title`) with a sharp one-line value proposition,
  and exactly one subhead paragraph (≤ ~140 chars). Remove any tertiary copy/badges that compete for
  attention.
- [ ] **Step 2:** Provide exactly two CTAs: primary **Get started** (→ `/guides` or docs quick-start)
  and secondary **GitHub** (existing `GITHUB_HREF`). Keep the `npx cascivo add button` copy command
  (existing `CopyCommand`) directly under the CTAs.
- [ ] **Step 3:** Include **one** product visual only. Prefer a static, instantly-painting element
  (component preview / theme swatches) so it does not delay LCP or add JS. No carousel, no autoplay.

### Task 2: Minimal hero layout + spacing

**Files:** `apps/landing/src/landing.css`

- [ ] **Step 1:** Increase vertical breathing room and constrain the content measure (readable line
  length) to match the minimal references. Single-column on mobile; optional two-column (copy | visual)
  at `lg` (`64rem`) using a `@container` or `@media (min-width: 64rem)` query — canonical scale only.
- [ ] **Step 2:** Ensure the LCP element (headline) renders eagerly with no web-font layout shift; the
  visual must reserve its space (no CLS).
- [ ] **Step 3:** Touch targets on both CTAs reach ≥44px under `@media (pointer: coarse)` via
  `var(--cascivo-target-min-coarse, 2.75rem)`.
