# v31 — Tranche 4: Layout Primitives (Join, Indicator, Stack)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship three CSS-first layout utility components: `Join` (border-radius collapsing for adjacent grouped children), `Indicator` (absolutely-positioned badge overlay), and `Stack` (z-axis stacked layout). All three require minimal React and no signals — they are pure layout primitives.

**Architecture:** `Join` uses only CSS `:first-child`/`:last-child` selectors — no JS children inspection. `Indicator` uses `position: relative` on the wrapper and `position: absolute` on the overlay child. `Stack` uses `position: relative` on the wrapper and stacked `position: absolute` on children with configurable pixel offsets via a CSS custom property.

**Tech Stack:** CSS Modules, no signals, no state. `Join` and `Indicator` use CSS logical properties for RTL support.

- [ ] **Step 2: Create `join.module.css`**

  ```css
  @layer cascade.component {
    .join {
      display: inline-flex;
    }

    .join[data-orientation='horizontal'] { flex-direction: row; }
    .join[data-orientation='vertical']   { flex-direction: column; }

    /* collapse inner radii for horizontal join */
    .join[data-orientation='horizontal'] > :first-child:not(:last-child) {
      border-start-end-radius: 0 !important;
      border-end-end-radius: 0 !important;
    }
    .join[data-orientation='horizontal'] > :last-child:not(:first-child) {
      border-start-start-radius: 0 !important;
      border-end-start-radius: 0 !important;
    }
    .join[data-orientation='horizontal'] > :not(:first-child):not(:last-child) {
      border-radius: 0 !important;
    }

    /* collapse inner radii for vertical join */
    .join[data-orientation='vertical'] > :first-child:not(:last-child) {
      border-end-start-radius: 0 !important;
      border-end-end-radius: 0 !important;
    }
    .join[data-orientation='vertical'] > :last-child:not(:first-child) {
      border-start-start-radius: 0 !important;
      border-start-end-radius: 0 !important;
    }
    .join[data-orientation='vertical'] > :not(:first-child):not(:last-child) {
      border-radius: 0 !important;
    }

    /* collapse duplicate borders — only keep the leading border */
    .join[data-orientation='horizontal'] > :not(:first-child) {
      border-inline-start-width: 0;
    }
    .join[data-orientation='vertical'] > :not(:first-child) {
      border-block-start-width: 0;
    }
  }
  ```

  The `!important` overrides are intentional and scoped: they affect only direct children
  of `.join`, overriding the child component's own border-radius tokens within the group
  context. This is the accepted technique for layout composition utilities.

    children: React.ReactNode
    /** The badge/dot to overlay on the corner. Accepts any node. */

  }

- [ ] **Step 1: Create `stack.tsx`**

  ```tsx
  import styles from './stack.module.css'

  export interface StackProps {
    children: React.ReactNode
    /** Per-layer offset in px. Default: 4 */
    offset?: number
    className?: string
  }

  export function Stack({ children, offset = 4, className }: StackProps) {
    const items = Array.isArray(children) ? children : [children]
    return (
      <div
        className={[styles.stack, className].filter(Boolean).join(' ')}
        style={{ '--cascivo-stack-offset': `${offset}px` } as React.CSSProperties}
      >
        {items.map((child, i) => (
          <div
            key={i}
            className={styles.layer}
            style={{ '--cascivo-stack-index': i } as React.CSSProperties}
          >
            {child}
          </div>
        ))}
      </div>
    )
  }
  ```

- [ ] **Step 2: Create `stack.module.css`**

  ```css
  @layer cascade.component {
    .stack {
      display: grid;
      grid-template-areas: 'stack';
      isolation: isolate;
    }

    .layer {
      grid-area: stack;
      transform: translate(
        calc(var(--cascivo-stack-index, 0) * var(--cascivo-stack-offset, 4px)),
        calc(var(--cascivo-stack-index, 0) * var(--cascivo-stack-offset, 4px))
      );
      z-index: calc(10 - var(--cascivo-stack-index, 0));
    }
  }
  ```

  The `grid-template-areas: 'stack'` + all children at `grid-area: stack` stacks all
  children into the same grid cell. Each layer shifts by `index × offset` in both axes.
  Earlier children (lower index) have higher z-index — the first child is on top.
