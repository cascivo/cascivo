# v48 — Tranche 5: Charts Adoption Hardening — Color (ship/doc/test) + Preact Compat + Bridge Recipe

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Resolve the charts feedback's **headline blocker** — per-datum / per-series `color` (C1/C7) — which is
**already implemented in source** (`PieChartDatum.color`, `BarChartSeries.color`) but unpublished, **undocumented**
(absent from the component metas), and untested. This tranche makes it real: add `color` to both metas, add explicit
override tests, and document the resolution. It also adds the two cross-cutting adoption guards the feedback asks
for: a **chart Preact-compat smoke test** (mount `PieChart` + `BarChart` under the `preact/compat` +
`@preact/signals-react → @preact/signals` bridge, as done for `Button`) and a **LifeOS bridge cookbook** mapping
`--cascivo-chart-1..8`, `--cascivo-chart-grid`, `--cascivo-chart-axis` onto a consumer palette (C9). No component
logic changes here — verify, document, test.

**Architecture:** `pie-chart.tsx:134`/`:157` and `bar-chart.tsx:195`/`:281` already do `d.color ?? COLORS[i % …]`,
so a provided `color` beats the positional palette — that's the behavior to pin with tests. The metas
(`*.meta.ts`) list props but omit `color`; adding it makes the MCP/registry/docs surfaces advertise the override.
The Preact bridge is the established `preact/compat` alias + `@preact/signals-react → @preact/signals` mapping +
`useSignals()` no-op shim used by the `Button` spike; charts already call `useSignals()` first, so the test confirms
they mount and update under that bridge. The bridge tokens live in the **consumer's** `cascivo.css`; we ship a
worked recipe doc, not consumer code.

**Tech Stack:** owned charts metas/tests; the existing `preact/compat` bridge config (locate the `Button` smoke
test/spike for the exact alias setup); Vitest + @testing-library/react (+ preact/compat); a Markdown cookbook;
vite+ (`vp`). No new runtime deps, no component logic changes.
