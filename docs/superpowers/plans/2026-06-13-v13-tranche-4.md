> - `CLAUDE.md` holds the token architecture + house component rules (the source of truth for
>   most design decisions ‚Äî specs **reference and structure** these, never fork them).

**Goal:** Turn cascade's scattered, implicit design knowledge into machine-readable
infrastructure: structured spec files agents consume, a strict-vs-flexible boundary
registry, and an exceptions log preserving historical reasoning ‚Äî all folded into the
context bundle so agents read the WHY and the boundaries before generating.

**Architecture:** Human-authored markdown specs under `docs/specs/` + a generator that emits
`specs.json`, `boundaries.json` (from per-component `intent.flexibility` + a small global
section), and parses `exceptions.md` into structured entries. The context generator (T2) is
extended to fold these into `context.json`.

**Tech stack:** unchanged. Mostly authoring + a join generator.
