# v42 — Tranche 3: `relative-time` Component + `formatBytes`

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Adopt the two Web Awesome formatter ideas that are genuine gaps, and **only** those. Web Awesome
ships `wa-format-date`, `wa-format-number`, `wa-format-bytes`, and `wa-relative-time` as elements because HTML
can't call a function; cascivo already has `formatDate`/`formatNumber`/`formatRelativeTime` **functions**. So:

1. Add the **one missing Intl formatter**, **`formatBytes`**, to `@cascivo/i18n` (completes the set).
2. Ship **`relative-time`** as the one formatter that genuinely benefits from being a *component* — it
   **auto-ticks** ("2 minutes ago" → "3 minutes ago") via a `useSignalEffect` interval, reusing
   `formatRelativeTime`, with an injectable `now` so tests stay deterministic.

`format-date`/`format-number` stay **functions** (a React wrapper that calls them once adds nothing — avoid
bloat).

**Architecture:** `formatBytes` follows the existing `Intl`-backed pattern in `packages/i18n/src/format.ts`
(`Intl.NumberFormat` with `style: 'unit'`, a `unit` chosen from the byte magnitude, binary vs decimal option,
locale from the i18n store). `relative-time.tsx` derives a `now` signal (default `useSignal(Date.now())`,
overridable via prop for tests), updates it on an interval whose cadence scales with the magnitude (every
second under a minute, every minute under an hour, …) inside `useSignalEffect`; renders `<time datetime>` with
the relative phrase as text and the absolute date in `title`/`aria-label`. Per CLAUDE.md: no banned hooks; the
interval lives in `useSignalEffect` with cleanup, SSR-guarded; `useSignals()` first.

**Tech Stack:** `@cascivo/i18n` (`Intl`, the locale signal store) for `formatBytes`; signal-driven TSX for
`relative-time` (`useSignal`, `useSignalEffect`, `useComputed`, `useSignals`); Vitest + Testing Library.

### Task 3: Implement `relative-time.tsx` (auto-ticking, signal-driven)

**Files:** `packages/components/src/relative-time/relative-time.tsx`

- [ ] **Step 1:** Props: `date` (Date | number | ISO string, required), `sync?: boolean` (default `true` —
      auto-tick), `now?: number` (injectable current time for deterministic tests; default `Date.now()`),
      `format?` (passthrough to `formatRelativeTime` options), `className`/`...rest`.
- [ ] **Step 2:** `useSignals()` first. `const now = useSignal(props.now ?? Date.now())`. In a `useSignalEffect`
      (guarded by `sync` and `typeof window`), `setInterval` updates `now.value`; the cadence is derived from
      the |date − now| magnitude (1s under a minute, 60s under an hour, …); clear the interval in teardown. When
      `props.now` is provided, sync it into the signal during render and skip the interval (tests drive `now`).
- [ ] **Step 3:** `const text = useComputed(() => formatRelativeTime(date, now.value, format))`. Render
      `<time dateTime={isoString} title={absolute}>{text}</time>` with the absolute formatted date in
      `title`/`aria-label` (via `formatDate`). No `useState`/`useEffect`.
