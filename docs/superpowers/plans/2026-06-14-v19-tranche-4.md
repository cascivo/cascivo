# v19 Tranche 4 — Link & navigation semantics

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development or
> superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.
>
> Make navigation semantic and links safe. Today several CTAs are `<button onClick=window.location>`
> (not keyboard/middle-click friendly, not crawlable) and the footer renders external GitHub links with
> no `rel`/`target`. Replace button-navigation with real anchors; secure external links.
>
> Current-state assumptions — RE-VERIFY before starting:
>
> - `Footer.tsx` builds links from a `COLUMNS` model where `REPO = https://github.com/urbanisierung/cascivo`;
>   all links (internal + external) render via a single `<a href={link.href}>` (~line 53) — no
>   `rel`/`target` distinction.
> - Button-as-nav: `Hero.tsx` (~41–54), `CtaBand.tsx`, `pages/guides/GuidesCta.tsx`,
>   `pages/accessibility/A11yCta.tsx` use `onClick → window.location.href`.
> - `#axe` is VALID (`AxeComparison.tsx:17`). Do NOT "fix" it.

**Goal:** Every element that navigates is an `<a>` (styled as a button where needed). External links
carry `target="_blank"` + `rel="noopener noreferrer"`; internal links stay plain SPA anchors. Cross-app
footer targets are verified/documented.

**Architecture:** `Footer.tsx`, `Hero.tsx`, `CtaBand.tsx`, `pages/guides/GuidesCta.tsx`,
`pages/accessibility/A11yCta.tsx`, `landing.css` (anchor-as-button class if not already present). No
package changes.

**Tech stack:** unchanged. Anchors are static; no signals involved.

## Tranche exit criteria

1. No element that navigates is a `<button onClick=window.location>`; all are real anchors.
2. Every external link carries `target="_blank"` + `rel="noopener noreferrer"`; internal links don't.
3. Converted CTAs are keyboard-activatable and support open-in-new-tab; internal anchors still route
   client-side without a full reload.
4. Cross-app footer targets are documented; `#axe` untouched and still valid.
5. No hardcoded colors added; no orphaned handlers left behind.
