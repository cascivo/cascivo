# v56 — Tranche 1: Bounded Stream Buffer (core)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Supply the primitive the feedback needed but could not find — a **fixed-capacity stream buffer** with O(1)
append and **rAF-coalesced** flushing — and retire the feedback's O(n)-per-line anti-pattern
(`logsSignal.value = [...currentLogs.slice(1), newLine]`). This is the primitive T2 (`LogViewer`) and T3 (live
charts) both consume (Master Decision 1 / M-1).

**Architecture:** one module in `@cascivo/core`.
- **`createStreamBuffer<T>({ capacity, flush? })`** → `{ signal, append, appendMany, clear, size, capacity }`. Internally
  a ring (a pre-sized array + head index + length), so `append` is O(1) and never reallocates. Writes are **staged**
  and flushed to `signal.value` once per animation frame via `requestAnimationFrame`, so a burst of N appends in one
  frame yields **one** signal write and **one** render. `signal.value` exposes the current window oldest-first (a
  freshly materialized readonly array per flush). `flush: 'raf' | 'sync'` (default `'raf'`; `'sync'` for tests).
- **`useStreamBuffer<T>(options)`** → the same handle, created once per component (`useRef`-stable) and disposed on
  unmount (cancel any pending rAF). Reading `handle.signal.value` in render drives re-renders.

**Tech Stack:** `packages/core/src/` — `signal` from `./signals.ts`, `requestAnimationFrame`/`cancelAnimationFrame`,
`useRef` for the stable instance, `useSignalEffect` for unmount cleanup. No new dependency.

### Task 1: The ring buffer + rAF flush

**Files:** `src/stream-buffer.ts`

- [ ] **Step 1:** Define `StreamBufferOptions { capacity: number; flush?: 'raf' | 'sync' }` and
      `StreamBuffer<T> { signal: ReadonlySignal<readonly T[]>; append(item: T): void; appendMany(items: readonly T[]):
      void; clear(): void; readonly size: number; readonly capacity: number }`.
- [ ] **Step 2:** Implement the ring with a pre-sized `Array<T>(capacity)`, a `head` index, and a `length`. `append`
      writes at `(head + length) % capacity` when not full, else overwrites the oldest and advances `head` — **O(1),
      no reallocation, no per-line array copy**. `appendMany` loops `append` (or fast-paths when `items.length >=
      capacity`, keeping only the last `capacity`).
- [ ] **Step 3:** Stage writes: keep a `dirty` flag. On `append`/`appendMany`/`clear`, if `flush==='sync'` materialize
      immediately; if `'raf'` and no frame is pending, schedule one `requestAnimationFrame` that materializes the
      current ring into a readonly array (oldest-first) and assigns `signal.value`, then clears `dirty`/the handle.
      Coalesce: many appends in one frame → one assignment.
- [ ] **Step 4:** `clear()` resets `head`/`length` and flushes an empty array. Expose `size` (current length) and
      `capacity`. Guard `capacity < 1` with a thrown error (fail fast on invalid input).
