>   are test-safe. Verify by reading each component's test before editing.
> - Visual snapshots still deferred to T6.

**Goal:** Replace every left-border state indicator with the context-appropriate modern pattern:
alerts/toasts → full hairline border + colored icon/title + 6% tint; vertical nav/list
(SideNav, Switcher, ShellHeader dropdown items) → rounded background-tint + font-medium;
horizontal nav (ShellHeader top-level) → inset 2px bottom underline.

**Architecture:** CSS-module-only changes. No TSX edits, no API changes, no test rewrites
(unless a test asserts a style — re-verify). The patterns come from shadcn sidebar
(bg-tint + medium), Vercel/Radix top-nav (bottom underline), shadcn/Geist alert (full border +
tinted icon/title).

**Tech stack:** CSS modules under `@layer cascade.component`.

### Task 1: Alert

**Files:**
- Modify: `packages/components/src/alert/alert.module.css`

- [ ] **Step 1: Read the full file** to map the current structure (root rule with
      `border-inline-start-width: 3px`, variant blocks setting
      `border-inline-start-color`, plus where icon/title colors are set).

- [ ] **Step 2: Replace the pattern.** Root: uniform 1px border, surface radius. Variants:

```css
  .alert[data-variant='info'] .icon,
  .alert[data-variant='info'] .title {
    color: var(--cascade-color-info);
  }

  /* repeat for success / warning / destructive with their tokens */
```

Delete every `border-inline-start-*` declaration in the file.

### Task 5: ShellHeader — underline for horizontal nav, tint for dropdown items

This is the user-flagged "badly designed" case: the current 3px *left* border on items in a
*horizontal* header nav reads as a glitch. Horizontal nav gets a bottom underline
(Vercel/Radix pattern); the dropdown menu items (vertical) get the bg-tint pattern.

**Files:**
- Modify: `packages/components/src/shell-header/shell-header.module.css`

- [ ] **Step 1: Read the file fully.** Three elements use the left border: `.navLink` (~line
      100), `.navMenuTrigger` (~135), `.navMenuItem` (~206). The header has fixed height
      `var(--cascade-shell-header-block-size)` (3rem) — the underline must NOT change layout
      (edge case 4): use `box-shadow: inset 0 -2px 0`, not border.

- [ ] **Step 2: `.navLink` and `.navMenuTrigger`** (horizontal, top-level). For BOTH:

```css
  .navLink {
    /* DELETE: border-inline-start: 3px solid transparent; */
    /* items already fill header height — keep existing padding/display rules */
    color: var(--cascade-color-text-subtle);
  }

  .navLink:hover {
    color: var(--cascade-color-text);
    background: var(--cascade-color-bg-subtle);
  }

  .navLink[data-state='active'] {
    color: var(--cascade-color-text);
    font-weight: var(--cascade-font-medium);
    box-shadow: inset 0 -2px 0 var(--cascade-color-accent);
  }
```

(Repeat identically for `.navMenuTrigger`; it may share rules — combine selectors if the file
structure allows.)

- [ ] **Step 3: `.navMenuItem`** (inside the dropdown — vertical list):

```css
  .navMenuItem {
    /* DELETE: border-inline-start: 3px solid transparent; */
    border-radius: var(--cascade-radius-item);
  }

  .navMenuItem[data-state='active'] {
    background: var(--cascade-color-active-bg);
    font-weight: var(--cascade-font-medium);
  }
```

If the dropdown panel has zero inline padding, add `padding: var(--cascade-space-1)` to the
menu panel rule so item pills inset nicely (the shadcn menu look).

- [ ] **Step 4: RTL sanity.** `box-shadow: inset 0 -2px 0` is direction-neutral — nothing to
      mirror. Confirm no other physical properties were introduced.

- [ ] **Step 5: Run shell-header tests; rebuild react dist; commit:**

```bash

pnpm --filter @cascade-ui/react build
```
