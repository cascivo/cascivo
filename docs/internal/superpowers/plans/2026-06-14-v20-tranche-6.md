# v20 Tranche 6 — ConsoleApp flagship responsive rebuild

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development or
> superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax. **This is the acceptance demo** — the
> request names `ConsoleApp` as the block that "demonstrates the app," so this tranche is the visible
> proof the DNA work reached the surface. Pair with compound-engineering:frontend-design +
> screenshot-verify at 320/360/390/414.
>
> ConsoleApp today is fully inline-styled and renders a 7-column `<table>` in main plus an inline-styled
> aside. On a phone the table overflows horizontally and the aside is hidden by AppShell. Rebuild it so it
> is usable, legible, and tappable at 320px — table stacks into cards, aside content is reachable, nothing
> overflows — while keeping the desktop look identical.
>
> Current-state assumptions — RE-VERIFY before starting:
>
> - `packages/layouts/src/blocks/console-app/console-app.tsx`: no own CSS; inline styles throughout.
>   Composes `AppShell` (state via lazy `createShellState({ persistKey: false })`, `useSignals()` first),
>   `ShellHeader`, `SideNav`, two `HeaderPanel`s, an inline-styled `aside` (~92–117: dl of Status/Region/
>   CPU/Memory), and a main with an inline-styled 7-column table (Name/Status/Type/Region/IP/Created/
>   Actions, ~162–236).
> - `console-app.test.tsx` has no mobile assertions.
> - T1–T5 shipped: canonical scale + sweep + AppShell aside relocation are available to reuse.

**Goal:** ConsoleApp is fully usable at 320px — the instances table becomes a stacked card list below
`md`, the resource-details aside is reachable via the AppShell disclosure/drawer (T5) rather than hidden,
all controls are ≥44px on touch, and there is zero horizontal overflow. Desktop is pixel-unchanged. A
dedicated mobile e2e proves it. Signal patterns and the lazy shell-state init are preserved.

**Architecture:** Modify `console-app.tsx`; add `console-app.module.css` (extract the inline styles +
responsive rules); update `console-app.meta.ts` (tokens/`a11yRationale`); rewrite/extend
`console-app.test.tsx`; add a dedicated mobile e2e spec. Reuse the table→stack pattern and AppShell aside
relocation from T5.

**Tech stack:** unchanged. `useSignals()` stays first; the lazy `createShellState` pattern is preserved;
no `useState`/`useEffect`. CSS-first responsiveness; canonical `md` for the stack breakpoint.

### Task 1: Extract inline styles to a CSS module (no visual change yet)

**Files:** Create `packages/layouts/src/blocks/console-app/console-app.module.css`; modify
`console-app.tsx`

- [ ] **Step 1:** Move the inline styles (aside container, `dl`, headings, table, `th`/`td`, panel list
      items, content padding) into `console-app.module.css` under `@layer cascade.component`, using
      tokens (`--cascivo-color-border`, `--cascivo-space-*`, `--cascivo-text-*`) instead of literal
      rem/px where a token exists. Replace inline `style={...}` with `className={styles[...]}`.
- [ ] **Step 2:** Verify the desktop render is pixel-identical to before (screenshot diff). This step is a
      pure refactor — no responsive behavior yet — so a regression here is easy to spot.
- [ ] **Step 3:** Keep `useSignals()` as the first statement and the `stateStore.peek() === null` lazy
      init exactly as-is. Do not touch the `openPanel`/`HeaderPanel` signal logic.

### Task 2: Instances table → stacked cards below `md`

**Files:** Modify `console-app.tsx`, `console-app.module.css`

- [ ] **Step 1:** Keep the semantic `<table>` for desktop. Below `md` (40rem), transform it to a stacked
      card list via CSS: each `<tr>` becomes a card (`display: block`), each `<td>` a labeled row. Use the
      data-label pattern (`<td data-label="Status">` + `td::before { content: attr(data-label) }`) so each
      cell shows its column name on mobile and **no screen-reader semantics are lost**. Hide the `<thead>`
      visually below `md` (visually-hidden, not `display:none` removal from a11y tree — re-verify the
      project's visually-hidden utility/token).
- [ ] **Step 2:** Ensure the card list has zero horizontal overflow at 320px (long values like IPs wrap or
      truncate with a title). The "Manage" action becomes a tappable ≥44px control on coarse pointers.
- [ ] **Step 3:** Confirm only the canonical `md` value appears in any `@media`/`@container`; prefer
      `@container` (the block adapts to its slot) if the table has a container-type ancestor — otherwise
      `@media` is acceptable and documented.
