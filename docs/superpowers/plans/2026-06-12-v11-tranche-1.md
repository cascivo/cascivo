> Current-state assumptions — RE-VERIFY before starting:
>
> - `registry.json` (repo root) is `{ version, generatedAt, components[] }`, generated by
>   `scripts/registry/generate.ts` (`pnpm registry:generate`); entries have no per-item
>   `version`, `author`, `license`, or `registryDependencies`.
> - Registry types/validation are duplicated: `packages/cli/src/utils/registry.ts`
>   (`parseRegistry`, `findComponent`), `packages/mcp/src/registry.ts` (`loadRegistry`),
>   and the generate script each carry their own shape.
> - `apps/docs/public/` already hosts generated artifacts (`llms.txt`, `llms/<name>.md`)
>   and `packages/render/schema/view.v1.json` is published at `cascade-ui.dev/view.v1.json`
>   — the hosting path for JSON Schemas exists.

**Goal:** One shared registry package defines schema v2 (per-item semver, changelog,
author/license/homepage, `registryDependencies`, `theme`/`style` types), validates it,
builds flat static per-item JSON, and publishes JSON Schemas — while the monolithic
`registry.json` and every existing consumer keep working unchanged.

**Architecture:** New workspace package `packages/registry` (`@cascade-ui/registry`,
npm-published like core/tokens). It owns: TypeScript types, hand-rolled validators (house
style — no zod), `buildRegistry()` (index + sources → per-item JSON), and schema JSON files.
`scripts/registry/generate.ts` and the CLI/MCP refactor onto it in this tranche only where
free (full CLI consumption is T2).

**Tech stack:** unchanged. No runtime deps in the new package.

```ts
export interface RegistryItem {
  schemaVersion: 2
  name: string // 'button', 'layout/app-shell', '@ns prefix NOT stored here'
  type: 'component' | 'layout' | 'block' | 'chart' | 'section' | 'theme' | 'style'
  description: string
  category?: string
  version: string // per-item semver — the v11 headline
  changelog?: { version: string; note: string }[]
  author?: string
  license?: string // SPDX id
  homepage?: string
  files: RegistryFile[] // { url, target? } — url absolute or registry-relative
  install?: string
  dependencies: string[]
  registryDependencies?: string[] // 'input', '@ns/thing', URL, 'owner/repo/item#ref'
  tags: string[]
  meta?: ComponentMeta // required for directory listing (T3), optional in schema
}
export interface RegistryIndex {
  schemaVersion: 2
  name: string
  homepage?: string
  items: RegistryItem[]
}
```
