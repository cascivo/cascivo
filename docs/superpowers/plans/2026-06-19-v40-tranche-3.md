# v40 — Tranche 3: Streaming / Progressive Rendering

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Render generated UI **as the model streams it** — OpenUI's "the client does not need a complete
document before painting the skeleton." Add an incremental `cvl` parser and a streaming `CascadeView` mode
that paints each node once it parses cleanly, with a placeholder for in-flight nodes. Signal-driven,
obeying the cascivo reactivity rules (no `useEffect`/`useState`).

**Architecture:** `CascadeView` (`packages/render/src/cascade-view.tsx`) renders a complete `ViewConfig`.
The streaming layer adds: (1) `lang/stream.ts` — feed `cvl` text chunks, emit complete `ComponentNode`s as
their lines finish; (2) a `StreamingView` (or a `streaming` prop on `CascadeView`) that holds parsed nodes
in a signal and renders them progressively. Text leaves can reuse `@cascivo/ai` `StreamingText`
(`packages/ai/src/streaming-text.tsx`). Per CLAUDE.md: DOM/async side effects use `useSignalEffect`; React
apps need `useSignals()` when reading signals during render.

**Tech Stack:** Signal-driven TSX (`@cascivo/core` signals, `useSignal`/`useSignalEffect`/`useRef` only) +
CSS Modules; Vitest + @testing-library/react via vite+ (`vp`). Depends on T1's parser.

### Task 1: Incremental `cvl` parser

**Files:** `packages/render/src/lang/stream.ts`

- [ ] **Step 1:** Implement a stateful parser that accepts `cvl` text in chunks and, when a node's lines are
      complete, yields a fully-formed `ComponentNode` tagged with its region. Reuse T1's `parse` primitives per
      line; buffer incomplete trailing lines.
- [ ] **Step 2:** Never throw on partial input — buffer it. (Full invalid-input error handling is **T4**;
      here, partial-but-valid must accumulate cleanly.) Expose either an async iterator or a `push(chunk) =>
      node[]` interface — choose the simpler to test.

### Task 2: Streaming render component

**Files:** `packages/render/src/streaming-view.tsx`, `streaming-view.module.css`

- [ ] **Step 1:** Hold parsed nodes-per-region in a `useSignal`; as the stream yields nodes, append them.
      Render completed nodes by delegating to the existing node renderer used by `CascadeView` (share the
      mapping in `component-map.ts` — do not fork it).
- [ ] **Step 2:** Show a lightweight skeleton/placeholder for the region currently receiving a node. Use CSS
      for the placeholder; respect `prefers-reduced-motion`. **No `display:none`** data loss; placeholders are
      additive.
- [ ] **Step 3:** Drive stream consumption with `useSignalEffect` (subscribe to a source signal of incoming
      chunks), **not** `useEffect`. Add `useSignals()` as the first statement (React apps read signals during
      render). `useRef` only for DOM refs.

### Task 3: Tests
