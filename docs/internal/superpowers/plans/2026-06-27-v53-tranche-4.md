# v53 — Tranche 4: Crosshair, Shared Tooltip & Decimation

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make multi-series hover and large-series rendering match the engines: an **axis-pointer crosshair** + a
**shared (axis-trigger) tooltip** listing every series' value at the hovered x (E-5), and **LTTB / min-max
decimation** + progressive paint so a dense series renders fast and crisp (C-1). Today the tooltip is per-point
(one series), and v52's Canvas paints all points with no downsampling. Lens: **ECharts + Chart.js**. Builds on the
T1 zoomable plot.

**Architecture:**
- **Crosshair + shared tooltip** (`core/chart-frame.tsx` + `core/chart-tooltip.tsx`): extend `TooltipModel` with a
  `mode: 'item' | 'axis'`. In `axis` mode, on pointer move the frame finds the nearest **x** (not the nearest point),
  draws a vertical (and optional horizontal) crosshair rule, and renders **one tooltip** listing every series' value
  at that x (each in its series color). The `aria-live` readout announces the x + all series values. Keyboard
  Arrow steps the x-index and the same shared readout follows.
- **Decimation** (`engine/decimate.ts`): `lttb(points, threshold)` (Largest-Triangle-Three-Buckets) preserves visual
  shape; `minmax(points, threshold)` preserves spikes. A chart with a series longer than a threshold decimates before
  scaling/painting. For the Canvas path, paint **progressively** in chunks (a `useSignalEffect` loop that yields) so a
  very large series doesn't block. The fallback `<table>` still lists the full data (a11y unaffected by decimation).

**Tech Stack:** `core/chart-frame.tsx` + `core/chart-tooltip.tsx`; new `engine/decimate.ts`; signals +
`useSignalEffect`. No new dependency.

### Task 1: Decimation engine

**Files:** `engine/decimate.ts`

- [ ] **Step 1:** `lttb(points, threshold)` — bucket the points, keep the one forming the largest triangle with the
      previous kept point and the next bucket's average; always keep first/last. `minmax(points, threshold)` — per
      bucket keep the min and max y.
- [ ] **Step 2:** No-op when `points.length <= threshold`; guard tiny inputs.
- [ ] **Step 3:** `decimate.test.ts`: LTTB returns ≤ threshold points, keeps endpoints, and preserves a known spike's
      approximate extent; min-max keeps the true min/max of each bucket.
