> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land the closing polish: **enter/update/exit transitions** (N-4 — nivo's react-spring motion, but CSS/SVG-
native and reduced-motion gated, Decision 3) and the **"cascivo vs nivo vs visx" comparison** + the toolkit/recipe
docs that make the whole v52 surface discoverable. v51 added only a fade-in; this adds data-change transitions and the
honest positioning page. Lens: **nivo + docs**.

**Architecture:** transitions are CSS keyframes + SVG attribute transitions, all inside
`@media (prefers-reduced-motion: no-preference)` (a static, correct frame when suppressed). `core/transitions.module.css`
defines `line-draw` (`stroke-dashoffset`), `bar-grow` (`transform: scaleY`), `arc-sweep`, and `fade`; charts tag marks
with the classes. **Update** transitions (data changes) interpolate the same properties via a CSS `transition` keyed on
the resolved geometry; any JS timing uses `useSignalEffect` (no react-spring, no `useEffect`). The comparison page is
an honest feature matrix (cascivo wins a11y/zero-deps/themes/owned-code; nivo leads on type-count/maturity; visx leads
on primitive breadth/ecosystem) plus side-by-side "same chart, three libraries" framing. The recipe gallery folds in
v51's deferred docs.

**Tech Stack:** `core/transitions.module.css` + the charts; `apps/site` (Preact) comparison + recipe pages reusing the
existing code-block/playground. No new dependency.
