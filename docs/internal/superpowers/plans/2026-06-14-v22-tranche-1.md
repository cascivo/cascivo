# v22 Tranche 1 — Mount-portability + assembly pipeline

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development or
> superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.
>
> Start here. This tranche makes the five demos mount-portable (relative base) and builds the pipeline
> that assembles them into the landing's `dist/demos/<name>/`. Nothing in T2–T5 works until a single
> local command produces a landing `dist/` that previews the homepage AND every live demo.
>
> Current-state assumptions — RE-VERIFY before starting:
>
> - `apps/examples/{deploy,pay,flow,track,pulse}` each have a `vite.config.ts` with **no `base`** and
>   source aliases to `packages/*/src` + `../kit/src/index.ts`. Packages
>   `@cascivo/example-{deploy,pay,flow,track,pulse}`.
> - Each demo's `src/main.tsx` mounts `<App/>` on `#root`; no internal client routing exists.
> - Demos persist via `@cascivo/storage` with app-prefixed keys (e.g. `deploy.theme`). Re-verify each
>   demo's keys.
> - Root scripts run via `vp run`/`pnpm`; `scripts/` holds repo-level Node scripts. Re-verify the dir.

**Goal:** Every demo builds with `base: './'` and runs correctly both standalone (`vp preview` at `/`)
and assembled at `/demos/<name>/`. A new assembly script builds all five and copies their output into
`apps/landing/dist/demos/<name>/`, and one command yields a landing `dist/` serving the homepage and all
five live demos. Storage keys are confirmed collision-free across the now-shared origin.

**Architecture:** Edit each demo's `vite.config.ts` (`base: './'`). Add `scripts/assemble-demos.mjs`
(build each demo → copy `dist` → `apps/landing/dist/demos/<name>/`). Add a root convenience script.

**Tech stack:** unchanged.

### Task 1: Relative base on all five demos

**Files:** `apps/examples/{deploy,pay,flow,track,pulse}/vite.config.ts`

- [ ] **Step 1:** Add `base: './'` to each demo's `defineConfig({...})`. Keep all existing `resolve.alias`

- [ ] **Step 2:** Verify standalone still works: for one demo, `pnpm exec vp run @cascivo/example-deploy#build`
      then `vp preview` — the app loads at `/` with assets resolving (relative URLs). Confirm no asset

- [ ] **Step 3:** Confirm the existing per-demo Playwright specs still target `/` and pass (relative base

      runs green._

### Task 2: Storage-key collision audit (shared origin)

**Files:** read-only sweep across `apps/examples/*/src`; record findings in this tranche's notes.

- [ ] **Step 1:** Grep every `persistedSignal(` / storage-key literal across the five demos. List each
      demo's keys.
- [ ] **Step 2:** Assert every key is uniquely prefixed by its demo (`deploy.*`, `pay.*`, `flow.*`,
      `track.*`, `pulse.*`). If any unprefixed/shared key exists, prefix it (smallest possible change in
      that demo only).
- [ ] **Step 3:** Add a tiny check (extend an existing test or a script invoked by the gate) that fails
      if two demos declare the same storage key. Deterministic, no wall-clock.

### Task 3: Assembly script

**Files:** Create `scripts/assemble-demos.mjs`; edit root `package.json` (add a script).

- [ ] **Step 1:** `assemble-demos.mjs` — for each of the five demos: run its build
      (`vp run @cascivo/example-<name>#build`) if a `--build` flag is passed, then copy
      `apps/examples/<name>/dist` → `apps/landing/dist/demos/<name>/` (recursive, clean target first).
      Pure Node (`node:fs`/`node:path`), no new deps. Fail loudly if a source `dist` is missing.
- [ ] **Step 2:** Confirm the script writes ONLY under `apps/landing/dist/demos/<name>/` and never under
      `apps/landing/dist/examples/` (collision guard from the master plan).
- [ ] **Step 3:** Root `package.json` script, e.g. `"build:landing-demos": "vp run @cascivo/landing#build && node scripts/assemble-demos.mjs --build"`
      (build landing first so `dist/` exists, then build + copy demos into it). Match existing script
      naming/quote style.

### Task 4: Local end-to-end verification

- [ ] **Step 1:** Run `pnpm build:landing-demos`. Confirm `apps/landing/dist/index.html` and
      `apps/landing/dist/demos/{deploy,pay,flow,track,pulse}/index.html` all exist.
- [ ] **Step 2:** Serve `apps/landing/dist` from one static server (e.g. `vp preview` in landing, or a

      Deploy app (data loads, theme toggles, a mutation persists across reload); spot-check one other
      demo (`/demos/pulse/`).
- [ ] **Step 3:** Confirm a hard reload of `/demos/deploy/` (deep) serves the demo, not the landing's
