# v53 — Tranche 5: Toolbox — Export, Data View & Restore

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add ECharts' `toolbox` (E-6) — the dashboard utility belt users expect: **export to PNG/SVG**, a
**data-view** `<table>` toggle, and a **restore/reset** control. Today there's none. Lens: **ECharts**. Self-contained
and a11y-friendly.

**Architecture:** a `chrome/toolbox.tsx` button cluster rendered in the chart's corner, plus `core/export.ts`
helpers.
- **Export SVG:** serialize the live `<svg>` (`new XMLSerializer().serializeToString`), inline the computed values of
  the `--cascivo-*` custom properties it uses (so the standalone file renders without the page's CSS), and download a
  `Blob`.
- **Export PNG:** render the serialized SVG into the existing Canvas layer (or an offscreen `<canvas>`) at
  `devicePixelRatio` and `toBlob('image/png')`; download.
- **Data view:** toggle a visible, readable `<table>` of the series (the chart already builds a visually-hidden
  fallback table — reuse its shape, shown on demand).
- **Restore:** reset zoom (T1), the visualMap range (T2), and any legend filters to defaults.

Buttons are real `<button>`s (keyboard + focus-visible), labelled from `@cascivo/i18n`, ≥44px under `pointer: coarse`.
All side effects (serialize, `toBlob`, download) run in event handlers / `useSignalEffect`, never `useEffect`.

**Tech Stack:** `chrome/toolbox.tsx` + `core/export.ts` (`XMLSerializer`, `Blob`, the Canvas layer); `@cascivo/i18n`
for labels; `useRef` for the SVG/canvas elements. No new dependency.
