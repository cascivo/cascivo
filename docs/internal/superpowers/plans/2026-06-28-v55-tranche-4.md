# v55 — Tranche 4: Marketplace Directory & Aggregate Index

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make templates **listed and catalogued** with no backend (M-5). Extend the existing directory
(`directory/registries.json`) so a registry can advertise it carries templates, then have a **GitHub Action** fetch
every listed registry's index, filter `type:'template'`, and write a static `marketplace.json` — the catalog is a CI
**build artifact**, not a live query (ROADMAP-V55 Q3 / Master Decision 3). Submission stays PR-based with CI
validation, reusing the registry's tiered-trust model (Decision 5).

**Architecture:** three pieces.
- **Directory facet** (`registries.v1.json` + `directory/registries.json`): add an optional `provides: ('component' |
  'template' | 'theme' | …)[]` to each `RegistryDirectoryEntry`, so the catalog builder knows which registries to scan
  for templates. Backward-compatible (absent ⇒ `['component']`).
- **Catalog builder** (`scripts/registry/build-marketplace.ts`): for each listed registry, fetch its index, keep
  `type:'template'` items, attach `{ registry, verified, repoUrl, stars }` (stars baked at CI-time via the GitHub API —
  **never** at the user's runtime), and write `apps/site/public/marketplace.json` (a flat, gallery-ready array +
  facets: categories, frameworks, tags).
- **PR validation** (`scripts/registry/validate-directory.ts` + a workflow): on a directory PR, validate the new entry
  — schema, reachability (the index responds `schemaVersion:2`), ≥1 `type:'template'` item, every template passes
  `validateTemplate` (screenshots + license present), and no duplicate `namespace`.

**Tech Stack:** `packages/registry/schema/registries.v1.json`, `directory/registries.json`,
`scripts/registry/{build-marketplace,validate-directory}.ts`, `.github/workflows/marketplace.yml`. CI-time fetch + the
GitHub API. No runtime dependency, no backend.

### Task 1: Directory `provides` facet

**Files:** `registries.v1.json`, `directory/registries.json`

- [ ] **Step 1:** Add `provides?: string[]` to the `RegistryDirectoryEntry` schema (enum of item types; default
      semantics: absent ⇒ component-only). Keep all existing entries valid.
- [ ] **Step 2:** Mirror the type in `packages/registry/src/types.ts` (a `RegistryDirectoryEntry` interface if not
      already present) and export it.
- [ ] **Step 3:** Test: an entry with `provides:['component','template']` validates; an unknown `provides` value is
      rejected; an entry without `provides` still validates.

### Task 2: The catalog builder (`marketplace.json`)

**Files:** `scripts/registry/build-marketplace.ts`

- [ ] **Step 1:** For each directory entry whose `provides` includes `'template'`, fetch its registry index (the
      `registryUrl` index endpoint), parse, and keep items where `type === 'template'`.
- [ ] **Step 2:** For each template, project a gallery record: `{ name, namespace, description, category, framework,
      tags, screenshots, demoUrl, version, license, verified, repoUrl, installSpec, stars }`. Resolve `installSpec`
      (`@ns/name` or `owner/repo/name`) and bake `stars` via a single CI-time GitHub API call per repo (skip silently
      on rate-limit — stars are decorative).
- [ ] **Step 3:** Write `apps/site/public/marketplace.json`: `{ generatedAt, templates: [...], facets: { categories,
      frameworks, tags } }`. Sort `verified` first, then stars, then name. Test with fixture indexes: filtering,
      projection, facet derivation, and stable sort are correct; an unreachable registry is skipped with a logged
      warning (not a hard failure).

### Task 3: PR validation + workflow

**Files:** `scripts/registry/validate-directory.ts`, `.github/workflows/marketplace.yml`

- [ ] **Step 1:** `validate-directory.ts`: validate the directory file against `registries.v1.json`; for each entry
      that `provides` templates, fetch the index, assert reachability (`schemaVersion:2`), assert ≥1 `type:'template'`
      item, run `validateTemplate` on each (screenshots + license required), and assert no duplicate `namespace`.
- [ ] **Step 2:** `.github/workflows/marketplace.yml`: (a) on PRs touching `directory/registries.json`, run
      `validate-directory` (block on failure); (b) on push to `main` / a schedule, run `build-marketplace` and commit
      the regenerated `apps/site/public/marketplace.json` (first-party catalog auto-regenerates — Decision 5).
- [ ] **Step 3:** Test `validate-directory` against good/bad fixtures (unreachable, zero templates, missing
      screenshots, duplicate namespace each fail with a specific message).
