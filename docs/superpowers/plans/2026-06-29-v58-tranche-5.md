# v58 — Tranche 5: Canonical Tokens for Agents — "Exactly One Correct Token" on the AI Surfaces (post 1 F3)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stop the AI surfaces from offering an agent **more than one** correct token for the same purpose. cascivo
ships intentional backward-compat aliases — `color-bg`↔`color-background`, `color-text`↔`color-foreground`,
`color-error`↔`color-destructive`, `color-foreground-muted`↔`color-text-muted`, … (the `ALIASES` map in
`scripts/tokens/generate-manifest.mjs`). The data already records which name is `canonical` and which is an
`aliasOf`, but the MCP `get_tokens` tool returns them undifferentiated, so an agent sees two equally-valid names —
the opposite of post 1's *"exactly one correct token."* Surface canonical-only by default, flag aliases, add the new
`typography` family to `get_variant_matrix`, and have `validate_component` nudge generated code toward canonical
names. The CSS aliases **stay** (removing them is a breaking change — Master Decision 6). (M-5.)

**Architecture:** three small MCP-surface changes over data/machinery that already exists.

- **`get_tokens` (`packages/mcp/src/tokens.ts` + schema in `server.ts`)**: the token catalog the tool reads already
  carries (or can carry, from `generate-manifest.mjs`) `canonical`/`aliasOf`. Default the tool to **canonical-only**;
  add an opt-in `includeAliases?: boolean` that returns aliases too, each tagged `{ aliasOf: '<canonical>' }`. So the
  default agent experience is one name per purpose; the escape hatch exists for migration/lookup.
- **`get_variant_matrix` (`packages/mcp/src/variants.ts`)**: T1 adds a `typography` family to
  `tokens.variants.json`; confirm the tool returns it so an agent can ask "type token for a button label" and get
  `--cascivo-text-ui` deterministically (the same way it asks "accent hover" today).
- **`validate_component` (`packages/mcp/src/validate-component.ts`)**: it already flags banned hooks, off-scale
  breakpoints, missing fallbacks, and hallucinated tokens. Add a **warning** (not an error — aliases are valid CSS)
  when a manifest's `tokens[]` or candidate CSS uses an alias for which a canonical token exists, suggesting the
  canonical name. This closes the loop: agents writing components are steered to canonical at authoring time.

**Tech Stack:** `packages/mcp/src/{tokens,variants,validate-component}.ts` + their tests; `packages/mcp/src/server.ts`
(tool schema/description). Depends on **T1** (the `typography` family must exist in `tokens.variants.json`). Pure
TypeScript. No new dependency.
