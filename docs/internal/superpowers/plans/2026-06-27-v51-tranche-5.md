# v51 — Tranche 5: New Chart Types — RadialBar + Funnel

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the two Recharts chart types that are both genuinely missing and tractable within the existing engine —
**RadialBar** (concentric/radial gauge bars) and **Funnel** (progressive-stage trapezoids). cascivo already *leads*
Recharts on chart-type count (17 vs ~11) and uniquely owns Sparkline/KPI/Bullet/Meter/Boxplot/Histogram, so this is the
roadmap's only new-type tranche. **Sankey and Sunburst are explicitly out of scope** — each needs a new layout engine
(flow routing / radial hierarchy) and is a future roadmap. Lens: **feature parity** (C-F11).

**Architecture:** both reuse existing scales rather than new engines.
- **RadialBar:** map a category band to an angular sweep and a linear value scale to radius. Each series is an arc
  (`<path>` describing an annular segment) from a start angle to `valueScale.map(value)`. This is the same
  arc-from-angle math the PieChart/Donut already does (`pie-chart.tsx`), generalized to multiple concentric rings (one
  ring per category) or stacked sweeps. A center label slot mirrors the donut's `centerValue`/`centerLabel`.
- **Funnel:** a vertical stack of trapezoids, each stage's top width ∝ its value and bottom width ∝ the next stage's
  value (so the silhouette narrows). A linear scale maps value → width; stages stack down the block axis. Optional
  per-stage conversion % label (reuses the T2 data-label helper). This is straightforward polygon geometry — no new
  scale type.

Both follow the established chart contract: `'use client'`, `useSignals()` first, render inside `ChartFrame`
(`role="img"`, title, fallback table, optional tooltip), color from `--cascivo-chart-*`, i18n strings, a sibling
`*.meta.ts` with a populated `intent` block, tests, and exports from `@cascivo/charts` + `@cascivo/react`.

**Tech Stack:** SVG arcs/polygons + existing `linearScale`/`bandScale` + the pie arc math; `ChartFrame`;
`@cascivo/i18n`; `--cascivo-chart-*` tokens. No new engine module, no dependency.

      `centerValue?`/`centerLabel?`, `tooltip?`, `max?` (domain top), `className?`, `plain?`.
- [ ] **Step 2:** Build a `bandScale` over the categories for the radial bands (one ring each) and a `linearScale`

      arc-path helper from `pie-chart.tsx` (generalize to an annular segment with inner/outer radius and angle range).
- [ ] **Step 3:** Center slot (donut-style); tooltip via `ChartFrame`'s `TooltipModel` (points at each arc's mid-angle

- [ ] **Step 4:** Tests: N data → N arcs; an arc's sweep ∝ its value; center label renders; empty data → placeholder;
      a11y `role="img"` + title.

### Task 2: Funnel

**Files:** `charts/funnel/*`

- [ ] **Step 1:** Define `FunnelProps` — `data: { id; label; value; color? }[]` (descending), `title`,
      `description?`, `width?`/`height?`, `showConversion?` (per-stage % of previous and of first), `tooltip?`,
      `className?`, `plain?`.
- [ ] **Step 2:** `linearScale [0, data[0].value] → [0, innerW]` for widths; stack trapezoids down the height, each
      stage's top width = `scale(value)`, bottom width = `scale(next.value)` (last stage is a rectangle or point).
      Center each stage horizontally.
- [ ] **Step 3:** Optional conversion labels (reuse the T2 `DataLabel` helper) — value + `% of first` (+ `% of
      previous`). Tooltip per stage; fallback table.
- [ ] **Step 4:** Tests: N stages → N trapezoids with monotonically non-increasing widths; conversion % computed
      correctly; empty data → placeholder; a11y title.
