> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development or
> superpowers:executing-plans. ALSO use compound-engineering:frontend-design for the visual quality
> bar and screenshot verification. Steps use checkbox (`- [ ]`) syntax.
>
> T1 (accuracy fixes) is done. Now build the biggest missing piece on the landing: a dedicated
> charts showcase section. `@cascivo/charts` ships 17 types, CVD-safe palettes across all 10
> themes, and keyboard-navigable tooltips — and it is entirely invisible on the landing page today.
>
> Current-state assumptions — RE-VERIFY before starting:
>
> - `@cascivo/charts` is imported in `apps/landing/src/sections/ProofTeasers.tsx` for the perf
>   bar chart. Check whether the vite alias `@cascivo/charts` exists in
>   `apps/landing/vite.config.ts`. If it does NOT exist and the existing import works, the package
>   exports source directly (no alias needed). If it DOES exist, the alias format is the pattern
>   for adding new ones.
> - Available chart components: `BarChart`, `LineChart`, `AreaChart`, `ScatterChart` (or similar
>   names — re-verify from `packages/charts/src/index.ts`). The theme-aware CVD-safe palette is
>   applied via `--cascivo-chart-1` … `--cascivo-chart-8` tokens on the active `data-theme`.
> - Section order in `App.tsx`: ThemeDemo → QuickStart. The new ChartShowcase goes between them.
> - The active theme is set via the Header's theme switcher, which applies `data-theme` to the
>   document root. Charts that use `--cascivo-chart-*` tokens automatically reflect it.

**Goal:** A new `ChartShowcase` section on the landing that:

1. Shows ≥3 chart types with live data, styled with the active theme's CVD-safe palette.
2. Calls out CVD safety explicitly: all 10 themes are Okabe-Ito-grounded and verified under
   protanopia, deuteranopia, and tritanopia.
3. Demonstrates keyboard tooltip traversal (arrow keys navigate between data points).
4. Links to the charts docs page for the full gallery.
5. Renders with zero horizontal overflow at 320–414px and looks world-class at all widths.

**Architecture:** New `apps/landing/src/sections/ChartShowcase.tsx` + CSS in
`apps/landing/src/landing.css`. No new packages; `@cascivo/charts` is already a workspace dep.

**Tech stack:** unchanged. React 18+ + `@preact/signals-react` (`useSignals()`). Mobile-first CSS.
