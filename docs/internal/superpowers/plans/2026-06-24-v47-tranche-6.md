**Architecture:** The editor package is `@cascivo/editor`. `README.md` is **generated** from
`packages/editor/readme.body.md` via `pnpm regen` — do **not** hand-edit `README.md`. The registry entry is
`editor/code-editor` in `registry.json`, also **regenerated** (edit the source/meta, regenerate; never hand-patch
the generated entry). Storybook stories live in `apps/storybook/stories/editor/`; the docs editor page is
`apps/docs/src/pages/EditorPage.tsx`. Storybook is a **React** app (no signals Babel transform → call
`useSignals()` first in any render that reads a signal; no `useState`/`useEffect`/`useContext`/`useReducer`); docs
is **Preact** (signals natively reactive, still no banned hooks). Both apps must build **without** a prior full
build, so the `@cascivo/editor` source alias must be present in `apps/storybook/.storybook/main.ts` (`viteFinal`)
and `apps/docs/vite.config.ts` — **verify, don't assume**.

**Tech Stack:** `packages/editor/PERFORMANCE.md` + `readme.body.md` (→ `README.md` via regen); `CHANGELOG.md` +
`package.json` version (+ `VERSION` const if present); `code-editor.meta.ts`; `registry.json` (generated);
Storybook (`@storybook/react-vite` — `Meta`/`StoryObj`, `argTypes`, render functions) with `@cascivo/core` signals
(`useSignal`, `useSignals`) for the interactive demo; the docs Preact `EditorPage`; the v47 engine surface
(`tokenizeRange`, `LineStateIndex`) from T2; `pnpm regen` + the full `pnpm ready` gate + `breakpoint:check` +
`fallback:check`. No new runtime deps. Depends on **T1–T5** (the windowed path must be landed).
