# v40 ‚Äî Tranche 1: Cascade View Language (`cvl`) ‚Äî Compact DSL ‚Üî `ViewConfig`

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give cascivo a compact, line-oriented serialization of the render tree ‚Äî the lesson from studying
OpenUI Lang, which is up to 67% smaller than the equivalent JSON. Add `encode(ViewConfig): string` and
`parse(string): ViewConfig` with a **lossless round-trip**, keeping `ViewConfig` JSON as the canonical
in-memory form. This is the wire/LLM-output format T2 (grammar), T3 (streaming) and T4 (validation) build
on. Additive only ‚Äî no change to `view.v1.json`, `CascadeView`, or the `ViewConfig` type.

**Architecture:** `packages/render/src/types.ts` defines `ViewConfig`/`ComponentNode` (`component`, `props`,
`bind: $data.*`, `events: $actions.*`, `children: ComponentNode[] | string | TranslationRef`). `cvl` is a
1:1 lossless projection of that tree into a line-oriented DSL inspired by OpenUI Lang's
`identifier = Expression` form (e.g. `root = Stack([title, tbl])` / `title = Text("Hello")`). Bindings,
events, and `$t` translation refs must survive the round-trip exactly.

**Tech Stack:** Hand-written TypeScript encoder/parser (no Zod, no parser-generator dep unless trivial);
Vitest via vite+ (`vp`).

### Task 1: Pin the `cvl` grammar against `ViewConfig`

**Files:** `packages/render/src/types.ts` (read), `packages/render/src/lang/grammar.ts`

- [ ] **Step 1:** Re-read `types.ts` and enumerate every shape that must round-trip: `ViewConfig.$schema`,
      `version`, `view.layout`, `view.regions` (named arrays), and `ComponentNode.{component, props, bind,
      events, children}`; `PropValue` (string/number/boolean/null/`TranslationRef`/array/object);
      `DataRef`/`ActionRef` string forms; `children` as array | string | `TranslationRef`.
- [ ] **Step 2:** Define the line-oriented grammar in `grammar.ts` (comment-documented): a root `view`
      entry, named regions, component calls `Name(propsObj?, children?)`, and literal forms for each
      `PropValue`. Choose explicit sigils so encode/parse are unambiguous: e.g. `@data.users` ‚Üî `$data.users`
      bindings, `@act.openUser` ‚Üî `$actions.openUser` events, `t"key"{params}` ‚Üî `TranslationRef`. Record the
      exact mapping table in a comment ‚Äî it is the contract for T2's grammar generator.

### Task 2: Implement `encode(config): string`

**Files:** `packages/render/src/lang/encode.ts`

- [ ] **Step 1:** Serialize `ViewConfig` to `cvl`: emit `$schema`/`version`/`layout` headers (only when
      present), one block per region, each node as a component call with its props, `bind`, `events`, and
      `children` in the grammar from Task 1.
- [ ] **Step 2:** Encode every `PropValue` variant (incl. nested arrays/objects and `TranslationRef`) and
      both ref kinds. Keep output deterministic (stable key ordering) so the round-trip test and the token
      benchmark are reproducible.

### Task 3: Implement `parse(src): ViewConfig`

**Files:** `packages/render/src/lang/parse.ts`

- [ ] **Step 1:** Parse `cvl` back into a `ViewConfig`, reconstructing `bind`/`events`/`children`/`$t` exactly
      as `encode` produced them. (Partial/invalid tolerance and structured errors are **T4**; here, valid
      input must parse exactly ‚Äî but do not `throw` on the happy path.)
- [ ] **Step 2:** Export `encode`, `parse`, and the lang types from `packages/render/src/lang/index.ts`, and
      re-export from `packages/render/src/index.ts`.

### Task 4: Lossless round-trip tests
