# v19 Tranche 3 — Crawlability (prerender) + real 404

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development or
> superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax. **HIGHEST-RISK TRANCHE — spike
> first, decide explicitly, document what shipped.**
>
> The SPA renders client-side only: a crawler, AI answer-engine, or agentic crawler that skips JS sees
> an empty `#app` shell with the same head for every route. This tranche makes each route ship real
> HTML with its own head (built on T1's per-route head model), and turns "unknown path silently renders
> home" into a real 404.
>
> Current-state assumptions — RE-VERIFY before starting:
>
> - `vite.config.ts:11–23` `deepLinkCopies()` copies the built `index.html` into `accessibility/`,
>   `performance/`, `guides/` at `closeBundle`. Same (empty) shell for each; no per-route head.
> - `App.tsx` routing is history-API based; unknown paths fall back to HOME (no 404 view).
> - T1 added `src/seo.ts` with per-route `{ title, description, canonical }` and a `setMeta` helper.
> - The build can run headless Chromium (the OG flow in T2 already does).

**Goal:** Each public route ships static HTML whose `<head>` contains that route's title, description,
canonical, and og tags — viewable with JS disabled / `curl`. An unknown path renders a real `NotFound`
view, and static hosts serve a `404.html`.

**Architecture:** `apps/landing/vite.config.ts` (prerender step replacing/extending `deepLinkCopies`),
the emitted per-route `dist/<route>/index.html`, `App.tsx` (`NotFound` view + route match), new
`apps/landing/src/pages/NotFound.tsx`, `public/404.html` (or emitted at build). No package changes.

**Tech stack:** unchanged. `NotFound` is static markup (no signals unless it reads one).

---

### Task 0: Spike the prerender path (decide before building)

- [ ] **Step 1:** Spike whether the build can prerender route bodies under alpha `vp`/Rolldown. Two
      candidate approaches:
      **(A) Full prerender** — after `vite build`, boot `vite preview`, use headless Chromium (already
      available via the OG flow) to load each route, capture `document.documentElement.outerHTML`, and
      write it to `dist/<route>/index.html`. Crawlers get real content + real head.
      **(B) Head-only static** — extend `deepLinkCopies` to inject each route's `<title>`/description/
      canonical/og (from `src/seo.ts`) into the copied shell. Body stays the app shell, but social/AI
      crawlers get correct per-route previews.
- [ ] **Step 2:** Pick (A) if it's stable and deterministic; otherwise ship (B). **Record the decision
      and rationale** in the tranche commit and in a short comment in `vite.config.ts`. Either way, the
      acceptance test (Task 3) is the same: per-route head is present in static HTML.

### Task 1: Implement the chosen prerender in the build

**Files:** `apps/landing/vite.config.ts`

- [ ] **Step 1:** Replace/extend `deepLinkCopies()` to emit `dist/<route>/index.html` for `/`,
      `/accessibility`, `/performance`, `/guides` with the route's head from `src/seo.ts`. (Root `/`
      stays `dist/index.html`.)
- [ ] **Step 2:** Make the step deterministic (fixed viewport, animations/reveal disabled during
      capture if using (A)) so `pnpm regen` + `git diff` stays clean. Keep `/og` out of the route set.
- [ ] **Step 3:** Ensure the JSON-LD, canonical, and og tags differ correctly per route (canonical and
      og:url must be the route's absolute URL, not always `/`).

### Task 2: Real NotFound view + 404.html

**Files:** Create `apps/landing/src/pages/NotFound.tsx`; modify `App.tsx`; add `public/404.html` (or
emit it at build)

- [ ] **Step 1:** Build `NotFound.tsx` — `Header` + `main` with an h1 ("Page not found"), a short line,
      and links back to `/` and `/guides`. Mobile-first, token-based, reuse existing section classes.
- [ ] **Step 2:** In `App.tsx`, route any path not in `ROUTES` to `NotFound` (instead of silently
      rendering HOME). Set `document.title` to "Not found — cascivo" and a `noindex` robots meta for it.
- [ ] **Step 3:** Emit `public/404.html` (a prerendered NotFound, or a copy that boots the SPA which
      then resolves to the NotFound view) so static hosts (e.g. GitHub Pages/Netlify) serve it for
      unknown deep links instead of a generic host 404.

### Task 3: Acceptance check + commit

- [ ] **Step 1:** After build, `curl`/grep each `dist/<route>/index.html` for that route's `<title>`
      and `meta name="description"` — they must be route-specific, not the home defaults.
- [ ] **Step 2:** Load a built route with JS disabled; confirm the head is correct (and, under (A), the
      body has visible content).
- [ ] **Step 3:** Navigate to `/nonexistent` in the running app → `NotFound` view renders; confirm
      `dist/404.html` exists.
- [ ] **Step 4: Commit** — `fix(landing): prerender per-route head + real 404 (v19-t3)` (note which
      prerender approach shipped).

## Tranche exit criteria

1. Each of `/`, `/accessibility`, `/performance`, `/guides` ships a static `index.html` whose head has
   that route's title + description + canonical + og (verified via `curl`/grep, no JS).
2. The prerender approach (A or B) is recorded in `vite.config.ts` and the commit message.
3. Unknown paths render a real `NotFound` view (h1 "Page not found", `noindex`); `dist/404.html` exists.
4. `pnpm regen` + `git diff --exit-code` stays clean (deterministic prerender).
5. `/og` is excluded from prerender + sitemap; no per-route head regressions for existing routes.
