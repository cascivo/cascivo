# v2 Tranche 5 — JSON-Configured UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship Phase 5 — `@cascivo/render`: a versioned view schema (TypeScript types + a JSON Schema artifact generated from `registry.json`), strict validation with actionable errors, the `<CascadeView>` runtime renderer with signal-driven live patching, upgraded MCP tools (`validate_view`, `scaffold_view`), a json-playground example app, and `cascade generate` codegen with a round-trip CI test.

**Architecture:** runtime renderer first, codegen second, one schema shared by both (locked roadmap decision). The renderer resolves component names against `@cascivo/react` (prebuilt package) through an explicit name→component map generated into the package — no dynamic `import()` magic. Data binding (`$data.*`) and actions (`$actions.*`) resolve against host-provided objects so configs stay JSON-serializable. `{ "$t": "key" }` resolves through a new key-based lookup added to `@cascivo/i18n`. The JSON Schema artifact derives its component enum + per-component prop checks from `registry.json` manifests automatically and is regenerated under the drift gate.

**Conventions (same as previous tranches):** `vp check` before commits; no semicolons, single quotes; GPG broken — `git -c commit.gpgsign=false commit …`; new-package README generated + committed.

- [ ] **Step 2: Implement** — in `messages.ts`, add a module-global `defaults = new Map<string, MessageValue>()`; `defineMessages` registers every message's default there (one added line in its loop). Then:

```ts
/** Key-string lookup for serialized references (JSON configs). Untyped by design. */
export function translateKey(key: string, params?: Record<string, string | number>): string {
  void catalogVersion.value
  const locale = currentLocale()
  const value = catalogs.get(locale)?.get(key) ?? defaults.get(key)
  if (value === undefined) return key
  return t({ key, value }, ...([params] as never))
}
```

(Reuse `t`'s resolution by constructing a transient Message; the cast silences the variadic-tuple type since params are untyped here — keep the cast local and commented.) Export from index.

- [ ] **Step 3:** `vp run @cascivo/i18n#test && vp check` → commit `feat(i18n): translateKey for serialized key references`.

- [ ] **Step 2: The v1 schema types** (`src/types.ts`):

```ts
/** A reference into the host-provided data context: "$data.users" → context.data.users */
export type DataRef = `$data.${string}`
/** A reference into the host-provided actions: "$actions.openUser" */
export type ActionRef = `$actions.${string}`
/** A translatable string reference resolved via @cascivo/i18n translateKey. */
export interface TranslationRef {
  $t: string
  params?: Record<string, string | number>
}

export type PropValue =
  | string
  | number
  | boolean
  | null
  | TranslationRef
  | PropValue[]
  | { [key: string]: PropValue }

export interface ComponentNode {
  /** Registry component name, e.g. "DataTable" (case-insensitive match against registry `meta.name`). */
  component: string
  props?: Record<string, PropValue>
  /** Prop name → data ref. Bound values come from the host context, never from JSON. */
  bind?: Record<string, DataRef>
  /** Event prop name → action ref. */
  events?: Record<string, ActionRef>
  children?: ComponentNode[] | string | TranslationRef
}

export interface ViewConfig {
  $schema?: string
  version?: 1
  view: {
    /** A layout/block registry name ("dashboard" → layout/dashboard) or "none". */
    layout?: string
    regions: Record<string, ComponentNode[]>
  }
}
```

```tsx
export interface CascadeViewProps {
  config: ViewConfig | Signal<ViewConfig>
  data?: Record<string, unknown>
```

Implementation: `useSignals()`; unwrap `config` (`isSignal(config) ? config.value : config` — type guard on `.value`/`.peek` presence); `validateView` first (throw with the formatted error list, or render an error panel per `onInvalid`); walk regions → for each node resolve component from `componentMap`, build props: spread `props` (resolving `TranslationRef` values via `translateKey`, recursively through arrays/objects), then `bind` entries via dot-path lookup into `data` (`getPath(data, 'users.active')` — implement with tests, no lodash), then `events` entries from `actions`. `children` recurse. Because reading `config.value` subscribes, patching the signal re-renders the view — the live-preview loop.
