# v22 Tranche 4 — Deterministic screenshot assets

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development or
> superpowers:executing-plans.
>
> Depends on T1 (demos buildable + assemblable) and T2/T3 (hub + detail pages reference screenshot
> paths). This tranche generates the real, deterministic screenshots those surfaces consume.
>
> Current-state assumptions — RE-VERIFY before starting:
>
> - The `og:generate` pattern in `apps/landing` (`vp preview --port 4180 --strictPort & sleep 3 && node
scripts/gen-og.mjs; kill %1`) is the existing build-time browser-capture precedent. Playwright is a
>   devDependency in the demos and landing.
> - Demos use seeded fixtures (v21 determinism rule) and a pausable simulation engine
>   (`createSimulation` from `@cascivo/example-kit`).
> - `DEMOS` (T2 `pages/examples/data.ts`) references screenshot asset paths (light/dark/mobile) that this
>   tranche must populate.

**Goal:** A deterministic screenshot script that, for each demo, captures light + dark (desktop +
mobile) images, written to a committed assets location the hub and detail pages already reference.
Re-running the script produces a no-op git diff (determinism), because fixtures are seeded and the
simulation engine is paused before capture.

**Architecture:** Add `scripts/gen-demo-screenshots.mjs` (Playwright) that, per demo: serves the
standalone demo preview (or the assembled `/demos/<slug>/`), sets the theme, pauses the simulation,
waits for data-loaded state, and screenshots desktop + mobile viewports in both themes. Output to
`apps/landing/public/screenshots/<slug>/<theme>-<viewport>.<ext>` (path must match `DEMOS`). Add a
`screenshots:generate` script.

**Tech stack:** unchanged. Playwright (existing devDep). No new runtime deps.

---

### Task 1: Capture script

**Files:** Create `scripts/gen-demo-screenshots.mjs`; edit root or landing `package.json` (add a script).

- [ ] **Step 1:** For each demo slug: launch its preview (reuse the `og:generate` approach — start
      `vp preview` for the demo, or serve the assembled landing `dist` and hit `/demos/<slug>/`), wait
      for the demo's "data loaded" signal (a stable selector — re-verify per demo), and explicitly pause
      the simulation (drive the demo into a deterministic frame; if no pause hook is exposed, set the
      theme + wait for first paint of seeded data only).
- [ ] **Step 2:** Capture matrix per demo: themes `{light, dark}` × viewports `{desktop 1280×800,
mobile 390×844}`. Use Playwright `page.screenshot`. Disable animations (`prefers-reduced-motion`
      emulation or a capture flag) so frames are identical run-to-run.
- [ ] **Step 3:** Write to `apps/landing/public/screenshots/<slug>/<theme>-<viewport>.png` (or `.webp` if
      the landing already favours webp — re-verify). Filenames must match the paths referenced in
      `DEMOS`.

### Task 2: Determinism

- [ ] **Step 1:** Confirm the demos render from seeded fixtures with no `Math.random()` in committed data
      (v21 rule) and that the sim engine is paused/seeded at capture.
- [ ] **Step 2:** Run the script twice; `git status` shows no diff on the second run. If pixels differ,
      eliminate the nondeterminism (timestamps, relative-time strings, animation frames) at the capture
      step (freeze clock / pick a fixed "now" used by fixtures).
- [ ] **Step 3:** Keep image count/size reasonable (these ship in the landing bundle's public assets);
      prefer compressed PNG/webp.

### Task 3: Wire assets into hub + detail pages

**Files:** `apps/landing/src/pages/examples/data.ts`, plus verify `ExamplesPage.tsx` (T2) +
`ExampleDetailPage.tsx` (T3)

- [ ] **Step 1:** Point `DEMOS` screenshot paths at the generated files; replace any T2/T3 placeholders.
- [ ] **Step 2:** Verify the hub thumbnails and detail-page galleries render the real images with proper
      `width`/`height` (no CLS) and `alt` text (i18n), in both light and dark theme contexts.
- [ ] **Step 3:** Screenshot-verify the hub + one detail page now show real captures at mobile + desktop.

### Exit criteria

- [ ] `scripts/gen-demo-screenshots.mjs` exists and a `screenshots:generate` script runs it.
- [ ] Committed light/dark/desktop/mobile captures exist for all five demos under the referenced path.
- [ ] Re-running the script is a no-op diff (determinism proven).
- [ ] Hub + detail pages render the real screenshots without CLS, with i18n alt text.
- [ ] Gate green. Committed.
