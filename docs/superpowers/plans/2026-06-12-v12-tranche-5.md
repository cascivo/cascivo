**Goal:** The "broken keyboard effect" — customize a copied dropdown, silently lose arrow
navigation — gets a tripwire in the **user's** repo: every component ships a portable
keyboard/ARIA contract test that `cascade add` copies alongside the source, and
`cascade audit --a11y` runs those tests (plus axe) against the user's possibly-customized
components. Nested composition (menu-in-modal, etc.) becomes a tested, documented matrix.

**Architecture:** Contract tests are new `<name>.contract.test.tsx` files per component —
self-contained (relative imports into the component folder + @testing-library only),
listed in registry `files`, copied on `add` (default on, opt-out). `audit --a11y` runs
vitest with a minimal CLI-shipped config against the installed components' contract
tests. Composition tests live in the components package (`src/_composition/`).

**Tech stack:** unchanged. `axe-core` (or `vitest-axe`) as a dev dep where the audit
runner needs it — evaluate making it a lazy install in user projects.

   test, and passes after restore.

   documented; any browser-only gaps noted.
