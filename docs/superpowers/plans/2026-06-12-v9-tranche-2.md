**Goal:** `AppShell` renders a thin top progress bar that clients drive through
`ShellState` — show at 0, update determinate progress, finish (auto-hide), or fail (bar
turns destructive + dismissible `role="alert"` message below the header).

**Architecture:** All state in `shell-state.ts` (two new plain signals + five methods —
never persisted). All rendering in `app-shell.tsx` + module CSS: an absolutely positioned
3px strip overlaying the top edge of the shell, fill width driven by a private
`--_progress` custom property (logical `inline-size`, so RTL is correct by construction).
No new registry entry — copy-paste users get it with `app-shell`.

**Tech stack:** unchanged — signals only (no hooks beyond the allowed set), CSS Modules
under `@layer cascade.component`, vitest + @testing-library/react.

    expect(shell.loadingProgress.value).toBe(0.6)
    shell.setLoadingProgress(1.7)
    expect(shell.loadingProgress.value).toBe(1)

    expect(shell.loadingProgress.value).toBe(0)

- [ ] **Step 3: Implement.** Extend the interface and factory in `shell-state.ts`:

```ts
export interface ShellState {
  // …existing members unchanged…
  /** Top loading bar: null = hidden, 0–1 = determinate progress. Never persisted. */
  loadingProgress: Signal<number | null>
  /** Loading error message; non-null renders the dismissible alert strip. */
  loadingError: Signal<string | null>
  /** Show the bar at 0. Clears any error and any pending hide timer. */
  startLoading: () => void
  /** Update progress; clamped to [0, 1]. No-op if the bar is hidden. */
  setLoadingProgress: (fraction: number) => void
  /** Snap to 1 and hide after a short fade (400ms). */
  finishLoading: () => void
  /** Keep the bar (destructive style) and show a dismissible message. */
  failLoading: (message?: string) => void
  /** Dismiss the error and hide the bar. */
  clearLoadingError: () => void
}
```

      Factory additions inside `createShellState` (plain `signal()`, NOT
      `persistedSignal` — loading is transient):

```ts
const loadingProgress = signal<number | null>(null)
const loadingError = signal<string | null>(null)
let hideTimer: ReturnType<typeof setTimeout> | undefined

return {
  // …existing members…
  loadingProgress,
  loadingError,
  startLoading: () => {
    if (hideTimer !== undefined) clearTimeout(hideTimer)
    loadingError.value = null
    loadingProgress.value = 0
  },
  setLoadingProgress: (fraction) => {
    if (loadingProgress.value === null) return
    loadingProgress.value = Math.min(1, Math.max(0, fraction))
  },
  finishLoading: () => {
    loadingProgress.value = 1
    if (hideTimer !== undefined) clearTimeout(hideTimer)
    hideTimer = setTimeout(() => {
      loadingProgress.value = null
    }, 400)
  },
  failLoading: (message) => {
    loadingError.value = message ?? 'Loading failed'
    // bar stays at its current progress, styled destructive via data-state
  },
  clearLoadingError: () => {
    loadingError.value = null
    loadingProgress.value = null
  },
}
```

      Note: the `'Loading failed'` literal is a fallback only — the rendered default lives
      in the i18n catalog (Task 2 wires `t()` at the render site; `failLoading()` callers
      normally pass their own message). If reviewers prefer zero literals here, make
      `message` required — decide at implementation, keep tests in sync.

{
  shell.loadingProgress.value !== null && (
    <div
      className={styles['loading-bar']}
      role="progressbar"
      aria-valuemin={0}

      aria-valuenow={Math.round(shell.loadingProgress.value * 100)}
