**Goal:** The landing renders world-class and with **zero horizontal overflow** at 320/375/390/414px
portrait, mobile-first, with an off-canvas nav, fluid type, and deliberate mobile treatments for the
heavy demos. Verified by Playwright screenshots + an overflow assertion, attached to the PR.

**Architecture:** Mobile-first refactor of `landing.css` (base = mobile, `min-width` to enhance),
a signal-driven off-canvas nav, `container-type` on sections that benefit, and per-demo mobile
variants. No new packages; reuse `@cascivo/layouts` breakpoints where shared.

**Tech stack:** modern CSS (`clamp()`, `@container`, `env(safe-area-inset-*)`, logical props),
Preact signals, Playwright.

### Task 2: Foundations — type scale, container queries, safe-area, nav shell

**Files:**

- Modify: `apps/landing/src/sections/Header.tsx` (+ a new mobile-nav signal/component)

- [ ] **Step 1:** Introduce the fluid type scale and spacing rhythm at the top of `landing.css`;

      to the header, any sticky CTA, and the footer.
- [ ] **Step 2:** Build the **off-canvas mobile nav**: a hamburger in `Header` toggles a
      `useSignal(false)` open state; a slide-in drawer + scrim. Requirements (house + a11y/v14):
  - `useSignals()` first in the React component; state in `useSignal`; DOM side effects (focus trap,
    body scroll lock, Escape listener) in `useSignalEffect` — **no `useEffect`**.
  - Focus trapped while open; focus restored to the trigger on close; closes on Escape, scrim click,
    and route/anchor navigation; `aria-expanded`/`aria-controls` on the trigger; ≥44px targets;
    respects `prefers-reduced-motion`.
- [ ] **Step 3:** Add `container-type: inline-size` to sections that should respond to their own
      width (not just the viewport) and convert their internal grids to `@container` queries where it
      reads better than `@media`.
