> - CLI integration tests use a fixture-project harness (e.g. `add.integration.test.ts`).

**Goal:** `cascade audit --ai <paths…>` checks the code an agent wrote in the **user's**
repo against cascade's published contract: hard-coded values that should be tokens, props
invented on known components, missing required props, and raw strings where i18n is
expected — with honest, low-false-positive reporting and safe `--fix` for the unambiguous
cases.

**Architecture:** A new `--ai` mode on the existing `audit` command. It loads the token
catalog + the registry component/prop index (through the injectable fetch, cached), then
runs four checkers over the target files: a CSS/inline-literal checker (reusing the v12
value classifier), a JSX component/prop checker (lightweight element + attribute walk), a
required-prop checker, and a raw-string checker. Findings render like v12 audit; `--json`
and `--fix` supported.

**Tech stack:** unchanged. CSS parsing reuses the v12 classifier. JSX detection is a
**lightweight** element-name + attribute walk — NOT full type inference (master-plan risk #6).

### Task 2: CSS / inline literal checker

**Files:**

- Create: `packages/cli/src/audit-ai/css-literals.ts` (+ test)

- [ ] **Step 1:** `findLiteralViolations(source, contract)` over `.css`/`.module.css` files
      and `style={{…}}` inline objects in `.tsx`:
  - reuse the v12 value classifier to find _visual_ declaration values (color, background,
    border-color, box-shadow, border-radius, spacing, font-size) that are **literals** (not
    `var(--cascade-*)`, not structural keywords)
  - for each literal, look up `contract.tokensByValue` — if **exactly one** token matches,
    emit an `error` with `{ file, line, value, suggestedToken }`; if multiple match, emit
    `info` (ambiguous); if none match, emit nothing (can't suggest — don't flag arbitrary
    brand values the user legitimately chose)
- [ ] **Step 2: Test:** `color: #3b82f6` where one token matches → error with suggestion;
      `gap: 13px` with no catalog match → no finding; a value matching two tokens → info.
- [ ] **Step 3: Commit** — `feat(cli): audit-ai CSS/inline literal checker`

### Task 3: JSX component + prop checker

**Files:**

- Create: `packages/cli/src/audit-ai/jsx-props.ts` (+ test)

- [ ] **Step 1:** Detect cascade components in a `.tsx` file: collect imported names from
      `@cascade-ui/react` (and the user's configured components dir, if discoverable from
      `cascade` config); walk JSX opening elements whose name is in that set. For each such
      element:
  - **unknown prop:** an attribute whose name is not in that component's `props` (and not a
    valid passthrough like `className`, `style`, `id`, `data-*`, `aria-*`, event handlers
    `on*`) → `error { file, line, component, prop }`
  - **spread suppression:** if the element has `{...spread}`, skip the missing/unknown checks
    for it and emit one `info` noting the audit can't see spread props (master-plan risk #6)
- [ ] **Step 2:** Use a lightweight parse — prefer an AST the repo already has access to via
      the vp/oxc toolchain (oxc parser is Rust-backed and likely available); if no AST parser
      is exposed, fall back to a scoped regex over JSX opening tags + attributes and document
      the limitation in `--help`. Decide and record which at implementation.
- [ ] **Step 3: Test:** `<Button frobnicate>` → unknown-prop error; `<Button variant="primary">`
      → clean; `<Button {...rest}>` → info (suppressed); `<div foo>` → ignored (not a cascade
      component).
- [ ] **Step 4: Commit** — `feat(cli): audit-ai JSX component/prop checker`

## Tranche exit criteria

1. `cascade audit --ai` flags, on the `dirty/` fixture: a single-match hard-coded value with
   the suggested token, an invented prop on a known component, a missing required prop, and a
   raw chrome string — and reports **clean** on the `clean/` fixture.
2. Ambiguous literals and spread-prop elements degrade to `info`, never false `error`s.
3. `--json` and `--level` behave like v12 audit; unreachable contract exits 2.
4. `--fix` rewrites only unambiguous full-value literal→token cases and never touches props
   or shorthand-embedded literals; proven by test.
5. Root gate + drift check exit 0.
