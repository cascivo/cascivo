# v35 — Tranche 2: `inert` Accessibility Fix

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix both `[aria-hidden="true"] elements contain focusable descendants` violations. Two elements set `aria-hidden` when closed but do not prevent their interactive children from receiving focus, making screen reader users encounter "hidden" controls in their tab order.

**Root cause:** `aria-hidden="true"` removes an element from the accessibility tree but does not affect the tab order. Links and buttons inside an `aria-hidden` element are still reachable with Tab. The fix is the `inert` HTML attribute, which suppresses both a11y visibility **and** focus/interaction for the entire subtree.

**Browser support:** `inert` is supported in Chrome 102+, Firefox 112+, Safari 15.5+. All supported by the cascivo browser target matrix (last 2 Chrome/Firefox/Safari). No polyfill needed.

**JSX note:** React 18 passes unknown boolean DOM attributes through. `inert={true}` renders `inert=""` on the DOM element; `inert={false}` or `inert={undefined}` omits the attribute. React 19 has native `inert` support.

### Task 1: Fix console sidebar — `RelayConsole.tsx`

**File:** `apps/landing/src/demo/RelayConsole.tsx`

The offending element is at line ~135–138:

```tsx
<div
  id="console-sidebar"
  className={`console-sidebar${sidebarOpen.value ? ' console-sidebar--open' : ''}`}
  aria-hidden={!sidebarOpen.value ? true : undefined}
>
```

- [ ] **Step 1:** Replace `aria-hidden={!sidebarOpen.value ? true : undefined}` with `inert={!sidebarOpen.value || undefined}`.

  The expression `!sidebarOpen.value || undefined` evaluates to:
  - `true` (truthy) when sidebar is closed → `inert=""` on the DOM element
  - `undefined` when sidebar is open → attribute omitted

  Alternatively: `inert={!sidebarOpen.value ? true : undefined}` — either form works; pick the form consistent with the existing `aria-hidden` pattern.

Target state:

```tsx
<div
  id="console-sidebar"
  className={`console-sidebar${sidebarOpen.value ? ' console-sidebar--open' : ''}`}
  inert={!sidebarOpen.value ? true : undefined}
>
```

- [ ] **Step 2:** Confirm no other `aria-hidden` on this element or its wrapping elements that would conflict.

### Task 2: Fix mobile nav drawer — `Header.tsx`

**File:** `apps/landing/src/sections/Header.tsx`

The offending element is at line ~262–267:

```tsx
<nav
  ref={drawerRef}
  id="mobile-nav-drawer"
  className={`mobile-nav-drawer${isNavOpen.value ? ' mobile-nav-drawer--open' : ''}`}
  aria-label="Main navigation"
  aria-hidden={!isNavOpen.value}
>
```

- [ ] **Step 1:** Replace `aria-hidden={!isNavOpen.value}` with `inert={!isNavOpen.value ? true : undefined}`.

  Note: Unlike `aria-hidden`, `inert` should not be a plain boolean prop without the `undefined` fallback — React renders `inert=""` for truthy and omits for falsy, but passing `false` would render `inert="false"` which the DOM treats as `inert` (browsers ignore the value). Use `!isNavOpen.value ? true : undefined` to ensure the attribute is absent when the drawer is open.

Target state:

```tsx
<nav
  ref={drawerRef}
  id="mobile-nav-drawer"
  className={`mobile-nav-drawer${isNavOpen.value ? ' mobile-nav-drawer--open' : ''}`}
  aria-label="Main navigation"
  inert={!isNavOpen.value ? true : undefined}
>
```

- [ ] **Step 2:** Confirm `aria-label="Main navigation"` is still present — `inert` suppresses the element from the a11y tree when closed, but the label is still needed when the drawer is open.

### Task 4: TypeScript notes

If TypeScript emits an error for `inert` on a `<div>` or `<nav>` (depends on `@types/react` version), add the prop via cast:

```tsx
{...({ inert: !sidebarOpen.value ? '' : undefined } as React.HTMLAttributes<HTMLDivElement>)}
```

Or extend the type locally in `css-modules.d.ts` / `globals.d.ts`:

```ts
declare namespace React {
  interface HTMLAttributes<T> {
    inert?: '' | undefined
  }
}
```

Prefer the declaration augmentation (cleaner, project-wide). Only needed if type errors appear.
