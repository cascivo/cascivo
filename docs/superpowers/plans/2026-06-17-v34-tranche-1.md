# v34 — Tranche 1: Release-set Lockdown

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Pin down exactly which packages ship to npm. Mark the three not-yet-public packages (`@cascivo/ai`, `@cascivo/charts`, `@cascivo/render`) as `private: true`, and verify the Tier‑1 publish closure is self-contained (no Tier‑1 package depends on a private one).

**Architecture:** Only `private` flags change here. No metadata, no workflow, no source. Setting `private: true` excludes a package from changesets versioning/publishing and from `pnpm pack`, while leaving it importable by workspace apps.

**Tech Stack:** pnpm workspaces, vite+ (`vp`).

### Task 1: Mark Tier‑2 packages private

**Files:** `packages/ai/package.json`, `packages/charts/package.json`, `packages/render/package.json`

- [ ] **Step 1:** In each of the three files, set `"private": true` (replace `"private": false` where present; `ai` may need the key added). Do not touch versions, exports, or scripts.
- [ ] **Step 2:** Confirm no other package lists any of these three in `dependencies` or `peerDependencies`. Expected dependents:
  - `@cascivo/render` is depended on by **nothing** publishable (it depends on react/i18n/core, not the reverse) — verify.
  - `@cascivo/charts` / `@cascivo/ai` are consumed by apps (`apps/landing`, `apps/docs`) as workspace deps — those are apps (private), so this is fine.
  - If any **Tier‑1** package depends on one of these three, STOP — that package cannot ship until the dependency is resolved (move to Tier 1, or remove the dependency). Record the finding.

### Task 2: Verify the Tier‑1 publish closure

**Files:** read-only audit of the ten Tier‑1 `package.json` files (`core`, `tokens`, `themes`, `icons`, `i18n`, `storage`, `react`, `mcp`, `registry`, `cli`)

- [ ] **Step 1:** For each Tier‑1 package, list its `dependencies` and `peerDependencies`. Confirm every `@cascivo/*` entry there is itself Tier‑1. Known-good edges:
  - `react → core, i18n`
  - `i18n → core`
  - `themes → tokens`
  - `storage → core`
  - `cli → registry`
  - `core`, `tokens`, `icons`, `mcp`, `registry` → no `@cascivo/*` runtime deps.
- [ ] **Step 2:** Confirm private-package usage appears **only** in `devDependencies` (expected: `@cascivo/react` dev-depends on `@cascivo/components`; that is bundled at build and must not leak into the published `dependencies`).
- [ ] **Step 3:** If any invariant is violated, do not "fix" by silently re-tiering — record the conflict in the commit message / a note and surface it; the maintainer decides.
