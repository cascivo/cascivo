# v51 — Tranche 7: Interactive Examples + Recipe Gallery + Engine Reference

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the charts docs as good as the reference everyone benchmarks against. Recharts' docs are loved for
three things cascivo lacks: **live-editable, copyable examples** (C-D5), a **task-oriented examples/recipe gallery**
(C-D6), and discoverability of the **composition primitives**. cascivo's examples are static; there is no recipe
gallery; and the public engine API (scales, shapes, `toStackedSeries`, `stats`) is exported but undocumented (C-D7).
This is the final tranche; it documents the full surface T1–T6 produced. Lens: **docs depth**.

**Architecture:** the docs app already renders chart demos (`ChartsPage.tsx`) and almost certainly has a code-block /
playground component used elsewhere in the site (the landing has copy-command affordances; per the audit history the
docs app shows code samples). Reuse that capability for charts rather than building new infra. The recipe gallery is a
set of small, focused, named compositions — each a runnable demo + its source — organized by the *task* a user is
trying to accomplish, mirroring how Recharts' examples are categorized. The engine reference is a doc page sourced
from `packages/charts/src/index.ts`'s exports.

**Tech Stack:** `apps/site` (Preact) docs routes/components; the existing code-block/playground; the charts package's
public exports. No new docs framework (out of scope per the master plan).
