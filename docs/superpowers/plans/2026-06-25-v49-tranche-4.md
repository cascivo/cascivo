# v49 — Tranche 4: Showcase Fidelity (twelve themes + carousel motion)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the showcase show what it claims. The themes panel headline reads "Twelve themes, one token swap" but
renders only **three** cards (light/dark/warm) — the system's single biggest visual differentiator vs shadcn is told,
not shown. And the advantage carousel auto-advances every 6.5s via a raw `setInterval` with **no
`prefers-reduced-motion` guard**, overriding a stated motion preference on the very page that sells accessibility.
This tranche renders all twelve themes as live, theme-scoped swatches and makes the carousel motion-preference-safe.
Lens: **designer.**

**Architecture:** `ThemesPanel` (`AdvantageCarousel.tsx:80`) maps a literal `(['light','dark','warm'] as const)` into
`.adv-theme-card` elements, each a real `data-theme={t}` container holding a name, a "live" `Badge`, and a `Button` —
so the cards genuinely restyle. The full set is `THEMES` in `apps/site/src/theme.ts` (12 entries). The carousel
(`AdvantageCarousel`) runs `useSignalEffect(() => { const id = setInterval(() => { if (paused.value) return; …advance
}, ROTATE_MS); return () => clearInterval(id) })` with `ROTATE_MS = 6500`; `paused` is set on hover/focus. There is no
`prefers-reduced-motion` read. `useMediaQuery` is available from `@cascivo/core` (used in `Header.tsx`). `landing.css`
already has `@media (prefers-reduced-motion: reduce)` blocks for reveal animations.

**Tech Stack:** Preact + `@cascivo/core` signals (`useSignal`/`useSignalEffect`/`useMediaQuery`); `Badge`/`Button`
components; `landing.css`; Vitest + @testing-library/react for the new test. No new deps.
