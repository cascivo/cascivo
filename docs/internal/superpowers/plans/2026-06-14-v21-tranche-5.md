> Build the fifth dashboard by copying T1 patterns. This app's emphasis is **live data** via
> `createSimulation` and **heavy `@cascivo/charts` composition** (LineChart, Heatmap, Sparkline, Meter,
> Bullet, ProgressCircle). It closes most remaining chart-coverage gaps for T6.
>
> Current-state assumptions — RE-VERIFY before starting:
>
> - T1 kit + reference patterns shipped; `createSimulation`/`useSimulation` proven deterministic and
>   pausable. `@cascivo/charts` exports LineChart, Heatmap, Sparkline, Meter, Bullet (and ProgressCircle
>   is in `@cascivo/react`) — re-verify names in `packages/charts/src/index.ts` and `packages/react/src/index.ts`.

**Goal:** A functional **Cascade Pulse** observability dashboard: a metrics grid that **ticks live**
(LineChart latency p50/p95/p99, request Heatmap, per-host `Sparkline` grid, `Meter`/`Bullet` SLOs,
`ProgressCircle` resource usage), a time-range selector, an alerts list that fires/clears as the sim
runs, and an auto-scrolling log stream. A global pause control freezes the simulation.

**Architecture:** New `apps/examples/pulse/` mirroring T1 (vite config + aliases incl.
`@cascivo/example-kit` + `@cascivo/charts`, `data/`, `api/`, `App.tsx`, `sections/`, `test/`). No kit
changes expected.

**Tech stack:** unchanged. Live updates run through `useSimulation`/`useSignalEffect`; `useSignals()`
first; no `useState`/`useEffect`/`useContext`. Strings via `@cascivo/i18n`.

### Task 2: Metric model + live simulation

**Files:** Create `apps/examples/pulse/src/data/*.ts`, `src/sim/metrics.ts`

- [ ] **Step 1:** Seeded fixtures: hosts/services, an initial backfill of time-series points (latency,
      throughput, error rate, CPU/mem), a request density grid (for the Heatmap), SLO targets, and seed
      log lines. From `seededRandom`.
- [ ] **Step 2:** A `createSimulation` that on each tick appends a new point to each series (sliding
      window), updates the heatmap grid, may fire/clear an alert when a threshold is crossed, and appends
      log lines. Deterministic under the seed; pausable. Drive it with `useSimulation` mounted once at the
      app root.

### Task 4: Metrics grid (heavy chart composition)

**Files:** Create `src/sections/Metrics.tsx`, `src/sections/SloStrip.tsx` (+ CSS)

- [ ] **Step 1:** `LineChart` of latency p50/p95/p99 (multi-series), `Heatmap` of request density,

- [ ] **Step 2:** SLO strip: `Meter`/`Bullet` for SLO attainment, `ProgressCircle` for CPU/mem. Above-the-

### Task 5: Alerts + log stream

## Tranche exit criteria

1. `apps/examples/pulse` runs and builds with no prior full build (charts + kit aliases present).
2. Metrics tick live via `createSimulation` (LineChart/Heatmap/Sparkline/Meter/Bullet/ProgressCircle);
   the global Pause freezes them; updates are deterministic under the seed and don't cause CLS.
3. Alerts fire/clear with the sim; acknowledging one persists across reload; the log stream auto-scrolls
   and respects manual scroll-up (handled via `useSignalEffect`).
4. Playwright smoke + axe pass; bundle audit acceptable.
5. No `useState`/`useEffect`/`useContext`; `useSignals()` first; strings via `@cascivo/i18n`; mobile-clean
   320–414.
