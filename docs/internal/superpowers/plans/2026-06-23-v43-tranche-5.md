# v43 — Tranche 5: Interactive Connect + Chrome

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the graph interactive and navigable. Ship **interactive connect** — drag from a `source`
handle to a `target` handle to create an edge (a live connection-in-progress SVG path follows the pointer;
dropping on a valid handle fires `onConnect`) — and the **chrome**: `<FlowControls>` (zoom in / out / fit,
built from cascivo `icon-button`s, i18n-defaulted labels), `<FlowMiniMap>` (a scaled SVG overview with a
draggable viewport rectangle), and `<FlowPanel>` (an absolutely-positioned slot for custom UI such as a
legend).

**Architecture:** `useConnection` wires a pointer drag (via `useDraggable`/pointer math in `useSignalEffect`)
that starts on a `source` handle, tracks a live path to the pointer, and on `pointerup` over a `target` handle
calls `onConnect({ source, target, sourceHandle, targetHandle })`. `<FlowControls>` calls the T2 `useViewport`
actions (`zoomTo`, `fitView`). `<FlowMiniMap>` renders a scaled SVG of `graphBounds` + node rects + a viewport
rectangle; dragging the rect pans the main `viewport` (reuse `useDraggable`). `<FlowPanel>` is a positioned
slot. Per CLAUDE.md: `useSignals()` first; no `useState`/`useEffect`; all pointer side effects in
`useSignalEffect`; controls are real buttons (WCAG AA, ≥44px coarse).

**Tech Stack:** signal-driven TSX + CSS Modules + SVG; `@cascivo/core` (`useDraggable`, `useSignal`/
`useComputed`/`useSignalEffect`/`useSignals`, `cn`); the existing cascivo `icon-button` (or `button`) for
controls; `@cascivo/icons`; the T1–T4 engine (`useFlow`, `useViewport`, `geometry`, `path`); `@cascivo/i18n`
for control labels; Vitest + Testing Library.
