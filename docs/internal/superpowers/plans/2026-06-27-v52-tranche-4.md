# v52 — Tranche 4: Canvas Rendering Path (high-density data)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give cascivo nivo's one thing SVG fundamentally can't do — **render tens of thousands of marks** (N-1).
nivo ships `LineCanvas`/`ScatterPlotCanvas`/`HeatMapCanvas` because SVG DOM nodes choke past ~5–10k. Add an opt-in
`renderer="canvas"` (or auto, past a density threshold) for Line/Scatter/Heatmap that paints the dense marks to a
`<canvas>` — **while keeping the SVG chrome (axes/grid/legend/annotations) and the entire a11y tree in the DOM**
(Decision 2). Lens: **nivo**. Heaviest and most isolated tranche — a parallel paint behind a flag.

**Architecture:** today `chart-frame.tsx` renders one `<svg>`. Add a `renderer?: 'svg' | 'canvas' | 'auto'` prop. For
`canvas`/auto-over-threshold, the frame stacks a `<canvas>` (absolutely positioned, sized to the plot, `devicePixelRatio`
scaled) **under** the existing SVG, which now carries only chrome + the focusable/`aria-live`/fallback layers (no
marks). A chart provides a `paint(ctx, { width, height, scales })` callback; `core/canvas-layer.tsx` runs it inside a
`useSignalEffect` on every size/data/zoom change. The keyboard layer, `aria-live` readout, focus ring, and
visually-hidden fallback `<table>` are unchanged — Canvas is an *extra paint*, not a replacement for the a11y tree.
`auto` switches to canvas when the mark count exceeds a threshold (e.g. > 2000 points) and `log()`-style notes the
switch in dev.

**Tech Stack:** Canvas 2D via a `<canvas ref>` (the one allowed `useRef`-for-DOM) + `useSignalEffect` for the paint;
the existing scales; `devicePixelRatio` handling. No new dependency. No `useEffect`.

### Task 1: The canvas layer

**Files:** `packages/charts/src/core/canvas-layer.tsx`

- [ ] **Step 1:** `CanvasLayer({ width, height, paint })` renders an absolutely-positioned `<canvas>` sized to the plot;
      in a `useSignalEffect`, set the backing store to `width*dpr × height*dpr`, scale the context by `dpr`, clear, and
      call `paint(ctx, { width, height })`. Re-runs when width/height (or any signal `paint` reads) change.
- [ ] **Step 2:** `aria-hidden` on the canvas (it's a pixel paint; the data lives in the SVG/DOM a11y tree).
- [ ] **Step 3:** `canvas-layer.test.tsx`: the canvas mounts at the right CSS size and `paint` is invoked (mock the 2D
      context); a size change re-invokes it.

### Task 2: Renderer switch in the frame

**Files:** `core/chart-frame.tsx`

- [ ] **Step 1:** Add `renderer?: 'svg' | 'canvas' | 'auto'` (default `'svg'`) and an optional `paint` prop. When
      `canvas` (or `auto` past the density threshold), render `<CanvasLayer paint={paint} …>` beneath the SVG and have
      the SVG render chrome + the focusable/`aria-live`/fallback layers **only** (charts pass `null` marks in that mode).
- [ ] **Step 2:** The focus ring + tooltip overlay still draw in the DOM at the focused point's `cx/cy` (they read the
      same `TooltipModel`), so keyboard nav looks identical over the canvas.
- [ ] **Step 3:** `auto`: switch when `points.length` exceeds the threshold; keep SVG otherwise.

### Task 3: Canvas paint for Line / Scatter / Heatmap

**Files:** `line-chart.tsx`, `scatter-chart.tsx`, `heatmap.tsx`

- [ ] **Step 1:** Each chart, when `renderer !== 'svg'`, supplies a `paint(ctx, { width, height })` that draws the same
      geometry it draws in SVG (line strokes / point dots / heat cells) using the resolved scales and the
      `--cascivo-chart-*` colors (resolve the CSS var to a concrete color via `getComputedStyle` once per paint).
- [ ] **Step 2:** Keep the SVG path (default) byte-for-byte unchanged; canvas is purely additive.
- [ ] **Step 3:** The fallback `<table>` + tooltip model are identical in both renderers (a11y parity).
- [ ] **Step 4:** Test (jsdom, mocked 2D ctx): a `renderer="canvas"` Line invokes `paint` and still renders the
      fallback table + keyboard layer; a 5k-point `auto` Scatter chooses canvas.
