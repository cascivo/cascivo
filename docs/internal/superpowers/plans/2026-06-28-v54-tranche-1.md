# v54 — Tranche 1: Caret Coordinate Primitive

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the one genuinely new piece a textarea slash menu needs and the editor lacks: a **caret → pixel
mapping** (S-1). Every reference editor (CodeMirror's `view.coordsAtPos`, ProseMirror's `coordsAtPos`) has this
built-in; cascivo has none (verified: 0 hits for `coordsAtPos`/`caretCoords`/`getCaretCoordinates` in
`packages/editor/src`). Lens: **CodeMirror/ProseMirror + the plain-textarea libraries.** Foundational — T3 anchors the
menu against this. The editor is **monospace** (`--cascivo-font-mono`, `code-editor.module.css:10`) and already measures
`lineHeight` (`code-editor.tsx:285`) + `scrollTop` (`code-editor.tsx:288-294`) and ships `offsetToLineCol` (`find.ts`),
so the default path is cheap arithmetic — **no mirror-div hack** (S-7).

**Architecture:** one pure module, two coordinate paths + one measurement.
- **`caretCoords(text, offset, metrics)`** (the default path): compute `{ line, col } = offsetToLineCol(text, offset)`,
  expand `col` to a **visual** column honoring tabs (`tabSize`), then
  `top = line * lineHeight - scrollTop + padTop`, `left = visualCol * charWidth - scrollLeft + padLeft`. Pure,
  DOM-free, unit-testable. Correct for the `insertSpaces:true` / `wrap:false` default (S-9).
- **`caretRectFromPre(pre, offset)`** (the fidelity path): place a DOM `Range` at the corresponding text node in the
  already-rendered, pixel-aligned `.pre` highlight layer and read `getClientRects()` — handles tabs and soft-wrap for
  free. Used when `wrap` is on or tabs are present; returns a viewport-relative rect the caller converts to
  `.codeArea`-relative.
- **`measureCharWidth(ta)`**: one-time measure of a monospace glyph advance (a hidden probe span with the textarea's
  computed `font`), cached; recomputed only if the font changes.

**Tech Stack:** a new pure `caret.ts` (no React); reuses `offsetToLineCol` from `find.ts`. No new dependency.

### Task 1: Arithmetic caret coordinates (the default path)

**Files:** `caret.ts`

- [ ] **Step 1:** Define `interface CaretMetrics { charWidth: number; lineHeight: number; scrollTop: number;
      scrollLeft: number; padTop: number; padLeft: number; tabSize: number }` and
      `caretCoords(text: string, offset: number, m: CaretMetrics): { top: number; left: number }`.
- [ ] **Step 2:** Use `offsetToLineCol(text, offset)` (import from `./find.ts`) for `{ line, col }`. Expand the column
      to a **visual** column: walk the current line's first `col` chars, advancing `visualCol` by `tabSize - (visualCol % tabSize)`
      for a `\t` and by `1` otherwise (so tab stops line up with the rendered `tab-size`).
- [ ] **Step 3:** Return `top = line * m.lineHeight - m.scrollTop + m.padTop`,
      `left = visualCol * m.charWidth - m.scrollLeft + m.padLeft`. Clamp nothing here (the caller clamps to the plot);
      keep it a pure transform.

### Task 2: `.pre` Range path (tabs / soft-wrap fidelity)

**Files:** `caret.ts`

- [ ] **Step 1:** `caretRectFromPre(pre: HTMLElement, offset: number): DOMRect | null` — walk the `.pre` text nodes
      accumulating length until `offset` falls inside a node, set a collapsed `Range` there, and return the first
      `range.getClientRects()[0]` (fallback to `range.getBoundingClientRect()`). Return `null` if the offset isn't
      rendered (e.g. windowed-out — never the case while typing).
- [ ] **Step 2:** Document at the call-site contract that this returns a **viewport-relative** rect; the caller
      subtracts the `.codeArea` `getBoundingClientRect()` origin to get a local position. Keep the DOM walk allocation-free
      (no `querySelectorAll` of the whole tree — a `TreeWalker` over text nodes).

### Task 3: One-time char-width measurement

**Files:** `caret.ts`

- [ ] **Step 1:** `measureCharWidth(ta: HTMLTextAreaElement): number` — create a detached `<span>` with the textarea's
      computed `font`/`letterSpacing`/`tabSize`, fill it with a known run (e.g. `'0'.repeat(10)`), measure
      `getBoundingClientRect().width / 10`. Guard SSR (`typeof document === 'undefined'` → return 0; the caller skips
      positioning until measured).
- [ ] **Step 2:** Note in a comment that the caller caches the result (a ref) and only re-measures when the font
      changes; this module stays stateless.
