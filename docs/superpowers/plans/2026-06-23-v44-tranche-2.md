# v44 — Tranche 2: `CodeEditor` + `Highlight` (textarea overlay)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the two React surfaces on the T1 engine: **`Highlight`** (read-only highlighted `<pre><code>`)
and **`CodeEditor`** (a native `<textarea>` overlaid on a `Highlight` layer). Ship the **basic set**:
controllable `value`, `language`, line numbers, Tab/Shift-Tab indent/dedent, read-only, placeholder, word-wrap
toggle, `tabSize`, current-line highlight, scroll-sync. Signal-driven, a11y-complete (the textarea **is** the a11y
surface), each with a `component.meta.ts` (incl. `intent`) and tests.

**Architecture:** `Highlight` maps `tokenizeDocument(getGrammar(language), value)` (from T1) to per-line
`<span class={kind}>` inside `<pre aria-hidden><code>`. `CodeEditor` stacks, in one positioned container: (1) the
`Highlight` layer (visual), (2) a transparent native `<textarea>` (the editing surface, on top), and (3) an
optional `aria-hidden` line-number gutter. The `value` is a `useControllableSignal`
(`value`/`defaultValue`/`onValueChange`). On input, the textarea updates the signal; the `Highlight` layer
re-renders from the signal (rAF-debounced highlight is added in T4 — here a straight signal read is fine).
Scroll-sync (textarea → pre + gutter) and any imperative focus run in **`useSignalEffect`** with cleanup — never
`useEffect`. `Tab` inserts `tabSize` spaces (or `\t`) at the selection and `Shift+Tab` dedents, both via the
textarea's `setRangeText`/selection API. Per CLAUDE.md: `useSignals()` first; no `useState`/`useEffect`/
`useContext`/`useReducer`; visual states (focus/hover/disabled) via CSS; `data-*` only for what CSS can't express.

**Tech Stack:** signal-driven TSX + CSS Modules; `@cascivo/core` (`useControllableSignal`, `useSignal`,
`useComputed`, `useSignalEffect`, `useSignals`, `useRef`, `cn`); `@cascivo/i18n` for default `aria-label`/
placeholder; the T1 engine; Vitest + Testing Library.

### Task 2: Implement `Highlight` (read-only)

**Files:** `packages/editor/src/editor/highlight/highlight.tsx`

- [ ] **Step 1:** Props: `value: string`, `language?: string` (default `'plaintext'`), `lineNumbers?: boolean`,
      `wrap?: boolean`, `tabSize?` (default 2), `label?` (a11y), `className`/`...rest`. `useSignals()` first.
- [ ] **Step 2:** Compute `const lines = useComputed(() => tokenizeDocument(getGrammar(language), value))`.
      Render `<pre><code>` with one block per line, each token a `<span className={styles[token.kind]}>`; a
      trailing empty line renders a zero-width placeholder so blank lines keep height.
- [ ] **Step 3:** Optional `aria-hidden` gutter column (line indices) when `lineNumbers`. The whole block is a
      read-only display: `role` left default on `<pre>`; expose `aria-label` from `label`/i18n default when
      provided.

### Task 3: Implement `CodeEditor` (editable overlay)

**Files:** `packages/editor/src/editor/code-editor/code-editor.tsx`

- [ ] **Step 1:** Props: `value?` + `defaultValue?` + `onValueChange?` (controllable), `language?`,
      `lineNumbers?` (default true), `readOnly?`, `disabled?`, `placeholder?`, `tabSize?` (default 2),
      `insertSpaces?` (default true), `wrap?` (default false), `spellCheck?` (default false), `label?`/
      `aria-label`, `name`, `id`, `className`, plus pass-through textarea props. `useSignals()` first.
- [ ] **Step 2:** `const text = useControllableSignal({ value, defaultValue: defaultValue ?? '', onChange:
      onValueChange })`. Render the container with: a `<Highlight value={text.value} language={language}
      lineNumbers={lineNumbers} wrap={wrap} />` layer (`aria-hidden`), the optional gutter, and a transparent
      native `<textarea value={text.value} onChange={e => (text.value = e.currentTarget.value)} readOnly
      disabled spellCheck placeholder aria-label={…} />` on top. Container carries `data-wrap`,
      `data-line-numbers`, and a `--tab-size` custom property.
- [ ] **Step 3:** `onKeyDown`: `Tab` → insert `insertSpaces ? ' '.repeat(tabSize) : '\t'` at the selection (or
      indent each selected line) via `setRangeText`, write back to `text`, and `preventDefault`; `Shift+Tab` →
      dedent the selected lines. (Optional, simple) `Enter` auto-indent to the current line's leading whitespace.
      No `useEffect` — this is a React `onKeyDown` prop mutating the signal.
- [ ] **Step 4:** Scroll-sync: a `useRef` on the `<textarea>` and the `<pre>`/gutter; in a **`useSignalEffect`**,
      attach a `scroll` listener that copies `scrollTop`/`scrollLeft` from the textarea to the highlight layer +
      gutter; return a cleanup that removes it. SSR/no-DOM guarded. Current-line highlight is derived from the
      caret line (a `useComputed` from selection) and exposed as a CSS-driven row marker (no per-keystroke React
      re-render of the document).

- [ ] **Step 3:** `data-wrap` toggles `white-space: pre` ↔ `pre-wrap` on **both** layers in lock-step. Any

      chrome (e.g. a wrap toggle, if surfaced) is ≥44px under `@media (pointer: coarse)`.
