# v27 — Tranche 1: SPA Client-Side Router

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Convert the landing from a full-reload MPA to a SPA. Every same-origin anchor click
must call `history.pushState` instead of triggering a browser navigation. A `currentPath` signal
drives `App`'s route selection. `document.startViewTransition` wraps path updates so the existing
`@view-transition` CSS animations continue to fire; browsers without the API get an instant swap.

**Architecture:** New `router.ts` module (signal + interceptor). `App.tsx` reads from the signal
instead of `window.location.pathname`. `Header.tsx` uses `navigate()` for in-page links.

**Tech Stack:** `@preact/signals-react` (signal, direct import), `useSignals`, `useSignalEffect`,
`history.pushState`, View Transitions API.

### Task 2: Create the router module

- [ ] **Step 1: Create `apps/landing/src/router.ts`**

  ```ts
  import { signal } from '@preact/signals-react'

  function normPath(p: string) {
    return p.replace(/\/+$/, '') || '/'
  }

  export const currentPath = signal(
    typeof window !== 'undefined' ? normPath(window.location.pathname) : '/',
  )

  export function navigate(href: string) {
    const target = normPath(new URL(href, location.origin).pathname)
    if (target === currentPath.value) return
    history.pushState(null, '', href)
    if ('startViewTransition' in document) {
      ;(document as Document & { startViewTransition: (cb: () => void) => unknown }).startViewTransition(
        () => { currentPath.value = target },
      )
    } else {
      currentPath.value = target
    }
  }

  export function initRouter() {
    document.addEventListener('click', (e) => {
      const a = (e.target as Element | null)?.closest('a')
      if (!a?.href) return
      const url = new URL(a.href)
      if (url.origin !== location.origin) return
      if (a.target === '_blank') return
      if (a.hasAttribute('download')) return
      e.preventDefault()
      navigate(url.pathname)
    })

    window.addEventListener('popstate', () => {
      currentPath.value = normPath(location.pathname)
    })
  }
  ```

  Notes:
  - `normPath` strips trailing slashes and ensures `/` for the root.
  - The `startViewTransition` cast is needed because TypeScript's lib may not include the API yet.
  - The `popstate` handler handles browser back/forward buttons.
  - No `import { useSignals }` here — this is a plain TS module, not a React component.

### Task 3: Update App.tsx to use the router

- [ ] **Step 1: Replace `window.location.pathname` read**

  Find the line:
  ```ts
  const pathname =
    typeof window !== 'undefined' ? window.location.pathname.replace(/\/+$/, '') || '/' : '/'
  ```

  Replace with:
  ```ts
  useSignals()
  const pathname = currentPath.value
  ```

  Add `useSignals` to the imports from `@cascivo/core`. Add `{ currentPath, initRouter }` to
  imports from `'./router'`.

- [ ] **Step 2: Wire `initRouter` on mount**

  Inside the `App` function, after `useSignalEffect(() => initReveal())`, add:

  ```tsx
  useSignalEffect(() => {
    initRouter()
  })
  ```

  `useSignalEffect` with no signal reads runs once on mount and cleans up on unmount — this is
  the correct pattern for a one-time DOM side effect.

- [ ] **Step 3: Ensure `App` calls `useSignals()`**

  `App` now reads `currentPath.value` during render, so it must call `useSignals()` as its
  first statement. If `App` doesn't already call it, add it:

  ```tsx
  export function App() {
    useSignals()
    useSignalEffect(() => initReveal())
    useSignalEffect(() => { initRouter() })
    // ...rest unchanged
  }
  ```
