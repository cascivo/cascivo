> Current-state assumptions ŌĆö RE-VERIFY before starting:
>
> - `packages/charts/src/charts/` contains 16 charts; `core/chart-frame.tsx` exposes
>   `ChartFrame` (`title`, `description`, `width`, `height = 300` default, `fallback`,
>   `className`, `data-state`); `core/use-chart.ts` exports `DEFAULT_MARGINS`.
> - Chrome lives in `chrome/axis.tsx`, `chrome/grid-lines.tsx`, `chrome/legend.tsx`;
>   cartesian charts render them unconditionally (see `area-chart.tsx`), legend defaults to
>   `series.length > 1`.
> - `sparkline`, `meter`, `kpi` render no chrome (reference implementations ŌĆö untouched).
> - Each chart ships `<name>.meta.ts` + `<name>.test.tsx`; charts are npm-distributed
>   (registry `install` field), so no registry file lists change ŌĆö only regenerated metas.
> - Docs Charts page: `apps/docs/src/pages/ChartsPage.tsx`, route `/charts`.
> - Storybook stories are hand-written in `apps/storybook/stories/`.

**Goal:** Every chrome-bearing chart accepts `plain?: boolean`. Plain mode renders only the
marks: no axes, no grid lines, no legend, margins collapsed to 2px, default height 48,
`data-plain` attribute for styling hooks ŌĆö while keeping `role="img"`, title/desc, and the
table fallback. Demonstrated in docs at table-cell size.

**Architecture:** One shared `PLAIN_MARGINS` constant in `core/use-chart.ts`; each chart
resolves `margins`/`height`/chrome conditionally from its own `plain` prop. `ChartFrame`
gains an optional `plain` passthrough that emits `data-plain=""` on the frame element.
No forked components, no behavior change when `plain` is absent.

**Tech stack:** unchanged ŌĆö `@cascivo/charts` (published npm package), vitest +
@testing-library/react.
