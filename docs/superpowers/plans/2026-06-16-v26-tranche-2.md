# v26 — Tranche 2: Layout Stability — Fixed Heights, Bar Chart Labels, QuickStart Width

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Eliminate three layout-shift problems on the landing home page: (1) animated section
containers shift their neighbours up and down as content changes height, (2) the BarChart in
"Numbers, not adjectives." clips its y-axis labels, (3) the import code block in "Up and running
in three steps" overflows the page width on narrow viewports.

**Architecture:** Mostly CSS changes in `landing.css`. One prop change in `ProofTeasers.tsx` for
the BarChart label width. No signal or component logic changes.

- Modify: `apps/landing/src/landing.css`
- Modify: `apps/landing/src/sections/ProofTeasers.tsx`

### Task 1: Fix ExamplesGallery carousel info height

The `.examples-carousel-info` block contains a title, description paragraph, coverage chips, and
a footer link. Descriptions across the five demos vary significantly in length. When the carousel
auto-advances, the block height changes and everything below the carousel jumps.

- [ ] **Step 1: Measure the tallest description**

  Open the dev server:
  ```sh
  pnpm exec vp run @cascivo/landing#dev
  ```

  On the home page, open DevTools → Elements. Inspect `.examples-carousel-info` for each of the
  five demos (either wait for auto-advance or click the dots). Note the tallest rendered height
  in pixels across all five.

- [ ] **Step 2: Set `min-block-size` on `.examples-carousel-info`**

  In `apps/landing/src/landing.css`, find the `.examples-carousel-info` rule. Add:

  ```css
  .examples-carousel-info {
    /* existing rules … */
    min-block-size: 14rem; /* adjust to the tallest measured height + small buffer */
  }
  ```

  Use `rem` not `px`. Divide the pixel measurement by 16 (the root font size). Round up to the
  nearest 0.5 rem.

- [ ] **Step 3: Verify no layout shift on auto-advance**

  With the dev server running, watch the carousel auto-advance through all five demos. The
  content below "Drive it, don't read about it." must not move. Check at 375 px, 640 px, and
  1280 px widths.

### Task 2: Fix ThemeDemo card height stability

`ThemeDemo` cycles through ten themes. The `SignupCard` rendered inside may vary in height
across themes due to different token sizes (line-heights, padding, border-radius). When height
changes, everything below `ThemeDemo` shifts.

- [ ] **Step 1: Measure the tallest card across all themes**

  In the dev server, open the home page. Scroll to the "One form, ten personalities" section.
  Open DevTools → Elements. Inspect the theme card container for each theme (click dots or
  wait for auto-advance). Note the tallest rendered height.

- [ ] **Step 2: Set `min-block-size` on the card container**

  In `apps/landing/src/landing.css`, find the `.theme-demo-card` (or whatever class wraps the
  single card in `ThemeDemo.tsx`). Add `min-block-size` matching the tallest observed height
  plus a small buffer.

  If no dedicated card wrapper class exists, open `apps/landing/src/sections/ThemeDemo.tsx`,
  find the outermost card wrapper `<div>`, and add a class `theme-demo-card-wrap` to it. Then
  in CSS:

  ```css
  .theme-demo-card-wrap {
    min-block-size: 22rem; /* adjust to measurement */
  }
  ```

- [ ] **Step 3: Verify no layout shift on theme cycle**

  Watch all ten theme transitions. The sections below ThemeDemo must not shift vertically.
  Check at 375 px and 1280 px.

### Task 3: Fix BarChart y-axis label clipping in ProofTeasers

The horizontal BarChart in `ProofTeasers.tsx` shows `cascivo`, `shadcn`, `carbon` on the left
axis. These labels are clipped because the chart container does not reserve enough horizontal
space for them.

- [ ] **Step 1: Open ProofTeasers.tsx and find the BarChart usage**

  Read `apps/landing/src/sections/ProofTeasers.tsx`. Locate:

  ```tsx
  <BarChart
    orientation="horizontal"
    title="Total gzip size: cascivo vs shadcn vs Carbon"
    series={BUNDLE_SERIES}
    x={(d) => d.x}
    y={(d) => d.y}
    height={72}
  />
  ```

- [ ] **Step 2: Check BarChart props for a label-width control**

  Read `packages/charts/src/bar-chart/bar-chart.tsx` (or wherever the BarChart is defined).
  Look for props like `yLabelWidth`, `labelWidth`, `marginLeft`, or similar.

  - If a `yLabelWidth` (or equivalent) prop exists: add it to the usage, set to `72` (px).
    ```tsx
    <BarChart yLabelWidth={72} … />
    ```
  - If no such prop exists: go to Step 3 for the CSS wrapper approach.

- [ ] **Step 3: CSS wrapper fallback (only if no prop exists)**

  In `apps/landing/src/sections/ProofTeasers.tsx`, wrap the `BarChart` in a div:

  ```tsx
  <div className="proof-chart-wrap">
    <BarChart … />
  </div>
  ```

  In `apps/landing/src/landing.css`, add:

  ```css
  .proof-chart-wrap {
    padding-inline-start: 5rem;
    overflow: visible;
  }
  .proof-chart-wrap svg {
    overflow: visible;
  }
  ```

  Check if this causes the labels to appear. If not, investigate the chart's internal SVG
  clip path and find the correct approach.

- [ ] **Step 4: Verify labels are fully visible**

  In the dev server, scroll to "Numbers, not adjectives." → "Performance" card. Hover a bar
  to confirm the tooltip also appears (this was fixed in v25 — confirm it is still working).
  Confirm all three y-axis labels (`cascivo`, `shadcn`, `carbon`) are fully visible at 375 px,
  640 px, and 1280 px.

### Task 4: Fix QuickStart import block overflow

In `QuickStart.tsx`, the prebuilt pathway renders:

```tsx
<CopyCommand
  command={`import { Button } from '@cascivo/react'\nimport '@cascivo/themes/light.css'`}
/>
```

The `CopyCommand` component expands to the width of its content, which can overflow the
`.quickstart-prebuilt-steps > div` container on narrow viewports.

- [ ] **Step 1: Identify the overflow**

  In the dev server at 320 px width, scroll to "Up and running in three steps". Inspect the
  second step in the prebuilt block. If horizontal scrollbar appears, the overflow is confirmed.

- [ ] **Step 2: Add CSS fix in landing.css**

  In `apps/landing/src/landing.css`, find or add a rule for the prebuilt steps container:

  ```css
  .quickstart-prebuilt-steps > div {
    max-inline-size: 100%;
    overflow-x: auto;
  }

  .quickstart-prebuilt-steps .copy-command {
    max-inline-size: 100%;
  }
  ```

  If `.copy-command` already has a max-inline-size rule, check whether it has `100%` or a fixed
  value. Change to `100%` inside the prebuilt context.

- [ ] **Step 3: Verify no overflow at 320 px**

  In the dev server at 320 px, confirm no horizontal scrollbar on the page when the QuickStart
  section is visible. The code block should wrap or scroll internally if the content is long.
