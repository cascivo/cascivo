# v54 — Tranche 2: Trigger Detection + Menu State

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Decide *when* the slash menu is open and *what* it filters to (S-2) — the equivalent of Lexical's
`useBasicTypeaheadTriggerMatch` / `@tiptap/suggestion`'s match, but reactive over the editor's existing signals.
Lens: **Lexical/TipTap.** Today there is no `/`-trigger scan and **no public `onInput`/`onSelect` seam** — so detection
must live *inside* `CodeEditor`, driven by the `caretOffset` + `text` signals (`code-editor.tsx:236, 297-324`). The
critical correctness rule (Decision 2): the `/` **types into the textarea normally** and is detected *after* input —
it is **never** a keymap binding (that would `preventDefault` and swallow the slash).

**Architecture:** a pure detector + pure filter + four signals.
- **`detectTrigger(text, caret)`** (`slash-trigger.ts`): scan backwards from `caret` to the nearest `/`; the trigger is
  active iff the char before that `/` is a line start or whitespace (a word boundary) **and** the span from `/`+1 to
  `caret` contains no whitespace. Returns `{ start, query }` (where `start` is the `/` index, `query` the text after it)
  or `null`.
- **`filterCommands(commands, query)`**: case-insensitive match on `label` + `keywords`; empty query → all commands;
  stable order (matches keep input order). Pure.
- **Signals in the component:** `slashOpen` / `slashStart` / `slashQuery` / `slashIndex` (mirroring the `findOpen`/
  `findQuery` template at `code-editor.tsx:228-233`), recomputed in a `useSignalEffect` keyed on `caretOffset.value` +
  `text.value`. `slashIndex` resets to 0 whenever the filtered list changes.

**Tech Stack:** a new pure `slash-trigger.ts` (no React) + signal state in `code-editor.tsx`; `useSignalEffect` for the
reactive detection. No new dependency.

### Task 1: Pure trigger detection

**Files:** `slash-trigger.ts`

- [ ] **Step 1:** `interface Trigger { start: number; query: string }` and
      `detectTrigger(text: string, caret: number): Trigger | null`. Scan backward from `caret-1`: stop at the first
      `/`; if a whitespace/newline is hit first (or the start of text without a `/`), return `null`.
- [ ] **Step 2:** Word-boundary guard: the char at `slashIndex-1` must be `undefined` (start of document), `\n`, or
      whitespace — so a `/` inside `http://` or a path doesn't trigger. The query is `text.slice(slashIndex+1, caret)`;
      reject if it contains any whitespace.
- [ ] **Step 3:** Keep it allocation-light (index math, one `slice` for the query). Document that the caller treats a
      `null` as "close the menu".

### Task 2: Pure command filter

**Files:** `slash-trigger.ts`

- [ ] **Step 1:** `filterCommands<T extends { label: string; keywords?: string[] }>(commands: readonly T[], query: string): T[]`
      — lowercase the query; keep commands whose `label` or any `keyword` includes it; empty query keeps all. Preserve
      input order (no fuzzy ranking in v1).

### Task 3: Wire the signals + reactive detection into the component

**Files:** `code-editor.tsx`

- [ ] **Step 1:** Next to the find signals, add `const slashOpen = useSignal(false)`, `const slashStart = useSignal(0)`,
      `const slashQuery = useSignal('')`, `const slashIndex = useSignal(0)`. (Per CLAUDE.md these are signals, **not** an
      FSM — the signal *is* the state.)
- [ ] **Step 2:** Add a `useSignalEffect` that reads `caretOffset.value` + `text.value`, calls
      `detectTrigger(text.value, caretOffset.value)`, and sets `slashOpen`/`slashStart`/`slashQuery` accordingly
      (close when `null`). Gate on `editable` (no menu in `readOnly`/`disabled`) and on `commands?.length` (no menu
      with no commands — `commands` arrives in T5; until then guard against `undefined`).
- [ ] **Step 3:** When the filtered list (computed from `slashQuery` — the filtering call can live in render for now)
      changes length, clamp `slashIndex` into range (mirror the find-panel `currentMatch` clamp at
      `code-editor.tsx:383`). Confirm the `/` keystroke is **not** added to any keymap binding.
