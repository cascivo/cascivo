> Current-state assumptions — RE-VERIFY before starting:
>
> - `apps/landing/src/App.tsx` routes via a `ROUTES` record keyed by pathname; pages are `React.lazy`
>   chunks under `src/pages/`. Existing routes: `/`, `/accessibility`, `/performance`, `/guides`, `/og`.

> - `public/sitemap.xml` is a static file listing prerendered routes.

**Goal:** A `/examples` hub page listing all five demos (thumbnail/screenshot, name, "feels like",
one-line description, coverage chips, link to the detail page, and a direct "Open live demo" link to
`/demos/<name>/`). It is prerendered, in the sitemap, reachable from header nav + footer, and the
homepage `ExamplesGallery` cards now link to `/examples/<name>` (detail pages land in T3; until then
they may point at the hub anchor or the live demo — see Step notes).

**Architecture:** New `src/pages/ExamplesPage.tsx` (+ a `src/pages/examples/` data module shared with
T3). Register the route in `App.tsx`, `route-head.ts` (`ROUTE_HEAD['/examples']` + add `'examples'` to
`PRERENDER_ROUTES`), `sitemap.xml`, `Header.tsx`, `Footer.tsx`. Rewire `ExamplesGallery.tsx` links.

**Tech stack:** unchanged. `useSignals()` first where signals are read; tokens; mobile-first; i18n.
