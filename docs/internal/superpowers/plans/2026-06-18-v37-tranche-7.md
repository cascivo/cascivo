# v37 — Tranche 7: Generate the Consumer Hand-off Guide

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Produce a single, self-contained markdown file — `docs/v37-CONSUMER-CHANGES.md` — that the
**boringtools** project (or any downstream consumer) can read on its own to undo its workarounds and adopt
the v37 fixes. It is the mirror image of this sprint: for each of the 13 feedback items, "what cascivo
changed" + "what you should do on your side."

**Architecture:** This tranche runs **last**, after T1–T6 have actually shipped, so the guide documents
real, merged changes (new export, new layer names, new bundle/exports, new `AppShell`, new token
manifest, new docs) — not intentions. The file is written for an audience that does **not** have this
repo: it must be complete, copy-pasteable, and reference only published package surfaces
(`@cascivo/react`, `@cascivo/themes`, `@cascivo/tokens`) and version numbers, never internal paths.

**Tech Stack:** Markdown authoring; cross-checked against the shipped T1–T6 artifacts.
