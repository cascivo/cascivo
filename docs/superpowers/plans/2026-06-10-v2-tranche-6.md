### Task 6: Owned-code update story — lock file, three-way diff, outdated

**Files:** `packages/cli/src/utils/lock.ts`, `packages/cli/src/commands/{add,update}.ts`, new `packages/cli/src/commands/outdated.ts`; `scripts/registry/generate.ts` (per-component versions); tests for all

- [ ] **Step 1: Per-component versions in the registry**

In `scripts/registry/generate.ts`: component `version` = derived from conventional commits touching that component's directory — `git log --format=%s -- <dir>` count of `feat:`/`fix:` prefixed commits mapped to a synthetic `0.<feats>.<fixes>` version. Deterministic from git history; documented as "change counter, not semver promise". (Full semver per component would need release tooling — out of scope, recorded as the pragmatic choice.) The drift job runs in CI with full history? — **verify**: `actions/checkout@v4` defaults to depth 1 → versions would all collapse. Fix: set `fetch-depth: 0` on the drift job's checkout step. Test: regenerating twice yields identical output.

- [ ] **Step 2: `cascade.lock.json` + base copies at add time**

`lock.ts`:

```ts
export interface LockEntry {
  version: string
  files: Record<string, string> // relative path → sha256 of content AS INSTALLED
}
export interface LockFile {
  version: 1
  components: Record<string, LockEntry>
}
```

`add.ts`: after writing files, write/merge `cascade.lock.json` (project root) and store pristine copies under `.cascade/base/<name>/<file>` (gitignore advice printed: commit both — base copies are what make merges work; they're small). `sha256` via `node:crypto`.

- [ ] **Step 3: Three-way `update`**

`update.ts` rewrite: fetch latest files; for each file — local hash == lock hash (user never touched it) → overwrite + update lock/base; local differs → three-way merge: base = `.cascade/base/...`, ours = local, theirs = fetched, via `git merge-file -p` when git is available (temp files), conflict markers left in place with a clear summary; no git → print unified diffs of base→theirs and instruct manual merge. Update lock + base only for cleanly merged files. Tests: clean overwrite path, user-edit + upstream-change merge (non-conflicting hunks merge cleanly), conflict path leaves markers and reports.

- [ ] **Step 4: `cascade outdated`** — compare lock versions to registry versions, table output (name, installed, latest, drifted-files count from hash check). Test with a fixture registry.

- [ ] Gate + commit `feat(cli): lock file, three-way update, outdated (owned-code update story)`.
