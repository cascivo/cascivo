**Goal:** The floating overlay family renders through the native `popover` attribute with
CSS anchor positioning and `@starting-style` entry/exit transitions wherever the platform
supports it — and behaves exactly as today everywhere else. This makes pitch claim #3
("near-zero overlay JS") true and testable.

**Architecture:** Progressive enhancement in three layers per component: (1) the popover
attribute replaces manual top-layer/z-index management, (2) anchor positioning CSS inside
`@supports` replaces JS measurement on supporting engines while the existing positioning
path remains as fallback, (3) `@starting-style` + `transition-behavior: allow-discrete`
adds entry/exit motion (reduced-motion-gated). Dialog-family components stay on `<dialog>`;
`toast` migrates only if the a11y suite stays green (decision 14).

**Tech stack:** unchanged. Zero new dependencies — that is the point.

## Tranche exit criteria

1. Audit doc exists with per-component decisions and final outcomes (including any honest
   partial migrations).
2. Migrated overlays use `popover` + anchor CSS under `@supports`, with `@starting-style`
   motion, reduced-motion-gated; fallback path verified in a non-supporting engine.
3. All pre-existing overlay tests pass; keyboard suites for menu/context-menu unmodified
   and green.
4. Orphaned JS positioning/dismiss code deleted where migration landed; bytes-saved table
   recorded.
5. Root gate + drift check exit 0.
