# v30 — Tranche 1: Registry Bundle Fix

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove the 619 KB `registry.json` from the landing JS bundle. `Hero.tsx` and `StatsBand.tsx` currently import the full registry file just to read `components.length` — one integer. Replace both imports with build-time constants injected by Vite's `define` option.

**Architecture:** Extends the existing `componentCount()` / `themeCount()` helpers in `vite.config.ts` (already used for HTML injection) to also inject `__CASCIVO_COMPONENT_COUNT__` and `__CASCIVO_THEME_COUNT__` as TypeScript-ambient constants. No new npm packages needed.

**Tech Stack:** Vite `define`, TypeScript ambient declarations.

### Task 1: Add `define` entries to vite.config.ts

**Files:**

- Modify: `apps/landing/vite.config.ts`

- [ ] **Step 1: Read `apps/landing/vite.config.ts` in full**

  Locate the `export default defineConfig({ ... })` call. The `componentCount()` and
  `themeCount()` functions are already defined and exported above it.

- [ ] **Step 2: Add `define` to `defineConfig()`**

  Inside the `defineConfig({ ... })` object, add a `define` key alongside `plugins`,
  `preview`, `server`, and `resolve`:

  ```ts
  define: {
    __CASCIVO_COMPONENT_COUNT__: componentCount(),
    __CASCIVO_THEME_COUNT__: themeCount(),
  },
  ```

  The values are evaluated once at build/dev-server start — the same timing as the existing
  HTML injection. No import changes needed; `componentCount` and `themeCount` are already
  in scope.

- [ ] **Step 3: Verify the build still works**

  ```bash
  pnpm exec vp run @cascivo/landing#build
  ```

  Expected: exits 0. The constants are injected as integer literals in the bundle
  (visible in the dist output as `__CASCIVO_COMPONENT_COUNT__` → bare number).
