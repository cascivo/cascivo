**Goal:** All four content pages have skip-nav; the mobile drawer traps focus while open (or `inert`s
the background) and restores focus to the toggle on close, without breaking Esc/scrim close. axe is
wired to cover `/guides`.

**Architecture:** `App.tsx` (HomePage skip-nav), `PerformancePage.tsx` (skip-nav), `Header.tsx` (focus
management via `useSignalEffect`), `landing.css` (focus-visible/inert styles if needed),
`test/a11y.spec.ts` (add `/guides` — assertion lands in T7). No package changes.

**Tech stack:** unchanged. Focus management is a DOM side effect → **`useSignalEffect`, never
`useEffect`**.

- [ ] **Step 1:** When `isNavOpen.value` is true, prevent focus from reaching background content. Pick
      the simplest correct mechanism:
      (a) set `inert` on the main page container while open (and remove it on close), or
      (b) a minimal focus trap that cycles Tab/Shift-Tab between the drawer's first/last focusable.

      the toggle. Ensure `aria-hidden`/`inert` aren't applied to an element that contains the currently

## Tranche exit criteria

1. Home and Performance pages render `SkipNavLink`/`SkipNavTarget`; the skip link is first-focusable and
   jumps to `main`.
2. While the mobile drawer is open, focus cannot reach background content; on close, focus returns to
   the menu toggle. Esc + scrim close still work.
3. Focus management uses `useSignalEffect` (no `useEffect`); the toggle ref uses `useRef` for DOM only.
4. axe covers `/guides` (light + dark) and passes alongside the existing routes.
5. No hardcoded colors added; no banned hooks introduced.
