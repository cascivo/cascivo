**Goal:** Make the collapsed SideNav a true icon rail: icons-only layout, grapheme fallback badge
for icon-less items, real tooltips, anchored flyouts for groups (today sublinks are unreachable
when collapsed — the critical bug), opt-in expand-on-hover, and a footer slot. Plus ~24 new
console-staple icons. All changes API-additive; the expanded state must be pixel-identical.

**Architecture:** Rail rendering branches inside the existing component (no new component). In
rail mode: leaf items render icon-only wrapped in `Tooltip`; group items render an icon trigger
that opens a `usePopover`-anchored flyout (`role="menu"`) listing the sublinks. `expandOnHover`
is pure CSS (`:hover` widens the nav as an absolutely-positioned overlay) and suppresses flyouts
(master plan edge case 4). Icon fallback uses `Intl.Segmenter` when available.

**Tech stack / Conventions:** identical to T1 (signals only, `useSignalEffect` for DOM effects,
four-file pattern already in place for side-nav, `vp check` before each commit).

**Context paths:**

- The component: `packages/components/src/side-nav/`

- [ ] **Step 2: Implement**

Add a helper and use it for leaf links and group triggers when `rail` is true:

```tsx
function firstGrapheme(label: string): string {
  if (typeof Intl !== 'undefined' && 'Segmenter' in Intl) {
    const seg = new Intl.Segmenter(undefined, { granularity: 'grapheme' })
    for (const s of seg.segment(label)) return s.segment
  }
  return [...label][0] ?? ''
}
```

Rendering changes (leaf item shown; group trigger analogous):

```tsx
<a
  href={item.href}
  aria-current={item.active ? 'page' : undefined}
  aria-label={rail ? item.label : undefined}
  data-state={item.active ? 'active' : undefined}
  className={styles['link']}
>
  {item.icon ? (
    <span className={styles['icon']} aria-hidden="true">
      {item.icon}
    </span>
  ) : rail ? (
    <span className={styles['fallbackIcon']} data-fallback-icon aria-hidden="true">
      {firstGrapheme(item.label)}
    </span>
  ) : null}
  <span className={styles['label']} data-rail-hidden={rail || undefined}>
    {item.label}
  </span>
</a>
```

CSS: in `[data-state='collapsed']`, `.label[data-rail-hidden] { display: none; }`, links become
`justify-content: center` squares; `.fallbackIcon` is a `1.25rem` circle
(`background: var(--cascivo-color-surface-hover); border-radius: 50%; font-size: 0.6875rem;`
centered). Remove the old `title={rail ? ... : undefined}` attributes (replaced by Tooltip in
Task 3).

The nav keeps its width transition between `--cascivo-sidenav-inline-size` and
`--cascivo-sidenav-rail-inline-size` (`3rem` — add the token if the current rail width differs).

```tsx
describe('SideNav rail flyout', () => {
  const items = [
    {
      label: 'Manage',
      icon: <svg />,
      items: [
        { label: 'Users', href: '/users' },
        { label: 'Billing', href: '/billing' },
      ],
    },
  ]

    render(<SideNav items={items} collapsed />)

    await userEvent.click(trigger)

    expect(flyout).toBeVisible()
    expect(screen.getByRole('menuitem', { name: 'Users' })).toBeVisible()
  })

  it('closes the flyout on Escape and restores trigger focus', async () => {
    render(<SideNav items={items} collapsed />)
    const trigger = screen.getByRole('button', { name: 'Manage' })
    await userEvent.click(trigger)
    await userEvent.keyboard('{Escape}')
    expect(trigger).toHaveAttribute('aria-expanded', 'false')
    expect(trigger).toHaveFocus()
  })

  it('keeps inline expansion when not collapsed', async () => {
    render(<SideNav items={items} />)
    await userEvent.click(screen.getByRole('button', { name: 'Manage' }))
    // inline sublist, not a menu popover
    expect(screen.queryByRole('menu')).not.toBeInTheDocument()
    expect(screen.getByRole('link', { name: 'Users' })).toBeVisible()
  })
})
```
