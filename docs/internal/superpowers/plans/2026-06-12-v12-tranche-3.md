> Current-state assumptions — RE-VERIFY before starting:
>
> - Docs app (Preact, signals natively reactive) has `PlaygroundPage.tsx` as the closest
>   prior art for an interactive tool page; demos render real components.
> - Shared theme generation (`generateThemeCss`) + DTCG export/import (v11-T6) exist;
>   T2's `theme create` wraps them.
> - 5 first-party themes; semantic-token parity test (`packages/themes/src/parity.test.ts`)
>   defines the canonical token key set — the studio's editable surface IS that key set.
> - Directory + theme-type registry items exist (v11); `cascade add @ns/<theme>` works.
> - Tokens: three-level architecture (primitive → semantic → component); themes override
>   the semantic layer only.

**Goal:** Two cascade apps with five minutes of configuration each should be
_recognizably different products_. The Theme Studio makes building a brand theme a
visual, shareable act; composable axes (palette × shape × density × type) multiply the
default-config space; the gallery surfaces community themes.

**Architecture:** Studio = one docs page + a pure theme-state module (signals in, CSS
string out — unit-testable without the page). Axes = small generated CSS partials the
studio (and `theme create`) compose into a theme file. Gallery = generated page over
first-party themes + directory theme items.

**Tech stack:** unchanged. URL-state compression may add one tiny dep to the docs app
only (master-plan risk #7) — evaluate before adding.

### Task 2: Axes as composable CSS partials

**Files:**

- Create: `packages/themes/src/axes/` — `shape-{sharp,soft,round}.css`,
  `density-{compact,comfortable,spacious}.css`, `type-{neutral,humanist,mono}.css`
  (exact preset names decided in design pass; palette is not a partial — it's the token
  colors themselves)
- Create: axis key-set test (sibling of parity.test.ts)

- [ ] **Step 1:** Design pass first: which token subsets do axes own? shape → radius
      tokens + border-width treatment; density → spacing scale multipliers; type → font
      stacks + scale. Axes must be disjoint (no token owned by two axes) — enforce by
      test. Values in each preset get a real design review (eyeball matrix on docs
      components), not just numbers.
- [ ] **Step 2:** Compose rule: theme CSS = palette block + one partial per axis. Update
      shared `generateThemeCss`/`theme create`/studio to accept axis selections.
- [ ] **Step 3:** First-party themes: attempt re-expression as palette + preset
      references **only if** the v9 parity test stays green AND a visual diff pass on
      docs/Storybook shows no unintended change (roadmap decision 8). If it fights,
      axes remain additive output for new themes — record the outcome here either way.
- [ ] **Step 4:** Tests: disjointness, parity of composed output, each preset
      individually applied to the docs theme demo (manual eyeball, recorded).
- [ ] **Step 5: Commit** — `feat(themes): composable axes — shape, density, type presets`

## Tranche exit criteria

1. Studio edits the full semantic key set + axes with live preview; all three exports
   work; share URL round-trips exactly; page passes its own a11y bar.
2. Axes are disjoint by test; composed themes pass parity; first-party re-expression
   outcome recorded honestly (done, or deferred with reason).
3. Gallery lists first-party + directory themes with working install snippets and
   open-in-studio.
4. Screenshot pair proving two default-effort cascade apps look like different products.
5. Root gate + drift check exit 0.
