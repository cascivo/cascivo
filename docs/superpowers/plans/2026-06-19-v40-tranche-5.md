# v40 — Tranche 5: `openui.json` Interop Export + Docs & Final Gate

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let the broader OpenUI / AI-tooling ecosystem discover and consume cascivo by emitting a
standard, framework-agnostic component spec — the "OpenUI = AI-Native Specification for UIs" idea
(`openui.yaml`/`openui.json` at repo root, OpenAPI-for-components). This is **additive interop**, exactly
like v39's shadcn-registry emitter: derive it from the manifests/registry and write it alongside the
existing outputs, with **no change** to `registry.json` or the v39 shadcn output. Then document the whole
v40 roadmap and run the full gate + drift.

**Architecture:** `packages/registry/src/build.ts` already emits cascivo's own `registry.json` and (from
v39) shadcn `/r/<name>.json` items. The manifests carry name/description/props (type/enum/default/required)
and examples — the exact fields the OpenUI spec wants. Add a second emitter that maps them into the
standard `openui.json` schema (`name`, `version`, `description`, `components: { <Name>: { description,
example, props: { <prop>: { type, description, enum, default, required } } } }`).

**Tech Stack:** TypeScript emitter + Vitest in `@cascivo/registry`; Markdown docs. vite+ (`vp`) for the
full gate. Additive; consumes the manifests like T2/T4.
