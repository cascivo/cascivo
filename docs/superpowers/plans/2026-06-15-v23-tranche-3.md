**Goal:** Selecting a project opens a faithful project detail surface: a header (project name, "Visit"
CTA → production URL, production domain, framework, repo), a tab bar (Overview / Deployments / Analytics
/ Settings) driven by `activeProjectTab`, an **Overview** showing the Production Deployment panel + active
preview branches, and a **Deployments** tab with a dense DataTable (status/environment filters). Rows
link to the deployment inspector (`openDeployment`, built in T4).

**Architecture:** `src/sections/ProjectDetail.tsx` reads `activeProjectId`; sub-sections
`ProjectHeader.tsx`, `ProductionPanel.tsx`, `DeploymentsTable.tsx`. Tab switching is the
`activeProjectTab` signal (Tabs component bound to it). All data comes from the T2 signals.

**Tech stack:** unchanged.
