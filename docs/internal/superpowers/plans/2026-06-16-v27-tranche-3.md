# v27 — Tranche 3: Compact Theme Switcher

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the three colored dot buttons (`header-themes`) with a single icon button that
cycles through light → dark → warm → light. A `Tooltip` shows the current theme name on hover.
This frees horizontal space in the header end slot and removes an unlabelled UI pattern.

**Architecture:** `Header.tsx` only (plus minimal CSS). The `THEMES` array and `setTheme` import
are unchanged. The `theme` signal remains the source of truth.

**Tech Stack:** `useSignals`, `theme` signal, `@cascivo/components/tooltip` (if available) or
a CSS-powered tooltip fallback.

**Prerequisite:** T2 must be complete (the `end` slot now holds the GitHub icon alongside the
theme dots).

- [ ] **Step 2: Add `cycleTheme` function**

  Below the icon declarations, add:

  ```ts
  function cycleTheme() {
    const idx = THEMES.indexOf(theme.value)
    setTheme(THEMES[(idx + 1) % THEMES.length] ?? THEMES[0])
  }
  ```

- [ ] **Step 1: Remove the `header-themes` div from the `end` slot**


- [ ] **Step 2: Add the cycle button**

  Replace with a single button. If `Tooltip` is available:

  ```tsx
  import { Tooltip } from '@cascivo/components/tooltip'

  end={
    <>
      <Tooltip content={`Theme: ${theme.value}`} side="bottom">
        <button
          type="button"
          className="header-theme-cycle"
          aria-label={`Switch theme, current: ${theme.value}`}
          onClick={cycleTheme}
        >
          {THEME_ICONS[theme.value]?.() ?? <SunIcon />}
        </button>
      </Tooltip>
      <a href="..." className="header-icon-link" aria-label="GitHub" ...>
        <GitHubIcon />
      </a>
    </>
  }
  ```

  If `Tooltip` is not available, use the CSS-only fallback (see Task 4 below) and omit the
  `<Tooltip>` wrapper.
